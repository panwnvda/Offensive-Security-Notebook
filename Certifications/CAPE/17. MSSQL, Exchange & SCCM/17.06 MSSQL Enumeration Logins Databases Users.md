# 17.06 MSSQL Enumeration Logins Databases Users

> Enumerate server logins/roles, database TRUSTWORTHY flags, and DB users to find impersonation targets and sysadmin holders.

## Overview

Before attacking MSSQL Server, enumerate server-level logins and their roles, databases and their trustworthy flag, and database-level users. These queries reveal impersonation targets, sysadmin holders, and trustworthy databases that enable privilege escalation.

## Steps

- Query sys.server_principals + sys.syslogins to list all logins and server-level roles (sysadmin, securityadmin, etc.)
- Query sys.databases to identify database owners and the is_trustworthy_on flag
- USE <database> then EXECUTE sp_helpuser to enumerate database-level users and roles
- Cross-reference trustworthy databases with db_owner membership to spot escalation paths

## Commands

### Enumerate logins + server roles

```sql
SELECT r.name, r.type_desc, r.is_disabled, sl.sysadmin, sl.securityadmin, sl.serveradmin, sl.setupadmin, sl.processadmin, sl.diskadmin, sl.dbcreator, sl.bulkadmin FROM master.sys.server_principals r LEFT JOIN master.sys.syslogins sl ON sl.sid = r.sid WHERE r.type IN ('S','E','X','U','G');
```

### Enumerate databases + trustworthy flag

```sql
SELECT a.name AS 'database', b.name AS 'owner', is_trustworthy_on FROM sys.databases a JOIN sys.server_principals b ON a.owner_sid = b.sid;
```

### Enumerate database users

```sql
USE webshop; EXECUTE sp_helpuser;
```

### Enumerate db_owner members (full query)

```sql
USE webshop; SELECT b.name, c.name FROM webshop.sys.database_role_members a JOIN webshop.sys.database_principals b ON a.role_principal_id = b.principal_id LEFT JOIN webshop.sys.database_principals c ON a.member_principal_id = c.principal_id;
```
