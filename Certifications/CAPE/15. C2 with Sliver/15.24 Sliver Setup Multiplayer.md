# 15.24 Sliver Setup Multiplayer

> Stand up Sliver server (unpacks on first run), generate operator configs, then explicitly enable multiplayer before clients can join.

## Overview

Sliver runs as separate server and client binaries. The server is started first; on first run it unpacks assets. Operators connect via imported config profiles. Multiplayer mode must be explicitly enabled before clients can join.

## Steps

- Download server and client binaries from BishopFox GitHub releases.
- Start server: `./sliver-server_linux`; first run unpacks assets and may take several minutes.
- Generate an operator profile with `new-operator -n <name> -l <lhost>`.
- Enable multiplayer mode on the server with `multiplayer`.
- Download client binary, import the generated .cfg, then launch client.
- Client connects to server; sessions/beacons appear in both operator clients.

## Commands

### Download server binary

```bash
wget -q https://github.com/BishopFox/sliver/releases/download/v1.5.42/sliver-server_linux
```

### Make executable and run

```bash
chmod +x ./sliver-server_linux && ./sliver-server_linux
```

### Generate operator profile

```text
[server] sliver > new-operator -n student -l 10.10.14.193
```

### Enable multiplayer

```text
[server] sliver > multiplayer
```

### Download client binary

```bash
wget -q https://github.com/BishopFox/sliver/releases/download/v1.5.42/sliver-client_linux
```

### Import profile and start client

```bash
chmod +x ./sliver-client_linux && ./sliver-client_linux import student_10.10.14.193.cfg && ./sliver-client_linux
```
