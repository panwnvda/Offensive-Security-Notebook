# 15.18 Listeners

> Bind C2 ports for implant callbacks — don't kill the default gRPC `:31337`; pivot listeners start from inside a session, not top-level.

## Overview

Listeners bind the C2 server to a port to accept callbacks from implants. Multiple listeners can run concurrently. The default gRPC listener on TCP/31337 must not be stopped. Named-pipe pivot listeners are started from within an active session, not the top-level console.

## Steps

- Start HTTP listener with `http -L <ip> -l <port>`; omit -L to listen on all interfaces.
- `jobs` shows all active listeners with their IDs, protocols, and ports.
- Named-pipe pivot listener is started in a session context with `pivots named-pipe --bind <name>`.
- TCP pivot listener is started with `pivots tcp --bind <ip>` and defaults to port 9898.
- Stop a listener/job with `jobs -k <id>`.

## Commands

### Start HTTP listener

```text
sliver > http --lport 8088
```

### Start HTTP listener on specific IP

```text
sliver > http -L 10.10.14.62 -l 9001
```

### List running listeners

```text
sliver > jobs
```

### Start named-pipe pivot listener (from session)

```text
[server] sliver (http_beacon) > pivots named-pipe --bind academy
```

### Start TCP pivot listener (from session)

```text
sliver (http-beacon) > pivots tcp --bind <TARGET>
```
