# 15.08 Execute Assembly Inline Rubeus

> Run .NET in a sacrificial `notepad.exe` via `execute-assembly` or stealthier in-process via `inline-execute-assembly` (coff-loader).

## Overview

`execute-assembly` runs .NET binaries in a sacrificial child process (default: notepad.exe). `inline-execute-assembly` (a coff-loader extension) runs the binary in the current beacon process — stealthier but riskier. Use `-i/-E/-M` flags on `execute-assembly` for in-process AMSI/ETW bypass.

## Steps

- `execute-assembly <path> [args]` — spawns notepad.exe (or --process override) and injects .NET.
- `--process` flag overrides the sacrificial process to reduce OPSEC footprint.
- `inline-execute-assembly <path> 'args'` — runs in beacon process; no child process spawned.
- Both accept full paths to binaries from SharpCollection or locally compiled tools.
- Use `-t <seconds>` to extend timeout for long-running .NET tools like SharpView.
