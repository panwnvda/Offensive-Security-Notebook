# 15.22 Sessions Beacons Management

> Drive interactive sessions (red) vs async beacons (blue) — `use` switches by UUID prefix, `interactive` upgrades a beacon.

## Overview

Sessions (red in CLI) are interactive; beacons (blue) execute commands asynchronously on a callback interval. `use` switches context to either type by UUID prefix. `interactive` upgrades a beacon to a session. `tasks` shows pending and completed beacon task queue.

## Steps

- `sessions` lists interactive sessions; `beacons` lists async beacons.
- `use <UUID-prefix>` activates a session or beacon; supports TAB autocomplete.
- `interactive` inside a beacon context spawns an interactive session from it.
- `tasks` inside a beacon shows task state: pending, sent, completed.
- `info` displays full details: hostname, user, OS, PID, active C2, interval.
- `pivots` lists active pivot listeners and how many implants are chained through them.

## Commands

### List sessions

```text
sliver > sessions
```

### Switch to session

```text
sliver > use 06ff8ed9-5a6b-4eca-ba88-7b49e8637dd3
```

### Convert beacon to interactive session

```text
sliver (http-beacon-9001) > interactive
```

### Show beacon info

```text
sliver (http-beacon-9001) > info
```

### List beacons

```text
sliver (HIGH_RISER) > beacons
```

### View beacon task queue

```text
sliver (beacon) > tasks
```

### List pivot listeners

```text
sliver (http-beacon) > pivots
```
