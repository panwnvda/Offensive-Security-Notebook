# 05.09 SSH Lateral Movement Windows

> SSH into Windows hosts running OpenSSH with `nxc ssh` — keys must be owner-readable only, no network-logon token is created.

## Overview

OpenSSH can be installed on Windows, enabling SSH-based lateral movement where RDP/SMB may be blocked. Credential testing uses netexec with the ssh protocol. Authentication works with both passwords and private keys; key files must be readable only by the owning user (use `copy` + `icacls` to fix permissions). SSH does not produce a network logon so it limits network-based authentication but works for local enumeration.

## Steps

- Enumerate: `nmap <IP> -p 22 -sCV -Pn` to confirm OpenSSH for Windows.
- Test credentials: `netexec ssh <IP> -u <user> -p <pass>` — "Windows - Shell access!" confirms access.
- Connect from Linux: `ssh user@IP`; accept host key; enter password.
- Connect from Windows: `ssh ambioris@SRV01` — prompts for host key confirmation then password.
- Private key auth: `ssh -i <keyfile> -l <user@domain> -p 22 <host>` — key must not be world-readable.
- Fix key permissions: `copy <keyfile> C:\Users\<user>\` — inherited ACL removes BUILTIN\Users read access.
- Verify permissions: `icacls.exe <keyfile>`.

## Commands

### nmap SSH scan

```bash
nmap <TARGET> -p 22 -sCV -Pn
```

### netexec SSH credential test

```bash
netexec ssh <TARGET> -u ambioris -p Ward@do9049
```

### SSH connect from Linux

```bash
ssh [email protected]
```

### SSH connect from Windows

```powershell
ssh ambioris@SRV01
```

### SSH private key auth (Windows)

```powershell
ssh -i C:\helen_id_rsa -l [email protected] -p 22 SRV01
```

### icacls check permissions

```powershell
icacls.exe C:\helen_id_rsa
```

### Copy key to fix permissions

```powershell
copy C:\helen_id_rsa C:\Users\Ambioris\
```
