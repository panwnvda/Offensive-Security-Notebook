# 05.07 Remote Management Tools MeshCentral

> Pivot through RMM platforms (MeshCentral, PDQ, Intune) that hold persistent agent connections over 443, bypassing SMB/RDP ACLs.

## Overview

Software deployment and RMM tools (MeshCentral, SCCM, PDQ Deploy, Intune, Kaseya VSA, etc.) maintain persistent agent connections to managed hosts, often over port 443, bypassing most SMB/RDP firewall rules. Compromising RMM credentials grants GUI-based terminal access to all managed devices. MeshCentral is open-source and identifiable by its login page on port 443.

## Steps

- Scan for port 443 and inspect TLS/HTTP title: `nmap -p443 <IP> -sC -sV` — look for "MeshCentral - Login" or similar RMM title.
- Check installed software locally if you have access to a host: `Get-WmiObject -Class Win32_Product` or review Add/Remove Programs.
- Obtain RMM admin credentials (from password reuse, secrets in config files, etc.).
- Log in to MeshCentral web portal (default path `/`; credentials `admin / RemoteManagement01` in lab).
- Navigate My Devices → select target device → Terminal tab → Connect.
- MeshCentral agent connects outbound to server on port 443 — works even when inbound ports to the device are all blocked.

## Commands

### nmap MeshCentral discovery

```bash
nmap -p443 <TARGET> -sC -sV
```
