# 06.08 Discovery Relay List Generation

> Sweep SMB hosts with `nxc smb --gen-relay-list` to produce a signing-disabled target list for `ntlmrelayx`.

## Overview

Sweep subnets to identify live SMB hosts with OS info, SMB signing status, and domain membership. --gen-relay-list filters hosts that do NOT have SMB signing enforced, producing a target list for ntlmrelayx. Unauthenticated scan; no credentials required.

## Steps

- Run `nxc smb <subnet>/24` to identify live hosts, Windows version, SMB signing, and domain.
- Hosts where `signing:False` are candidates for SMB relay attacks.
- Use `--gen-relay-list <file>` to automatically output only the relay-eligible IPs to a file.
- Feed the relay list directly to `ntlmrelayx.py -tf relay.txt`.
- Domain Controllers will show `signing:True`; workstations often show `signing:False`.
- Output includes architecture (x86/x64), which is useful for payload selection.

## Commands

### SMB subnet sweep

```bash
nxc smb 192.168.133.0/24
```

### Generate relay target list

```bash
nxc smb 192.168.1.0/24 --gen-relay-list relaylistOutputFilename.txt
```

### View relay list contents

```bash
cat relaylistOutputFilename.txt
```
