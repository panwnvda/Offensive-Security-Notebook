# 06.01 Additional CME Functionality Audit Mode IPv6

> Enable audit mode to mask passwords in screenshots, target IPv6 hosts natively, and press Enter mid-scan for progress.

## Overview

CME 5.3.0+ includes audit mode, which replaces passwords/hashes in output with a custom character (useful for screenshots in reports). CME also supports IPv6 targets natively — use an IPv6 address as the target after discovering it via `get_netconnections`. Press Enter during a running scan to see completion percentage and remaining host count.

## Steps

- Edit `~/.cme/cme.conf` and set `audit_mode = #` (or any character) to mask credentials in output.
- To avoid the command itself showing the password, store the password in a file and use `-p <file>`.
- Discover IPv6 addresses with `get_netconnections` module (returns all IPs including IPv6).
- Pass IPv6 address directly as the CME target: `nxc smb dead:beef::... -u <user> ...`
- Press Enter while a multi-host scan is running to get `[*] completed: X.XX% (Y/Z)` progress.

## Commands

### Enable audit mode in cme.conf

```text
audit_mode = #
```

### Use password from file (audit mode)

```bash
nxc smb <TARGET> -u robert -p robert_password.txt
```

### Discover IPv6 via get_netconnections

```bash
nxc smb <TARGET> -u robert -p robert_password.txt -M get_netconnections
```

### Execute command over IPv6

```bash
nxc smb dead:beef::8c8a:5209:5876:537d -u robert -p robert_password.txt -x "ipconfig"
```
