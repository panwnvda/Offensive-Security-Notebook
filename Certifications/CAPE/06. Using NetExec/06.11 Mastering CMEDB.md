# 06.11 Mastering CMEDB

> Browse stored creds, hosts, shares, and groups in `~/.cme/workspaces/` via the `cmedb` CLI — replay creds by `-id <CredID>`.

## Overview

CME stores all credentials, hosts, shares, and groups in SQLite databases at `~/.cme/workspaces/default/`. Interact with `cmedb` CLI. Supports multiple workspaces. Use `-id <CredID>` in CME commands to re-use stored credentials. Export data as CSV. Import credentials from Empire. SMB and MSSQL have the richest database features (creds, hosts, shares, groups).

## Steps

- `cmedb` launches the interactive database shell; `proto smb` enters the SMB database.
- `creds` lists all credentials; `creds hash` / `creds plaintext` / `creds <username>` to filter.
- `hosts` lists compromised hosts with OS, signing, and admin count; `hosts <hostname>` to filter.
- `shares` lists enumerated shares with read/write access per user.
- `workspace create <name>` creates a new workspace; `workspace list` shows all; `workspace <name>` switches.
- `creds add <domain> <user> <pass>` manually adds credentials; `creds remove <CredID>` removes them.
- `export creds detailed <filename.csv>` / `export shares detailed <filename.csv>` / `export local_admins detailed <filename.csv>` for CSV export.
- Use `-id <CredID>` in any CME command to reference stored credentials by ID.

## Commands

### Launch cmedb

```bash
cmedb
```

### Enter SMB database

```text
cmedb (default) > proto smb
```

### List all credentials

```text
cmedb (default)(smb) > creds
```

### Filter credentials by user

```text
cmedb (default)(smb) > creds julio
```

### List only hashes

```text
cmedb (default)(smb) > creds hash
```

### List only plaintext creds

```text
cmedb (default)(smb) > creds plaintext
```

### List compromised hosts

```text
cmedb (default)(smb) > hosts
```

### Add credentials manually

```text
cmedb (default)(smb) > creds add <DOMAIN> john Password3
```

### Remove credentials

```text
cmedb (default)(smb) > creds remove 45
```

### Export detailed credentials to CSV

```text
cmedb (default)(smb) > export creds detailed detailed_creds.csv
```

### Use stored CredID in CME command

```bash
nxc smb <TARGET> -id 4 -x whoami
```

### Create a new workspace

```text
cmedb (default) > workspace create <DOMAIN>
```
