# 06.13 Null Session Enumeration

> Try null and guest sessions with `nxc` to pull password policy, RID brute-force users, and list shares without credentials.

## Overview

Attempt null (`''`) and guest sessions to extract password policy, domain user list, RID brute-force, and share enumeration without valid credentials. Exported files are JSON and require `sed`/`jq` post-processing. RID brute forces up to RID 4000 by default and can be capped with `--rid-brute [MAX_RID]`.

## Steps

- Use `-u '' -p ''` for true null session; fall back to `-u guest -p ''` if null is blocked.
- `--pass-pol` dumps lockout threshold, complexity, and history — essential before spraying.
- `--users` enumerates domain users; some DCs expose this with null auth via SAMRPC fallback.
- `--rid-brute` works even when `--users` is restricted; enumerates users AND groups by SID.
- `--export $(pwd)/file.txt` writes JSON; use `sed -i "s/'/\"/g"` then `jq` to parse.
- `--shares` lists shares; errors from null can be resolved by trying `-u guest -p ''`.
- Extract a clean user list with: `jq -r '.[]' users.txt > userslist.txt`

## Commands

### Null session password policy

```bash
nxc smb <TARGET> -u '' -p '' --pass-pol
```

### Export password policy to file

```bash
nxc smb <TARGET> -u '' -p '' --pass-pol --export $(pwd)/passpol.txt
```

### Null session user enumeration

```bash
nxc smb <TARGET> -u '' -p '' --users --export $(pwd)/users.txt
```

### Format exported JSON

```bash
sed -i "s/'/\"/g" users.txt && jq -r '.[]' users.txt > userslist.txt
```

### RID brute-force (no creds)

```bash
nxc smb <TARGET> -u '' -p '' --rid-brute
```

### Null share enumeration

```bash
nxc smb <TARGET> -u '' -p '' --shares
```

### Guest share enumeration

```bash
nxc smb <TARGET> -u guest -p '' --shares
```
