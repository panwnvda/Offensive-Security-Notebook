# 04.01 Attack Path Analysis Methodology

> Mark owned nodes, audit Domain Users privileges, then pathfind to Domain Admins — sessions reveal where privileged users currently sit.

## Overview

Systematic approach to analyzing BloodHound data: start with the domain overview, enumerate Domain Users group privileges, run all pre-built queries, mark owned nodes, then use pathfinding from owned nodes to Domain Admins. Sessions reveal where privileged users are currently active — pivotal for planning lateral movement.

## Steps

- **Step 1** — Search `domain:<DOMAIN>.HTB` for domain stats (users, groups, computers, functional level).
- **Step 2** — Search Domain Users group → Outbound Object Control → Transitive Object Control (all inherited privileges for every domain user).
- **Step 3** — Pathfinding: Domain Users → Domain Admins to check if all users have a path to DA.
- **Step 4** — Run 'Find all Domain Admins' (direct and unrolled nested members).
- **Step 5** — Run 'Find Shortest Paths to Domain Admins'.
- **Step 6** — Run 'Find Principals with DCSync Rights'.
- **Step 7** — Check sessions on Domain Admins group → sessions count → which computers DA members are logged into.
- **Step 8** — Mark compromised nodes as Owned (right-click → Mark as Owned).
- **Step 9** — Re-run 'Shortest Path from Owned Principals' after each new compromise.
- **Step 10** — If no GUI path exists for a compromised node, run a custom `allshortestPaths` Cypher query or fall back to PowerView `Get-DomainObjectAcl`.

## Commands

### Pre-built Analysis queries (run in order)

```text
Find all Domain Admins
Find Shortest Paths to Domain Admins
Find Principals with DCSync Rights
Users with Foreign Domain Group Membership
Groups with Foreign Domain Group Membership
Map Domain Trusts
Shortest Paths to Unconstrained Delegation Systems
Shortest Paths from Kerberoastable Users
Shortest Path from Owned Principals
Shortest Paths to Domain Admins from Owned Principals
Shortest Paths to High-Value Targets
```

### Get-DomainObjectAcl fallback (when BH shows no path)

```powershell
PS C:\htb> Import-Module c:\tools\PowerView.ps1
PS C:\htb> Get-DomainObjectAcl -Identity peter -domain <DOMAIN>.HTB -ResolveGUIDs
```
