# 04.08 Cypher Query Language

> Write Cypher: `MATCH p=(...) WHERE ... RETURN p` for path queries — enable Query Debug Mode to see the Cypher behind built-ins.

## Overview

Cypher is Neo4j's graph query language. MATCH describes a graph pattern, WHERE filters results, RETURN specifies output. Nodes use parentheses `()`, relationships use dashes with brackets `[r:TYPE]->`. In BloodHound's Raw Query bar use `MATCH p=(...) RETURN p` — path variable form. The Neo4j web console at http://localhost:7474/ supports non-path queries (e.g., `RETURN u.name`). Enable Query Debug Mode in Settings to see the Cypher behind every built-in query.

## Steps

- **MATCH (n:Label)** — match all nodes of a type.
- **MATCH (n:User {name:'X'})** — match node by property.
- **MATCH (n)-[r:TYPE]->(m)** — match directed relationship.
- **`*1..`** — any depth traversal. **`*1..2`** — depth 1 or 2 hops.
- **CONTAINS** — substring match. **`=~`** — regex match. **`(?i)`** — case-insensitive.
- **shortestPath()** — single shortest path. **allshortestPaths()** — all shortest paths (slower, more results).
- **BloodHound Raw Query** requires `p=` path variable and `RETURN p`. Non-path queries only work in the Neo4j console.

## Commands

### Cypher basics — users and group memberships

```text
-- Return all users:
MATCH (u:User) RETURN u

-- Return specific user by name:
MATCH (u:User {name:"[email protected]"}) RETURN u

-- Same using WHERE:
MATCH (u:User) WHERE u.name = "[email protected]" RETURN u

-- Peter's direct group memberships (Neo4j console):
MATCH (n:User {name:"[email protected]"})-[r:MemberOf]->(g:Group) RETURN n,r,g

-- Same query for BloodHound Raw Query bar (path form):
MATCH p=((n:User {name:"[email protected]"})-[r:MemberOf]->(g:Group)) RETURN p

-- All MemberOf at any depth:
MATCH p=((u:User {name:"[email protected]"})-[r:MemberOf*1..]->(g:Group)) RETURN p

-- Filter where first group name contains 'ITSECURITY':
MATCH p=(n:User)-[r1:MemberOf*1..]->(g:Group) WHERE nodes(p)[1].name CONTAINS 'ITSECURITY' RETURN p

-- Same with case-insensitive regex:
MATCH p=(n:User)-[r1:MemberOf*1..]->(g:Group) WHERE nodes(p)[1].name =~ '(?i)itsecurity.*' RETURN p

-- Depth-limited (1 or 2 hops):
MATCH p=(n:User)-[r1:MemberOf*1..2]->(g:Group) RETURN p
```

### Built-in 'Shortest Paths to DA' (from Query Debug Mode)

```text
MATCH p=shortestPath((n)-[*1..]->(m:Group {name:"DOMAIN [email protected]"})) WHERE NOT n=m RETURN p
```
