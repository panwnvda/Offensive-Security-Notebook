# 04.09 Neo4j Setup

> Stand up Neo4j 4.4.16 (NOT 5.x) and rotate the default `neo4j/neo4j` creds before BloodHound will connect.

## Overview

Neo4j 4.4.16 is the required graph database backend for BloodHound. Neo4j 5 has severe performance regressions and must be avoided. Initial credentials are neo4j/neo4j and must be changed before BloodHound can connect. BloodHound config and custom queries live in ~/.config/bloodhound (Linux) or AppData\Roaming\bloodhound (Windows).

## Steps

- **Do NOT install Neo4j 5** — severe performance regression; pin to 4.4.16.
- **JAVA_HOME must point to JDK 11** — Neo4j will fail to start with other JDK versions.
- **`sudo neo4j console`** — run in foreground for initial password setup.
- **`sudo systemctl start neo4j`** — background service for normal use.
- **Remote access**: uncomment `dbms.default_listen_address=0.0.0.0` in `/etc/neo4j/neo4j.conf`.
- **Password recovery**: disable auth in neo4j.conf, reset via Cypher, re-enable.

## Commands

### Linux — install Neo4j 4.4.16

```bash
wget -O - https://debian.neo4j.com/neotechnology.gpg.key | sudo apt-key add -
echo 'deb https://debian.neo4j.com stable 4.4' | sudo tee -a /etc/apt/sources.list.d/neo4j.list
sudo apt-get update
sudo apt install neo4j=1:4.4.16 -y
# Select Java 11 as default:
sudo update-alternatives --config java   # enter 1 for java-11-openjdk
sudo neo4j console                       # foreground — change password on first run
sudo systemctl start neo4j               # background service
```

### Linux — run BloodHound GUI

```bash
unzip BloodHound-linux-x64.zip
cd BloodHound-linux-x64/
./BloodHound --no-sandbox
```

### Windows — install Neo4j as service

```powershell
PS C:\htb> .\jdk-11.0.17_windows-x64_bin.exe /s
PS C:\htb> Expand-Archive .\neo4j-community-4.4.16-windows.zip .
PS C:\htb> .\neo4j-community-4.4.16\bin\neo4j.bat install-service
PS C:\htb> net start neo4j
# Then open http://localhost:7474/ — change neo4j/neo4j password
```

### Password recovery — disable auth, reset via Cypher

```bash
sudo systemctl stop neo4j
# In /etc/neo4j/neo4j.conf — uncomment:
#   dbms.security.auth_enabled=false
sudo neo4j console
# Navigate to http://localhost:7474/ — click Connect with no credentials
# In Cypher query box:
ALTER USER neo4j SET PASSWORD 'Password123';
# Then stop, re-comment dbms.security.auth_enabled=false, restart
```

### PwnBox shortcuts

```bash
sudo neo4j console    # or: sudo systemctl start neo4j
bloodhound            # launch BloodHound GUI
# Lab neo4j credentials: neo4j / Password123
```
