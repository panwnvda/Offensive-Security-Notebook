# 04.02 AzureHound

> Collect Entra ID and ARM objects with `AzureHound` — re-run after escalation since basic roles can't see subscription resources.

## Overview

AzureHound is a Go binary that collects Azure AD (Entra ID) and Azure Resource Manager objects via the MS Graph and Azure REST APIs. Output is a single JSON file imported into BloodHound alongside AD data. A user with only basic permissions (like Password Administrator) will not enumerate subscription resources — re-run as a more privileged user after escalation to see additional objects.

## Steps

- **list --tenant** — collect all Azure objects for the specified tenant.
- **-o all.json** — write output to JSON for BloodHound import.
- **Re-run after privilege escalation** — re-import as a more privileged user to see subscription objects.
- **TheEdgeMaker.ps1** — creates an Azure lab environment with vulnerable edges for practice.
- **Azure node prefixes in BH search**: AZTenant:, AZUser:, AZGroup:, AZApp:, AZSubscription:, AZResourceGroup:, AZVM:, AZDevice:, AZServicePrincipal:, AZKeyVault:.
- **Azure edges**: AZResetPassword, AZOwns, AZAddMembers, AZContributor, AZGlobalAdmin, AZGetSecrets, AZExecuteCommand, AZVMAdminLogin — see BloodHound edge help for abuse info.
