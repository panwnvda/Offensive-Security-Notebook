# 04.10 PlumHound Path Analyzer

> Translate BloodHound paths into remediation reports with `PlumHound -ap` — pinpoints the exact relationship to remove to break a path.

## Overview

PlumHound wraps BloodHound Neo4j queries into reports and includes a Path Analyzer (-ap) that identifies exactly which relationship to remove to break a given attack path — giving AD administrators concrete, actionable remediation steps. It connects to bolt://localhost:7687.

## Steps

- **`--easy -p <password>`** — test connectivity, confirms Neo4j is reachable.
- **`-ap "<start>" "<end>"`** — Path Analyzer: finds the specific edge to remove to break the path.
- **Note**: some relationships may not be found by the path analyzer (known limitation).
- **Note**: not all remediation suggestions are safe — e.g., removing DC01 from Domain Controllers container would break DC functionality. Review suggestions before applying.

## Commands

### Install and test connectivity

```text
C:\Tools\PlumHound> python -m pip install -r requirements.txt
C:\Tools\PlumHound> python PlumHound.py --easy -p Password123
```

### Path Analyzer — find edge to break an attack path

```text
C:\Tools\PlumHound> python PlumHound.py -p Password123 -ap "DOMAIN [email protected]" "WS01.<DOMAIN>.HTB"
```
