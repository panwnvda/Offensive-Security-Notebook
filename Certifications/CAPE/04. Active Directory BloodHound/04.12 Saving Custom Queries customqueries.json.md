# 04.12 Saving Custom Queries customqueries.json

> Persist custom queries to `customqueries.json` — supports interactive two-stage queries that pipe `$result` into stage two.

## Overview

Custom queries are saved to customqueries.json in the BloodHound config directory and appear under 'Custom Queries' in the Queries tab. Queries can be static (single-stage) or interactive two-stage, where stage one returns a list for the user to select from and stage two executes with $result as the selected value. Enable Query Debug Mode to use the Raw Query box.

## Steps

- **Config directory (Linux)**: `/home/<user>/.config/bloodhound` or `/root/.config/bloodhound`
- **Config directory (Windows)**: `AppData\\Roaming\\bloodhound`
- **Static query**: `"final": true` — single Cypher query, result displayed directly.
- **Two-stage interactive**: first entry has `"final": false` (returns list); second has `"final": true` and uses `$result`.
- **Reload**: click the pencil icon's refresh button while BloodHound is open.
- **PrebuiltQueries.json** in the BloodHound install directory contains all built-in Analysis queries.

## Commands

### Config directory

```bash
ls /root/.config/bloodhound
# blob_storage  Cache  Code Cache  config.json  customqueries.json  Cookies ...
```

### customqueries.json — static query

```json
{
    "queries": [
        {
            "name": "From Peter to Anything",
            "category": "Shortest Paths",
            "queryList": [
                {
                    "final": true,
                    "query": "MATCH p = allshortestPaths((n)-[*1..]->(c))  WHERE n.name =~ '(?i)peter.*' AND NOT c=n  RETURN p",
                    "allowCollapse": true
                }
            ]
        }
    ]
}
```

### customqueries.json — interactive two-stage query

```json
{
    "queries": [
        {
            "name": "Search From Owned to Anything",
            "category": "Shortest Paths",
            "queryList": [
                {
                    "final": false,
                    "title": "Select the node to search...",
                    "query": "MATCH (n {owned:true}) RETURN n.name",
                    "props": { "name": ".*" }
                },
                {
                    "final": true,
                    "query": "MATCH p = allshortestPaths((n)-[*1..]->(c))  WHERE n.name = $result AND NOT c=n  RETURN p",
                    "allowCollapse": true,
                    "endNode": "{}"
                }
            ]
        }
    ]
}
```
