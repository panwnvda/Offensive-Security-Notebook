# 04.05 bloodhound-python Linux

> Run `bloodhound-python` from Kali — pin DC FQDN in `/etc/hosts` for Kerberos, add `--zip` to bundle the JSON output.

## Overview

bloodhound-python (fox-it, Dirk-jan Mollema) is a Python/Impacket-based BloodHound collector. It runs from Kali without touching the target Windows filesystem. Kerberos is the default auth method with NTLM fallback — but Kerberos requires the DC's FQDN to be in /etc/hosts (--nameserver alone is not enough). Output is individual JSON files (not auto-zipped); add --zip for a ZIP. Supports all SharpHound collection methods except GPOLocalGroup.

## Steps

- **Add DC FQDN to /etc/hosts** before using --kerberos — nameserver flag alone is insufficient.
- **-c DCOnly** — skips connecting to workstations; only queries the DC. Fastest + stealthiest.
- **-c All** — full collection (except LoggedOn). Same scope as SharpHound's All.
- **Supported auth**: password, NTLM hash (-hashes), AES key (--aesKey), ccache (--ccache-file), Kerberos (--kerberos).
- **Output**: timestamped JSON files per object type. Import all JSON files or use --zip.

## Commands

### Install

```bash
pip install bloodhound
# or from source:
git clone https://github.com/fox-it/BloodHound.py -q
cd BloodHound.py/ && sudo python3 setup.py install
```

### Fix DNS for Kerberos auth

```bash
echo -e "\n10.129.204.207 dc01.<DOMAIN>.htb dc01 <DOMAIN> <DOMAIN>.htb" | sudo tee -a /etc/hosts
```

### DCOnly collection (stealth)

```bash
bloodhound-python -d <DOMAIN>.htb -c DCOnly -u <USER> -p HTBRocks! -ns <TARGET> -k
```

### Full collection with Kerberos (after /etc/hosts fix)

```bash
bloodhound-python -d <DOMAIN>.htb -c DCOnly -u <USER> -p HTBRocks! -ns <TARGET> --kerberos
```

### Collection methods reference (v1.6.0)

```text
Group, LocalAdmin, Session, Trusts, Default (all above),
DCOnly (no computer connections), DCOM, RDP, PSRemote,
LoggedOn, Container, ObjectProps, ACL,
All (everything except LoggedOn)

Output files produced:
  *_computers.json  *_domains.json  *_groups.json
  *_users.json  *_containers.json  *_gpos.json  *_ous.json
```
