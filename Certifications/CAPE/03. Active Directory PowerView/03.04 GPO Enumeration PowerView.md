# 03.04 GPO Enumeration PowerView

> Enumerate GPOs and their DACLs to find writable policies abusable via `SharpGPOAbuse`.

## Overview

Enumerate all GPOs in the domain to understand security controls (AppLocker, LAPS, PowerShell logging, Defender) and identify GPOs applied to specific computers. Check GPO permissions to find writable GPOs that Domain Users (or other non-admin principals) can modify for GPO abuse with SharpGPOAbuse.

## Steps

- List all GPO display names with Get-DomainGPO | select displayname.
- Check GPOs applied to a specific host with -ComputerName or -ComputerIdentity.
- Use gpresult /r /S or gpresult /r /user: to get applied GPOs without PowerView.
- Check GPO ACLs using Get-DomainGPO | Get-ObjectAcl filtered by a SID.
- Confirm a writable GPO name using the built-in Get-GPO -Guid.

## Commands

### List all GPO names (SharpView)

```powershell
.\SharpView.exe Get-DomainGPO | findstr displayname
```

### List all GPO names (PowerView)

```powershell
Get-DomainGPO | select displayname
```

### GPOs applied to a specific computer

```powershell
Get-DomainGPO -ComputerName WS01 | select displayname
```

### Check GPO permissions for Domain Users

```powershell
Get-DomainGPO | Get-ObjectAcl | ? {$_.SecurityIdentifier -eq 'S-1-5-21-2974783224-3764228556-2640795941-513'}
```

### Confirm writable GPO name by GUID

```powershell
Get-GPO -Guid 831DE3ED-40B1-4703-ABA7-8EA13B2EB118
```

### GPO report for a user (built-in)

```text
gpresult /r /user:harry.jones
```

### GPO report for a computer (built-in)

```text
gpresult /r /S WS01
```
