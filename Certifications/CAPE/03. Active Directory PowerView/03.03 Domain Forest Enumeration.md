# 03.03 Domain Forest Enumeration

> Use `Get-Domain`, `Get-DomainSID`, and `Get-DomainTrustMapping` to recursively walk every reachable trust including cross-forest.

## Overview

Get-Domain, Get-DomainSID, and trust-mapping functions are the starting point for every PowerView engagement. Get-Domain returns the full domain object including child domains and DC roles. Get-DomainTrustMapping recursively enumerates all trusts reachable from the current domain, including cross-forest trusts.

## Steps

- Run Get-Domain / SharpView Get-Domain to get the current domain name, DCs, forest root, and domain mode.
- Run (Get-DomainUser).count to quickly measure the number of users in scope.
- Run Get-DomainSID to retrieve the domain SID needed for ticket-forging attacks.
- Run Get-DomainTrust to enumerate direct trust relationships (type, direction, target).
- Run Get-DomainTrustMapping to recursively walk all reachable trusts across forests.
- Use Get-DomainOU / SharpView with findstr /b "name" to map the OU structure.

## Commands

### Get full domain object (SharpView)

```powershell
.\SharpView.exe Get-Domain
```

### Count domain users

```powershell
(Get-DomainUser).count
```

### Get domain SID

```powershell
Get-DomainSID
```

### Enumerate direct trusts

```powershell
Get-DomainTrust
```

### Recursively map all trusts

```powershell
Get-DomainTrustMapping
```

### List all OU names (SharpView)

```powershell
.\SharpView.exe Get-DomainOU | findstr /b "name"
```
