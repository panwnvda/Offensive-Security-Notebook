# 03.08 SharpView Help Parameter Discovery

> Hit `-Help` on any SharpView function — it's the only way to discover its parameter names since pipeline objects aren't supported.

## Overview

Any SharpView function accepts the -Help flag to display all supported parameters. This is the primary way to discover filtering, credential, and output options since SharpView does not support PowerShell pipeline objects. Understanding parameter names is essential because SharpView uses slightly different naming conventions.

## Steps

- Pass -Help to any SharpView function to list all accepted parameters.
- SharpView uses strings, not PS objects — cannot use Select-Object or pipe to Where-Object.
- Key shared parameters across most functions: -Domain, -Server, -Credential, -LDAPFilter, -Properties, -SearchBase.
- For SharpView, use OS-level tools like findstr to filter text output.

## Commands

### Get help for Get-DomainUser

```powershell
.\SharpView.exe Get-DomainUser -Help
```

### Get help for Get-DomainGroupMember

```powershell
.\SharpView.exe Get-DomainGroupMember -Help
```

### Get help for Get-DomainComputer

```powershell
.\SharpView.exe Get-DomainComputer -Help
```
