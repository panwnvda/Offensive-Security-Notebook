# 09.09 Kerberos User Enumeration Password Spraying Kerbrute

> Userenum and password-spray over Kerberos with `kerbrute` — userenum is lockout-safe, passwordspray still increments failure counts.

## Overview

Kerbrute abuses AS-REQ responses to enumerate valid usernames and spray passwords against AD without triggering traditional event 4625 lockout alerts. userenum sends TGT requests and detects PRINCIPAL_UNKNOWN vs pre-auth-required responses. passwordspray tests one password against a list of users — this does increment failure counts and can lock accounts, generating events 4768 and 4771. Kerbrute binary is downloaded from the releases page and made executable.

## Steps

- Download kerbrute_linux_amd64, chmod +x.
- Run userenum with a usernames wordlist, --dc, and -d to identify valid accounts — no lockout risk.
- Run passwordspray with a users list and a single candidate password to validate credentials.
- Username enumeration generates event 4768 (if Kerberos logging enabled); password spraying generates 4768 and 4771.
- Filter results: VALID USERNAME lines confirm existing accounts; VALID LOGIN lines confirm credentials.

## Commands

### Kerbrute user enumeration

```bash
kerbrute userenum users.txt --dc dc01.<DOMAIN>.local -d <DOMAIN>.local
```

### Kerbrute password spray

```bash
kerbrute passwordspray users.txt <DOMAIN> --dc dc01.<DOMAIN>.local -d <DOMAIN>.local
```
