# 16.13 Open-Source Tool Obfuscation

> Iterate ThreatCheck → rename signatures (`ticket` strings, `DiffieHellmanKey` class, TypeLib GUID for Rubeus) → recompile until clean.

## Overview

Iterate ThreatCheck → rename flagged strings → recompile. Key obfuscations for Rubeus: rename 'ticket' strings, rename DiffieHellmanKey class, and replace the TypeLib GUID (used in YARA rules).

## Steps

- **ThreatCheck → flagged string** — e.g. 'ticket' triggers detection.
- **Ctrl+Shift+H** in VS — Replace in Files: `ticket` → `tekcit`.
- **DiffieHellmanKey** class → rename to `DHKey` via VS Rename.
- **[GUID]::NewGUID()** — generate new TypeLib GUID; update in Project Properties → Assembly.

## Commands

### Generate new GUID (replaces TypeLib GUID in Rubeus)

```powershell
PS C:\> [GUID]::NewGUID()
# e.g. 0fb06558-5168-4398-a791-fa485f4f7325
# Then: Project > Properties > Application > Assembly Information > GUID
```

### ThreatCheck iteration workflow

```powershell
# Step 1: Build Rubeus-Obf in Release mode
# Step 2: Run ThreatCheck
ThreatCheck.exe -f .\Rubeus.exe
# Step 3: Identify flagged string (e.g. 'ticket')
# Step 4: Ctrl+Shift+H in VS → Replace 'ticket' with 'tekcit'
# Step 5: Rebuild → ThreatCheck again → repeat
# Common fixes: rename DiffieHellmanKey → DHKey, change TypeLib GUID, obfuscate 'Rubeus' namespace
```
