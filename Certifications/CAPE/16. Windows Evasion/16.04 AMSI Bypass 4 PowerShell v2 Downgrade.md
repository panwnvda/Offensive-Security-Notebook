# 16.04 AMSI Bypass 4 PowerShell v2 Downgrade

> Downgrade to PowerShell 2.0 (pre-AMSI, pre-CLM) — needs .NET 2.0/3.5 enabled, increasingly rare since 2020.

## Overview

PowerShell 2.0 predates AMSI (introduced in PS 5) and ConstrainedLanguage mode. Downgrading disables both. Requires .NET 2.0/3.5 still enabled — less common since 2020 but worth checking.

## Steps

- **powershell -version 2** — launches v2 interpreter.
- Bypasses AMSI and CLM simultaneously.
- Check if .NET 3.5 is enabled first — required for PS v2.

## Commands

### PS v2 downgrade

```powershell
powershell -version 2
# Verify: $PSVersionTable.PSVersion
```
