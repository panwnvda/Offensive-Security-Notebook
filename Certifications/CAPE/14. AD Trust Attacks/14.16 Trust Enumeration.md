# 14.16 Trust Enumeration

> Map every trust direction + type with `Get-ADTrust`, PowerView, and Adalanche — flag SIDHistory and TREAT_AS_EXTERNAL bits.

## Overview

Enumerate forest and domain trusts using built-in cmdlets, PowerView, and visual tools. Trust direction and type (intra-forest vs cross-forest, bidirectional vs one-way) determine available attack paths. The current card is shallow — it is missing Get-ADTrust, Adalanche, and SIDHistory flag checks.

## Steps

- **Import-Module activedirectory; Get-ADTrust -Filter \*** — built-in cmdlet for all trust relationships
- **Get-DomainTrust / Get-DomainTrustMapping** — PowerView recursive trust map
- **Get-ForestTrust** — inter-forest trusts
- **Adalanche collect activedirectory** — fast visual trust mapping tool
- **.\Adalanche.exe analyze** — opens browser visualizer on port 8080
- **BloodHound / SharpHound** — graphical trust path visualization
- Check TrustAttributes field for TREAT_AS_EXTERNAL (SID History enabled) and FOREST_TRANSITIVE
