# 10.02 DACL Enumeration dsacls PowerShell DirectoryServices

> Inspect raw DACLs with `dsacls.exe` or `[DirectoryServices.DirectorySearcher]` + SecurityMasks to understand what PowerView abstracts.

## Overview

Before using automated tools, DACLs can be inspected manually. dsacls.exe (built-in Windows binary) displays ACLs for AD objects in human-readable form. PowerShell's System.DirectoryServices.DirectorySearcher with SecurityMasks retrieves the raw binary security descriptor, and ActiveDirectorySecurity.SetSecurityDescriptorBinaryForm parses it into ACE objects. These techniques are useful for understanding what automated tools are doing under the hood.

## Steps

- dsacls.exe with the full LDAP Distinguished Name displays all ACEs for an object.
- Pipe dsacls output through Select-String to filter for a specific principal's rights.
- In PowerShell, use DirectorySearcher with SecurityMasks::Dacl|Owner to fetch the raw binary descriptor.
- Parse it with a new ActiveDirectorySecurity object and SetSecurityDescriptorBinaryForm.
- Filter .Access by IdentityReference to see a specific user's rights.
- AccessChk from Sysinternals can show security descriptors for processes with -l flag.

## Commands

### dsacls — show all ACEs for Yolanda

```text
PS C:\Users\Administrator> dsacls.exe "cn=Yolanda,cn=users,dc=<DOMAIN>,dc=local"
```

### dsacls — filter to Pedro's rights

```powershell
dsacls.exe "cn=Yolanda,cn=users,dc=<DOMAIN>,dc=local" | Select-String "Pedro"
```

### DirectorySearcher — get binary security descriptor

```powershell
$directorySearcher = New-Object System.DirectoryServices.DirectorySearcher('(samaccountname=Yolanda)')
```

### Parse and view all ACEs

```powershell
$parsedSecurityDescriptor = New-Object System.DirectoryServices.ActiveDirectorySecurity
```

### Filter ACEs to Pedro only

```powershell
$parsedSecurityDescriptor.Access | Where-Object {$_.IdentityReference -like '*Pedro*'}
```

### AccessChk — process security descriptor

```text
PS C:\Users\Admin\Downloads\AccessChk> .\accesschk64.exe -p "explorer.exe" -l
```
