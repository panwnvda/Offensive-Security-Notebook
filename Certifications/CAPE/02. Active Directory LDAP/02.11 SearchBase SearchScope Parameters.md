# 02.11 SearchBase SearchScope Parameters

> Narrow AD queries with `-SearchBase` to an OU DN and `-SearchScope Base/OneLevel/SubTree` to slash result noise.

## Overview

SearchBase scopes an AD query to a specific OU distinguished name, dramatically reducing noise in large environments. SearchScope controls depth: Base returns the OU object itself, OneLevel returns only direct children, SubTree recurses all child containers. The numeric (0/1/2) and name forms are interchangeable.

## Steps

- SearchBase accepts a DN string such as "OU=Employees,DC=<DOMAIN>,DC=LOCAL".
- SearchScope Base (0) — returns only the container object itself via Get-ADObject; returns nothing with Get-ADUser.
- SearchScope OneLevel (1) — returns direct children of the OU only.
- SearchScope SubTree (2) — full recursive traversal of the OU and all sub-OUs (default behavior).
- Combine with -Filter * and .count to benchmark total object counts before narrowing scope.
- Numeric and name forms (1 and OneLevel) are interchangeable in all cmdlets.

## Commands

### Count all users in OU (SubTree)

```powershell
(Get-ADUser -SearchBase "OU=Employees,DC=<DOMAIN>,DC=LOCAL" -Filter *).count
```

### SearchScope Base — returns the OU object

```powershell
Get-ADObject -SearchBase "OU=Employees,DC=<DOMAIN>,DC=LOCAL" -SearchScope Base -Filter *
```

### SearchScope OneLevel — direct children only

```powershell
Get-ADUser -SearchBase "OU=Employees,DC=<DOMAIN>,DC=LOCAL" -SearchScope OneLevel -Filter *
```

### SearchScope SubTree — all recursive

```powershell
(Get-ADUser -SearchBase "OU=Employees,DC=<DOMAIN>,DC=LOCAL" -SearchScope Subtree -Filter *).count
```
