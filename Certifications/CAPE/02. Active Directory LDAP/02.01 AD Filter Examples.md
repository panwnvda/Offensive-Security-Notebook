# 02.01 AD Filter Examples

> Use AD module `-Filter` syntax — wildcards, escaped characters, and pipe into `Get-CimInstance` for installed-software priv-esc leads.

## Overview

PowerShell AD module -Filter syntax for common enumeration: hostname patterns, adminCount, DoesNotRequirePreAuth, SPN-set users. Filters can use curly braces, single quotes, or double quotes interchangeably. The asterisk wildcard is supported; certain characters must be escaped. Use get-ciminstance with -Filter to enumerate installed software and spot local privilege-escalation vectors.

## Steps

- Filter "DNSHostName -like 'SQL*'" — find SQL servers by hostname pattern.
- adminCount -eq 1 — enumerate protected groups (AdminSDHolder-protected).
- DoesNotRequirePreAuth -eq 'True' — find AS-REP-roastable accounts combined with adminCount.
- servicePrincipalName -ne $null — identify Kerberoastable admin accounts.
- get-ciminstance win32_product — enumerate installed software; filter with -Filter to exclude vendors.
- Filter * -SearchBase — scope enumeration to a specific OU path.

## Commands

### Filter for SQL servers

```powershell
Get-ADComputer -Filter "DNSHostName -like 'SQL*'"
```

### Filter administrative groups

```powershell
Get-ADGroup -Filter "adminCount -eq 1" | select Name
```

### Filter AS-REP-roastable admin users

```powershell
Get-ADUser -Filter {adminCount -eq '1' -and DoesNotRequirePreAuth -eq 'True'}
```

### Find admin users with SPN set (Kerberoastable)

```powershell
Get-ADUser -Filter "adminCount -eq '1'" -Properties * | where servicePrincipalName -ne $null | select SamAccountName,MemberOf,ServicePrincipalName | fl
```

### Enumerate installed software

```powershell
get-ciminstance win32_product | fl
```

### Filter out Microsoft software

```powershell
get-ciminstance win32_product -Filter "NOT Vendor like '%Microsoft%'" | fl
```

### All users in OU

```powershell
Get-ADUser -Filter * -SearchBase 'OU=Admin,DC=<DOMAIN>,dc=local'
```
