# 02.02 Built-in AD Enumeration dsquery WMI ADSI

> Living-off-the-land AD recon with whitelisted `dsquery+dsget`, `Get-WmiObject`, and `[adsisearcher]` ADSI queries.

## Overview

Beyond the AD PowerShell module, several native Windows tools enumerate AD: dsquery+dsget (DS Tools, available on all modern Windows without install), WMI via Get-WmiObject, and ADSI via [adsisearcher]. These are whitelisted in mature environments and useful for living off the land when third-party tools are unavailable.

## Steps

- dsquery user | dsget user — find users with non-expiring passwords scoped to an OU.
- Get-WmiObject -Class win32_group — enumerate domain groups using WMI with a domain filter.
- [adsisearcher] typeaccelerator — COM-based LDAP query returning object paths.
- DS Tools are available by default on modern Windows and require only domain connectivity.
- WMI supports complex -Filter strings (e.g., Domain='<DOMAIN>') for scoped queries.
- ADSI FindAll() returns LDAP paths for all matched objects.

## Commands

### dsquery+dsget — users with non-expiring passwords

```bash
dsquery user "OU=Employees,DC=<DOMAIN>,DC=local" -name * -scope subtree -limit 0 | dsget user -samid -pwdneverexpires | findstr /V no
```

### WMI — enumerate domain groups

```powershell
Get-WmiObject -Class win32_group -Filter "Domain='<DOMAIN>'" | Select Caption,Name
```

### ADSI — enumerate all computers via [adsisearcher]

```powershell
([adsisearcher]"(&(objectClass=Computer))").FindAll() | select Path
```
