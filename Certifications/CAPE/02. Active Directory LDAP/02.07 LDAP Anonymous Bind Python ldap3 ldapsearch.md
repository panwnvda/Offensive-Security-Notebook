# 02.07 LDAP Anonymous Bind Python ldap3 ldapsearch

> Test NULL LDAP binds against Linux DCs and vCenter with `ldap3`, then enumerate users/groups/policy via `ldapsearch` unauthenticated.

## Overview

LDAP anonymous binds allow unauthenticated retrieval of AD objects including users, groups, computers, and the password policy. Linux hosts and vCenter appliances are commonly misconfigured to permit this. Use Python ldap3 to quickly test if NULL bind succeeds and read DSE info, then ldapsearch or windapsearch to enumerate all objects.

## Steps

- Python ldap3 — test NULL bind programmatically; c.bind() returns True if anonymous bind is allowed.
- s.info — dumps DSA info including naming contexts, functional levels, DC hostname.
- ldapsearch -H -x -b — retrieve all AD objects via anonymous bind from Linux.
- Successful anonymous bind enables password spraying list building and AS-REP-roasting candidate identification.
- NULL bind confirmed by Binded as: None in windapsearch output.

## Commands

### Python ldap3 — test anonymous bind

```bash
python3 -c "from ldap3 import *; s = Server('<TARGET>',get_info = ALL); c = Connection(s, '', ''); print(c.bind()); print(s.info)"
```

### ldapsearch — anonymous bind retrieve all objects

```bash
ldapsearch -H ldap://<TARGET> -x -b "dc=<DOMAIN>,dc=local"
```

### windapsearch — anonymous functionality check

```bash
python3 windapsearch.py --dc-ip <TARGET> -u "" --functionality
```

### windapsearch — anonymous user enumeration

```bash
python3 windapsearch.py --dc-ip <TARGET> -u "" -U
```

### windapsearch — anonymous computer enumeration

```bash
python3 windapsearch.py --dc-ip <TARGET> -u "" -C
```
