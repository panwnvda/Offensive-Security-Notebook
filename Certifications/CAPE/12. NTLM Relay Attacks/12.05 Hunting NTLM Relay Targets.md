# 12.05 Hunting NTLM Relay Targets

> Find signing-disabled relay targets with `RunFinger.py`, `nxc --gen-relay-list`, or `nmap --script smb2-security-mode` — DCs always sign.

## Overview

Before relaying, identify machines with SMB signing disabled — those are valid relay targets. Three tools cover this: RunFinger.py (Responder toolkit), NetExec with --gen-relay-list, and nmap with the smb2-security-mode script. Domain controllers always require signing and are not valid SMB relay targets.

## Steps

- Use RunFinger.py to sweep the subnet and report signing status per host
- Use nxc --gen-relay-list to build a target file automatically
- Use nmap smb2-security-mode.nse to confirm per-host signing requirements
- Save results to relayTargets.txt for use with ntlmrelayx -tf
- Hosts with "signing:False" are valid SMB relay targets; DCs are not

## Commands

### RunFinger sweep

```bash
python3 RunFinger.py -i 172.16.117.0/24
```

### CME relay list

```bash
nxc smb 172.16.117.0/24 --gen-relay-list relayTargets.txt
```

### nmap SMB signing

```bash
nmap -Pn --script=smb2-security-mode.nse -p 445 172.16.117.0/24 --open
```
