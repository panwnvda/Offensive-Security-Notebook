# 08.04 bloodyAD Enumeration

> Map an instant ACL attack surface with `bloodyAD get writable` — lists every object the current user can modify.

## Overview

bloodyAD provides flexible enumeration via get object, get membership, get search, and get writable. get writable is particularly useful: it shows every object the current user can modify — an instant ACL attack surface map.

## Steps

- get object: fetch specific attributes of any AD object by sAMAccountName or DN
- get membership: list all members of a group (with --no-nested for direct only)
- get search: raw LDAP search with custom filter and attribute list
- get writable: enumerate all objects where current user has write permissions — key recon step
- get children: list child objects of an OU or container
- Use --attr to specify comma-separated attributes; default returns all

## Commands

### get object — query user / computer / group

```bash
bloodyAD -u user -p pass -d DOMAIN --host DC_IP get object Administrator
bloodyAD -u user -p pass -d DOMAIN --host DC_IP get object DC01$
bloodyAD -u user -p pass -d DOMAIN --host DC_IP get object "Domain Admins"
# Specific attributes:
bloodyAD -u user -p pass -d DOMAIN --host DC_IP get object svcaccount --attr servicePrincipalName,userAccountControl,memberOf
```

### get membership — group members

```bash
bloodyAD -u user -p pass -d DOMAIN --host DC_IP get membership "Domain Admins"
bloodyAD -u user -p pass -d DOMAIN --host DC_IP get membership user1
bloodyAD -u user -p pass -d DOMAIN --host DC_IP get membership "Domain Admins" --no-nested
```

### get search — raw LDAP filter

```bash
# All users with SPN (Kerberoastable)
bloodyAD -u user -p pass -d DOMAIN --host DC_IP get search --filter '(servicePrincipalName=*)' --attr sAMAccountName,servicePrincipalName
# Accounts with no preauth (AS-REP Roastable)
bloodyAD -u user -p pass -d DOMAIN --host DC_IP get search --filter '(userAccountControl:1.2.840.113556.1.4.803:=4194304)' --attr sAMAccountName
# All computers
bloodyAD -u user -p pass -d DOMAIN --host DC_IP get search --filter '(objectClass=computer)' --attr dNSHostName,operatingSystem
```

### get writable — instant attack surface map

```bash
# Show ALL objects current user can write to
bloodyAD -u user -p pass -d DOMAIN --host DC_IP get writable
# Filter by object type
bloodyAD -u user -p pass -d DOMAIN --host DC_IP get writable --otype user
bloodyAD -u user -p pass -d DOMAIN --host DC_IP get writable --otype computer
bloodyAD -u user -p pass -d DOMAIN --host DC_IP get writable --otype group
```

### Delegation enumeration

```bash
# Unconstrained delegation
bloodyAD -u user -p pass -d DOMAIN --host DC_IP get search --filter '(userAccountControl:1.2.840.113556.1.4.803:=524288)' --attr sAMAccountName,userAccountControl
# Constrained delegation
bloodyAD -u user -p pass -d DOMAIN --host DC_IP get search --filter '(msDS-AllowedToDelegateTo=*)' --attr sAMAccountName,msDS-AllowedToDelegateTo
# RBCD
bloodyAD -u user -p pass -d DOMAIN --host DC_IP get search --filter '(msDS-AllowedToActOnBehalfOfOtherIdentity=*)' --attr sAMAccountName
```

### LAPS password read (if rights exist)

```bash
bloodyAD -u user -p pass -d DOMAIN --host DC_IP get object COMPUTER$ --attr ms-Mcs-AdmPwd,ms-Mcs-AdmPwdExpirationTime
# Windows LAPS:
bloodyAD -u user -p pass -d DOMAIN --host DC_IP get object COMPUTER$ --attr msLAPS-Password,msLAPS-EncryptedPassword
```
