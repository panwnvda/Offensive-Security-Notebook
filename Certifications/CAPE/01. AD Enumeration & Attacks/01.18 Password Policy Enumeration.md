# 01.18 Password Policy Enumeration

> Pull lockout threshold and observation window via `net accounts`, `enum4linux-ng`, or `Get-DomainPolicy` before any spray.

## Overview

Always enumerate the password policy before spraying to learn lockout threshold and observation window. Combine Linux tools (CME, enum4linux-ng, rpcclient, ldapsearch) with Windows tools (net accounts, PowerView Get-DomainPolicy) and null-session SMB.

## Steps

- CME --pass-pol — fastest authenticated check from Linux.
- rpcclient getdompwinfo — null-session fallback.
- enum4linux-ng -P with -oA for structured output.
- ldapsearch pwdHistoryLength — raw LDAP.
- Windows: `net accounts` and `net accounts /domain`.
- PowerView: `Get-DomainPolicy` on Windows.
- Null-session SMB: `net use \\DC01\ipc$ "" /u:""`.

## Commands

### CME pass-pol

```bash
nxc smb 172.16.5.5 -u avazquez -p Password123 --pass-pol
```

### enum4linux-ng

```bash
enum4linux-ng -P 172.16.5.5 -oA ilfreight
```

### rpcclient getdompwinfo

```bash
rpcclient -U "" -N 172.16.5.5
```

### net accounts (Windows)

```cmd
net accounts /domain
```

### PowerView GetDomainPolicy

```powershell
Import-Module .\PowerView.ps1; Get-DomainPolicy
```

### Null session SMB

```cmd
net use \\DC01\ipc$ "" /u:""
```
