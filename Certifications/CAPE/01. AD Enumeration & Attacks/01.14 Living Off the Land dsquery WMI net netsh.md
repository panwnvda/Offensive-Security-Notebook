# 01.14 Living Off the Land dsquery WMI net netsh

> Enumerate AD with built-ins (`dsquery`, `wmic`, `net`, `netsh`) when third-party binaries are blocked.

## Overview

When custom tools are blocked, use built-in Windows binaries for AD enumeration. dsquery with LDAP filter bitmasks finds accounts matching UAC flags. WMI queries expose domain and computer info. PowerShell downgrade bypasses AMSI/logging.

## Steps

- dsquery user/computer — basic AD object listing.
- dsquery with UAC bitmask — find accounts with specific flags (e.g., DONT_REQ_PREAUTH=32, DCs=8192).
- wmic ntdomain / computersystem — domain and machine context.
- net group/user/accounts — domain group and user info.
- sc query windefend; netsh advfirewall; qwinsta; arp -a; route print.
- powershell.exe -version 2 — downgrade to bypass ScriptBlock logging.

## Commands

### dsquery users

```cmd
dsquery user
```

### dsquery no preauth

```cmd
dsquery * -filter "(&(objectCategory=person)(objectClass=user)(userAccountControl:1.2.840.113556.1.4.803:=32))" -attr distinguishedName userAccountControl
```

### dsquery DCs

```cmd
dsquery * -filter "(userAccountControl:1.2.840.113556.1.4.803:=8192)" -limit 5 -attr sAMAccountName
```

### wmic domain info

```cmd
wmic ntdomain get Caption,Description,DnsForestName,DomainName,DomainControllerAddress
```

### net commands

```cmd
net accounts /domain
```

### PS downgrade

```cmd
powershell.exe -version 2
```
