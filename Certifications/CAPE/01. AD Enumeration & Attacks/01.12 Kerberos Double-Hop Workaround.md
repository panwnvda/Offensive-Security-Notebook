# 01.12 Kerberos Double-Hop Workaround

> Pass an explicit `PSCredential` (or register a named PSSession config) to dodge the WinRM double-hop NTLM block.

## Overview

WinRM sessions do not cache credentials, so authenticated AD queries from inside a PSSession fail. Fix by passing an explicit PSCredential object with each command, or by registering a named PSSession configuration that impersonates the user on the remote host.

## Steps

- Detect the issue: `klist` inside the session shows only the HTTP service ticket, not a TGT.
- Workaround 1 (evil-winrm): Create PSCredential object and pass -credential to PowerView functions.
- Workaround 2 (Windows): Register-PSSessionConfiguration with -RunAsCredential, restart WinRM, reconnect with -ConfigurationName.

## Commands

### Detect double-hop

```text
klist
```

### PSCredential workaround

```powershell
$SecPassword = ConvertTo-SecureString '!qazXSW@' -AsPlainText -Force; $Cred = New-Object System.Management.Automation.PSCredential('<DOMAIN>\backupadm', $SecPassword); get-domainuser -spn -credential $Cred | select samaccountname
```

### Register PSSession

```powershell
Register-PSSessionConfiguration -Name backupadmsess -RunAsCredential <DOMAIN>\backupadm
```

### Connect with config

```powershell
Enter-PSSession -ComputerName DEV01 -Credential <DOMAIN>\backupadm -ConfigurationName backupadmsess
```
