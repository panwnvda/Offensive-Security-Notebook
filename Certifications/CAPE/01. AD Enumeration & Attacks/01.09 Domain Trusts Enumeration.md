# 01.09 Domain Trusts Enumeration

> Map every trust with `Get-ADTrust` and BloodHound's Map Domain Trusts to plan cross-domain Kerberoasting and foreign-group hops.

## Overview

Enumerate all trust relationships before attacking. Get-ADTrust, PowerView Get-DomainTrust, and netdom query all expose trust type and direction. BloodHound "Map Domain Trusts" shows the full picture. From trusted domains, check for foreign group membership and cross-forest Kerberoasting opportunities.

## Steps

- Get-ADTrust -Filter * — lists all trusts with ForestTransitive, IntraForest, Direction.
- PowerView Get-DomainTrust — quick trust view.
- Get-DomainTrustMapping — recursively maps all trusts in the forest.
- Get-DomainUser -Domain child — enumerate users in trusted domains.
- netdom query /domain: trust/dc/workstation — native trust info.
- Get-DomainForeignGroupMember — find users from other domains in local groups.

## Commands

### Get-ADTrust

```powershell
Import-Module activedirectory; Get-ADTrust -Filter *
```

### PowerView trust

```powershell
Get-DomainTrust
```

### Full trust mapping

```powershell
Get-DomainTrustMapping
```

### Users in child domain

```powershell
Get-DomainUser -Domain LOGISTICS.<DOMAIN>.LOCAL | select SamAccountName
```

### netdom trust query

```cmd
netdom query /domain:<DOMAIN>.local trust
```

### Foreign group members

```powershell
Get-DomainForeignGroupMember -Domain FREIGHTLOGISTICS.LOCAL
```
