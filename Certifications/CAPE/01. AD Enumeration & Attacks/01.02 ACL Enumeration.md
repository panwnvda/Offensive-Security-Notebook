# 01.02 ACL Enumeration

> Surface abusable ACEs via `Get-DomainObjectACL -ResolveGUIDs` and target controlled SIDs to cut noise.

## Overview

Enumerate ACLs on AD objects to find abusable rights. Use PowerView's Get-DomainObjectACL with -ResolveGUIDs to get human-readable ACE types. Target specific user SIDs to avoid noise. Also enumerate GPO DACL rights.

## Steps

- Convert-NameToSid — get the SID for a user to use as filter.
- Get-DomainObjectACL without -ResolveGUIDs — shows raw GUIDs.
- Get-DomainObjectACL -ResolveGUIDs — human-readable ACE types (ForceChangePassword, GenericWrite, etc.).
- Get-ADObject to reverse-lookup a GUID to an ACE name.
- Find-InterestingDomainAcl — broad sweep; slow but thorough.
- Get-DomainGPO + Get-ObjectAcl to find GPO write rights.

## Commands

### Get SID

```powershell
$sid = Convert-NameToSid wley
```

### Get ACLs raw

```powershell
Get-DomainObjectACL -Identity * | ? {$_.SecurityIdentifier -eq $sid}
```

### Get ACLs resolved

```powershell
Get-DomainObjectACL -ResolveGUIDs -Identity * | ? {$_.SecurityIdentifier -eq $sid}
```

### Reverse GUID lookup

```powershell
$guid= "00299570-246d-11d0-a768-00aa006e0529"; Get-ADObject -SearchBase "CN=Extended-Rights,$((Get-ADRootDSE).ConfigurationNamingContext)" -Filter {ObjectClass -like 'ControlAccessRight'} -Properties * |Select Name,DisplayName,DistinguishedName,rightsGuid| ?{$_.rightsGuid -eq $guid} | fl
```

### Find interesting ACLs

```powershell
Find-InterestingDomainAcl
```

### GPO ACL rights

```powershell
$sid=Convert-NameToSid "Domain Users"; Get-DomainGPO | Get-ObjectAcl | ?{$_.SecurityIdentifier -eq $sid}
```
