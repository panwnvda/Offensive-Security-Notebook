# 01.27 User Enumeration

> Validate domain accounts with `kerbrute userenum` over Kerberos pre-auth — no failed-logon events, no lockouts.

## Overview

Enumerate valid domain accounts before spraying. kerbrute validates via Kerberos pre-auth with no lockout risk. Add anonymous LDAP queries and windapsearch for unauthenticated paths. Use credentialed CME to expand the list.

## Steps

- kerbrute userenum — validates against DC Kerberos; no lockout.
- enum4linux -U, rpcclient enumdomusers — null-session RPC.
- ldapsearch anonymous bind to pull sAMAccountName list.
- windapsearch with empty credentials for anonymous user enumeration.
- CME --users with valid credentials for full list.

## Commands

### kerbrute userenum

```bash
kerbrute userenum -d <DOMAIN>.LOCAL --dc 172.16.5.5 jsmith.txt -o valid_ad_users
```

### rpcclient enumdomusers

```bash
rpcclient -U "" -N 172.16.5.5
```

### ldapsearch anonymous

```bash
ldapsearch -h 172.16.5.5 -x -b "DC=<DOMAIN>,DC=LOCAL" -s sub "(&(objectclass=user))"  | grep sAMAccountName: | cut -f2 -d" "
```

### windapsearch anonymous

```bash
./windapsearch.py --dc-ip 172.16.5.5 -u "" -U
```

### CME credentialed users

```bash
sudo nxc smb 172.16.5.5 -u <USER> -p Academy_student_AD! --users
```
