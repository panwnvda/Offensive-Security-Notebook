# 10.06 Raw LDAP ldapsearch ADSearch

> T1087.002 · LDAP queries

## Overview

When PowerShell is locked down, raw LDAP queries via `ldapsearch`, `ldapdomaindump`, `Get-ADComputer -LDAPFilter`, or C# ADSearch surface the same data with less telemetry than PowerView's `Get-NetXXX` wrappers.

## Steps

- **Bind anonymously / authenticated** — `ldapsearch -x -H ldap://dc -D ... -w ...`.
- **Filter by ObjectClass** — `(&(objectClass=user)(objectCategory=person))`.
- **Pick attributes** — minimize what you pull (sAMAccountName, memberOf, servicePrincipalName).
- **Quiet wins** — anonymous bind, or LDAP signing not enforced → ntlmrelay.

## Commands

### ldapsearch users + SPN

```bash
ldapsearch -x -H ldap://<DC> -D '<USER>@<DOMAIN>' -w '<PASS>' \\
  -b 'dc=corp,dc=local' '(&(objectClass=user)(servicePrincipalName=*))' samaccountname servicePrincipalName
```

### ldapdomaindump (full grab)

```bash
ldapdomaindump -u <DOMAIN>\\<USER> -p <PASS> <DC> -o /tmp/dump
```

### ADSearch (C#) via execute-assembly

```powershell
execute-assembly ADSearch.exe --search "(&(objectCategory=person)(objectClass=user))" --attributes samaccountname,memberof
```
