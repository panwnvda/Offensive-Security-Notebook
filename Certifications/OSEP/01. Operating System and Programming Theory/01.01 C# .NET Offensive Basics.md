# 01.01 C# .NET Offensive Basics

> Foundations · CLR · Reflection · P/Invoke

## Overview

C# compiles to MSIL inside a PE wrapper; the CLR JITs it at runtime. Offensive tooling (SharpHound, Rubeus, Seatbelt, Certify, SharpUp) ships as .NET assemblies and is loaded in-memory via execute-assembly, Assembly.Load, or `IEX` of PowerShell stagers. P/Invoke lets C# call Win32 APIs directly without C glue.

## Steps

- **Compile small** — `csc.exe /platform:x64 /target:exe Tool.cs` from any unprivileged user.
- **P/Invoke a Win32 function** — declare DllImport and call from managed code.
- **Load in-memory** — `[System.Reflection.Assembly]::Load([Convert]::FromBase64String($b64))`.
- **Beware AMSI on .NET** — Assembly.Load buffers are scanned (since Win10 1903); patch AMSI first.

## Commands

### Minimal C# P/Invoke

```csharp
using System;
using System.Runtime.InteropServices;
class P {
  [DllImport("user32.dll")] static extern int MessageBox(IntPtr h,string t,string c,uint type);
  static void Main(){ MessageBox(IntPtr.Zero,"hello","osep",0); }
}
```

### Compile with built-in csc

```powershell
$csc='C:\\Windows\\Microsoft.NET\\Framework64\\v4.0.30319\\csc.exe'
& $csc /platform:x64 /target:exe /out:tool.exe tool.cs
```

### Load assembly from base64 in PowerShell

```powershell
$b64 = [Convert]::ToBase64String([IO.File]::ReadAllBytes('Rubeus.exe'))
$asm = [Reflection.Assembly]::Load([Convert]::FromBase64String($b64))
$asm.EntryPoint.Invoke($null, (,@('triage')))
```
