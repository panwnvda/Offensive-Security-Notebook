# 01.03 Windows API Syscalls

> Foundations · user32/kernel32 → ntdll → syscall

## Overview

Win32 APIs in kernel32/user32 are thin wrappers over Nt*/Zw* in ntdll.dll. Those issue the actual `syscall` instruction to transition into the kernel. EDRs hook the user-mode Nt* functions in ntdll to inspect arguments before the syscall — direct syscalls or unhooking bypass these hooks.

## Steps

- **Identify the call chain** — e.g. `WriteFile` → `NtWriteFile` (ntdll) → `syscall` (kernel).
- **Locate the SSN (System Service Number)** — first DWORD-aligned `mov r10,rcx; mov eax,SSN` stub.
- **Choose evasion path** — direct syscall stub, unhook ntdll, or remap a fresh ntdll copy.
- **Beware version drift** — SSNs change per Windows build. Use Hell's Gate to resolve at runtime.

## Commands

### Direct syscall stub (NASM, x64)

```asm
; NtAllocateVirtualMemory direct syscall on Win11 23H2 (SSN 0x18 — verify!)
NtAllocateVirtualMemory:
    mov  r10, rcx
    mov  eax, 18h
    syscall
    ret
```

### Resolve SSN at runtime (Hell's Gate idea)

```c
// Walk EAT of ntdll.dll, find 'NtAllocateVirtualMemory',
// read first 8 bytes of stub: 4C 8B D1 B8 ?? ?? ?? ??
BYTE* fn = (BYTE*)GetProcAddress(GetModuleHandleA("ntdll"), "NtAllocateVirtualMemory");
DWORD ssn = *(DWORD*)(fn + 4);   // bytes 4..7 = SSN little-endian
```

### Quickly list NT functions in ntdll

```powershell
Get-Command -Module ntdll -ErrorAction SilentlyContinue # not directly possible
# Use rundll32 trick or dumpbin
dumpbin /EXPORTS C:\\Windows\\System32\\ntdll.dll | Select-String '^\\s+\\d+\\s+\\w+\\s+\\w+\\s+Nt'
```
