# 01.04 Windows PE Format Loader

> Foundations · PE32+ structure

## Overview

Every Windows EXE/DLL is a PE file: DOS stub → NT headers → section table → sections (.text, .rdata, .data, .rsrc, .reloc). The loader maps sections into memory, resolves imports via the IAT, applies relocations, and calls the entry point. Understanding this is the prerequisite for reflective loaders, manual mapping, and module stomping.

## Steps

- **Read DOS + NT headers** — e_lfanew points to PE\0\0 + FileHeader + OptionalHeader.
- **Walk section headers** — each maps a file range to RVA + memory size with characteristics flags (R/W/X).
- **Resolve imports** — IMAGE_IMPORT_DESCRIPTOR array → DLL names + INT/IAT for each function.
- **Apply base relocations** — fix .reloc entries if ImageBase is shifted (true for non-fixed DLLs).
- **Invoke entry point** — DllMain for DLLs (reason=DLL_PROCESS_ATTACH); main thunk for EXEs.

## Commands

### Inspect a PE in PowerShell

```powershell
# Show NT headers, sections, and imports of a binary
Get-Item C:\\Windows\\System32\\notepad.exe | Format-List Length, VersionInfo
# Better: use dumpbin from VS Build Tools
& 'C:\\Program Files (x86)\\Microsoft Visual Studio\\2022\\BuildTools\\VC\\Tools\\MSVC\\*\\bin\\Hostx64\\x64\\dumpbin.exe' /HEADERS /IMPORTS notepad.exe
```

### PE walk in C (skeleton)

```c
PIMAGE_DOS_HEADER dos = (PIMAGE_DOS_HEADER)base;
PIMAGE_NT_HEADERS nt  = (PIMAGE_NT_HEADERS)((BYTE*)base + dos->e_lfanew);
PIMAGE_SECTION_HEADER sec = IMAGE_FIRST_SECTION(nt);
for (WORD i = 0; i < nt->FileHeader.NumberOfSections; i++) {
    printf("%-8s VA=%08x Size=%08x Char=%08x\\n",
        sec[i].Name, sec[i].VirtualAddress, sec[i].Misc.VirtualSize, sec[i].Characteristics);
}
```

### Linux: read PE with pefile

```bash
pip install pefile
python3 - <<'EOF'
import pefile
p = pefile.PE('mimikatz.exe')
for s in p.sections:
    print(s.Name.decode().rstrip('\\x00'), hex(s.VirtualAddress), hex(s.SizeOfRawData))
for entry in p.DIRECTORY_ENTRY_IMPORT:
    print(entry.dll.decode())
EOF
```
