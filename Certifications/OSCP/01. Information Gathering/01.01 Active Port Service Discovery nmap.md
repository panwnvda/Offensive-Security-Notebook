# 01.01 Active Port Service Discovery nmap

> Run a fast SYN sweep then `nmap -sV -sC -oA` discovered ports — grep saved output later instead of re-scanning.

## Overview

nmap is the OSCP workhorse. Start with a fast all-ports SYN scan, then drill into discovered services with -sV/-sC. Always save output (-oA) so you can grep later without re-scanning.

## Steps

- **Full-port discovery** — `nmap -p- --min-rate 1000` to find every listener.
- **Service + script enum** — re-target found ports with `-sV -sC`.
- **UDP top-N** — `-sU --top-ports 50` (slow, but commonly forgotten).
- **Save all formats** — `-oA name` writes .nmap, .gnmap, .xml.
- **Re-scan over time** — services come and go in lab environments.

## Commands

### Standard OSCP nmap chain

```bash
nmap -p- --min-rate 1000 -oA scan-all <TARGET>
ports=$(grep -oP '\d+/open' scan-all.gnmap | cut -d/ -f1 | paste -sd,)
nmap -p$ports -sV -sC -oA scan-targeted <TARGET>
```

### UDP top-N

```bash
sudo nmap -sU --top-ports 50 -oA scan-udp <TARGET>
```

### Vulnerability NSE scripts

```bash
nmap --script vuln -p 80,443,445 <TARGET>
nmap --script smb-vuln-* -p 445 <TARGET>
```
