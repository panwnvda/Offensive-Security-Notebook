# 01.03 Passive OSINT

> Mine WHOIS, certificate transparency logs, GitHub, LinkedIn, and breach dumps to map domains, subdomains, and tech stack before sending a packet.

## Overview

Build a public-data picture of the target before sending a single packet. Combine WHOIS, certificate transparency logs, Google dorking, LinkedIn, GitHub, and breach datasets. Map domains, sub-domains, IP ranges, employee names, and tech stack.

## Steps

- **Whois + ASN** — registrar, name servers, IP ranges.
- **Certificate Transparency** — crt.sh enumerates HTTPS subdomains.
- **Google dorks** — `site:<TARGET-DOMAIN> -www`, `filetype:pdf`, `intitle:index.of`.
- **Employee enum** — LinkedIn, hunter.io for email patterns.
- **Public credentials** — Have-I-Been-Pwned API + dehashed for past leaks.

## Commands

### Whois + ASN + DNS

```bash
whois <TARGET-DOMAIN>
dig <TARGET-DOMAIN> any +short
dig +short txt <TARGET-DOMAIN>
nslookup -type=mx <TARGET-DOMAIN>
```

### Subdomain enum (passive)

```bash
curl -s 'https://crt.sh/?q=%25.<TARGET-DOMAIN>&output=json' | jq -r '.[].name_value' | sort -u
subfinder -d <TARGET-DOMAIN> -silent
amass enum -passive -d <TARGET-DOMAIN>
```

### Google dorks worth trying

```text
site:<TARGET-DOMAIN> -www
site:<TARGET-DOMAIN> filetype:pdf
site:<TARGET-DOMAIN> inurl:admin
site:<TARGET-DOMAIN> intitle:"index of"
site:linkedin.com "<TARGET-DOMAIN>"
```
