# 19.03 Public Network → Foothold

> Enum every external host with full nmap + service scan, pick the softest (often vulnerable WP plugin), pop RCE, then pivot internal.

## Overview

Treat the lab like a real engagement: enum every external host first (nmap full + service), pick the softest target (often a WordPress with a vulnerable plugin), get RCE, then internal pivot.

## Steps

- **Nmap all externals** — full ports + service.
- **Web triage** — wpscan / dir-brute.
- **Exploit + foothold** — webshell or known plugin CVE.
- **Pivot** — chisel/socat back to attacker.

## Commands

### Exam-like workflow

```bash
nmap -p- --min-rate 1000 -oA all <TARGET>
nmap -sV -sC -p $(grep -oP '\d+/open' all.gnmap | cut -d/ -f1 | paste -sd,) -oA targeted <TARGET>
nikto -h http://<TARGET>
wpscan --url http://<TARGET> --api-token <t>
```
