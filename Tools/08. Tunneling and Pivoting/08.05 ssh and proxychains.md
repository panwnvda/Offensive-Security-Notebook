# 08.05 ssh and proxychains

> Classic OpenSSH tunneling (-L, -R, -D) paired with proxychains for tunneling other tools through SOCKS.

## Overview

OpenSSH supports three tunnel types: local (-L) forwards a local port to a remote-side destination; remote (-R) does the reverse; dynamic (-D) creates a SOCKS5 proxy on a local port. Combined with proxychains (config file specifies the SOCKS proxy), you can route any client tool through the SSH-tunneled SOCKS proxy. Always available — no additional tooling needed.

## Steps

- **Step 1** - Establish SSH session to pivot host.
- **Step 2** - Pick tunnel type (-L/-R/-D).
- **Step 3** - Configure proxychains to use the SOCKS port.
- **Step 4** - Run tools through proxychains.

## Commands

### Dynamic SOCKS (most useful)

```bash
ssh -D 1080 -N <USER>@<PIVOT>
# Now SOCKS5 proxy listening on 127.0.0.1:1080
```

### Local port forward (-L)

```bash
ssh -L 8443:<INTERNAL>:443 -N <USER>@<PIVOT>
# Now browse https://127.0.0.1:8443 — it lands at INTERNAL:443
```

### Remote port forward (-R)

```bash
# From pivot, forward attacker:8443 back to pivot's internal:443
ssh -R 8443:<INTERNAL>:443 -N <USER>@<ATTACKER>
```

### proxychains config

```bash
sudo nano /etc/proxychains4.conf
# At the end:
socks5  127.0.0.1 1080
# Test:
proxychains -q curl http://<INTERNAL>/
```

### nmap through SOCKS

```bash
# Only TCP-connect scans work over SOCKS
proxychains -q nmap -sT -Pn -p 445,3389,5985 <INTERNAL-CIDR>
```

### Multi-hop through several pivots

```bash
ssh -J <USER1>@<HOP1>,<USER2>@<HOP2> <USER3>@<TARGET>
```

### ProxyJump in config

```bash
# ~/.ssh/config
Host pivot1
    HostName <HOP1>
    User <USER1>

Host internal
    HostName <TARGET>
    User <USER3>
    ProxyJump pivot1
# Then: ssh internal
```
