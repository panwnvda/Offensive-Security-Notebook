# 07.10 ThreatCheck and DefenderCheck

> Binary-split tools that locate the exact bytes in your payload that trigger Defender signatures.

## Overview

ThreatCheck (or DefenderCheck for Defender specifically) takes a payload and Defender's command-line scanner, then binary-splits the file recursively until it finds the exact offset/bytes triggering detection. Once you know which bytes get flagged, you can target obfuscation precisely instead of blanket-encoding everything.

## Steps

- **Step 1** - Install MpCmdRun.exe must be available (built into Windows Defender).
- **Step 2** - Build ThreatCheck from source.
- **Step 3** - Run against your suspect payload.
- **Step 4** - Output reveals the offending byte range — modify those bytes.

## Commands

### Get ThreatCheck

```bash
git clone https://github.com/rasta-mouse/ThreatCheck
# Build in Visual Studio or use pre-compiled release
wget https://github.com/rasta-mouse/ThreatCheck/releases/latest/download/ThreatCheck.exe
```

### Run against payload

```powershell
ThreatCheck.exe -f mimikatz.exe
# Outputs offset + 1000 bytes either side of the signature
```

### DefenderCheck alternative

```powershell
wget https://github.com/matterpreter/DefenderCheck/releases/latest/download/DefenderCheck.exe -OutFile DefenderCheck.exe
DefenderCheck.exe mimikatz.exe
```

### Workflow

```text
1. ThreatCheck.exe -f payload.exe
2. Note the offending bytes (often a string like 'Mimikatz' or specific shellcode)
3. Modify those bytes: rename variables/classes/functions, change shellcode encoding
4. Re-run ThreatCheck.exe — if still flagged, repeat
5. Iterate until clean
```
