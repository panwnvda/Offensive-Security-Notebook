# 05.02 Havoc

> Modern modular C2 framework — sleek Qt-based client, designed for OPSEC against modern EDR.

## Overview

Havoc is a newer open C2 (HavocFramework). Qt-based client, modular architecture, post-ex module library, supports custom payloads. Designed with EDR evasion as a first principle — indirect syscalls, sleep obfuscation, beacon object files. Less mature than Sliver but actively developed. Try as a Sliver alternative.

## Steps

- **Step 1** - Build from source (Linux).
- **Step 2** - Start teamserver + connect with client.
- **Step 3** - Configure listener.
- **Step 4** - Generate implant.

## Commands

### Install / build

```bash
git clone https://github.com/HavocFramework/Havoc.git
cd Havoc
sudo apt install -y golang-go cmake qtbase5-dev libqt5websockets5-dev python3-dev libboost-all-dev mingw-w64
make ts-build           # build teamserver
make client-build       # build client
```

### Start teamserver

```bash
sudo ./havoc server --profile profiles/havoc.yaotl -v --debug
```

### Configure listener (in client UI)

```text
View → Listeners → Add
  Name: http
  Protocol: Https
  Host: <C2-DOMAIN>
  PortBind: 443
  PortConn: 443
```

### Generate implant

```text
Attack → Payload → Demon
  Listener: http
  Architecture: x64
  Format: Windows Executable
  Sleep: 10
  Jitter: 30
```
