# 03.29 PSPKIAudit

> PowerShell module for PKI/ADCS auditing — read CA configs, identify mis-configurations, dump templates.

## Overview

PSPKIAudit (formerly part of PSPKI) is the blue-team-y / defensive equivalent of Certify/Certipy. Read-only by default — enumerates CAs, templates, certificate enrollment rights, and flags ESC1-8 conditions in a structured PowerShell pipeline. Useful for engagement reporting (defensive recommendations) and for quickly orienting yourself in an unfamiliar ADCS environment.

## Steps

- **Step 1** - Install module.
- **Step 2** - Run audit cmdlets.
- **Step 3** - Cross-reference findings with Certify/Certipy for attack.

## Commands

### Install

```powershell
iwr 'https://github.com/GhostPack/PSPKIAudit/archive/refs/heads/master.zip' -OutFile PSPKIAudit.zip
Expand-Archive PSPKIAudit.zip
Import-Module .\PSPKIAudit-master\PSPKIAudit.psm1
```

### Enumerate CAs

```powershell
Get-CertificationAuthority
Get-CertificateAuthority -Verbose | Format-List *
```

### Find vulnerable templates

```powershell
Invoke-PKIAudit
Invoke-PKIAudit -CAComputerName <CA-FQDN>
```

### Specific ESC checks

```powershell
Get-AllVulnerableTemplates
Get-CertificateTemplate | Where-Object { $_.EnrolleeSuppliesSubject -eq $true }    # ESC1
Get-CertificateTemplate | Where-Object { $_.PolicyOIDs -match 'Any Purpose' }     # ESC2
```
