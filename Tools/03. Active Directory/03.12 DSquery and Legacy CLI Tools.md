# 03.12 DSquery and Legacy CLI Tools

> Pre-PowerShell legacy AD CLI tools — dsquery, dsget, dsmod, net.exe, setspn.exe.

## Overview

Before PowerShell, AD admins used the legacy CLI: dsquery (LDAP search), dsget (get attributes), dsmod (modify), dsadd, dsrm, net.exe (Windows networking), setspn.exe (SPN management). These tools are present on every Windows host and don't trigger PowerShell logging. Use when PowerShell is locked down or you want to operate via cmd.exe only. CRTP material references them heavily for living-off-the-land enumeration.

## Steps

- **Step 1** - Already on every domain-joined Windows host (cmd.exe).
- **Step 2** - Use dsquery for searches.
- **Step 3** - Pipe to dsget for attribute reads.

## Commands

### dsquery — find users

```cmd
dsquery user -name *
dsquery user -samid <USER>
dsquery user -limit 0 -o samid                       # all users by samaccountname
```

### dsquery — find groups

```cmd
dsquery group -name *admin*
dsquery group -name 'Domain Admins'
```

### dsquery — find computers

```cmd
dsquery computer -name *
dsquery computer -o samid -limit 0
```

### dsquery — custom LDAP filter

```cmd
dsquery * -filter '(servicePrincipalName=*)' -limit 0 -attr samaccountname,servicePrincipalName
```

### dsget — get attributes of object

```cmd
dsget user 'CN=<USER>,CN=Users,DC=<DOMAIN>,DC=local' -memberof -expand
```

### setspn — manage SPNs

```cmd
setspn -L <USER>                                       # list SPNs for user
setspn -A http/<SERVER>.<DOMAIN> <USER>                # add SPN (requires write access)
setspn -X                                              # show duplicate SPNs
```

### net.exe

```cmd
net user /domain
net user <USER> /domain
net group 'Domain Admins' /domain
net localgroup Administrators
net view /domain
net view \\<TARGET>
net share
net session                                            # logged-on sessions (admin only)
net accounts /domain                                   # password policy
```
