# 03.01 ActiveDirectory PowerShell Module RSAT

> Microsoft's official AD cmdlets — Get-ADUser, Get-ADGroup, Get-ADComputer, etc.

## Overview

The ActiveDirectory PowerShell module is part of RSAT (Remote Server Administration Tools) and is the official Microsoft path for AD queries. Less feature-dense than PowerView for offensive work, but every cmdlet is signed Microsoft code — no AMSI/Defender triggers. Comes preinstalled on Server OSes; on Workstation, install via Capability Add. CRTP/CRTO and most enterprise blue-team docs reference it heavily.

## Steps

- **Step 1** - Install RSAT (Windows 10/11 via Optional Features).
- **Step 2** - Import the module.
- **Step 3** - Run Get-AD* cmdlets.

## Commands

### Install on Windows 10/11

```powershell
# Workstation: install RSAT capability
Get-WindowsCapability -Name 'Rsat.ActiveDirectory*' -Online | Add-WindowsCapability -Online
# Server: usually preinstalled, or:
Add-WindowsFeature RSAT-AD-PowerShell
```

### Import + verify

```powershell
Import-Module ActiveDirectory
Get-Module ActiveDirectory
```

### User enumeration

```powershell
Get-ADUser -Filter * -Properties *
Get-ADUser -Identity <USER> -Properties *
Get-ADUser -Filter {ServicePrincipalName -ne $null} -Properties ServicePrincipalName    # kerberoastable
Get-ADUser -Filter {DoesNotRequirePreAuth -eq $true} -Properties DoesNotRequirePreAuth   # asreproastable
```

### Group enumeration

```powershell
Get-ADGroup -Filter *
Get-ADGroupMember 'Domain Admins' -Recursive
Get-ADGroupMember 'Enterprise Admins' -Recursive
Get-ADPrincipalGroupMembership <USER>     # groups <USER> belongs to
```

### Computer enumeration

```powershell
Get-ADComputer -Filter * -Properties OperatingSystem,LastLogonDate
Get-ADComputer -Filter {OperatingSystem -like '*Server*'}
Get-ADComputer -Filter {TrustedForDelegation -eq $true}   # unconstrained delegation
Get-ADComputer -Filter * -Properties msDS-AllowedToActOnBehalfOfOtherIdentity
```

### OU + GPO + Trust

```powershell
Get-ADOrganizationalUnit -Filter *
Get-GPO -All
Get-ADTrust -Filter *
```

### ACL queries

```powershell
Get-Acl 'AD:CN=<USER>,DC=<DOMAIN>,DC=local' | Select -ExpandProperty Access
```
