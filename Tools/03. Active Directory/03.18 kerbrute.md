# 03.18 kerbrute

> Fast Kerberos username enumeration and password spraying — does NOT trigger AD account lockout by default.

## Overview

kerbrute is Go-based, runs against the KDC over UDP/88. Two modes: usernameenum (which usernames exist?) and passwordspray (try one password against many users). Critically: kerbrute uses AS-REQ with pre-auth, which does NOT increment the badPwdCount on AD by default in many configs (verify per-engagement) — so it's lower-lockout-risk than NetExec SMB spray.

## Steps

- **Step 1** - Download binary or build from source.
- **Step 2** - Identify the KDC (any DC).
- **Step 3** - Run username enumeration first, then spray validated users.

## Commands

### Install

```bash
go install github.com/ropnop/kerbrute@latest
# Or download binary from releases
```

### Username enumeration

```bash
kerbrute userenum -d <DOMAIN> --dc <DC> users.txt -o valid.txt
```

### Password spray

```bash
kerbrute passwordspray -d <DOMAIN> --dc <DC> valid.txt 'Spring2025!'
```

### Safe mode (caps attempts per user)

```bash
kerbrute passwordspray -d <DOMAIN> --dc <DC> --safe valid.txt 'Spring2025!'
```

### Brute (one user, many passwords)

```bash
kerbrute bruteuser -d <DOMAIN> --dc <DC> rockyou.txt <USER>
# Only do this against accounts that won't lock out (service accounts)
```
