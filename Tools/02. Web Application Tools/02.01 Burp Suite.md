# 02.01 Burp Suite

> The industry-standard web pentest proxy — request interception, replay, fuzzing, scanning, plugin ecosystem.

## Overview

Burp Suite is the Swiss Army knife of web testing. Use it for: intercepting traffic, replaying requests with modifications, fuzzing parameters (Intruder), authentication testing (Autorize), brute force, scanning, and OAST callbacks (Collaborator). Community Edition is free with reduced concurrency. Pro adds scanner, Collaborator-self-hosted, and Turbo Intruder.

## Steps

- **Step 1** - Install — Kali ships it. Or download from PortSwigger.
- **Step 2** - Configure browser proxy to 127.0.0.1:8080.
- **Step 3** - Install Burp CA cert in browser for HTTPS interception.
- **Step 4** - Common workflow: browse target → review Proxy history → Repeater for surgical testing → Intruder for fuzzing.
- **Step 5** - Extensions: Autorize (authz testing), Turbo Intruder (race conditions), Hackvertor (encoding), JSON Web Tokens, Param Miner, DOM Invader.

## Commands

### Launch on Kali

```bash
burpsuite
# or community version
burpsuite_community
```

### Browser proxy setup

```text
Firefox: Settings → Network → Manual proxy → 127.0.0.1:8080 (HTTP+HTTPS)
Or install FoxyProxy extension and toggle when needed.
Download Burp CA cert from http://burp/cert and add to browser trust store.
```

### Match-and-Replace rule

```text
Proxy → Options → Match and Replace
  Type: Request body
  Match: "user_id":1234
  Replace: "user_id":1235
→ Auto-rewrites every matching request
```

### Intruder cluster bomb

```text
1. Send a request to Intruder
2. Select payload positions (clear all, then add 2 positions: user + password)
3. Attack type: Cluster bomb
4. Payload set 1: usernames; Set 2: passwords
5. Start attack — every username paired with every password
6. Sort by response length to find auth success
```

### Turbo Intruder race

```text
1. Install Turbo Intruder extension via BApp Store
2. Right-click request → Send to Turbo Intruder
3. Use built-in race-attack template
4. Engine: HTTP/2 single-packet attack (sends all requests in one TCP frame)
5. Click Attack — 20 parallel requests within microseconds
```

### Collaborator OOB

```text
Burp → Collaborator → Copy to clipboard
# Use the unique subdomain (e.g. abc123.oastify.com) in SSRF/XXE/blind RCE payloads
# Poll Collaborator for received DNS/HTTP callbacks
```
