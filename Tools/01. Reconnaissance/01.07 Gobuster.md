# 01.07 Gobuster

> Go-based directory, DNS, vhost brute-force — the classic OSCP-era content discovery tool.

## Overview

Gobuster is older than ffuf but still common in CTF/OSCP write-ups. Three modes: `dir` (content discovery), `dns` (subdomain brute), `vhost` (vhost discovery). Simple, predictable, runs everywhere. ffuf is faster and more flexible — use Gobuster when documentation/scripts assume it.

## Steps

- **Step 1** - Install via apt or Go.
- **Step 2** - Pick mode: dir/dns/vhost/s3/fuzz.
- **Step 3** - Supply wordlist and target.

## Commands

### Install

```bash
sudo apt install -y gobuster
# or
go install github.com/OJ/gobuster/v3@latest
```

### Directory mode

```bash
gobuster dir -u https://<TARGET> -w /usr/share/wordlists/dirb/common.txt -x php,html,bak -t 20 -o gobuster_dir.txt
```

### DNS subdomain brute

```bash
gobuster dns -d <DOMAIN> -w /usr/share/wordlists/seclists/Discovery/DNS/subdomains-top1million-5000.txt -t 50 -o subs.txt
```

### Virtual host brute

```bash
gobuster vhost -u https://<TARGET> -w subdomains.txt --append-domain -t 20
```

### S3 bucket discovery

```bash
gobuster s3 -w bucket-names.txt -t 10
```
