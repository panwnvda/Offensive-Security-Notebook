# 01.01 amass

> OWASP DNS reconnaissance suite — passive subdomain enumeration across 20+ sources, plus active brute and graph correlation.

## Overview

amass aggregates passive DNS data from CT logs, public DNS feeds, VirusTotal, SecurityTrails, AlienVault OTX, GitHub, and dozens of others. Its 'enum' mode supports passive (no target contact), active (DNS resolution), and brute-force. The graph database lets you visualize attack surface over time. Use API keys to unlock the high-signal sources.

## Steps

- **Step 1** - Install via Go or homebrew.
- **Step 2** - Configure API keys in `~/.config/amass/config.ini`.
- **Step 3** - Run passive enum first (no target contact).
- **Step 4** - Active mode for DNS resolution + brute force.
- **Step 5** - Graph view: `amass viz` outputs Maltego/d3 formats.

## Commands

### Install

```bash
go install -v github.com/owasp-amass/amass/v4/...@master
# Or brew:
brew install amass
```

### Config API keys

```bash
mkdir -p ~/.config/amass
cat > ~/.config/amass/config.ini <<EOF
[data_sources.SecurityTrails]
[data_sources.SecurityTrails.account]
apikey = <ST-KEY>

[data_sources.Shodan]
[data_sources.Shodan.account]
apikey = <SHODAN-KEY>

[data_sources.VirusTotal]
[data_sources.VirusTotal.account]
apikey = <VT-KEY>
EOF
```

### Passive enum

```bash
amass enum -passive -d <DOMAIN> -o subs_passive.txt
```

### Active enum with brute force

```bash
amass enum -active -brute -d <DOMAIN> -w /usr/share/wordlists/amass/subdomains-top1mil-110000.txt -o subs_active.txt
```

### Graph database query

```bash
amass db -names -d <DOMAIN>
amass db -show -d <DOMAIN>
amass viz -d3 -d <DOMAIN> -o graph.html
```
