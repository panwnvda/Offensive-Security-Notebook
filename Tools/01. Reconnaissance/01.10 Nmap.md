# 01.10 Nmap

> Standard port scanner and service-version fingerprinter for everything from a single host to a /16 sweep.

## Overview

Nmap is the workhorse of network reconnaissance. Use it for port discovery, service version detection, OS fingerprinting, and as an NSE script runner for vulnerability identification and protocol-specific enumeration. Two-phase workflows pair it with masscan (masscan for fast all-port sweep, Nmap for depth on confirmed-open ports).

## Steps

- **Step 1** - Install via apt or build from source. On Kali it ships preinstalled.
- **Step 2** - Pick a scan type matching engagement context (stealth vs speed vs depth).
- **Step 3** - For service depth, follow up with `-sC -sV` on confirmed-open ports.
- **Step 4** - Use NSE scripts for protocol-specific enumeration (smb-enum, http-enum, ssl-cert).
- **Step 5** - OPSEC — adjust `-T` timing template, use `-D` decoys, fragment with `-f`, source port `--source-port 53` for firewall hopping.

## Commands

### Install

```bash
sudo apt install -y nmap
```

### Quick host discovery

```bash
sudo nmap -sn <CIDR>
sudo nmap -PE -PA21,22,80,443,3389 <CIDR>
```

### Standard CTF/lab scan

```bash
sudo nmap -sV -sC -p- -T4 --min-rate 5000 -oA full <TARGET>
```

### Targeted depth on confirmed open ports

```bash
sudo nmap -sV -sC -p 22,80,443,445,3389 --version-intensity 9 <TARGET> -oA depth
```

### SMB NSE enumeration

```bash
sudo nmap -p 445 --script 'smb-protocols,smb-security-mode,smb-os-discovery,smb2-time,smb-vuln*' <TARGET> -oA smb_recon
```

### Vulscan + vulners

```bash
sudo nmap -p- -sV --script vulners,vulscan/vulscan.nse <TARGET> -oA vuln
```

### Stealth OPSEC scan

```bash
sudo nmap -sS -T1 -D RND:10 --source-port 53 -f -iL targets.txt -oA stealth
```
