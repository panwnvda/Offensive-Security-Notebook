# 01.12 Subfinder

> ProjectDiscovery's passive subdomain enumerator — fast, focused, supports 30+ sources.

## Overview

Subfinder is amass's faster, leaner cousin. Pure passive — never touches the target's authoritative resolver. Configure with API keys for high-signal sources (Shodan, SecurityTrails, BinaryEdge, Chaos). Fits cleanly into pipelines with httpx/dnsx for live-host probing.

## Steps

- **Step 1** - Install via Go install.
- **Step 2** - Edit `~/.config/subfinder/provider-config.yaml` with your API keys.
- **Step 3** - Run with `-all -recursive` for full coverage.
- **Step 4** - Pipe to dnsx for resolution then httpx for live check.

## Commands

### Install

```bash
go install -v github.com/projectdiscovery/subfinder/v2/cmd/subfinder@latest
```

### Config API keys

```bash
mkdir -p ~/.config/subfinder
cat > ~/.config/subfinder/provider-config.yaml <<EOF
virustotal: [<VT-KEY>]
securitytrails: [<ST-KEY>]
shodan: [<SHODAN-KEY>]
censys: [<CENSYS-ID>:<CENSYS-SECRET>]
binaryedge: [<BE-KEY>]
chaos: [<CHAOS-KEY>]
github: [<GH-PAT>]
EOF
```

### Run with all sources

```bash
subfinder -d <DOMAIN> -all -recursive -silent -o subs.txt
```

### Multi-domain

```bash
subfinder -dL domains.txt -all -silent -o all_subs.txt
```

### Pipeline: enum → resolve → live

```bash
subfinder -d <DOMAIN> -all -silent | dnsx -silent | httpx -silent -mc 200,301,302,403 -o live_hosts.txt
```
