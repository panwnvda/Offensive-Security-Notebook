# 01.06 Gitleaks

> Scan Git repos for hardcoded secrets — API keys, tokens, credentials.

## Overview

Gitleaks scans git history (not just current state) for hardcoded credentials, API keys, private keys, and other secret patterns. Built-in ruleset covers AWS, GCP, Azure, GitHub, GitLab, Slack, Datadog, Twilio, etc. Run against your target org's public repos or pen-test target's source code.

## Steps

- **Step 1** - Install via brew/apt/Go.
- **Step 2** - `gitleaks detect` against a repo.
- **Step 3** - Review JSON output.

## Commands

### Install

```bash
brew install gitleaks
# or apt
sudo apt install -y gitleaks
# or go
go install github.com/zricethezav/gitleaks/v8@latest
```

### Scan a cloned repo + history

```bash
git clone https://github.com/<ORG>/<REPO>.git
cd <REPO>
gitleaks detect --source . --report-path leaks.json --no-banner
```

### Scan full history with options

```bash
gitleaks detect --source . --log-opts='--all' --report-path leaks-history.json --redact
```

### Custom config

```bash
gitleaks detect --source . --config ./custom-rules.toml --report-path leaks.json
```
