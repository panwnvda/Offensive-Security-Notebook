# 12.03 WinDbg

> Microsoft's official Windows debugger — kernel debugging, user-mode debugging, crash dump analysis.

## Overview

WinDbg is the canonical Windows debugger. WinDbg Preview is the modern GUI; WinDbg classic still in many lab guides. Use for: exploit development (analyzing crashes, finding offsets), malware reverse engineering, kernel debugging via VM, post-mortem analysis of mini-dumps. The Mona script extension automates many exploit-dev tasks (pattern_create, pattern_offset, !pvefindaddr).

## Steps

- **Step 1** - Install via Microsoft Store (WinDbg Preview) or SDK install.
- **Step 2** - For user-mode: attach to running process or launch under WinDbg.
- **Step 3** - For kernel debug: configure target VM, connect via pipe/NET.
- **Step 4** - Install Mona.py extension for exploit dev.

## Commands

### Install WinDbg Preview

```text
Microsoft Store → search 'WinDbg Preview' → install
# Or via PowerShell:
winget install Microsoft.WinDbg
```

### Common commands

```text
g                    Go (run)
bp <addr>            Set breakpoint
bl                   List breakpoints
bc <num>             Clear breakpoint
t                    Step into (trace)
p                    Step over
g @$ra               Run to return
u @rip               Disassemble at current RIP
r                    Show all registers
r rax                Show RAX
.dump /ma C:\dump.dmp    Full process dump
!analyze -v          Auto-analyze a crash
k                    Stack trace
!peb                 Print PEB
!teb                 Print TEB
```

### Install mona.py

```text
# Get mona.py from https://github.com/corelan/mona
# Copy to: C:\Program Files (x86)\Windows Kits\10\Debuggers\x64\winext\
# In WinDbg:
!py mona
!py mona config -set workingfolder c:\mona
!py mona pattern_create 1024
!py mona pattern_offset 35724134
!py mona modules
!py mona find -s '\xff\xe4' -m libspp.dll        # find 'jmp esp'
!py mona bytearray -b '\x00\x0a\x0d'             # generate bad-char array
```
